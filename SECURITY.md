# Security Policy

This repository is primarily a documentation and workflow knowledge base for Chinese web-novel writing agents. Security issues usually mean unsafe documentation content, privacy leakage, copyright leakage, or workflow guidance that could enable real-world harm.

## Supported scope

The active `main` branch is supported.

## What to report

Please report:

- Private personal information accidentally included in documents.
- Unlicensed private manuscript, reader-chat, editor-feedback, or paid-course content.
- Operational instructions for fraud, evasion, forgery, smuggling, privacy abuse, violence, or other real-world harm.
- Unsafe guidance involving minors, medical topics, financial topics, public safety, or legal procedures.
- Malicious files or scripts if code is added in the future.

## How to report

If the issue contains sensitive details, do not paste the full content into a public issue. Use the least amount of information needed to identify the file, section, risk type, and suggested mitigation.

If no private reporting channel is available, open a public issue with a minimal, non-sensitive summary and avoid reproducing the harmful content.

## Expected handling

Reports should be triaged by risk:

1. Privacy leakage and unsafe operational content.
2. Copyright or source-policy violations.
3. Outdated high-risk legal, platform, medical, financial, or public-safety material.
4. General documentation quality issues.

Unsafe content should be removed, generalized, or rewritten into non-operational form: motivation, evidence, procedure, and consequences instead of actionable methods.
