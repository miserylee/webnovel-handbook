# 路线图

本项目当前重点从“继续横向新增专题”转为“开源化、收束、提纯和 agent 可消费”。

## v0.1 开源基础

目标：让仓库可以安全公开、被阅读、被引用、被贡献。

- MIT License。
- 贡献规范、来源政策、安全政策和社区准则。
- GitHub PR/Issue 模板。
- README 突出关键文件和 AI agent 使用路径。
- 移除空占位目录和明显临时文件。

## v0.2 Agent 消费优化

目标：让 AI agent 少读、读准、能执行。

- 继续强化 `docs/workflows/57-knowledge-base-routing-consolidation-guide.md`。
- 把常用任务压缩为最小阅读包。
- 给过长专题补“十秒定位卡”。
- 抽取重复的投诉链、证据链、部门分流链和安全边界模板。

## v0.3 写稿闭环稳定

目标：让写稿、内部试读、审稿、改稿和台账回写成为默认闭环。

- 继续打磨 `docs/workflows/58-integrated-drafting-beta-review-revision-workflow.md`。
- 增加更多章节生产卡和交接包样例。
- 明确何时需要外部真实读者和编辑反馈。
- 优化前 10 章、前 30 章和卷级推进的调用路径。

## v1.0 稳定知识库

目标：形成可长期复用的 AI agents 中文网文创作权威参考。

- 核心入口稳定。
- 来源政策和安全边界稳定。
- 新专题准入机制稳定。
- 常用创作任务有清晰工作流、模板和停止条件。
- 文档结构能被人类作者和 AI agent 共同使用。
