# 学习写网文

这是一个“网文写作知识库”项目，不是单本小说项目。它的目标是把本地 `novel` skill、已有小说项目、编辑部审稿经验和读者群反馈规则沉淀成可长期复用的权威参考资料。

后续让 AI 起草、改稿、审稿或维护网文项目时，优先读取本项目文档，再结合具体小说项目的专属设定、人物、大纲和正文。

## 如何接续

每次新会话先读：

1. [项目索引](docs/00-index.md)
2. [资料来源清单](docs/01-source-inventory.md)
3. 当前任务对应的专题文档

如果任务是让 AI 写或改一章，至少读取：

- [网文核心原则](docs/02-webnovel-principles.md)
- [人物与对话](docs/04-character-and-dialogue.md)
- [AI 写作与降 AI 味指南](docs/06-ai-writing-guidelines.md)
- [模板与检查清单](docs/10-templates-and-checklists.md)

如果任务是搭新小说项目或维护长篇项目，额外读取：

- [项目流程与文档规范](docs/03-project-workflow.md)
- [连载结构与节奏](docs/05-plot-and-serial-rhythm.md)
- [连续性与伏笔台账](docs/07-continuity-ledger.md)

如果任务是审稿、试读或评估作品，读取：

- [审稿与读者反馈](docs/08-review-and-reader-feedback.md)
- [案例笔记](docs/09-case-notes.md)

## 维护原则

- 本项目允许沉淀通用写作方法，因为它本身就是知识库；单本小说项目仍只记录项目专属事实和例外。
- 结论必须尽量带“适用场景、操作方法、反例风险、检查问题”，不能只写口号。
- 新增经验优先写入对应专题文档；原始、未整理材料可以先放在 `notes/`。
- 所有中文文档统一使用 UTF-8。
