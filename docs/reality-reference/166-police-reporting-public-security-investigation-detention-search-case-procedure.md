# 166. 报警接处警、公安办案、侦查措施、拘留逮捕与案件程序专项

适用范围：当 AI agent 需要设计、起草、审稿或修订报警、报案、110 接处警、派出所接待、民警出警、现场处置、受案登记、立案/不立案、刑事案件侦查、治安行政案件、询问、讯问、笔录、传唤、拘留、逮捕、取保候审、监视居住、搜查、扣押、查封、冻结、鉴定、辨认、证人保护、被害人告知、案件移送、检察院审查、校园警情、家暴警情、诈骗报案、失踪走失、事故后警方介入和任何“角色把冲突交给公安/警务系统处理”的剧情时，使用本文件。

核心目标：让警务桥段不再是“报警立刻抓人、警察一句话定性、主角私查全案、派出所万能盖章”，而是形成可追踪的程序链：

`警情发生 -> 报警/报案 -> 接警分流 -> 现场处置 -> 受案登记 -> 刑事/行政/民事/求助分流 -> 证据固定 -> 调查/侦查 -> 强制措施 -> 权利告知 -> 案件移送/处罚/调解/不立案 -> 复议救济/申诉 -> 台账回写`

一句话规则：警察不是剧情橡皮章，而是有职权边界、程序条件、证据要求、时限压力、公众监督和后续移送链的制度角色。

---

## 1. 本专题解决什么问题

AI 写警务和办案常见坏味道：

1. 报警按钮化：主角说报警，警察秒到、秒抓、秒定罪，缺少接警、到场、取证和分流。
2. 警察万能化：派出所能随口调监控、查隐私、冻结账户、定罪判刑、威胁嫌疑人。
3. 主角越权神探化：主角私闯、偷拍、套话、拿走证据，最后警方无条件认可。
4. 立案误写：普通纠纷、治安案件、刑事案件、民事争议、求助警情混成一锅。
5. 讯问随意化：把询问、讯问、传唤、拘留、逮捕、取保候审等概念混用。
6. 证据污染：案发现场被角色随便翻、证据被拿走、手机被解锁，后续还一锤定音。
7. 强制措施轻飘：抓人、搜查、扣押、冻结没有条件、手续、权利告知和救济路径。
8. 被害人缺席：受害者只负责哭和提供线索，不写告知、保护、隐私、伤情、财产追赃和心理后果。
9. 未成年人和家暴警情冒犯：把校园欺凌、家暴、走失、自伤风险和未成年人隐私当普通吵架。
10. 违法教程化：为了显得真实，写如何逃避侦查、毁灭证据、作伪证、虚假报案、规避传唤或反侦查。

本专题把警务程序写成“制度压力 + 证据条件 + 人物权利 + 叙事阻力”。

## 3. 安全边界：只写程序可信度，不写规避侦查教程

警务、侦查和案件程序具有现实风险。AI agent 必须遵守：

1. 不输出逃避侦查、毁灭证据、作伪证、串供、规避传唤、摆脱监控、对抗搜查、逃避抓捕、威胁证人和诬告陷害的操作步骤。
2. 不提供真实法律建议、报案策略、诉讼策略或规避法律责任的方法。
3. 不替真实公安机关、检察院、法院或地方部门编造最新程序、时限、权限和内部规则；需要精确事实时查最新官方来源。
4. 不把非法私查、偷拍、窃听、跟踪、黑客取证、暴力逼供写成无代价正义。
5. 不把未成年人、家暴、性侵、自伤自杀、走失、老人失智、残障者受害等警情写成轻松梗或爽点。
6. 可以写概念性程序：报警、受案、调查、询问、讯问、传唤、立案、强制措施、鉴定、移送、告知、申诉、投诉。
7. 写架空世界也要有等价程序：官府受状、捕快勘验、宗门执法堂、族规调查、城防缉捕、神明审判前调查，都要有权限、证据、见证和责任。

可写的是“程序如何制造阻力、保护权利、固定证据和改变关系”。

不可写的是“现实中如何逃避或操控程序”。

## 4. 警务程序十线卡

设计警务桥段时，至少检查十条线。

| 线 | 要问什么 | 叙事功能 | 常见坏味道 |
| --- | --- | --- | --- |
| 警情线 | 发生了什么，谁报警，是否紧急 | 进入制度的触发点 | 报警秒解决 |
| 管辖线 | 哪个部门、地点、层级、警种处理 | 阻力、延迟、推诿、协作 | 派出所万能 |
| 分流线 | 刑事、治安行政、民事纠纷、求助、事故还是风险预警 | 类型判断 | 全部当刑案 |
| 现场线 | 到场后谁先控制现场、救人、隔离、记录 | 空间和证据保护 | 主角乱翻现场 |
| 证据线 | 报警记录、笔录、监控、物证、电子数据、鉴定如何形成 | 公平性和后续裁判 | 证据空降 |
| 人物线 | 报案人、被害人、嫌疑人、证人、家属、律师、未成年人 | 群像和权利 | 全员工具人 |
| 措施线 | 传唤、拘留、逮捕、取保、搜查、扣押、冻结有何条件 | 程序阻力 | 想抓就抓 |
| 告知线 | 权利义务、案件进展、家属通知、处罚决定如何送达 | 情绪和关系后果 | 消息凭空知道 |
| 监督线 | 检察监督、复议复核、投诉、申诉、舆论、内部审核 | 防止警察万能或黑箱 | 黑箱无边界 |
| 余波线 | 名声、工作、学校、家庭、财产、心理、安全如何改变 | 长篇连续性 | 结案清零 |

最小规则：每次报警至少回答“谁报、报什么、谁接、怎么分流、什么证据、下一步由谁承担”。

## 5. 报警与接处警：110 不是万能许愿机

报警场景要先区分紧急警情、非紧急求助、纠纷、刑事犯罪线索和行政违法线索。

### 5.1 报警四要素

| 要素 | 小说表达 |
| --- | --- |
| 地点 | 具体地址、楼栋房号、地标、路线、现场危险 |
| 事件 | 发生了什么，是否正在发生，是否有人受伤或被困 |
| 人员 | 报警人、被害人、嫌疑人、现场人数、是否有未成年人/老人/伤者 |
| 证据/风险 | 是否有武器、火灾、持续威胁、逃跑方向、监控、车辆、联系方式 |

### 5.2 接处警可以制造剧情

- 报警人紧张，地址说不清，拖慢到场。
- 同一事件多人报警，信息互相矛盾。
- 接警员要求确认是否有人受伤，主角才发现少了一个人。
- 出警民警先控制现场而不是立刻追问八卦。
- 纠纷警情暂时调解，但发现伤情或财物损失后转入案件。
- 网络诈骗报警后，关键不是“抓人”，而是止付、冻结、保留转账和聊天材料。
- 消防、医疗、交警、派出所等多部门联动，信息同步有时间差。

检查问题：报警之后，场景是否出现接警分流、到场时间、现场控制和下一步安排？

## 6. 受案、立案与分流：不是所有报案都叫立案

报案进入公安后，可能有不同路径。

### 6.1 常见分流

| 类型 | 例子 | 叙事结果 |
| --- | --- | --- |
| 刑事案件线索 | 盗窃、诈骗、伤害、侵财、严重暴力、重大事故犯罪线索 | 受案审查、立案侦查、强制措施可能出现 |
| 治安行政案件 | 打架、扰乱秩序、轻微盗窃、寻衅滋事等行政违法 | 询问、调查、行政处罚或调解 |
| 民事纠纷 | 借款、合同、婚恋财产、邻里纠纷 | 可能先调解或告知民事途径，但若有违法犯罪线索则另行处理 |
| 求助警情 | 走失、噪音、救助、危险预警 | 帮助联系、现场处置、转介相关部门 |
| 事故警情 | 交通事故、火灾事故、安全生产事故 | 现场保护、协同救援、事故调查或案件转化 |
| 风险警情 | 家暴、威胁、自伤风险、未成年人危险 | 保护、告诫、转介、通知监护或专业机构 |

### 6.2 立案的戏剧价值

立案不是剧情结束，而是新阶段开始：

- 主角从“自己怀疑”进入“需要证据支撑”的阶段。
- 对手开始销毁证据、找关系、施压证人。
- 警方可依法采取更多侦查措施，但也受程序限制。
- 被害人期待快速结果，现实侦查需要时间。
- 不立案或暂不立案可以形成申诉、复议、补证和舆论压力。

检查问题：这个报案为什么进入刑事、行政、民事或求助路径？角色是否知道路径不同带来的后果？

## 7. 现场处置与证据保护：先稳住现场，再谈真相

警务现场第一任务通常不是马上破案，而是控制危险、保护人员、固定现场和初步了解情况。

### 7.1 现场动作

| 动作 | 叙事功能 |
| --- | --- |
| 隔离危险 | 防止继续伤害、爆炸、坍塌、围观冲突 |
| 救助伤者 | 与医疗、消防、社区联动 |
| 控制人员 | 防止嫌疑人逃离、双方继续冲突、证人散失 |
| 保护现场 | 避免证据污染、物品丢失、围观拍摄 |
| 初步询问 | 确认身份、时间线、关键事实和风险 |
| 记录固定 | 照片、视频、笔录、扣押、登记、封存 |
| 通知家属/单位 | 未成年人、伤者、死亡、拘留等情境 |

### 7.2 主角能做什么

主角可以：

- 报警并提供准确地点。
- 保护自己和他人离开危险区域。
- 记住时间、人物、车牌、衣着、声音等观察。
- 保留原始聊天记录、转账记录、视频链接和票据。
- 告知警方现场还有谁、哪里有监控、谁可能受伤。
- 不随便触碰、移动、删除、转发关键证据。

主角不应无代价：

- 私自进入封控现场。
- 拿走物证。
- 逼供嫌疑人。
- 偷拍隐私并公开。
- 黑入系统取证。
- 威胁证人改口。

检查问题：现场证据是否因人物行为变得更可信，还是被作者方便地污染后仍被接受？

## 8. 询问、讯问、传唤与笔录：话语进入程序后会变重

警务对话不是普通聊天。进入笔录后，话语会影响后续案件。

### 8.1 角色身份不同，程序感不同

| 身份 | 常见互动 | 戏剧点 |
| --- | --- | --- |
| 报案人 | 陈述事件、提交材料、补充线索 | 期待与现实节奏冲突 |
| 被害人 | 伤情、损失、隐私、告知、保护 | 情绪、二次伤害、证据难度 |
| 证人 | 看到什么、听到什么、记忆限制、害怕报复 | 证言摇摆、利益冲突 |
| 嫌疑人 | 被讯问、权利告知、供述和辩解 | 沉默、狡辩、翻供、律师介入 |
| 未成年人 | 监护、合适成年人、隐私保护、学校家庭介入 | 安全边界高 |
| 家属 | 寻人、通知、探视、焦虑、舆论压力 | 信息差和情绪线 |

### 8.2 笔录的戏剧功能

- 前后说法不一致，形成疑点。
- 证人记错时间，主角通过物证校正。
- 报案人夸大事实，导致后续被反噬。
- 嫌疑人拒不承认，但电子数据和资金流水形成外部证据。
- 民警反复确认细节，让角色意识到自己忽略了关键点。
- 笔录签字前，角色发现记录不完整或表述有歧义。

检查问题：这场警务对话是否改变证据状态、人物身份或案件路径，而不是普通对话换个场地？

## 9. 强制措施：抓人不是警察随口决定

小说可以写拘留、逮捕、取保候审和监视居住，但必须把它们当作有条件、有后果的程序动作。

### 9.1 常见措施的叙事功能

| 措施 | 小说作用 | 注意 |
| --- | --- | --- |
| 传唤/拘传 | 让相关人员到案接受调查 | 不是定罪，不能等同坐牢 |
| 刑事拘留 | 对现行犯或重大嫌疑人等采取临时羁押措施 | 会改变家属、舆论、工作和后续辩护 |
| 逮捕 | 更强的羁押措施，通常涉及检察机关批准或决定 | 不能写成派出所一句话 |
| 取保候审 | 不羁押但限制和保证并存 | 不是“没事了”，也不是判轻了 |
| 监视居住 | 特定条件下限制居住和活动 | 会改变行动自由和剧情空间 |
| 行政拘留 | 治安行政处罚之一 | 与刑事拘留不同，不能混写 |
| 扣押/查封/冻结 | 控制物品、财产、账户或数据 | 要写清对象、清单、保管和争议 |

### 9.2 强制措施产生的剧情后果

- 家属收到通知后反应不同：找律师、找关系、恐慌、隐瞒。
- 公司停职或解除合作。
- 学校、社区、粉丝群出现舆论判断。
- 嫌疑人无法自由行动，旧计划被迫中断。
- 被害人既松一口气，又担心报复或证据不足。
- 主角发现“被抓”不等于“真相已定”。

检查问题：这项措施的条件、执行者、通知、限制和后续影响是否被写清？

## 10. 搜查、扣押、查封、冻结：权力越大，程序越重要

这些措施很适合制造爽点，但最容易写成警察万能或主角作弊。

### 10.1 正确写法

可写：

- 警方依法对相关场所、物品、账户或电子设备采取措施。
- 扣押物品有清单、封存、保管和后续鉴定。
- 冻结账户导致公司或家庭现金流骤停。
- 查封场所改变角色居住、经营和舆论状态。
- 搜查发现的不只是“铁证”，也可能是误导、无关隐私或新的疑点。

不可写：

- 主角随便要求警察搜别人家。
- 警察随口查所有人的隐私、账户、通信和定位。
- 非专业角色私自破解手机、复制数据、删除文件。
- 通过违法取证得到关键证据后被程序无条件认可。

### 10.2 物品和数据的后果

- 手机被扣押，角色失去通信和工作工具。
- 电脑被封存，公司项目停摆。
- 账户被冻结，工资、房租、货款和医疗费受影响。
- 车辆被扣，家庭通勤和逃亡计划改变。
- 证件被留置或丢失，身份线联动第 161 篇。

检查问题：被搜查、扣押、查封、冻结的对象是否改变了人物行动资格和生活状态？

## 11. 刑事、行政、民事交叉：很多现实冲突不是单线

好看的现代案件常是多路径交叉。

### 11.1 常见交叉

| 表面冲突 | 可能路径 |
| --- | --- |
| 打架 | 治安处罚、刑事伤害、民事赔偿、校园处分 |
| 欠钱不还 | 民事诉讼、诈骗嫌疑、非法集资、家庭伦理 |
| 家暴 | 公安处置、告诫书、人身安全保护令、离婚和抚养 |
| 网络曝光 | 治安/刑事、名誉侵权、平台投诉、隐私保护 |
| 交通事故 | 交警认定、保险理赔、刑事责任、民事赔偿 |
| 公司泄密 | 劳动纪律、商业秘密犯罪、民事赔偿、内部审计 |
| 校园欺凌 | 学校处分、公安介入、未成年人保护、民事责任 |
| 诈骗报案 | 刑事侦查、资金止付、平台申诉、民事追偿 |

### 11.2 写法

不要让一条路径包办一切。可以写：

- 民警处理现场，律师处理民事赔偿，学校处理纪律，平台处理内容。
- 刑事案件推进慢，但民事生活压力立刻到来。
- 行政处罚不等于赔偿完成。
- 民事和刑事证据互相影响，但使用边界不同。

检查问题：这个冲突到底需要公安、法院、学校、公司、平台、社区、医院中的哪些系统共同处理？

## 12. 特殊警情：家暴、未成年人、失踪走失、诈骗和精神危机

### 12.1 家庭暴力

家暴不是普通家务事。写作时要联动第 105 篇。

可写：

- 报警记录、出警记录、伤情、告诫、庇护、保护令、亲友支持和离开计划。
- 受害者犹豫、撤回、害怕经济报复或孩子受影响。
- 民警、社区、妇联、法院、学校等系统协作。

不可写：

- 劝和万能。
- 把暴力浪漫化。
- 让受害者为“不报警”被旁白羞辱。

### 12.2 未成年人

未成年人警情要提高隐私和保护阈值。

可写：

- 通知监护人或合适成年人。
- 学校、家长、公安、未保机构协同。
- 隐私保护、心理支持和避免二次传播。

不可写：

- 公开可识别信息。
- 让未成年人被审问式羞辱。
- 把校园暴力当恋爱误会。

### 12.3 失踪和走失

不要写“必须等 24 小时才能报警”这类未经核实的通用桥段。失踪、走失、老人儿童失联、自伤风险等都应按情境紧急程度处理，并查当地官方规则。

可写：

- 家属提供照片、衣着、最后位置、联系方式、病史、监控点和可能去向。
- 警方、社区、学校、物业、交通、平台和志愿者协作。
- 假线索、重复报警、网络寻人隐私风险和家属崩溃。

### 12.4 网络诈骗

诈骗报案要联动第 163、165 篇。

重点不是警察“马上抓到骗子”，而是：

- 保留聊天、转账、链接、账号、电话、平台订单。
- 尽快止付冻结和平台投诉。
- 受害者羞耻、家属责怪、二次诈骗风险。
- 资金跨地区、跨平台，侦查有协作和时间成本。

### 12.5 精神危机和自伤风险

涉及自伤、自杀、精神危机、极端羞辱时，必须联动第 105 篇，不写可模仿细节，不用警情当反转刺激。

检查问题：特殊警情是否被当成普通桥段处理，从而伤害真实读者或破坏可信度？

## 13. 警方信息公开与舆论边界

案件进入公安程序后，信息公开受到边界限制。

### 13.1 可写冲突

- 家属想知道更多，警方只能告知有限进展。
- 媒体和自媒体抢发未经证实细节。
- 公开视频片段误导公众，完整材料尚未公布。
- 官方通报措辞克制，网友觉得“没说清”。
- 警方不公开未成年人、被害人隐私和侦查细节。
- 主角想用舆论逼程序加速，却可能污染证据或伤害受害者。

### 13.2 信息公开四问

1. 谁有权公开：公安机关、检察机关、法院、当事人、媒体、平台。
2. 公开什么：事实、警情、进展、处罚、辟谣、寻人信息。
3. 不能公开什么：未成年人隐私、被害人隐私、侦查秘密、个人信息、未核实指控。
4. 公开后果：安抚公众、引发网暴、逼迫证人、刺激模仿、影响审判。

检查问题：案件信息公开是否服务真相和安全，而不是单纯让主角爽？

## 14. 警察角色写法：专业但不万能，有边界也有人味

### 14.1 警察角色六维

| 维度 | 要写什么 |
| --- | --- |
| 职责 | 接警、巡逻、刑侦、治安、经侦、网安、禁毒、交警、社区警务等不同岗位 |
| 权限 | 能做什么，不能做什么，什么时候要审批、协作或移送 |
| 专业过滤 | 看现场、时间线、证据来源、人物关系、风险等级 |
| 限制 | 证据不足、辖区管辖、工作量、程序时限、舆论压力、资源短缺 |
| 人性 | 疲劳、共情、挫败、谨慎、职业习惯、家庭生活 |
| 边界 | 不私刑、不剧透侦查、不随便泄露信息、不替法院定罪 |

### 14.2 常见岗位差异

| 岗位 | 叙事功能 |
| --- | --- |
| 接警员 | 信息入口、判断紧急程度、安抚报警人 |
| 派出所民警 | 现场处置、纠纷调解、治安案件、社区信息 |
| 刑警 | 刑事案件侦查、证据链、嫌疑人画像和追踪 |
| 经侦 | 经济犯罪、合同诈骗、资金流、公司账目 |
| 网安 | 网络犯罪、电子数据、平台协作、账号线索 |
| 交警 | 交通事故、道路秩序、酒驾、逃逸线索 |
| 法医/鉴定人员 | 专业鉴定，不等于万能答案机 |
| 社区民警 | 长期关系、人口信息、邻里线索、基层矛盾 |

检查问题：这位警察是不是有岗位职责和限制，而不是“知道一切的正义NPC”？

## 15. 反派和嫌疑人的程序压力

嫌疑人不是被抓就结束。程序会改变他能做什么。

可写压力：

- 被传唤后不敢联系同伙。
- 手机电脑被扣押，无法继续操控舆论。
- 账户被冻结，资金链断裂。
- 家属通知后，家庭和公司开始切割。
- 取保候审限制行动，计划被迫延后。
- 证人保护让威胁失效。
- 立案后同伙开始互相甩锅。
- 不起诉、撤案或证据不足释放后，受害者情绪更复杂。

检查问题：程序动作是否改变反派资源、行动、名声和同盟关系？

## 16. 类型文用法

### 16.1 悬疑刑侦

重点：案件程序、现场保护、询问讯问、证据链、移送审查。

好用冲突：证据不足但主角直觉强；警方方向与私家侦探方向冲突；嫌疑人被释放后再犯风险；被害人家属不理解侦查节奏。

### 16.2 都市家庭伦理

重点：家暴、失踪、借贷诈骗、老人被骗、邻里冲突、财产纠纷。

好用冲突：公安只能处理违法犯罪和治安问题，民事争议还要走法院、社区、律师和家庭协商。

### 16.3 校园青春

重点：未成年人保护、校园欺凌、失联、偷拍传播、网络谣言。

好用冲突：学校想压事，家长想曝光，公安要保护未成年人隐私和证据。

### 16.4 商战职场

重点：商业秘密、职务侵占、合同诈骗、数据泄露、财务舞弊。

好用冲突：内部调查材料不等于刑事证据，报警后公司也要承担合规和声誉后果。

### 16.5 娱乐圈公关

重点：造谣诽谤、偷拍视频、私生跟踪、账号冒用、粉丝集资诈骗。

好用冲突：报警记录不能立刻平息舆论，警方通报也不能替艺人完成情绪沟通。

### 16.6 古言权谋

等价系统：报官、递状、捕快查访、县衙受理、仵作验看、差役拘人、公堂审问、上级覆核。

重点：时代程序、身份等级、证人可信度、官府腐败、文书和见证。

### 16.7 玄幻仙侠

等价系统：宗门执法堂、戒律院、城防司、仙盟审查、天道誓约、留影石、魂灯、命牌、执法长老。

重点：超凡证据也要有权限、伪造可能、争议程序和执行代价。

## 17. 常见坏味道与修复

| 坏味道 | 读者体感 | 修复方式 |
| --- | --- | --- |
| 报警秒抓人 | 程序太假 | 加接警、现场处置、证据固定、受案审查和侦查阻力 |
| 派出所万能 | 像NPC盖章 | 区分派出所、刑警、经侦、网安、交警、检察院和法院 |
| 主角私查全案 | 爽但违法 | 改为提供线索、合法保存材料、配合调查或承担越权代价 |
| 立案等于定罪 | 程序混乱 | 写明立案只是侦查开始，最终还需证据和后续程序 |
| 证据随便拿 | 作者作弊 | 写来源、固定、清单、鉴定、质疑和程序边界 |
| 想抓就抓 | 警察万能 | 写传唤、拘留、逮捕、取保等措施的条件和后果 |
| 家暴被劝和 | 冒犯受害者 | 联动创伤和反家暴规则，写保护、告诫、证据和离开成本 |
| 未成年人信息乱曝 | 平台和伦理风险 | 写隐私保护、监护/合适成年人、学校和专业支持 |
| 官方通报万能 | 舆论廉价 | 写信息公开边界、未公开细节、谣言和公众误读 |
| 规避侦查教程化 | 安全风险 | 只写概念、后果和程序，不写逃避、毁证、串供步骤 |

## 18. 审稿检查清单

审稿时逐条检查：

1. 报警或报案是否写清地点、事件、人员、风险和证据。
2. 是否区分紧急警情、求助、治安行政案件、刑事案件、民事纠纷和事故警情。
3. 是否写出接警、出警、现场控制、受案登记和下一步安排。
4. 是否区分派出所、刑警、经侦、网安、交警、检察院、法院等职责。
5. 主角是否越权取证、污染现场或替代警务系统；若越权，是否有代价。
6. 证据是否有来源、固定、清单、保全、鉴定或被质疑的空间。
7. 询问、讯问、笔录、传唤、拘留、逮捕、取保候审、行政拘留是否混用。
8. 强制措施是否有条件、执行者、通知、限制和后续影响。
9. 被害人、证人、嫌疑人、家属和律师是否有不同信息权限和情绪状态。
10. 家暴、未成年人、失踪、诈骗、精神危机等特殊警情是否联动安全专题。
11. 警方信息公开是否有边界，不泄露隐私和侦查秘密。
12. 警察角色是否有岗位差异、专业判断和制度限制。
13. 案件是否有移送、处罚、调解、不立案、申诉或后续救济路径。
14. 是否避免逃避侦查、毁灭证据、虚假报案、作伪证、威胁证人等可模仿步骤。
15. 警务事件是否改变人物名声、工作、学校、家庭、安全、财产和后续行动。
16. 是否把报警记录、案件状态、强制措施、证据状态和权利告知写入台账。

## 21. 外部资料吸收点

本专题吸收外部资料时，只转化为小说程序可信度和安全边界。

- 中国《中华人民共和国刑事诉讼法》：刑事案件侦查、强制措施、证据、辩护、审查起诉和审判有明确程序；迁移为“立案不是定罪，侦查不是任意行动，强制措施要有边界”。
- 公安部《公安机关办理刑事案件程序规定》：公安机关办理刑事案件涉及受案、立案、回避、律师参与、证据、强制措施、侦查取证和移送审查起诉等；迁移为“刑事案件要有受案、立案、侦查和移送链”。
- 公安部《公安机关办理行政案件程序规定》：治安等行政案件有受案、调查取证、询问、传唤、证据、听证和处罚程序；迁移为“治安案件不是刑案，但也不是随便调解”。
- 中国《治安管理处罚法》：违反治安管理行为、处罚种类和程序边界；迁移为“打架、扰序、轻微违法等不必都升级成刑案，但要有行政后果”。
- 中国《人民警察法》：人民警察的任务、职权、义务和纪律；迁移为“警察角色有职权也有纪律，不能写成无边界正义机器”。
- 公安部《110 接处警工作规则》：110 接处警负责受理报警、紧急求助和投诉，并进行指挥调度；迁移为“报警不是万能许愿，而是信息入口和分流机制”。
- 中国《反家庭暴力法》：家庭暴力处置涉及公安机关制止、调查取证、协助就医鉴定、告诫书和保护令等机制；迁移为“家暴不是家务事，不能只写劝和”。
- 中国《未成年人保护法》：涉及未成年人案件和信息时有特殊保护要求；迁移为“校园、未成年人警情要保护隐私、监护和心理安全”。
- UNODC Crime Scene and Physical Evidence Awareness：犯罪现场和物证对非专业人员也有保护意识要求；迁移为“现场不能被主角乱翻后仍当铁证”。
- UNODC Police Accountability, Oversight and Integrity：警务权力需要问责、监督和廉洁边界；迁移为“警察既不能万能，也不能黑箱无代价”。

## 22. 最小执行规则

如果 AI agent 只能记住五条：

1. 报警之后不是直接抓人，而是接警分流、现场处置、受案/立案审查、证据固定和后续程序。
2. 必须区分刑事案件、治安行政案件、民事纠纷、求助警情和事故警情。
3. 警察有岗位、管辖、权限和程序限制；主角不能无代价替代警务系统或污染证据。
4. 询问、讯问、传唤、拘留、逮捕、取保、搜查、扣押、冻结等措施必须有条件、通知、限制和后果。
5. 只写程序可信度、风险和后果，不写逃避侦查、毁灭证据、虚假报案、作伪证、威胁证人或规避法律责任的可模仿步骤。
