## 1. 本专题解决的 AI 写作问题

AI 写税务和发票常见失真：

1. 开店只写营业执照，不写税务信息确认、申报、开票、账簿和社保费。
2. 把发票写成“报销小票”，忽略它是交易凭证、税务凭证、合同履约和审计证据。
3. 把增值税写成老板随口加的费用，忽略一般纳税人、小规模纳税人、进销项、价税分离和现金流压力。
4. 把“不要发票便宜点”写成聪明省钱，忽略无票成本、维权困难、税务风险和虚开发票风险。
5. 把税务稽查写成突击抓人，忽略案源、选案、检查、取证、审理、决定、送达、执行和救济。
6. 把补税写成认怂，忽略税款、滞纳金、罚款、纳税信用、失信公布、刑事移送和经营资格后果。
7. 把个体户写成“没有公司所以不用交税”，忽略登记、申报、核算、发票和经营数据。
8. 把员工社保费写成老板愿不愿意交，忽略缴费基数、单位责任、税务征收、人社医保权益和劳动争议。
9. 把逃税、虚开发票、刷流水、拆单、两套账、买票抵扣写成商业头脑，产生危险教程化。

本篇把税务桥段写成“经营事实 -> 票据账簿 -> 税费申报 -> 风险识别 -> 稽查处理 -> 信用和现金流后果”的制度链。

## 2. 总原则：经营收入不是到手利润，发票不是装饰道具

县域税务的戏剧力量不在算税率，而在经营者第一次意识到：钱进了账户、货出了仓、发票开出去、员工发了工资、工程验了收，都会留下制度痕迹。

标准经营税费链：

```text
开店、接单、直播、承包工程或提供服务
-> 确认经营主体：个体工商户、公司、合作社、个人劳务、平台主体
-> 税务信息确认、纳税人身份、申报期限和税费种认定
-> 建立账簿、收付款记录、合同、订单、库存、工资和成本资料
-> 发生交易：报价、合同、收款、交付、验收、开票或不开票
-> 纳税申报：增值税、所得税、附加税费、代扣代缴、社保费等
-> 税务风险：发票异常、收入和申报不匹配、成本异常、平台数据、银行流水、投诉举报、审计发现
-> 税务提示、约谈、纳税辅导、检查或稽查
-> 补税、滞纳金、处罚、信用调整、失信公布、行政救济或刑事移送
-> 现金流、融资、招投标、平台合作、家庭关系和公司声誉回写
```

AI agent 写税务桥段时，先问：

- 这是经营收入、工资薪金、劳务报酬、稿酬、直播佣金、股权分红、房屋租售、工程款，还是赔偿款？
- 主体是个人、个体工商户、公司、合作社、平台、MCN，还是挂靠/承包关系？
- 有没有合同、订单、发票、收据、流水、账簿、工资表、验收单和申报记录？
- 争议是不开票、虚开发票、少申报收入、成本不实、社保费欠缴、税务稽查，还是纳税信用后果？

## 3. 法律与制度底座

### 3.1 税收征收管理法：申报、账簿、检查和法律责任的主干

税收征收管理法为税收征收、缴纳、账簿凭证、纳税申报、税务检查、税收保全、强制执行和法律责任提供基础。它的写作价值是：税务不是“老板良心问题”，而是纳税人、扣缴义务人和税务机关之间有程序、有材料、有责任的制度关系。

写作迁移：

- 纳税人要按规定办理申报、设置账簿、保管凭证、如实提供资料。
- 税务机关不能靠传闻直接罚，检查、处理、处罚、强制和救济都要有程序。
- 逃避缴纳税款、拒绝检查、隐匿账簿、虚假申报、欠税不缴会产生不同后果。

### 3.2 增值税法：交易会产生价税、发票和现金流压力

《中华人民共和国增值税法》已于 2024 年 12 月 25 日通过，自 2026 年 1 月 1 日起施行。它将货物、服务、无形资产、不动产等应税交易纳入增值税制度框架。

写作迁移：

- 店铺卖货、企业服务、工程施工、平台交易、房产租售都可能涉及增值税和发票。
- “含税价”和“不含税价”会影响报价、利润和合同争议。
- 一般纳税人与小规模纳税人的开票、抵扣、申报、客户要求不同，能制造经营压力。
- 不要在小说里提供现实税率测算和避税方案，只写价税分离、票据、申报和现金流后果。

### 3.3 发票管理办法和实施细则：发票是证据，也是风险

发票管理办法和实施细则规范发票印制、领用、开具、取得、保管、检查和数据安全。发票能证明交易，也能成为虚假交易、虚开发票、报销舞弊和税务风险的入口。

写作迁移：

- 发票不是“想开就开”，要与真实交易、合同、收付款、交付和主体相匹配。
- 购买方索要发票、销售方拒开发票、抬头错误、金额错误、作废红冲、电子发票重复报销，都能成为现实冲突。
- 数电票和发票数据让税务、审计、财务、平台和企业内控更容易交叉验证。
- 虚开发票和买票抵扣只写风险、后果和证据痕迹，不写操作路径。

### 3.4 个人所得税、企业所得税和个体户：身份决定税务路径

个人取得工资薪金、劳务报酬、稿酬、特许权使用费、经营所得、利息股息红利等，可能进入个人所得税链。企业经营利润进入企业所得税链。个体工商户作为重要市场主体，既有发展扶持，也要依法经营、记账、申报、开票和接受监管。

写作迁移：

- 主播、网文作者、设计师、装修工、临时讲师、外包程序员的收入类型不同，不能一律写成工资。
- 公司盈利、老板分红、员工工资、个人劳务和个体户经营所得要区分。
- 个体户不是“没账没税”，小店也有进货、销售、租金、人工、税费和发票。
- 税收优惠和减免只能作为剧情背景，不提供现实筹划建议。

### 3.5 社保费征收：工资成本不是只发到手工资

社会保险费征缴制度说明，单位和个人缴费、登记、申报、征缴、滞纳和法律责任都有规则。实践中很多地区税务机关承担社会保险费征收职责，税务、人社、医保、社保经办机构之间有信息衔接。

写作迁移：

- 小老板说“工资给高点但不交社保”，不是单纯好心，可能埋下工伤、养老、医疗、劳动争议和社保费追缴。
- 员工到手工资、单位用工成本、社保缴费基数和个税申报会互相影响。
- 欠缴社保费、少报工资基数、把员工写成劳务或外包，都可能连到劳动仲裁、人社、税务和法院。

### 3.6 税务稽查和处罚：不是举报秒罚，而是证据和程序

税务稽查案件办理程序规定强调案源、选案、检查、审理、执行等程序，并要求全过程记录。税务行政处罚裁量权行使规则强调合法、合理、公正、公开、程序正当、处罚和教育相结合等原则。

写作迁移：

- 税务稽查不是主角一举报就冲进公司搬电脑，而是要有案源、立案、检查通知、调查取证、电子数据固定、审理、决定、送达和执行。
- 纳税人有陈述申辩、听证、行政复议、行政诉讼等权利。
- 首违不罚、责令限期改正、从轻减轻、重大案件审理等可成为剧情里的“不是所有错误都同一后果”。

### 3.7 税收违法检举、纳税信用和重大税收违法失信

税收违法行为检举有接收、受理、处理和保密管理要求。重大税收违法失信主体信息公布管理办法用于约束严重涉税违法失信行为，并与纳税信用、联合惩戒和信用修复相关。

写作迁移：

- 税务举报不是网上发帖审判，要走检举渠道、材料、受理和保密。
- 严重税收违法可能影响融资、招投标、政府采购、平台合作、企业法定代表人声誉和家庭关系。
- 缴清税款、滞纳金、罚款和符合条件后的信用修复，可以让“犯错企业如何回到市场”形成长线剧情。

| 角色 | 叙事功能 | 常见压力 | 写作抓手 |
| --- | --- | --- | --- |
| 个体户老板 | 开店、收款、开票、申报 | 不懂规则、现金流紧、熟人赊账 | 从“会做生意”到“会留下合规痕迹” |
| 小微企业财务 | 记账、报税、开票、工资社保 | 老板压成本、票据不齐、系统期限 | 她知道哪张发票会炸雷 |

| 税务稽查人员 | 调查取证、审理、处理处罚 | 证据复杂、电子数据、企业对抗 | 通过票、账、货、款还原真实交易 |
| 代账公司/涉税服务人员 | 帮助申报、记账和合规 | 客户不配合、灰色要求、职业风险 | 便宜代账可能把风险藏到下一年 |
| 直播主播/MCN | 收入结算、佣金、个税和企业税 | 流水多、主体乱、分成复杂 | 热度越高，税务痕迹越多 |
| 工程承包商 | 合同、验收、发票、成本票 | 垫资、挂靠、人工费、开票压力 | 工程款和税票卡住工资链 |
| 员工/农民工 | 工资、个税、社保费、欠薪 | 到手钱和长期权益冲突 | 不交社保省下的是未来保障 |
| 审计/纪检人员 | 发票、合同、资金、验收交叉验证 | 账实不符、虚假票据 | 发票日期能刺破项目假象 |
| 银行/平台 | 资金流水、风控和数据接口 | 异常交易、税务证明、贷款材料 | 税务信用影响融资和合作 |

## 5. 常见税务桥段分流表

| 场景 | 先判断 | 关键证据 | 可能路径 | 戏剧张力 |
| --- | --- | --- | --- | --- |
| 小店开业 | 个体户还是公司 | 营业执照、税务信息、收款码、进货票 | 办税服务、申报、开票 | 开门做生意后才发现账也要开门 |

## 6. 程序节点写法

### 6.1 开店后：营业执照不是经营合规终点

低质写法：

> 主角办完营业执照，当天开店赚钱，从此只管进货卖货。

高质写法：

```text
主角返乡开奶茶店
-> 政务服务中心办营业执照和食品经营相关手续
-> 税务信息确认后进入电子税务局
-> 收款码、进货票、租金、水电、人工和发票需求开始累积
-> 第一个公司客户要求开具发票
-> 主角才意识到，卖一杯奶茶不只是收钱，还要留下交易、成本和税费痕迹
```

### 6.2 发票争议：一张票连接报销、付款、税务和信任

低质写法：

> 对方不给发票，主角骂一顿就解决。

高质写法：

```text
民宿接待研学团队
-> 合同写明含税价和发票要求
-> 团队离店后财务要求开票才能付款尾款
-> 民宿老板发现当初报价没算税费和平台佣金
-> 代账提醒不能找别家代开发票
-> 尾款、利润、客户关系和税务风险同时压上来
```

### 6.3 增值税和现金流：利润不是账户余额

```text
小工厂接到大客户订单
-> 客户要求专票和账期
-> 原材料供应商不开合规票或票晚到
-> 工厂先垫货款、发工资、缴社保，再等客户回款
-> 财务提醒含税报价、进项抵扣和申报期限会影响现金流
-> 老板第一次发现销售额增长也可能把企业拖垮
```

重点：不必教读者算税，只要写出“票、款、货、账期”互相锁住。

### 6.4 税务风险提示：不是所有接触税务局都等于出事

```text
直播间销量突然翻倍
-> 平台结算和公户流水明显增加
-> 税务系统出现风险提示
-> 办税人员联系企业补充说明收入类型、开票情况和申报口径
-> 主播团队以为被针对，财务却知道问题在合同主体混乱
-> 整理后台、分成协议、退款记录和发票后，部分风险可解释，部分需要补申报
```

写作重点：风险提示、纳税辅导、约谈、检查和稽查是不同层级，别一接电话就写成大案。

### 6.5 税务稽查：票、账、货、款四条线互相咬合

```text
县城建材公司被举报虚开发票
-> 税务稽查立案检查
-> 调取合同、发票、银行流水、出库单、运输记录、仓库库存和聊天记录
-> 账面显示卖出大量钢材，仓库和物流却没有对应痕迹
-> 法定代表人说是财务操作失误
-> 稽查人员通过电子数据、资金回流和关联公司还原交易链
-> 后果可能是补税、滞纳金、罚款、失信公布或涉嫌犯罪移送
```

安全边界：只能写调查逻辑和证据类型，不能写如何虚开、刷流水、买票抵扣。

### 6.6 社保费：便宜用工会在受伤或离职时反噬

```text
餐馆老板和服务员约定“多发五百，不交社保”
-> 服务员平时觉得到手钱多
-> 一次厨房摔伤后医疗费用和工伤保障成争议

-> 劳动仲裁、人社、税务和医保链同时出现
-> 老板才明白省下的不是成本，是未来纠纷
```

## 8. 高质量桥段示例

### 8.1 返乡民宿老板第一次被发票卡住

```text
返乡青年把老宅改民宿
-> 第一批游客是公司团建
-> 对方财务要求发票和对公付款
-> 主角原本按散客价格报价，没有算平台佣金、税费、保洁人工和早餐成本
-> 代账提醒合同主体、开票内容和实际服务要一致
-> 主角为了不丢客户，只能重新拆报价和成本
-> 爽点不是逃税，而是他学会真正经营
```

### 8.2 直播带货税务风险提示

```text
县城女孩靠短视频卖农产品爆红
-> 平台打款进入个人账户和亲戚个体户账户
-> MCN 抽佣、退货退款、赠品和坑位费混在一起
-> 税务风险提示要求说明收入和申报情况
-> 家人觉得“赚点辛苦钱也要管”
-> 财务朋友帮她整理平台后台、合同、分成和成本
-> 她第一次意识到流量越大，账越要清楚
```

### 8.3 工程发票和农民工工资连锁

```text
乡镇道路维修项目完工
-> 承包商等财政尾款
-> 付款条件要求验收资料和发票齐全
-> 发票因主体和合同金额问题被退回
-> 工人工资被拖，村里开始上访
-> 财政、审计、税务、欠薪治理和项目验收同时出现
-> 主角拆清工程款、人工费、税票和财政支付边界
```

### 8.4 税务稽查不是舆论打脸

```text
竞争对手举报县城建材店偷税
-> 网上先起舆论
-> 税务机关按程序接收检举并研判
-> 稽查人员调取发票、合同、流水、库存和物流
-> 发现部分举报属实，部分是竞争对手夸大
-> 处理决定既让企业补税受罚，也让恶意造谣者失去舆论优势
```

### 8.5 老板不缴社保的现实代价

```text
小餐馆员工摔伤
-> 老板拿出微信聊天称双方说好不交社保
-> 员工家属拿出排班、工资、监控和工作服证明劳动关系
-> 劳动仲裁、人社、税务征收和工伤认定链条被触发
-> 老板发现“省社保”不是省钱，而是把风险押到事故那天
```

## 10. 安全边界

允许写：

- 开店、申报、开票、补税、纳税辅导、税务检查、稽查、处罚和信用后果的概念性程序。
- 企业因不懂规则、现金流紧、票据不齐、社保费少缴而产生的现实冲突。
- 税务违法被发现、调查、补税、处罚、信用受损和司法移送的后果。

不要写成教程：

- 如何逃税、避税筹划、拆分收入、隐匿收入、刷流水、两套账或虚构成本。
- 如何虚开发票、买卖发票、找票抵扣、红冲洗票或伪造发票数据。
- 如何规避税务稽查、删除账册、转移电子数据、串供、抗拒检查。
- 如何少缴社保费、伪装劳务外包、虚假工资、骗取税收优惠或补贴。
- 如何通过空壳公司、挂靠主体、关联交易转移利润。

反派可以这么做，但文本必须呈现风险、证据痕迹和后果，不能提供可复制步骤，也不能把涉税违法写成值得模仿的商业智慧。

## 11. 审稿检查问题

1. 经营主体是否明确：个人、个体户、公司、合作社、平台店铺、主播还是工程承包商？
2. 收入类型是否明确：销售、服务、工程、租金、佣金、工资、劳务、稿酬、分红还是补贴？
3. 是否写出合同、订单、发票、流水、账簿、工资表、库存、验收或平台后台中的至少两类证据？
4. 发票是否与真实交易、主体、金额、内容和付款相匹配？
5. 是否把不开票便宜、虚开发票、买票抵扣、两套账写成聪明爽点？
6. 增值税是否影响报价、合同、进销项、账期或现金流，而不是只有一个税率数字？
7. 个体户是否被误写成不用申报、不用记账、不用开票？
8. 税务风险提示、检查和稽查是否区分层级？
9. 稽查是否有案源、检查、取证、审理、决定、送达、执行和救济，而不是举报秒罚？
10. 社保费是否和工资、劳动关系、工伤、养老医保权益相连？
11. 纳税信用、重大税收违法失信、融资、招投标和平台合作后果是否回写？
12. 是否避免现实税务筹划、逃税、虚开发票、骗税和规避稽查教程化？
