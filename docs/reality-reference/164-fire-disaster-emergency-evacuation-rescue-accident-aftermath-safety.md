# 164. 火灾消防、灾害事故、应急疏散、救援响应与灾后余波专项

适用范围：当 AI agent 需要设计、起草、审稿或修订火灾、烟雾、燃气泄漏、爆炸事故、工厂事故、商场/学校/医院/酒店事故、高层住宅事故、地震、洪水、台风、暴雨内涝、山体滑坡、泥石流、停电断水、群体疏散、消防救援、避难安置、灾后恢复、事故调查、责任追究、灾难救援大场面和任何“突发事故改变人物命运、资源和社会秩序”的剧情时，使用本文件。

核心目标：让灾害事故不再是“爆炸按钮、主角冲进去、所有人哭喊、救完就翻篇”，而是形成可追踪的叙事链：

`隐患 -> 触发 -> 预警 -> 现场判断 -> 报警/通报 -> 疏散/避险 -> 专业响应 -> 次生风险 -> 救援分工 -> 安置救助 -> 事故调查 -> 责任与恢复 -> 长篇台账`

一句话规则：灾害不是单个大场面，而是环境、制度、空间、身体、信息、组织和余波同时失控后的连锁压力系统。

---

## 1. 本专题解决什么问题

AI 写灾害事故常见坏味道：

1. 爆炸按钮化：需要剧情转折就突然爆炸、起火、塌楼，前文没有隐患和预警。
2. 英雄单刷：主角一个人替代报警、疏散、消防、医疗、安置和调查。
3. 烟火装饰化：只写火光冲天，不写烟、热、噪音、断电、视线受限和空间阻断。
4. 疏散缺席：所有人只尖叫，不走安全出口，不点名，不照顾老人儿童伤者。
5. 专业救援降智：消防、应急、医疗、公安、物业、学校、企业只在结尾出现或完全消失。
6. 次生灾害缺席：火后没有坍塌、断电、停水、交通封控、二次泄漏、谣言和心理创伤。
7. 事故无责任：隐患没人负责，调查没人问，赔偿和整改没人提。
8. 受害者工具化：死伤只用于刺激主角成长，不写家属、安置、医疗、生活中断和长期影响。
9. 教程化危险：为了真实，细写纵火、爆炸、燃气排险、破拆、危险化学品、违规操作或逃避监管方法。
10. 灾后清零：下一章恢复日常，伤亡、财产、舆论、制度和关系都不进台账。

本专题把事故写成“风险链 + 组织链 + 人物选择链 + 余波链”。

## 3. 安全边界：只写叙事可信度，不写危险操作教程

灾害事故题材容易带出现实危险。AI agent 必须遵守：

1. 不输出纵火、爆炸、燃气破坏、危险化学品混合、电路改造、破拆、规避消防检查、逃避事故责任的操作步骤。
2. 不把燃气泄漏、火灾、爆炸、危险品、坍塌、群体踩踏写成可模仿教程。
3. 不鼓励非专业人员进入高危现场救人、破门、排险、关闭复杂设备或与危险源对抗。
4. 涉及真实救援、急救、用药、烧伤、窒息、中毒和伤情判断时，联动第 131 篇，不写医疗操作细节。
5. 涉及真实事故改编时，联动第 105 篇和第 111 篇，避免消费受害者、传播未证实细节和二次伤害家属。
6. 可以写概念性安全动作：预警、报警、撤离、封控、等待专业救援、配合调查、保留证据、安置救助、心理支持。
7. 架空世界也必须有边界：法术灭火、系统救援、机甲救援和神力护城都要有资源消耗、信息限制、误判风险和余波。

可写的是“角色怎样在危机中判断、选择、协作、承担后果”。

不可写的是“读者怎样现实复刻危险行为”。

## 4. 灾害事故十线卡

设计事故时，至少检查十条线。

| 线 | 要问什么 | 叙事功能 | 常见坏味道 |
| --- | --- | --- | --- |
| 隐患线 | 事故前已有何风险 | 铺垫公平性、责任追究 | 爆炸空降 |
| 触发线 | 什么事件让隐患失控 | 第一转折、误判、倒计时 | 一句“突然”解决一切 |
| 预警线 | 谁先发现异常，信号是否被忽视 | 悬念、责任、人物判断 | 所有人同时惊叫 |
| 空间线 | 出口、楼梯、通道、封控、堵点在哪里 | 疏散和救援调度 | 场景像空白盒子 |
| 人群线 | 谁在场，谁行动不便，谁负责组织 | 群像、照护、冲突 | 群众背景板化 |
| 信息线 | 报警、广播、群聊、平台、谣言如何传播 | 误会、延迟、舆论 | 电话刚好万能 |
| 专业线 | 消防、应急、医疗、公安、物业、学校、企业如何响应 | 现实可信度 | 主角替代一切机构 |
| 次生线 | 火后烟、塌、断电、停水、污染、传染、拥堵、谣言 | 连锁升级 | 救完就结束 |
| 证据线 | 报警记录、监控、巡检、维修、审批、聊天如何留痕 | 调查反杀、责任归属 | 责任空白 |
| 恢复线 | 安置、赔偿、重建、心理、名声和制度如何变化 | 长篇后果 | 下一章清零 |

最小规则：每场事故至少写清“事故怎么来、谁发现、谁组织、谁被困、谁救援、谁承担后果”。

## 5. 隐患不是背景：事故前的可回看伏笔

事故要让读者觉得“意外但不作弊”。前文可以埋下低调隐患。

### 5.1 常见隐患类型

| 类型 | 可写伏笔 | 后续用途 |
| --- | --- | --- |
| 空间隐患 | 通道堆物、出口上锁、楼梯灯坏、门禁失灵 | 疏散受阻、责任追究 |
| 设备隐患 | 老旧线路、报警器失灵、消防设施缺维护 | 事故扩大、调查证据 |
| 管理隐患 | 培训缺失、值班空岗、巡检造假、演练敷衍 | 组织责任、职场线 |
| 人群隐患 | 老人儿童多、病人多、粉丝拥挤、学生晚自习 | 疏散难度、照护冲突 |
| 环境隐患 | 高温干燥、暴雨积水、台风停电、地震余震 | 次生灾害 |
| 信息隐患 | 信号差、广播坏、群消息误传、报警延迟 | 误判和恐慌 |
| 资源隐患 | 备用电不足、车辆不够、药品不够、避难点满员 | 救援瓶颈 |
| 权力隐患 | 领导压事故、商家怕停业、物业推责 | 伦理和调查线 |

### 5.2 伏笔写法

不要在事故前大喊“这里有重大隐患”。可以写成日常：

- 保安让外卖车停在消防通道，主角皱眉但赶时间。
- 宿管说楼梯灯坏了报修三天没人来。
- 商场促销活动临时加座，通道越来越窄。
- 酒店演练时，员工只拍照打卡，没人真的走路线。
- 老人说楼下总闻到怪味，家人以为他多心。
- 暴雨预警弹窗被角色随手划掉。

检查问题：事故发生后，读者能否回想起至少一个可解释风险扩大的前文细节？

## 6. 触发与预警：危机先改变信息，再改变空间

事故开始不一定是爆炸。更有张力的是“异常信号被识别、误读或忽略”。

### 6.1 预警信号

| 信号 | 可写成 | 剧情作用 |
| --- | --- | --- |
| 感官异常 | 焦糊味、烟味、震动、异响、刺鼻气味、灯闪 | 视角人物先发现 |
| 设备提示 | 警报、广播、断电、门禁异常、监控黑屏 | 组织反应开始 |
| 人群反应 | 孩子哭、老人咳、员工对视、顾客回头 | 气氛变化 |
| 外部预警 | 气象预警、地震警报、洪水通知、社区广播 | 倒计时和选择 |
| 线上信息 | 群消息、热搜、直播弹幕、导航封路 | 信息真假和传播 |
| 专业判断 | 安保、消防员、医生、老师、工人发现不对 | 职业可信度 |

### 6.2 误判的价值

危机初期的误判要来自合理人性：

- 商家怕影响营业，先让员工“确认一下”。
- 学校怕家长恐慌，广播措辞含糊。
- 住户以为是楼上装修，不愿下楼。
- 主角以为只是普通停电，直到听见楼道喊声。
- 反派故意压下异常报告。
- 专业人员判断正确，但缺少授权或资源。

检查问题：预警被谁看见？为什么没有立刻转化为正确行动？这个延迟由谁承担后果？

## 7. 疏散与避险：写路线、秩序和照护，不写乱跑

疏散场景不是“大家冲出去”。它要同时管理空间、信息、人群和身体状态。

### 7.1 疏散四问

1. 从哪里撤：楼层、房间、地下空间、舞台、教室、病房、工厂车间。
2. 往哪里撤：安全出口、楼梯、空地、集合点、避难层、临时安置点。
3. 谁走得慢：老人、儿童、残障者、病人、孕产妇、伤者、宠物、惊恐者。
4. 谁来组织：老师、物业、保安、护士、班干部、现场负责人、志愿者、消防救援人员。

### 7.2 疏散中的冲突

可用冲突：

- 有人逆行拿财物或找人。
- 有人占用通道拍视频。
- 老人不愿离开家中药物和证件。
- 家长冲进学校找孩子。
- 领导要求先抢设备，员工要求先撤人。
- 主角必须选择留下引导还是跟家人先走。
- 有人散布错误路线导致拥堵。
- 残障角色需要无障碍路线，但平时被杂物挡住。

### 7.3 写作边界

可以写“角色听从广播、寻找出口、互相点名、照顾行动不便者、到集合点确认人数”。

不要写具体高危逃生技巧、危险攀爬、破窗破门、穿越火场、非专业排险等可模仿操作，更不要把冒险行为写成唯一正确选择。

检查问题：疏散后是否有人数清点、失联名单、伤者状态、集合点和下一步安排？

## 8. 专业响应：救援不是神兵天降，而是组织协同

现实事故处置通常需要多方协作。小说不必写成行政流程，但要保留“分工”。

### 8.1 常见响应角色

| 角色 | 叙事功能 |
| --- | --- |
| 消防救援 | 灭火、搜救、破拆、排险、危险源控制、现场警戒配合 |
| 应急管理 | 统筹事故处置、安全生产、救援协调、后续调查衔接 |
| 公安/交警 | 封控、秩序、交通疏导、身份核查、协助调查 |
| 医疗急救 | 分流伤员、转运、医院接收、家属通知 |
| 物业/学校/企业 | 启动预案、广播、点名、提供平面图和人员名单 |
| 社区/街道 | 安置、物资、老人儿童照护、居民沟通 |
| 电力/燃气/水务 | 切断或恢复公共设施，处理次生风险 |
| 新闻/平台 | 信息发布、谣言治理、公众解释 |
| 志愿者/邻里 | 协助安置、寻人、照顾弱势群体，但不能替代专业排险 |

### 8.2 救援中的真实阻力

- 到场需要时间，路被堵、消防通道被占。
- 现场信息不完整，平面图缺失或过期。
- 人员名单不准，有临时访客和外包人员。
- 通信拥堵，家属群和媒体同时施压。
- 二次风险未排除，专业人员不能盲目进入。
- 伤员数量超过现场资源，需要分流。
- 领导、家属、围观者和直播者干扰现场。
- 天气、夜间、断电、积水、浓烟、余震改变救援节奏。

检查问题：专业救援为什么不能立刻解决全部问题？主角能提供的价值是什么，而不是替代整个系统？

## 9. 火灾、烟雾与燃气事故写法

火灾戏重点不是火苗多大，而是烟、热、视线、声音、空间和时间压力。

### 9.1 火灾场景的叙事要素

| 要素 | 写作功能 |
| --- | --- |
| 烟 | 视线变差、呼吸困难、声音变钝、路线判断困难 |
| 热 | 逼迫后退、门把手/墙体变化、距离变成压力 |
| 声音 | 警报、玻璃碎裂、广播、喊人、消防车、喷淋 |
| 光 | 断电、应急灯、火光、手机电量、黑暗恐惧 |
| 水 | 喷淋、消防水、积水、湿滑、文件设备损失 |
| 空间 | 楼梯、电梯、走廊、门、窗、屋顶、地下室 |
| 人群 | 逆行、拥堵、点名、失散、救助弱者 |
| 财物 | 证件、药物、宠物、合同、服务器、展品 |

### 9.2 燃气与爆炸事故边界

燃气泄漏、爆炸和危险品事故只能写成风险和后果，不写操作细节。

可以写：

- 角色闻到异常、停止争吵、撤离并通知专业人员。
- 物业或燃气公司与消防救援进入现场。
- 现场被封控，居民等待通知。
- 事故调查发现长期报修、设备老化、违规施工或管理失职。
- 角色因没有及时重视邻居提醒而承担情绪和现实后果。

不要写：

- 如何制造泄漏、引爆、破坏管线。
- 如何私自排险、拆装设备、检测浓度。
- 如何规避监管、逃避监控和毁灭证据。

检查问题：这场火灾或燃气事故的重点是人物选择和制度后果，还是危险技巧展示？

## 10. 自然灾害写法：灾害会先打断基础生活线

地震、洪水、台风、暴雨、山体滑坡、泥石流和极端天气不能只写“天灾来了”。它们首先改变基础生活线。

### 10.1 基础生活线

| 生活线 | 灾害后变化 | 可生成剧情 |
| --- | --- | --- |
| 安全 | 建筑损坏、余震、积水、塌方、飞散物 | 是否转移、谁留下 |
| 交通 | 封路、桥断、地铁停运、航班取消 | 赶路失败、救援延迟 |
| 通信 | 断网、信号拥堵、谣言扩散 | 失联、误传、寻人 |
| 能源 | 停电、燃料不足、充电困难 | 信息断层、医疗设备压力 |
| 饮水食物 | 供应中断、超市排队、安置点配给 | 资源分配、冲突和互助 |
| 医疗 | 伤员增多、医院拥挤、药物短缺 | 角色旧病、照护压力 |
| 住所 | 房屋受损、临时安置、租住困难 | 家庭关系和财产线 |
| 工作学校 | 停课停工、线上通知、复工压力 | 阶层差和制度选择 |

### 10.2 次生灾害

自然灾害后常有次生风险，适合拉长单元：

- 地震后的余震、断水断电、道路中断、房屋安全评估。
- 暴雨后的内涝、地下空间风险、交通瘫痪、污水和疾病风险。
- 台风后的停电、玻璃破损、树木倒伏、物流延迟。
- 洪水后的饮水、清理、消杀、财产损失和安置。
- 山火后的空气污染、撤离令、宠物牲畜安置和返家检查。

检查问题：这场自然灾害除了“吓人”，是否改变了交通、通信、食物、住所、工作和关系？

## 11. 群体伤亡和医疗分流：写资源承压，不写医疗教程

大事故可能造成大量伤员。此时重点不是治疗细节，而是资源承压。

### 11.1 可写内容

- 现场先按危险区域和安全区域分开。
- 救援人员优先确认谁还在危险中、谁需要转运、谁能自行撤离。
- 医院急诊、救护车、血库、床位、家属联络同时承压。
- 家属找人，名单更新慢，信息不完整。
- 主角能做的是提供名单、引导家属、照顾轻伤者、翻译沟通、保留证据或协助秩序。
- 医护角色有专业压力和情绪代价，但不神化。

### 11.2 不写内容

- 不写具体急救操作步骤、药物剂量、手术细节。
- 不让非专业主角凭勇气解决医学问题。
- 不用死伤数字制造廉价震撼。
- 不把家属痛苦当围观 spectacle。

检查问题：伤亡是否转化为资源压力、人物选择、家属关系和长期后果，而不是单纯刺激？

---

## 12. 弱势群体、可及性和照护责任

灾害会放大平时被忽略的生活障碍。

### 12.1 需要主动检查的人群

- 儿童、学生、婴幼儿。
- 老人、失智老人、独居老人。
- 残障者、行动不便者、视听障碍者、慢性病患者。
- 孕产妇、伤病者、照护者。
- 外地游客、语言不通者、新员工、临时工。
- 住地下室、出租屋、宿舍、工地、养老院、医院的人。
- 宠物、牲畜、工作动物。

### 12.2 叙事重点

- 谁平时负责照护？事故时是否在场？
- 疏散路线是否适合轮椅、担架、婴儿车？
- 安置点是否有药物、隐私、无障碍和信息可及？
- 角色是否有拒绝权和尊严，而不是被当作负担？
- 照护者是否也会疲劳、恐惧、犯错和需要支援？

检查问题：这场事故是否只默认“身体健康、行动迅速、信息充足”的人存在？

---

## 13. 信息发布、谣言和公众沟通

事故发生后，信息是第二现场。

### 13.1 信息链

`现场目击 -> 报警/上报 -> 内部通知 -> 官方通报 -> 媒体报道 -> 社媒扩散 -> 谣言反噬 -> 后续澄清`

### 13.2 信息冲突

可用冲突：

- 官方通报慢，短视频先爆。
- 家属想知道名单，现场还在核对。
- 企业想淡化事故，员工爆料。
- 平台剪辑夸大，救援被围观干扰。
- 谣言说主角见死不救，完整视频在别人手机里。
- 志愿者群热心但信息混乱，造成二次拥堵。

### 13.3 写作边界

- 不传播真实事故未经证实的细节。
- 不公开可识别受害者隐私。
- 不把救援现场直播、围堵救援通道、拍摄遗体伤者写成无害热闹。
- 舆论线必须联动第 61、142、163 篇。

检查问题：事故信息如何被确认、误传、压制、澄清？谁因信息不对称做错选择？

---

## 14. 事故调查与责任：灾害结束后才开始另一场戏

事故调查不是一句“相关责任人被处理”。它可以是卷后半段的证据线、职场线、商战线和情感线。

### 14.1 调查材料

| 材料 | 可证明什么 |
| --- | --- |
| 报警记录 | 发现时间、报警人、响应时间 |
| 监控视频 | 人流、异常、责任行为、时间线 |
| 巡检记录 | 设备是否按期检查，是否造假 |
| 维修记录 | 隐患是否早已报告 |
| 培训/演练记录 | 组织是否有准备，是否走形式 |
| 审批文件 | 场所是否合规、活动是否超员 |
| 聊天记录 | 是否有人压下风险、要求继续营业 |
| 现场照片 | 通道堵塞、设备状态、标识缺失 |
| 证人证言 | 现场反应、通知过程、责任分歧 |
| 物证残留 | 事故源、损坏范围、次生风险 |

### 14.2 责任层级

| 层级 | 叙事问题 |
| --- | --- |
| 个人失误 | 谁当时犯错，为什么犯错，是否可预见 |
| 岗位失职 | 谁负责巡检、值班、培训、审批、上报 |
| 组织制度 | 安全制度是否被 KPI、成本、面子工程压垮 |
| 监管漏洞 | 检查是否走过场，整改是否拖延 |
| 商业压力 | 停业损失、活动档期、客户投诉如何压过安全 |
| 命运偶然 | 即使有人尽责，也可能有不可控损失 |

好事故调查不是找替罪羊，而是拆清“哪些可防、哪些不可控、哪些人必须承担”。

检查问题：事故调查是否让前文隐患、人物选择、组织制度和最终后果连成闭环？

---

## 15. 灾后恢复：住所、钱、证件、身体和心理都要入账

救援结束不是剧情结束。灾后恢复才决定事故是否有重量。

### 15.1 恢复项目

| 项目 | 可写后果 |
| --- | --- |
| 住所 | 临时安置、搬家、房屋评估、租金压力 |
| 钱 | 财产损失、保险、赔偿、募捐、停工收入 |
| 证件 | 身份证件、合同、病历、学籍材料、公司资料丢失 |
| 身体 | 烧伤、吸入伤、骨折、慢病加重、疲劳 |
| 心理 | 惊吓、失眠、内疚、创伤反应、幸存者负罪感 |
| 关系 | 谁救谁、谁抛下谁、谁隐瞒风险、谁照顾善后 |
| 工作学校 | 停课停工、项目延期、岗位责任、考试补安排 |
| 舆论 | 表彰、问责、网暴、谣言、英雄化压力 |
| 制度 | 整改、演练、处罚、通报、换物业、换负责人 |

### 15.2 灾后情绪

灾后情绪不能只写“大家哭了”。可以写：

- 平静后才开始发抖。
- 看到警报灯就失眠。
- 反复检查煤气、门锁、充电器。
- 对救命恩人感激，但也怨他当时先救了别人。
- 幸存者不愿被媒体采访。
- 英雄叙事压住了受害者复杂感受。
- 家属需要真相，不只需要赔偿。

检查问题：事故后，哪些人物的日常习惯、关系位置、经济状态和安全感改变了？

## 16. 类型文用法

### 16.1 都市现实与职场

可用：公司火灾、工厂事故、商场踩踏风险、物业失职、租房隐患、活动安保、直播事故。

重点：安全制度、岗位责任、企业声誉、赔偿、舆论和调查。

### 16.2 校园青春

可用：宿舍火情、实验室事故、暴雨停课、校车事故、体育馆疏散、匿名墙谣言。

重点：未成年人、老师组织、家长焦虑、学校通报、心理支持和隐私保护。

### 16.3 古言宅斗/权谋

可用：走水、粮仓失火、堤坝决口、瘟疫以外的灾后赈济、庙会拥挤、宫宴火情。

重点：时代消防能力、仆役分工、家族责任、救灾粮、官府文书、名声和民心。

### 16.4 玄幻仙侠

可用：雷劫引火、秘境崩塌、护山大阵失效、妖兽潮导致撤离、灵矿事故。

重点：超凡能力也要有资源限制、阵法权限、宗门救援和事后清算。

### 16.5 末世灾变

可用：基地火灾、避难所停电、洪水围困、物资仓库事故、通信中断。

重点：基础生命线、资源分配、秩序维护、弱者照护和长期恢复。

### 16.6 娱乐圈/直播

可用：舞台火情、演唱会疏散、片场事故、粉丝拥挤、直播救援争议。

重点：主办方预案、安保、平台传播、粉丝安全、艺人责任和公关余波。

### 16.7 悬疑刑侦

可用：伪装事故、火灾灭证、事故现场隐藏矛盾、报警时间疑点。

重点：现场证据、电子数据、证人时间线、专业调查和反作者作弊。

## 17. 常见坏味道与修复

| 坏味道 | 读者体感 | 修复方式 |
| --- | --- | --- |
| 爆炸空降 | 作者硬转折 | 前文埋隐患、预警、管理疏漏和触发条件 |
| 主角单刷救援 | 假，机构降智 | 写主角的局部贡献，同时保留专业响应和组织分工 |
| 所有人只会尖叫 | 群像扁平 | 给人群分层：组织者、拖慢者、伤者、逆行者、协助者 |
| 火灾只写火 | 现场悬浮 | 写烟、热、断电、警报、空间封锁、通信和人群 |
| 疏散像跑步比赛 | 不真实 | 写出口、堵点、点名、照护、集合点和失联名单 |
| 灾害无次生后果 | 轻飘 | 加断电、封路、停水、安置、谣言和二次风险 |
| 救完就结束 | 没重量 | 写安置、医疗、赔偿、调查、整改和心理后果 |
| 事故没人负责 | 爽点廉价 | 拆个人失误、岗位失职、组织制度和监管漏洞 |
| 伤亡数字堆砌 | 猎奇 | 回到具体人物、家属、照护和生活中断 |
| 危险操作教程化 | 安全风险 | 只写概念、后果和专业介入，不写步骤 |

## 18. 审稿检查清单

审稿时逐条检查：

1. 事故前是否有可回看的隐患或预警。
2. 事故触发是否有明确原因，而不是一句“突然”。
3. 现场空间是否清楚：出口、通道、楼梯、封控区、集合点。
4. 人群是否分层：老人、儿童、残障者、伤者、工作人员、围观者。
5. 报警、广播、群消息、官方通报和谣言传播是否有时间差。
6. 专业救援是否存在，并有真实限制和分工。
7. 主角贡献是否局部、合理、有代价，而不是替代整个救援系统。
8. 是否避免可模仿的纵火、爆炸、燃气、危险品、破拆和排险教程。
9. 火灾是否写到烟、热、断电、视线、声音和空间阻断。
10. 自然灾害是否改变交通、通信、能源、饮水、住所、工作学校。
11. 是否写出次生灾害或二次风险。
12. 伤亡是否被尊重，是否避免猎奇和工具化。
13. 是否处理未成年人、老人、残障者、病人和照护者的可及性需求。
14. 是否有安置、赔偿、医疗、心理支持或生活恢复。
15. 是否有事故调查线：报警记录、监控、巡检、维修、审批、证人。
16. 责任是否分层，而不是随便找替罪羊。
17. 事故是否改变人物关系、职业、名声、钱、住所或制度。
18. 是否把事故状态、失联名单、伤病、财产损失、舆论认知和整改结果写入台账。

## 21. 外部资料吸收点

本专题吸收外部资料时，只转化为小说叙事规则和安全边界。

- 中国《中华人民共和国突发事件应对法》：突发事件应对包括预防与准备、监测预警、应急处置与救援、恢复重建；迁移为“事故不是一个瞬间，而是前中后全链条”。
- 中国《国家突发事件总体应急预案》：强调统一指挥、分类管理、分级负责、属地为主、人员搜救、抢险救灾、医疗救治、疏散转移、临时安置、应急保障和防止次生衍生事件；迁移为“救援必须有组织、资源和次生风险”。
- 中国《突发事件应急预案管理办法》：应急预案要覆盖事前、事中、事后，包含风险防控、监测预警、处置救援、应急保障、恢复重建、培训演练和评估修订；迁移为“演练和预案可以成为伏笔、打脸和责任线”。
- 中国《中华人民共和国消防法》及应急管理部消防科普：消防安全涉及火灾预防、消防组织、灭火救援、监督检查和公共消防安全；迁移为“火灾现场要写通道、设施、组织和责任，不只写火”。
- 中国《生产安全事故应急条例》《生产安全事故报告和调查处理条例》：生产安全事故后要启动预案、控制危险源、组织撤离、通知影响单位和人员、防止次生灾害、维护现场秩序、保护证据并调查处理；迁移为“工厂/公司/片场事故必须有报告、保护现场、调查和责任”。
- Ready.gov 和 FEMA：家庭与社区准备、应急计划、物资、预警、疏散、避难和社区生命线是灾害响应基础；迁移为“灾害会打断安全、食物、医疗、能源、通信、交通、水等基础生活线”。
- American Red Cross Home Fire Safety：家庭火灾强调预防、逃生计划、警报、撤离和灾后恢复；迁移为“火灾前有准备，火灾后有恢复，不是只写现场刺激”。
- NFPA 火灾教育资料：烟雾报警和逃生计划是火灾早期预警与人员行动的重要部分；迁移为“警报、演练和路线会改变人物命运”。
- UNDRR Sendai Framework：灾害风险治理强调理解风险、加强治理、投资韧性、强化准备并在恢复重建中更好重建；迁移为“灾后恢复可以改变城市、组织和人物长线结构”。
- WHO Mass Casualty Management：大规模伤亡事故会让本地医疗资源承压，需要准备、响应和协调；迁移为“事故伤亡不是数字，而是资源分流、信息核对和家属压力”。
- CDC/NIOSH 一氧化碳和停电安全资料：灾害后的发电、取暖、通风和室内环境可能产生次生健康风险；迁移为“灾后不是安全结束，隐藏风险会继续改变行动”。

## 22. 最小执行规则

如果 AI agent 只能记住五条：

1. 灾害事故必须有链条：隐患、触发、预警、疏散、救援、次生风险、调查、恢复。
2. 主角只能做合理的局部贡献，不能替代消防、应急、医疗、公安、物业、学校或企业系统。
3. 事故现场必须写空间和人群：出口、通道、堵点、集合点、失联者、老人儿童残障者和伤者。
4. 救援结束后必须有余波：安置、医疗、赔偿、心理、舆论、责任、整改和台账回写。
5. 只写叙事可信度和安全边界，不写纵火、爆炸、燃气、危险品、破拆、排险和规避责任的可模仿步骤。
