# 179. 知识产权组合、版权商标专利、商业秘密、软件品牌与侵权维权链专项

状态：已确认  
日期：2026-06-09  
用途：供 AI agents 在创作现代都市、文娱、网文作者、娱乐公司、游戏公司、科技创业、商战、直播电商、品牌打假、设计师、药械研发、软件平台、豪门企业、重生赚钱等中文商业网文时，处理知识产权类型识别、权利归属、登记/申请/保密、授权许可、商标品牌、专利技术、商业秘密、软件作品、平台盗版、侵权证据、行政投诉、平台下架、仲裁诉讼和维权余波。

## 1. 本专题解决什么问题

AI 写知识产权最容易犯四种错：

- 把所有东西都叫“版权”，不知道版权、商标、专利、商业秘密保护的是不同对象。
- 把“我先想到”写成“我拥有权利”，忽略创作证据、登记申请、合同归属和保密措施。
- 把商战维权写成热搜喊话，不写平台规则、权利凭证、侵权比对、证据保全、行政投诉、仲裁诉讼和商业后果。
- 把改名、换皮、洗稿、扒代码、盗图、仿包装、抢注商标、挖客户名单写成聪明，而不是高风险侵权或不正当竞争。

本专题把知识产权拆成一条可执行链：

`成果/标识/技术/秘密信息出现 -> 权利类型识别 -> 权利人和形成时间 -> 登记/申请/授权/保密措施 -> 商业使用渠道 -> 侵权或混淆信号 -> 证据保全与比对 -> 平台/行政/仲裁/诉讼/刑事分流 -> 停止使用/下架/赔偿/许可/整改 -> 商业余波 -> 台账回写`

商业级 IP 戏的核心不是“谁喊原创更大声”，而是“谁能证明权利、证明对方使用、证明混淆或损害，并把结果落到合同、市场、平台、投资和读者信任上”。

## 3. 安全边界

本专题是小说创作资料，不是法律、知识产权代理、投资、技术、平台申诉或诉讼建议。

AI 不得提供以下现实可操作策略：

- 规避版权检测、洗稿、换皮、扒代码、破解 DRM、绕过技术保护措施。
- 抢注他人商标、仿冒包装装潢、制造山寨品牌、伪造授权链。
- 绕开专利、规避权利要求、拆解竞品技术、窃取源码或设计图。
- 诱导员工泄露商业秘密、客户名单、报价、算法、配方、未公开商业计划。
- 伪造登记证书、时间戳、合同、公证、聊天记录、平台通知或证据保全材料。
- 恶意投诉竞争对手、批量下架、虚假差评、刷单刷评、滥用平台规则。

写侵权、盗版、商业秘密、专利规避、品牌碰瓷、平台下架时，只写风险、证据、后果和被追责路径，不写可复制操作步骤。

## 5. 四类核心权利的叙事区别

| 类型 | 保护对象 | 典型证明 | 常见误区 | 叙事抓手 |
| --- | --- | --- | --- | --- |
| 著作权 | 文字、音乐、美术、摄影、视听、软件等具体表达 | 草稿、源文件、发表记录、登记、合同 | 以为保护想法、套路、题材 | 抄袭、洗稿、改编授权、盗版下架 |
| 商标 | 区分商品或服务来源的标志 | 注册证、申请记录、使用证据、商品类别 | 以为公司名、书名、品牌名天然都能排他 | 品牌碰瓷、山寨、联名授权、抢注 |
| 专利 | 技术方案或外观设计 | 申请号、授权公告、权利要求、设计图 | 以为保密技术和创意点自动是专利 | 技术壁垒、侵权比对、竞品绕不开 |
| 商业秘密 | 不为公众所知、有商业价值、采取保密措施的信息 | 保密制度、权限记录、NDA、研发日志、访问日志 | 以为“公司内部资料”全是商业秘密 | 员工跳槽、客户名单、配方、算法、报价泄露 |

最小区分：

- 版权保护表达，不保护纯想法和题材套路。网文平台签约和改编授权先转第 167 篇。
- 商标保护识别来源，核心是消费者会不会误以为是同一家或有关联。
- 专利保护公开换排他，必须看申请、授权、权利要求或设计范围。
- 商业秘密保护未公开信息，必须有保密措施；一旦公开或无人保密，戏剧张力会改变。

## 6. 著作权和网文接口

著作权部分主要调用第 167 篇，本篇只保留商业项目中的接口。

AI 写文娱版权时要区分：

- 作者身份：谁写的，是否共同创作、委托创作、职务创作或 AI 辅助。
- 作品类型：文字作品、音乐作品、美术作品、摄影作品、视听作品、软件等。
- 权利类型：复制、发行、信息网络传播、改编、摄制、翻译、汇编等。
- 授权边界：平台连载、电子出版、纸书、有声、漫画、短剧、影视、游戏、周边、海外。
- 授权方式：独家、非独家、买断、分成、保底、期限、地域、可否转授权。
- 证据链：草稿、发布时间、编辑记录、签约合同、授权书、付款和平台通知。

常见剧情：

- 作者只授权平台电子连载，影视公司却声称拿到全版权。
- 经纪公司把作者短篇包装成短剧，却没有改编权或摄制权。
- 同人作品爆火后想商业化，必须回到第 102 和第 167 篇检查原创化和授权风险。
- AI 辅助封面、字体、配乐、素材没有留授权记录，项目上线前被卡。

## 7. 商标、品牌和混淆

商标线适合写创业、餐饮、电商、美妆、服装、娱乐公司、游戏、文创周边、直播带货。

AI 写商标时先问：

- 标志是什么：文字、图形、字母、数字、三维标志、颜色组合、声音等。
- 用在哪类商品或服务上：同名不必然全行业通吃，类别和实际使用很重要。
- 是否已经注册或申请，申请到哪一步。
- 是否存在在先权利、近似商标、有一定影响的名称、包装、装潢或字号。
- 是否会让消费者误以为同一家、授权联名、官方周边或同一系列。
- 是否在电商平台、短视频账号、直播间、包装、门店招牌、广告中使用。

好用场景：

- 女主重生后抢先注册自家老字号的核心商标，阻止亲戚拿同名开山寨店。
- 反派把爆款网文的书名、角色名、门派名做成周边品牌，平台以商标或版权投诉卡住上架。
- 餐饮品牌改一个字、包装配色高度近似，消费者误认后引发不正当竞争与商标争议。
- 娱乐公司推出“官方联名”，但授权链只覆盖宣传，不覆盖商品类目。

不要写：

- “我先用了这个名字，所以所有领域都是我的”。
- “公司名注册了，就等于商标注册了”。
- “商标证一拿，别人所有同名使用都违法”。

## 8. 专利、技术成果和外观设计

专利线适合科技创业、医药器械、智能硬件、工业设计、游戏设备、AI 产品、制造业商战。

AI 写专利时先区分：

- 发明：更偏技术方案，审查周期和技术门槛较高。
- 实用新型：常用于产品形状、构造等实用技术方案。
- 外观设计：保护产品外观设计，不保护纯功能或抽象审美。
- 申请中、已授权、无效程序、许可实施、质押融资、专利池等状态不同。

专利戏的关键不是“他有一个神奇发明”，而是：

- 这个技术是否已经公开，公开会影响新颖性和商业秘密策略。
- 谁是发明人，谁是专利申请人或专利权人。
- 是职务发明、委托研发、合作开发，还是个人研发。
- 权利要求保护到哪里，竞争对手的产品是否落入保护范围。
- 公司是否用专利融资、许可、诉讼、禁令、谈判或防御。

常见剧情：

- 主角研发日志证明自己是核心发明人，但专利申请人是公司。
- 反派抢先公开论文或发布会，破坏专利布局。
- 竞品外观几乎一样，消费者误认和外观设计专利同时形成压力。
- 投资人尽调发现核心技术只有宣传，没有专利、保密或源代码权属。

## 9. 商业秘密和不正当竞争

商业秘密常是商战里最好用、也最容易被写歪的线。它不是“所有内部资料”，而是“不为公众所知悉、具有商业价值、并经权利人采取相应保密措施”的技术信息、经营信息等商业信息。

可写商业秘密：

- 配方、工艺参数、源代码、模型参数、研发路线、算法细节。
- 客户名单、报价策略、供应商价格、渠道政策、投标方案。
- 未公开新品计划、营销方案、并购方案、融资材料。
- 医药、材料、芯片、游戏数值、直播投流模型等行业资料。

保密措施要写具体：

- 保密协议、员工手册、权限分级、访问日志、文件水印、加密存储。
- 会议纪要标密、离职交接、设备回收、账号关闭。
- 客户名单和报价不是群里随手发，而是有访问权限和使用记录。

2025 年修订的反不正当竞争法强化了混淆、商业秘密、网络不正当竞争、数据不正当获取、平台规则滥用、拖欠中小企业账款等场景。写电商、平台、算法、数据和品牌碰瓷时，要优先检查这条制度链。

## 10. 软件、代码、游戏和数字产品

软件既可能受著作权保护，也可能涉及专利、商业秘密、合同和数据权益。

AI 写软件/游戏公司时要区分：

- 源代码：通常作为软件表达、商业秘密和合同交付物管理。
- 算法思想：不一定受著作权保护，可能通过专利、商业秘密或合同保护。
- 游戏美术、剧情、音乐、角色、UI：可能分别涉及版权、商标、外观设计和素材授权。
- 账号、服务器、后台数据、玩家日志：涉及平台规则、个人信息和电子证据。
- 开源组件：需要看许可证，不是“网上找到就能商用”。

常见剧情：

- 技术合伙人离职带走代码库，争点不是“谁敲的代码多”，而是职务开发、合同归属、代码仓库记录、商业秘密和竞业边界。
- 游戏换皮被玩家发现，平台下架，投资人追问素材授权和代码来源。
- 小团队用了未授权字体、图标、音效，上线后收到投诉。
- 竞品通过不正当方式获取数据和参数，触发商业秘密和网络不正当竞争线。

## 11. 授权许可和合同边界

调用第 122 篇和第 167 篇。IP 授权不是一句“卖给你了”，而是范围、期限、地域、媒介、品类、费用、结算、审核、署名、转授权、违约和终止。

授权合同检查项：

- 授权标的：哪部作品、哪个商标、哪项专利、哪套素材、哪份技术资料。
- 权利范围：复制、发行、信息网络传播、改编、摄制、游戏、商品化、商标使用、专利实施。
- 使用场景：平台、地区、语言、品类、渠道、线下/线上、广告和周边。
- 期限：开始、结束、续约、终止后库存/存量内容如何处理。
- 费用：买断、分成、保底、里程碑、最低保证金、审计权。
- 质量控制：品牌授权和改编授权常需要审核权。
- 转授权：能否给第三方，第三方责任如何承担。
- 违约：超范围使用、逾期付款、未署名、质量不合格、未下架。

高质量授权戏会让每个条款都能变成钩子：期限快到、地域超限、周边品类未授权、平台擅自转授权、审核权卡住上线、分成报表造假。

## 12. 侵权信号和证据链

IP 维权不是看一眼“像不像”。AI 要写出证据链。

证据类型：

- 权利证明：登记证、注册证、专利证、申请号、授权书、合同、研发记录、保密制度。
- 原始材料：草稿、源文件、设计稿、代码提交记录、样品、生产文件。
- 被控使用：网页、商品链接、直播回放、店铺页面、包装、实物样品、广告投放、软件下载包。
- 时间线：首次创作、首次发表、申请日、注册日、授权日、首次销售、投诉日期。
- 商业影响：销量、流量、客户误认、下架损失、授权费、市场份额、品牌声誉。
- 取证方式：截图、录屏、购买样品、物流记录、平台后台、电子数据、公证/可信时间戳、法院证据保全。

写作技巧：

- 先让读者知道主角留了原始文件，再让反派上线山寨品。
- 侵权比对要抓核心相似点，不要只写“大家都觉得像”。
- 让证据带来商业后果：平台下架、投资暂停、客户退单、主播解约、融资尽调失败。

## 13. 维权和争议分流

常见路径：

- 平台投诉：盗版、盗图、假货、商标侵权、商品混淆、未授权素材。
- 行政投诉：市场监管、版权、知识产权等行政路径，适合打假、混淆、商标、专利行政处理等情节。
- 商务谈判：补授权、改包装、停止使用、和解赔偿、公开声明。
- 仲裁或诉讼：合同纠纷、授权费、侵权赔偿、停止侵害、消除影响。
- 刑事报案：严重盗版、假冒注册商标、侵犯商业秘密、非法经营等达到相应风险时转第 166 篇。
- 海关保护或跨境平台投诉：外贸、跨境电商、海外盗版时可作为情节方向，但具体规则需另查。

不要写：

- 律师函一发，对方立刻破产。
- 平台投诉不需要权利凭证。
- 法官只凭“像”就判赔天价。
- 行政机关、平台、法院、仲裁委混成同一个万能机构。

## 14. 类型文用法

- 网文作者文：第 167 篇管平台签约和版权，本篇补充商标周边、盗版平台、短剧盗改、作者品牌和证据链。
- 娱乐公司文：艺人姓名、组合名、粉丝周边、歌曲、舞台、肖像、商标、合同和平台传播一起跑。
- 科技创业文：专利布局、商业秘密、软件著作权、开源合规、员工离职和投资尽调形成主线压力。
- 商战文：品牌混淆、假货、客户名单、报价泄露、恶意投诉、数据不正当获取，比“砸钱收购”更有真实阻力。
- 重生赚钱文：主角可以预判商机，但必须提前做主体、商标、专利、版权、供应链和授权布局。
- 设计师/美妆/服装文：外观设计、商标、包装装潢、摄影作品、模特肖像和电商打假常常交叉。
- 游戏文：代码、美术、音乐、剧情、角色、商标、版号/平台规则、玩家数据和开源组件都要分开。
- 古代/架空：不直接套现代 IP 法，但可以保留“匠作秘方、招牌字号、官府印契、行会规矩、师门秘传、抄书盗版、仿冒货”的功能链。

## 15. 常见坏味道与修正

| 坏味道 | 读者体感 | 修正方法 |
| --- | --- | --- |
| 什么都叫版权 | 外行、混乱 | 先区分版权、商标、专利、商业秘密 |
| 有灵感就等于有权利 | 悬浮 | 写创作记录、申请登记、合同或保密措施 |
| 改名就能规避侵权 | 洗稿味、风险高 | 进入第 102 篇做原创性和高识别表达隔离 |
| 商标证万能 | 过度简化 | 写类别、使用场景、近似混淆和在先权利 |
| 专利等于发明家光环 | 空泛 | 写申请、授权、权利范围、侵权比对和商业化 |
| 商业秘密只是口头机密 | 不可信 | 写保密制度、权限、日志、NDA 和离职交接 |
| 平台秒下架对手 | 机械爽 | 写权利凭证、投诉材料、反通知和商业影响 |
| 律师函等于胜诉 | 降智 | 律师函只是节点，后续还有谈判、仲裁或诉讼 |
| 维权只为打脸 | 后劲弱 | 写授权费、市场份额、融资、读者信任和品牌重建 |

## 17. 给 AI agent 的生成提示词

起草 IP 维权场景：

```text
请按第179篇“知识产权组合、版权商标专利、商业秘密、软件品牌与侵权维权链专项”写一场 IP 维权戏。先填 ip_scene_card，明确资产类型、权利人、形成证据、保护状态、授权范围、侵权行为、证据链、商业利害和程序路径。场景必须让一个权利证据触发一个商业后果，避免只靠热搜喊原创。
```

审稿知识产权线：

```text
请审稿本章知识产权线是否悬浮。检查：是否混淆版权/商标/专利/商业秘密，权利人和合同是否明确，登记/申请/保密措施是否存在，授权范围是否写清，侵权比对和证据链是否足够，平台/行政/仲裁/诉讼路径是否混乱。逐项给出风险和修改建议。
```

写商标品牌冲突：

```text
请写一场商标或品牌混淆冲突。不能只写“名字像”。必须写清使用场景、商品或服务类别、消费者误认、包装装潢、平台投诉材料、销售数据和下一步商业谈判或程序。
```

写商业秘密泄露：

```text
请写一场商业秘密泄露戏。必须写清秘密信息是什么、为什么有商业价值、公司采取过哪些保密措施、谁有访问权限、泄露证据如何出现、泄露后对订单/融资/竞品造成什么后果。不要写可复制的窃密方法。
```

## 18. 自检清单

- 争议对象到底是作品表达、商品来源标识、技术方案、外观设计、软件代码、客户名单还是商业秘密？
- 权利人是谁？作者、公司、平台、委托方、员工、共同创作者、投资方是否混淆？
- 权利何时形成？有草稿、源文件、登记、申请、注册、合同、研发日志或使用证据吗？
- 是否需要调用第 167 篇处理网文著作权、平台签约和改编授权？
- 是否需要调用第 102 篇处理抄袭、洗稿、融梗和原创性风险？
- 商标是否写清类别、使用场景、消费者混淆和在先权利？
- 专利是否写清申请/授权状态、权利范围和技术比对？
- 商业秘密是否写清不公开、商业价值和保密措施？
- 授权合同是否写清媒介、地域、期限、品类、独占、费用和转授权？
- 侵权证据是否有权利证明、被控使用、时间线和商业影响？
- 维权路径是平台投诉、行政投诉、谈判、仲裁、诉讼还是报案？
- 结果是否改变项目上线、收入、融资、品牌、粉丝信任或后续授权？

## 19. 最小规则

如果 AI 只能记住五条：

1. 版权、商标、专利、商业秘密保护对象不同，先分类再写剧情。
2. IP 爽点来自权利证据、授权边界、侵权比对和商业后果，不来自“我说原创”。
3. 网文版权和平台授权先接第 167 篇，原创性和防洗稿先接第 102 篇。
4. 商业秘密必须有保密措施，商标必须看混淆和类别，专利必须看申请/授权和权利范围。
5. 不写规避侵权、伪造授权、窃取秘密、恶意投诉和盗版传播的现实操作方法。

## 20. 主要来源

- 《中华人民共和国著作权法》，用于作品、著作权人、权利类型、许可转让和侵权责任底座。
- 《中华人民共和国商标法》，用于商标注册、商标专用权、混淆、商标侵权和品牌识别底座。
- 《中华人民共和国专利法》，用于发明、实用新型、外观设计、专利申请授权、实施许可和侵权纠纷底座。
- 《中华人民共和国反不正当竞争法》，2025 年修订，2025 年 10 月 15 日施行，用于混淆、商业秘密、网络不正当竞争、数据不正当获取、平台规则滥用等情节。
- 《信息网络传播权保护条例》，用于网络平台传播、通知删除、断开链接和信息网络传播权纠纷。
- 《计算机软件保护条例》，用于软件著作权、开发传播、侵权责任和软件商业化情节。
- 国家知识产权局商标、专利和知识产权公共服务资料，用于商标申请、专利申请、公告、注册证明、维权援助等现实流程背景。
