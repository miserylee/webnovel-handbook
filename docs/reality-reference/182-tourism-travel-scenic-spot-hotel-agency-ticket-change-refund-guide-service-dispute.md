# 182. 旅游出行、景区酒店旅行社、票务改签、导游服务与旅游纠纷链专项

状态：已确认  
日期：2026-06-09  
用途：供 AI agents 在创作现代都市、旅行治愈、公路文、综艺旅行、亲子游、老年团、蜜月旅行、出境游、研学游、旅行社商战、OTA 平台、景区经营、导游角色、旅游事故、票务退改、强制购物和旅游投诉等中文商业网文时，处理旅游合同、旅行社资质、在线旅游平台、导游服务、景区门票、行程变更、酒店交通组合、退票改签、游客安全、投诉处理和旅游市场监管。

## 1. 本专题解决什么问题

旅游线最容易被 AI 写成三类假：

- 旅游只剩风景打卡，没有订单、合同、门票、交通、住宿、导游、行程安排、退改规则和安全责任。
- 跟团游只写“低价团强制购物”，不写旅行社资质、旅游合同、购物安排、导游职责、证据和投诉处理。
- OTA 平台、旅行社、酒店、航空铁路、景区、导游、地接社、保险、监管部门混成一个万能客服。

本专题把旅游服务拆成一条可执行链：

`出行动机 -> 旅游产品/自由行组合 -> 经营主体 -> 宣传页面/行程单 -> 合同与付款 -> 票务交通 -> 住宿餐饮 -> 景区游览 -> 导游/领队/地接 -> 行程变更/安全事件 -> 退改投诉 -> 平台/旅行社/监管/诉讼 -> 赔付整改 -> 口碑与台账回写`

商业级旅游戏不是“换个地方谈恋爱”，而是让读者看见：角色为什么去、谁组织、钱付给谁、路线怎么卡住人物、服务承诺如何兑现或落空、风险如何改变关系和行动。

## 3. 安全边界

本专题是小说创作资料，不是法律、旅游、签证、交通、保险、投诉或安全建议。

AI 不得提供以下现实可操作策略：

- 伪造机票、火车票、酒店订单、签证材料、旅游合同、导游证、行程单或保险凭证。
- 逃避实名制、安检、边检、签证、旅客身份核验或景区安全管理。
- 制造低价团、强制购物、虚假宣传、恶意退票、骗保、刷单刷评、假投诉。
- 私自组织无资质旅游、非法包车、黑导游、黑旅行社、非法跨境线路。
- 教唆游客进入未开放区域、危险区域、野景点、禁区或规避救援/安全提示。
- 泄露游客身份、行程、酒店房号、护照、支付信息或位置信息。

写黑导游、低价团、非法包车、景区事故、出境受困、野游失联时，只写风险、证据、伤害和追责路径，不写可模仿操作。

## 5. 旅游产品类型先分类

| 类型 | 叙事功能 | 必写约束 |
| --- | --- | --- |
| 自由行 | 自主选择、情侣/朋友磨合、迷路和选择压力 | 订单分散，交通酒店门票各有规则 |
| 跟团游 | 群像、导游、时间表、低价团、强制购物 | 旅游合同、行程单、旅行社资质、导游职责 |
| 半自由行 | 平台组合、接送机、酒店交通打包 | 哪些由平台负责，哪些游客自理 |
| 定制游 | 高端服务、管家、私密路线 | 合同细，改动成本高，隐私和安全要求高 |
| 研学游 | 学生、家长、老师、机构、安全责任 | 未成年人保护、课程目标、保险和带队责任 |

| 自驾游 | 路线自由、车辆、疲劳驾驶、事故风险 | 转第 168 篇，写驾驶资格、保险和道路条件 |
| 出境游 | 护照签证、领队、当地规则、汇率和风险 | 转第 161 篇，身份和出入境必须查最新要求 |
| 综艺旅行 | 摄制、赞助、路线任务、嘉宾冲突 | 肖像、拍摄许可、平台传播和安全边界 |

## 6. 旅行社、OTA 和平台责任

旅行社、在线旅游平台、酒店航空铁路、景区和地接社不是同一个主体。

AI 写旅游产品时先问：

- 谁是合同相对方：旅行社、平台自营、平台内经营者、酒店、航空公司、铁路运输企业还是景区？
- 页面展示的是产品宣传、广告、行程单、确认单还是正式合同？
- 是否有旅行社业务经营许可，线上经营旅行社业务是否展示许可信息？
- OTA 是撮合交易、代理销售、平台规则处理，还是自己作为旅游经营者销售产品？
- 平台是否保存交易信息、合同履行情况和投诉处理信息。
- 地接社、导游、司机、酒店、餐厅是旅行社安排还是游客自选。

好用冲突：

- OTA 页面写“五星酒店”，实际入住的是同名低档酒店或远郊合作酒店。
- 平台把多个单项服务打包，出问题时各方互相推责。
- 旅行社转团给地接社，游客直到当地才发现行程缩水。
- 低价产品靠购物返点维持成本，导游在行程中施压。
- 游客私下加项目，事故后旅行社和游客责任边界不清。

## 7. 合同、行程单和付款

旅游纠纷最重要的材料通常不是风景照，而是合同和行程单。

旅游合同/订单要写：

- 产品名称、出发地、目的地、出行日期、人数。
- 交通、住宿、餐饮、景区、导游、自由活动和购物安排。
- 团费包含什么，不包含什么，自费项目是否自愿。
- 成团人数、退团、改期、取消、不可抗力和违约责任。
- 老人、儿童、孕妇、残障游客、慢病游客的特别说明。
- 保险、紧急联系人和安全告知。

付款链：

- 定金/订金、尾款、平台支付、线下转账、发票、收据。
- 是否有旅游意外险或旅行社责任保险，保险范围是什么。
- 退款是原路退回、平台余额、代金券，还是分阶段退。

AI 不要让“朋友圈转账给熟人团长”自动等同合法旅行社合同；这正是黑团和证据弱点。

## 8. 票务、预约、退改签

旅行中的票务至少分四类：交通票、住宿订单、景区门票、演出/活动票。每类规则不同。

交通票：

- 航空票务要区分自愿退改和非自愿退改，航班取消、延误、提前、舱位变化等会改变处理路径。
- 铁路旅客运输有实名、车票、乘车、改签、退票、投诉等规则；具体退改费和时限按当时铁路官方规则核对。
- 道路客运和旅游客运涉及客运站、包车、旅游客运、安检和禁止限制携带物。
- 水路、邮轮、跨境交通另有承运、安检、天气停航和港口规则。

景区票：

- 是否实名预约、分时段入园、限流、闭园、特殊天气、优惠票、老人儿童票。
- 套票、联票、年卡、二次入园、索道/观光车/游船等附加项目要分开。
- 景区关闭、天气原因、设备停运、游客迟到、证件不符会触发不同处理。

写法规则：

- 票务改签不是“客服动动手”。要写余票、时间、费用、身份核验、原支付渠道和下游订单连锁。
- 一个航班取消会影响酒店、景区预约、接送机、旅行社成团、亲友接站和工作安排。
- 退改损失是戏剧货币：钱、时间、信用、工作、关系都可能被扣。

## 9. 导游、领队、地接和司机

导游不是风景讲解机器。导游连接合同、行程、游客安全、购物安排、地方资源和突发处置。

写导游职责：

- 按合同和行程单提供服务，讲解自然和人文资源。
- 提醒安全、集合时间、禁忌、目的地规则和风险。
- 协调酒店、餐厅、景区、交通和游客矛盾。
- 突发事件中协助报警、就医、联系旅行社和家属。
- 不得擅自变更行程、降低服务标准、强迫购物或另行付费。

角色用法：

- 好导游可以成为地方知识入口、群像协调者和危机时的稳定器。
- 黑导游不是单纯嘴坏，而是被低价团、购物返点、旅行社压价和地方灰色链条裹挟。
- 领队在出境团中承担集合、证件、沟通和风险提示功能。
- 司机关系到旅游客运安全，不能让导游随便替代司机决策。

## 10. 景区、地方和游客安全

景区不是布景板。它有入口、票务、限流、路线、安保、救援、保洁、商业摊位、索道游船、厕所、医疗点和投诉入口。

安全节点：

- 天气：暴雨、台风、高温、冰雪、山洪、海浪、雷电。
- 地形：山地、峡谷、海边、湖泊、沙漠、高原、溶洞、古城小巷。
- 设施：索道、玻璃栈道、漂流、游船、观光车、游乐设备。
- 人群：老人、儿童、孕妇、残障游客、外地游客、语言不通者。
- 管理：公告、警示牌、封闭区域、救援点、广播、监控、票务记录。

禁止写法：

- 把私闯未开放区域写成浪漫冒险。
- 把野景点失联写成无代价刺激。
- 把工作人员安全劝阻写成扫兴反派。
- 把危险救援当主角耍帅舞台，不写专业救援和后果。

涉及火灾、踩踏、坠落、溺水、车辆事故、食物中毒、医疗急救时，分别联动第 164、168、169、170 篇。

## 11. 强制购物、低价团和购物纠纷

低价团的核心不是“游客贪便宜”，而是产品价格、购物返点、导游收入、旅行社成本和游客知情权之间的冲突。

AI 写低价团要写：

- 报名页面或合同是否写明购物点、自费项目和停留时间。
- 团费为什么低，哪些费用不包含，是否存在年龄或地域附加费。
- 导游是否明示或暗示不购物就不走、辱骂、冷落、缩短景点时间。
- 购物店和旅行社是否有关联，商品是否虚高、假货或不适用退换。
- 老年游客、外地游客、语言不通者是否被重点推销。
- 证据是录音录像、购物小票、行程单、导游通知还是其他游客证言。

修正方向：

- 不把所有购物都写成违法。自愿购物、明确安排和真实商品可以存在。
- 不把所有游客写成愚蠢。信息不对称、体力消耗、群体压力和返程依赖才是压迫点。
- 结果要落到退货、赔偿、旅行社处罚、导游处理、平台下架和口碑上。

## 12. 旅游投诉和纠纷处理

2026 年 3 月 15 日起施行的新《旅游投诉处理办法》强化了旅游投诉受理、处理、监督和保障。写 2026 年后的旅游投诉时，要按新办法处理；写旧时间线则按当时规则。

投诉要素：

- 投诉人是谁，是否是旅游者或合法权益受损方。
- 被投诉人是谁：旅行社、在线旅游经营者、导游、景区、酒店、其他旅游经营者。
- 投诉请求是什么：退费、赔偿、道歉、整改、处罚线索、合同履行。
- 事实和理由是什么：合同、行程单、付款、聊天、照片视频、票据、同行证言。
- 是否在受理范围和期限内，是否已有仲裁诉讼等处理。

路径分流：

- 旅行社/平台协商：最快但要有工单和处理记录。
- 旅游投诉处理机构：处理旅游经营者与旅游者纠纷。
- 市场监管/消协：虚假宣传、消费欺诈、价格和普通消费争议。
- 交通主管/民航/铁路：票务、承运、旅客运输服务。
- 公安/应急/消防/医疗：安全事故、失踪、伤害、诈骗、扰乱秩序。
- 仲裁/诉讼：合同、赔偿、侵权争议进入裁判路径。

不要写“投诉电话一打立刻封店赔十倍”。投诉处理需要材料、受理、调查、调解、转办或后续程序。

## 13. 出境游和跨境旅行

出境游必须联动第 161 篇。不要把护照、签证、边检、入境许可、当地法律和保险写成一句“飞过去”。

检查项：

- 护照有效期、签证或免签条件、入境目的、停留期限。
- 未成年人、老人、孕妇、慢病游客的材料和照护。
- 旅行社出境资质、领队、地接、紧急联系人。
- 当地交通、医疗、通信、支付、报警、使领馆协助和语言障碍。
- 汇率、漫游、境外刷卡、退税、购物争议和跨境售后。
- 涉及真实国家地区时必须查最新官方旅行、签证和安全资料。

## 14. 类型文用法

- 旅行治愈：景点不是明信片，而是人物在陌生规则下重新选择的场所。
- 都市言情：共同订票、误机、酒店取消、行李丢失、导游催集合都能制造关系压力。
- 家庭伦理：亲子游、老年团、谁订票谁付钱、老人购物被骗、孩子走失都能引发家庭责任。
- 综艺文：旅游路线、任务、赞助、拍摄许可、游客围观和平台舆论一起跑。
- 商战文：OTA、旅行社、景区、酒店、民宿、低价团、差评和投诉构成行业战场。
- 悬疑文：票务、酒店、景区监控、定位、导游通知、行李寄存、门票实名都是时间线证据。
- 重生文：主角能避坑，但要遵守票务、资质、合同和安全规则，不能用先知信息无成本套利。
- 古代/架空：不套现代旅游法，但可保留“路引、驿站、商队、客栈、向导、渡口、关卡、地陪、行程变更和旅途纠纷”的功能链。

## 15. 常见坏味道与修正

| 坏味道 | 读者体感 | 修正方法 |
| --- | --- | --- |
| 旅行像景点介绍 | 水、导游词 | 让路线改变人物选择、钱、时间和关系 |
| OTA 平台万能退款 | 权责混乱 | 分清平台、商家、旅行社、酒店、交通承运人 |
| 跟团游只有黑导游 | 扁平 | 写合同价格、购物安排、旅行社成本和导游压力 |
| 票务改签一句话解决 | 悬浮 | 写余票、时间、费用、实名核验和连锁订单 |
| 景区事故全靠主角救 | 不安全 | 写公告、封闭区域、救援、医疗和责任调查 |
| 投诉秒赔 | 过度爽 | 写材料、受理、调查、调解、转办或诉讼 |
| 出境游无证件 | 假 | 联动第 161 篇，写护照、签证、边检和当地规则 |
| 强制购物只靠吵架 | 证据弱 | 写合同、行程单、录音录像、小票、同行证言 |
| 民宿酒店重复写 | 边界不清 | 住宿细节转第 171 篇，本篇只管旅游组合责任 |
| 旅游后无余波 | 后劲弱 | 写差评、投诉、平台记录、保险、赔偿和关系变化 |

## 17. 给 AI agent 的生成提示词

起草旅游纠纷场景：

```text
请按第182篇“旅游出行、景区酒店旅行社、票务改签、导游服务与旅游纠纷链专项”写一场旅游纠纷戏。先填 tourism_service_scene_card，明确旅游产品、经营主体、合同订单、行程资产、票务状态、导游/领队、风险触发、证据链和投诉路径。场景必须让一个行程节点触发一个责任后果，不要只写游客和导游吵架。
```

审稿旅游线：

```text
请审稿本章旅游线是否悬浮。检查：旅行社/OTA/酒店/交通/景区主体是否分清，旅游合同和行程单是否存在，票务退改是否有规则，导游职责是否合理，强制购物是否有证据，投诉路径是否混乱，事故安全是否专业。逐项给出风险和修改建议。
```

写低价团翻车：

```text
请写一场低价团翻车戏。必须写清报名宣传、团费、行程单、购物点、自费项目、导游话术、游客证据、旅行社和平台态度、投诉路径和后续处罚或赔偿。不要把所有游客写成愚蠢，也不要提供黑团操作方法。
```

写票务改签连锁反应：

```text
请写一场旅游票务改签导致行程崩盘的戏。必须写清原票、改签原因、余票或舱位、费用、酒店/景区预约连锁影响、谁承担损失、平台或旅行社如何处理。不要让客服一句话恢复全部行程。
```

## 18. 自检清单

- 旅游目标是什么？度假、任务、探亲、综艺、调查、逃离还是商务顺路？
- 产品是自由行、跟团、半自由行、定制、研学、自驾还是出境？
- 合同或订单主体是谁？旅行社、OTA、平台内商家、酒店、交通承运人还是景区？
- 行程单是否写清交通、住宿、餐饮、景区、导游、购物和自费项目？
- 门票、机票、火车票、酒店和活动票的实名、预约、退改规则是否分开？
- 导游、领队、地接和司机是否有清楚职责？
- 是否涉及强制购物、低价团、行程缩水、酒店降级或虚假宣传？
- 证据链是否有宣传页、合同、行程单、付款、票据、聊天、照片视频和投诉记录？
- 投诉路径是平台、旅行社、旅游投诉、交通投诉、消协、诉讼还是报警救援？
- 涉及出境游时，护照、签证、入境、当地规则和使领馆协助是否查证？
- 场景结束后，钱、时间、路线、安全、关系、口碑或下一目标是否发生变化？

## 19. 最小规则

如果 AI 只能记住五条：

1. 旅游戏先看产品链：宣传、合同、行程、票务、导游、住宿、景区、退改投诉。
2. 旅行社、OTA、酒店、交通承运人、景区、导游和监管不是同一个角色。
3. 票务改签要写实名、余票、时间、费用和连锁订单，不要客服一键复原。
4. 强制购物和低价团要写合同、行程单、导游话术、购物票据和投诉处理。
5. 旅游场景的价值不是风景，而是陌生规则下人物选择、关系和风险的变化。

## 20. 主要来源

- 《中华人民共和国旅游法》，用于旅游者权益、旅游经营者义务、旅行社、导游、旅游安全、旅游合同和监督管理底座。
- 《旅行社条例》及《旅行社条例实施细则》，用于旅行社设立、经营范围、业务许可、服务网点、地接和旅游市场管理背景。
- 《在线旅游经营服务管理暂行规定》，用于 OTA、在线旅游平台、平台内经营者、交易记录、投诉处理、信息安全和产品服务责任。
- 《导游人员管理条例》《导游管理办法》和《导游等级考核管理办法》，用于导游证、导游执业、导游服务、强制购物、行程变更和职业能力背景。
- 文化和旅游部《旅游投诉处理办法》，2026 年 3 月 15 日施行，用于旅游投诉受理、处理、监督和保障。
- 《公共航空运输旅客服务管理规定》，用于航空客票销售、退票、变更、超售、旅客服务和投诉管理。
- 《铁路旅客运输规程》，用于铁路旅客实名、车票、乘车服务、退改投诉和铁路运输企业义务。
- 《道路旅客运输及客运站管理规定》，用于旅游客运、包车客运、客运站、安检和道路旅客运输安全背景。
- 《中华人民共和国民法典》合同编中的旅游合同规则，用于旅游合同履行、解除、转团、费用和违约责任底座。
