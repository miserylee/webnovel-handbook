## 1. 本专题解决的 AI 写作问题

很多 AI 写法院桥段时会犯以下错误：

1. 把法院写成“领导主持公道”的办公室，缺少诉状、证据、送达、调解、庭审、裁判和执行。
2. 把立案写成法官认可事实，把判决写成钱马上到账，把执行写成警察抓人。
3. 把诉前调解写成强迫和稀泥，忘了调解必须自愿、合法、可履行。
4. 把网上立案、在线开庭、电子送达写成万能便利，忘了身份核验、平台规则、程序同意和线下转换。
5. 把失信被执行人和限制消费写成羞辱式惩罚，忘了其前提是生效法律文书、执行程序和拒不履行风险。
6. 把“执行难”写成法院不作为，忽略查控、财产线索、执行和解、终本、恢复执行、异议复议和执行监督。
7. 把虚假诉讼、伪造证据、转移财产、规避执行写成聪明操作，破坏商业级现实感和安全边界。

本专题把基层法院写成一套有程序、有证据、有期限、有角色、有后果的叙事系统。

## 2. 总原则：法院不是情绪裁判器，而是程序化兑现权利的场

县域法院桥段的戏剧力量，不在“法官替主角骂反派”，而在：

- 私人冲突被转换成法律请求。
- 模糊口头约定被迫变成证据和争点。
- 熟人社会的面子被送达、开庭、裁判和执行重新排序。
- 胜诉不等于立刻拿钱，执行才是真正的兑现阶段。
- 法院能保护权利，也会暴露当事人的证据短板、道德灰区和策略成本。

标准链条：

```text
纠纷发生
-> 判断纠纷性质和法院管辖
-> 准备起诉材料、身份材料、证据和诉讼请求
-> 诉讼服务中心、网上立案或跨域立案提交
-> 登记立案、补正、不予受理或管辖处理
-> 诉前调解、多元解纷、司法确认或转立案审理
-> 送达、举证、答辩、保全、开庭或在线诉讼
-> 法庭调查、质证、辩论、调解可能
-> 判决、裁定、调解书或撤诉
-> 生效、自动履行或拒不履行
-> 申请执行、执行通知、财产报告、查控、冻结、扣划、拍卖、和解
-> 失信名单、限制消费、终本、恢复执行、执行异议或执行监督
-> 债务、信用、关系、名声、生意和家庭后果回写
```

AI agent 写作时，每个法院桥段至少要回答：

- 谁提出什么法律请求。
- 凭什么证据证明。
- 对方怎样答辩或反证。
- 法院当前处于哪个程序节点。
- 这个节点能产生什么结果，不能产生什么结果。
- 结果如何进入履行或执行。

## 3. 法律与制度底座

### 3.1 民事诉讼法：法院民事案件的主干程序

民事诉讼法提供起诉、受理、管辖、回避、证据、保全、调解、庭审、简易程序、小额诉讼、第二审、审判监督和执行的基础框架。2023 年修改决定已由全国人大常委会通过，自 2024 年 1 月 1 日起施行。

写作迁移：

- 民事纠纷不是“报官就判”，而是当事人提出请求、法院审查管辖和起诉条件、对方答辩、双方举证质证、法院裁判。
- 基层法院常处理婚姻家事、民间借贷、买卖合同、劳动争议后续诉讼、物业邻里、交通事故、侵权赔偿、小额合同和执行案件。
- 小额、事实清楚、权利义务明确的案件可以更快，但“快”不等于跳过送达、举证、审理和裁判。

### 3.2 登记立案：不是法官相信你，而是把案子纳入程序

登记立案制度强调依法、及时保障诉权。对起诉、自诉材料，法院要接收诉状、出具凭证、审查是否符合法定条件；符合条件的登记立案，不符合的说明和处理。

写作迁移：

- 立案不等于原告赢，也不等于法院认定事实成立。
- 立案窗口常见动作包括接收材料、核对身份、审查管辖、要求补正、缴费、生成案号、分流调解或转审判团队。
- 戏剧张力可以来自材料不齐、管辖争议、诉讼请求写错、证据与请求不匹配、对方躲避送达、是否走诉前调解。

### 3.3 一站式多元解纷和诉讼服务：法院门口不是只有开庭

最高人民法院推动一站式多元纠纷解决和诉讼服务体系，现实中基层法院往往把诉讼服务中心、诉前调解、在线调解、速裁快审、网上立案、跨域立案、12368 服务等整合在一起。

写作迁移：

- 当事人到法院后，第一站可能是诉讼服务中心，不一定马上见法官开庭。
- 适合调解的纠纷会进入诉前调解、特邀调解、行业调解或人民调解衔接。
- 调解成功后可能撤回起诉、出具调解协议、申请司法确认，或转成具有执行力的调解书。
- 调解失败后再进入正式审理，并且调解过程里暴露的底线、证据和关系会影响后续诉讼策略。

### 3.4 在线调解和在线诉讼：便利不是游戏化

在线调解规则和在线诉讼规则为身份认证、在线提交材料、在线调解、证据交换、在线庭审、电子送达、在线宣判等提供程序依据。

写作迁移：

- 在线诉讼有法律效力，但要尊重当事人程序权利、身份核验、平台操作和网络条件。
- 电子证据不是截图扔上去就赢，需要来源、真实性、完整性和关联性。
- 在线庭审可以压缩时间成本，也会制造新的戏剧点：老人不会操作、网络中断、代理权限、电子送达是否有效、证据上传格式、当事人是否同意在线方式。

### 3.5 执行制度：胜诉之后才进入兑现压力

生效法律文书确定义务后，一方不履行，另一方可以申请执行。执行阶段可能出现执行通知、报告财产、网络查控、冻结扣划、查封拍卖、执行和解、限制消费、纳入失信名单、终结本次执行、恢复执行、执行异议、复议和监督。

写作迁移：

- 判决不是自动转账，执行需要申请、立案和财产线索。
- 执行难不是单一“法院不干活”，可能是被执行人确无财产、财产登记复杂、第三人异议、企业经营困境、家庭共同财产边界或故意规避执行。
- 失信和限制消费要写清前提：生效法律文书、执行程序、拒不履行、违反财产报告或规避执行等。
- “失信”要区分“有履行能力而拒不履行”和“确无财产可供执行”的失能状态，避免把所有债务人都写成恶人。

## 4. 基层法院常见角色表

| 角色 | 叙事功能 | 常见压力 | 写作抓手 |
| --- | --- | --- | --- |
| 原告/申请执行人 | 提出请求、举证、推动程序 | 证据不足、成本、时间、对方拖延 | 诉讼请求从情绪变成数字和事实 |
| 被告/被执行人 | 答辩、反证、履行或抗拒履行 | 名声、资金、家庭、经营风险 | 是真没钱、争议未清，还是有能力拒不履行 |
| 立案窗口工作人员 | 接收材料、审查形式、补正引导 | 群众误解、材料混乱、管辖争议 | 冷静把吵闹变成清单 |
| 诉前调解员 | 促成和解、拆分诉求 | 双方不信任、熟人关系、履行保障 | 让情绪降温，让协议可履行 |
| 法官助理/书记员 | 程序安排、送达、笔录、材料流转 | 案多人少、送达难、当事人反复 | 程序的细节承压点 |
| 承办法官 | 审理、组织质证、主持调解、裁判 | 事实不清、证据冲突、舆论压力 | 不替任何一方演讲，只围绕争点判断 |
| 执行法官/执行员 | 查控、处置、催促履行、执行和解 | 财产难找、对抗执行、群体稳定 | “赢了怎么拿到” 的现实核心 |
| 律师/法律援助人员 | 帮助整理请求、证据和策略 | 代理权限、费用、沟通成本 | 把“我受委屈”改成“法院能支持什么” |
| 旁听群众/亲属 | 关系压力和乡土评价 | 面子、传言、亲情撕裂 | 裁判结果回到熟人社会 |
| 12368/诉讼服务平台 | 查询、联系、送达、服务 | 信息不对称、不会操作 | 程序可见性和数字化质感 |

## 6. 程序节点写法

### 6.1 起诉前：先把诉求从情绪翻译成请求

低质写法：

> “我要告他，让法院给我一个公道。”

高质写法：

> 主角在法律援助窗口被问住：要对方还多少钱，利息从哪天算，有没有转账记录，借条有没有写还款期限，起诉谁，管辖法院在哪。她第一次意识到，法院不审“他没良心”，法院审“这笔债能不能被证明”。

可用清单：

- 当事人身份和联系方式。
- 诉讼请求。
- 事实和理由。
- 证据目录。
- 管辖依据。
- 是否需要保全。
- 是否先经仲裁、复议或其他前置程序。

### 6.2 立案：程序入口，不是胜负判断

低质写法：

> 法院立案，所有人都知道反派完了。

高质写法：

> 立案窗口收了材料，给了回执，又让主角补一份被告身份信息和转账流水。案号出来那天，她只是获得了进入程序的资格。真正难的是，那个已经换了手机号的人，文书怎么送到。

可用节点：

- 现场立案。
- 网上立案。
- 跨域立案。
- 补正材料。
- 诉前调解分流。
- 不予受理或管辖异议。
- 缴纳诉讼费或申请缓减免。

### 6.3 诉前调解：不是和稀泥，而是低成本试探履行可能

诉前调解的商业价值很高，因为它既能保留人情，又能暴露底牌。

可写冲突：

- 一方只愿口头答应，不愿写清履行期限。
- 一方愿意分期，但要求撤诉后再还。
- 调解员要求把金额、日期、违约后果写清。
- 亲属在门口吵，调解室里反而只认票据。
- 调解成功但后续不履行，进入司法确认、调解书或执行。

风险边界：

- 不写调解员强迫当事人牺牲法定权利。
- 不写“你们各退一步”替代事实和法律。
- 不写调解协议含糊到无法执行。

### 6.4 审理：争点、证据和程序节奏比喊话重要

基层案件不一定都大庭广众、唇枪舌剑。很多案件的张力来自小证据、小矛盾和小人物的沉默。

审理链条：

```text
送达
-> 答辩
-> 举证期限
-> 证据交换
-> 开庭或在线庭审
-> 核对身份和权利义务告知
-> 法庭调查
-> 举证质证
-> 法庭辩论
-> 最后陈述
-> 调解可能
-> 裁判
```

写作抓手：

- 证人是亲戚，证明力被质疑。
- 转账备注含糊，对方说是合伙投资不是借款。
- 老人不会用手机，电子证据需要子女帮忙导出。
- 被告缺席不等于原告自动赢，法院仍要审查证据。
- 法官不负责替主角找证据，只在规则内组织审理和判断。

### 6.5 裁判与生效：判决不是剧情清零

裁判之后还有：

- 是否上诉。
- 文书是否生效。
- 是否主动履行。
- 是否分期履行。
- 是否申请执行。
- 是否出现新证据、再审申请或审判监督问题。

高质写法要把裁判后果回到生活：

- 村里人知道谁输了，但也知道谁拖了三年。
- 生意伙伴开始查信用和执行信息。
- 亲戚饭桌上不再谈钱，但每个人都记住了判决金额。
- 主角赢了官司，却发现对方名下没有可供执行财产。

### 6.6 执行：商业级现实感的关键

执行链条：

```text
申请执行
-> 执行立案
-> 执行通知和财产报告令
-> 网络查控和线索核查
-> 冻结、扣划、查封、评估、拍卖或交付
-> 履行、部分履行、执行和解
-> 拒不履行时限制消费、纳入失信或罚款拘留风险
-> 暂无财产时终结本次执行
-> 发现财产后恢复执行
-> 执行异议、复议或执行监督
```

戏剧重点：

- 申请执行人以为赢了就能拿钱，结果还要提供财产线索。
- 被执行人开着亲戚名下的车，执行员要判断是否可查可控。
- 小老板不还钱不是因为没钱，而是把款项挪到关联账户，触发信用和法律风险。
- 被执行人真没财产，胜诉人面对“法律文书确认了权利，但现实兑现仍需时间”。
- 执行和解既可能解决问题，也可能成为再次拖延的节点。

### 6.7 失信、限制消费和信用修复：不是羞辱，是履行压力

写作规则：

- 失信名单一般对应有履行能力而拒不履行、规避执行、违反财产报告、违反限制消费令等情形。
- 限制消费针对被执行人的高消费和非生活工作必需消费，不是把人从社会中抹掉。
- 企业被执行会影响招投标、融资、合作、市场信用和法定代表人压力。
- 自然人被限制消费会影响出行、子女高收费学校、酒店、旅游、商业合作等叙事后果。
- 履行完毕、执行和解履行完毕或符合法定情形后，应考虑删除失信信息、解除限制或信用修复。

不要写成：

- 主角随手把人“拉黑成老赖”。
- 法院把所有欠债人都公开羞辱。
- 被执行人教读者怎样转移财产、借名持有、逃避查控。
- 失信名单替代审判程序。

## 8. 高质量桥段示例

### 8.1 熟人借贷案：赢了判决才发现执行更难

低质写法：

> 法官看完聊天记录，当庭判反派还钱，反派立刻跪地转账。

高质写法：

```text
多年熟人借款
-> 主角拿着借条、转账、催款记录到法院
-> 立案窗口要求补被告身份信息
-> 诉前调解中对方承认借款但要求分期
-> 调解协议没履行
-> 转入审理并取得判决
-> 判决生效后对方仍不还
-> 申请执行
-> 查控发现账户余额少、名下车辆已转让
-> 执行员要求提供财产线索
-> 对方因有能力拒不履行被限制消费
-> 主角明白胜诉是确认权利，执行才是兑现权利
```

商业价值：熟人社会的“信任破产”与法律程序的“兑现难”叠加，主角成长不靠喊狠话，而靠学会把情绪变成证据和线索。

### 8.2 彩礼纠纷：两家人的脸面在调解室里变成清单

```text
订婚给付彩礼
-> 婚约解除
-> 双方亲属在村里互骂
-> 人民调解失败
-> 起诉到基层法院
-> 法官助理梳理给付项目、共同生活时间、登记情况、嫁妆、孕育、过错
-> 诉前调解尝试分项处理
-> 调解失败后开庭
-> 判决不是全退或不退，而是结合事实确定返还范围
-> 判后履行引发亲戚关系重排
```

关键：不要让法院替任何一方宣泄道德审判，要让事实和法律因素逐项浮出水面。

### 8.3 老宅通道纠纷：人民法庭把旧规矩写成可执行边界

```text
邻里因通道堵塞争吵
-> 村干部协调无果
-> 诉前调解进入人民法庭
-> 双方拿出老照片、宅基地资料和证人说法
-> 法官现场勘验或组织调解
-> 明确通行范围、修复方式和履行期限
-> 一方不履行时申请执行
-> 执行不是惩罚邻居，而是把可执行内容落到现场
```

关键：县域法院不是只在法庭里，它也可能通过人民法庭、巡回审判、现场勘验把法律带回乡土空间。

### 8.4 小额合同在线诉讼：快，但不是省略程序

```text
网店供货纠纷
-> 原告网上立案
-> 上传合同、发货单、签收记录、聊天记录和付款流水
-> 法院电子送达
-> 被告在线提交答辩称货物有瑕疵
-> 在线证据交换
-> 小额或简易程序审理
-> 法官围绕验收、质量异议和欠款金额发问
-> 判决后进入自动履行或申请执行
```

关键：数字化让程序更便捷，不让程序消失。

### 8.5 执行异议：第三人突然出现，爽感必须让位给权利边界

```text
申请执行人要求拍卖一辆车
-> 被执行人称车不是他的
-> 第三人拿出购买合同和付款记录提出执行异议
-> 执行程序暂停或审查相关权利
-> 申请执行人怀疑借名转移财产
-> 法院围绕真实权属、付款、占有使用和时间线审查
-> 结果可能继续执行、解除控制、另行诉讼或监督纠错
```

关键：执行不是“谁嗓门大谁拿走财产”，涉及第三人权利时，程序会变慢，但可信度会变强。

## 9. 与既有文档的联动

- 与 `204`：先看纠纷是否适合人民调解、司法所、法律援助、诉前调解或人民法庭；进入法院内部程序后调用本篇。
- 与 `141`：需要写庭审质证、证据证明力、法庭调查和裁判理由时调用第 141 篇；本篇负责基层法院场景和判后执行。
- 与 `166`：涉及报警、刑事报案、公安调查和刑事程序时调用第 166 篇；普通民事执行不能写成公安随意抓人。
- 与 `202/203`：婚恋彩礼、赡养老年、继承家庭争议可由本篇提供诉讼和执行出口。
- 与 `213/214`：消费维权、电诈追赃、银行卡冻结等情节进入法院或执行阶段时，本篇提供法院程序底座。
- 与 `227`：行政复议失败或直接行政诉讼进入法院时，本篇只处理法院节点，行政行为合法性审查仍看第 227 篇。

## 10. 安全边界

允许写：

- 法院程序、角色分工、材料清单、证据逻辑和裁判后果的概念性描写。
- 当事人因不懂程序、不会举证、送达困难、执行难而产生的戏剧冲突。
- 法官、调解员、律师、执行员依法履职时的压力和边界。
- 虚假诉讼、伪造证据、转移财产、规避执行造成的法律后果。

不要写成教程：

- 如何伪造借条、聊天记录、转账凭证、送达地址或证人证言。
- 如何通过虚假诉讼转移财产。
- 如何隐藏、转移、变卖、借名持有或规避法院查控。
- 如何逃避限制消费、失信名单、财产报告或执行传唤。
- 如何威胁法官、调解员、证人、申请执行人或执行人员。
- 如何买通法院、串通鉴定、操纵送达或干扰庭审。

写反派可以触碰这些行为，但必须呈现其风险、证据痕迹和后果，不能提供可模仿步骤，也不能奖励规避司法。

## 11. 审稿检查问题

1. 这个法院桥段是否先明确了纠纷性质、诉讼请求、当事人和管辖？
2. 立案是否被写成程序入口，而不是胜负判断？
3. 是否区分了诉前调解、人民调解、司法确认、调解书和判决？
4. 调解是否自愿、合法、具体、可履行，而不是强迫和稀泥？
5. 审理是否有送达、答辩、举证、质证、争点和裁判逻辑？
6. 在线诉讼是否写出身份核验、电子材料、平台限制和程序效力？
7. 判决后是否写出生效、履行、上诉或申请执行，而不是判完清零？
8. 执行是否写出申请、查控、财产线索、和解、限制消费、失信、终本或恢复执行？
9. 失信和限制消费是否有前提，是否区分失信与失能？
10. 是否避免把公安、法院、司法所、信访、纪检混成一个万能窗口？
11. 是否避免伪造证据、虚假诉讼、转移财产、规避执行和买通法院的教程化写法？
12. 判后是否有关系、信用、债务、家庭、经营和名声的后果回写？
