# 250 县域住房保障、公租房、保障性租赁住房、老旧小区改造、棚户区城中村改造与居住更新专项

适用对象：需要写县域住房保障、公租房、租赁补贴、保障性租赁住房、青年人才公寓、新市民住房、进城务工人员居住、老旧小区改造、棚户区改造、城中村改造、回迁安置、居民协商、改造资金、项目验收、住房困难家庭和住建部门工作的 AI agents。

本篇不是普通租房买房专题，也不是物业邻里纠纷专题。它负责把“住有所居”写成资格、程序、项目、资金、群众协商和长期管理的现实链条。

## 1. 本篇解决什么问题

AI 写住房保障和居住更新常见问题：

- 把公租房写成领导一句话分房，忽略申请条件、审核、公示、轮候、配租、租金、退出和动态管理。
- 把保障性租赁住房写成普通低价出租房，忽略新市民、青年人、产业园区、职住平衡、房源筹集和运营管理。
- 把老旧小区改造写成刷墙换门头，忽略居民意愿、加装电梯、管线改造、停车、养老托育、资金分担和后续维护。
- 把棚户区或城中村改造写成拆了就发财，忽略土地房屋权属、安置补偿、租户和小商户、过渡期、债务、公共服务和社区关系断裂。
- 把住建部门写成万能分房机关，忽略民政、人社、公安、不动产、财政、街道社区、国企平台、开发建设单位和群众监督。
- 把困难群众的居住问题写成单纯贫穷，忽略家庭结构、户籍、就业、收入、病残、教育、通勤、照护和尊严。

本篇目标：让 AI 在写住房保障和城市更新时，先拆清“谁有居住困难、依据什么申请、谁审核公示、房源从哪里来、钱怎么流、居民怎么参与、改造后谁维护”。

## 3. 制度锚点

写现实住房保障和居住更新时，至少记住：

1. 公租房面向符合条件的住房困难家庭、新就业无房职工、外来务工人员等，核心不是“谁可怜谁拿房”，而是条件、审核、公示、轮候、配租和退出。
2. 租赁补贴和实物配租是不同工具。没有房源时，补贴也可能成为过渡安排。
3. 保障性租赁住房重点解决新市民、青年人等群体阶段性住房困难，常见于产业园区、城市新区、存量房改造和企事业单位自有土地房源。
4. 老旧小区改造不是美化工程，基础类改造常涉及水电气热路、通信、消防、安防、停车、无障碍、养老托育、加装电梯和后续管理。
5. 棚户区、城中村改造会触及房屋权属、租赁关系、经营门面、过渡安置、补偿标准、回迁时间、债务资金和社区网络重组。
6. 住房保障和城市更新的冲突常在“公平”和“效率”之间：急需住房的人等不起，公示和审核又不能跳过；居民想改善环境，却不一定同意出钱、搬迁或改变生活方式。
7. 政策口径高度地方化。小说可以使用通用制度链，但具体资格、面积、租金、补贴金额、轮候规则和改造流程必须设定为“本地政策”。

## 4. 写作安全边界

可以写资格争议、项目腐败、优亲厚友和纠纷调查，但不要提供可执行违法方法。

禁止正向细化：

- 伪造收入、婚姻、户籍、居住证、社保、房产、残疾、低保、就业或无房证明来骗取保障房。
- 冒名申请、借名占房、转租转借公租房、空置占用、隐瞒收入房产变化的操作步骤。
- 伪造居民同意、补偿协议、房屋面积、评估报告、施工验收、审计材料和资金拨付凭证。
- 强拆、暴力逼迁、断水断电逼迁、恶意堵工、破坏施工和组织群体冲突的方法。
- 套取棚改、老旧小区改造、保障房建设、租赁补贴或维修资金的流程技巧。
- 非法调取或泄露申请人家庭收入、婚姻、房产、病残、低保和个人身份信息。

可写方向：资格核验、公开公示、群众解释、复核申诉、合同清退、补偿协商、审计巡察、工程整改和居住尊严修复。

## 5. 核心链条

### 5.1 住房保障链

`居住困难出现 -> 判断家庭或个人身份 -> 查询本地保障政策 -> 准备申请材料 -> 街道社区或窗口受理 -> 住建、民政、人社、公安、不动产等信息核验 -> 公示和异议处理 -> 轮候或配租/补贴 -> 签合同、缴租金、入住 -> 动态复核 -> 退出、续租、清退或救济 -> 家庭生活和公共信任回写`

### 5.2 居住更新链

`小区或片区问题暴露 -> 纳入改造计划或储备 -> 居民意愿征询 -> 方案设计和资金测算 -> 部门会商和项目审批 -> 招标采购或实施主体确定 -> 搬迁过渡或施工组织 -> 现场施工、监督和变更 -> 验收、资金结算和审计 -> 后续物业、业委会、运营维护和居民关系回写`

写作时最少回答：

- 人物是申请保障、等待轮候、被清退、参与改造、被搬迁安置，还是监督项目？
- 住房困难来自低收入、无房、就业迁移、家庭病残、灾后、离婚、进城务工、毕业就业还是照护压力？
- 房源是什么：公租房、保障性租赁住房、人才公寓、租赁补贴、回迁房、安置房、改造后住房？
- 权力和证据在哪里：申请表、公示名单、合同、会议记录、评估报告、补偿协议、施工验收、资金拨付还是群众投诉？
- 结局是改善居住、继续轮候、资格取消、补贴发放、搬迁过渡、项目停工、审计问责还是社区重建？

## 7. 常用场景模型

### 7.1 公租房申请和公示异议

可用桥段：一个外来务工家庭申请公租房，公示后有人举报他们“家里有车”。主角不能只靠同情判定，而要核对本地政策、收入、车辆性质、家庭成员、现住房、社保居住证和异议处理流程。

写法要点：

- 公示不是羞辱，而是保障公平，但要保护个人隐私边界。
- 车辆、商铺、离婚、成年子女、农村宅基地、共有房产都可能引发资格争议。
- 审核结果要能解释，不能靠干部一句“符合”。
- 真正的情绪是：一个家庭需要房子，另一个等待家庭也害怕被插队。

### 7.2 公租房清退和转租

可用桥段：公租房住户收入改善后长期不退，房子还被转租给亲戚。管理单位发清退通知，老人说自己当年等了五年才住进来。主角要把合同、复核、家庭变化和后面排队的人一起写出来。

写法要点：

- 清退不是冷血，公共房源长期占用会伤害轮候家庭。
- 也要写困难过渡：孩子上学、老人就医、市场租金压力、搬家成本。
- 证据包括合同、复核通知、租金记录、实际居住核查、水电使用和转租线索。

### 7.3 保障性租赁住房和青年居住

可用桥段：县里为产业园青年工人建设保障性租赁住房，房租低但离厂区近，生活配套不足。年轻人不是只要便宜，还要通勤、安全、网络、厨房、社交和未来选择。

写法要点：

- 保障性租赁住房适合写新市民、青年人、产业工人、毕业生和基层服务人员。
- 冲突不是“有房住就该感恩”，而是低租金、质量、通勤、公共服务和尊严的平衡。
- 企业、园区、政府平台、运营方和租户之间责任要分清。

### 7.4 老旧小区改造和加装电梯

可用桥段：老旧小区要改造，低楼层不同意加装电梯，高楼层老人下不了楼，商户担心施工影响生意，施工方又说管线太旧要变更预算。

写法要点：

- 老旧小区改造不要只写刷墙，重点是水电气热、排水、消防、道路、照明、无障碍、养老托育和后续维护。
- 加装电梯冲突要写楼层利益差异、采光噪声、出资比例、维护费用和合法程序。
- 居民参与不是一次举手，可能有多轮征询、方案调整、公示和投诉。

### 7.5 棚户区改造和过渡安置

可用桥段：棚改片区签约率卡在最后几户，已搬走的人等回迁，没签的人觉得评估不公，租户和小商户担心没有补偿。主角要看房屋权属、评估、补偿方案、过渡费、回迁时间和公共服务安置。

写法要点：

- 棚改不是“拆迁暴富”模板，可能伴随漫长等待、租房过渡、债务压力和社区关系断裂。
- 租户、小商户、无证建筑、共有产权和继承未分割都会制造复杂冲突。
- 不要用暴力逼迁制造爽点，真正强的戏是把方案、证据和利益重新摆上桌。

### 7.6 城中村改造和新市民空间

可用桥段：城中村改造启动，房东期待补偿，租客担心失去低价住处，外卖员、保洁、陪读家庭和小吃店一夜之间不知道搬去哪。县里宣传城市更新，主角看到的是低成本居住空间被重构。

写法要点：

- 城中村既有安全、消防、环境和治理问题，也承载低收入租住、就业和小商业生态。
- 改造不能只写业主和开发商，还要写租户、商户、学校、通勤、市场和社区服务。
- 好的剧情不是反对改造，而是追问“改造后的城市还能不能容纳这些人”。

### 7.7 改造工程质量和资金审计

可用桥段：老旧小区改造半年后外墙渗水、路面开裂，居民怀疑偷工减料。主角不是靠怒骂施工队，而是调取设计图、招标合同、变更单、验收记录、监理日志、支付凭证和质保责任。

写法要点：

- 工程问题要分清设计缺陷、施工质量、材料问题、居民私改、自然损耗和维护缺失。
- 资金线要接入预算、招投标、验收、审计和绩效评价。
- 居民信任修复比处罚一个包工头更重要。

## 8. 证据链清单

写住房保障和居住更新时，可从以下材料中选择：

- 住房保障申请表、身份证户口簿、居住证、婚姻材料、收入证明、社保证明、无房证明、不动产查询结果。
- 低保、残疾、优抚、就业、人才、毕业证、劳动合同等分类保障材料。
- 受理回执、审核意见、公示名单、异议处理记录、轮候排序、配租通知、摇号或选房记录。
- 公租房租赁合同、租金缴纳、入住验收、维修记录、续租复核、清退通知和实际居住核查。
- 保障性租赁住房项目认定、房源筹集、租金标准、运营合同、租户名册和投诉记录。
- 老旧小区改造居民意愿征询、方案公示、会议记录、设计图、预算、招标采购、施工日志、监理记录、验收材料。
- 棚改、城中村改造的房屋权属、面积测绘、评估报告、补偿安置方案、签约协议、过渡费、回迁房源和搬迁记录。
- 资金拨付、专项资金、专项债、银行贷款、平台公司融资、审计报告、绩效评价和整改报告。
- 12345 工单、信访材料、政府信息公开答复、行政复议诉讼材料、纪检巡察和审计处理文书。

```yaml
  conflict_hook: "公平轮候、资格争议、房源稀缺、低租金和尊严、居民不同意、补偿不均、过渡太久、工程质量翻车"
  consequence: "配租、补贴、清退、复核、方案调整、停工整改、回迁延期、资金追回、问责、社区关系重建"
  safety_note: "不得提供伪造材料骗取保障房、转租转借公租房、强拆逼迁、套取改造资金、伪造验收或泄露申请人隐私的具体方法。"
```

## 10. 审稿检查问题

- 这段住房线是普通租房买房，还是公共住房保障、城市更新或安置补偿？
- 申请人或居民的身份、家庭、收入、住房和本地政策口径是否清楚？
- 公租房是否写出申请、审核、公示、轮候、配租、合同、复核和退出？
- 保障性租赁住房是否区别于普通市场租房和人才福利房？
- 老旧小区改造是否超越刷墙美化，写到基础设施、居民意愿、资金和后续维护？
- 棚户区或城中村改造是否写到权属、评估、补偿、过渡、回迁和租户商户影响？
- 是否把住建、街道、民政、人社、不动产、财政、审计和实施单位职责混成一个万能机关？
- 是否有公示、合同、会议、评估、验收、资金和投诉证据？
- 如果写资格造假、优亲厚友、工程腐败，是否写出核查、清退、问责和补救，而不是把违规写成爽点？
- 是否保护申请人家庭贫困、病残、婚姻、收入、房产和身份隐私？

## 11. 人味写法提示

住房保障最好看的地方，不是“拿到钥匙”本身，而是钥匙背后的人生重量。

可写细节：

- 公示栏前，两户人家同时盯着名单，一个哭，一个沉默，因为轮候顺序只差一位。
- 老人住在六楼，听到加装电梯又要开会，先问的不是多少钱，而是自己还能不能等到。
- 青年工人搬进保障性租赁住房，第一晚最开心的是终于不用在厂外黑中介那里压一个月工资。
- 棚改片区的理发店搬走后，老人找不到剪头的人，才意识到改造拆掉的不只是房子。
- 社区干部入户核查时，看见申请人把所有药盒收进抽屉，因为不想让邻居知道家里有人长期病。
- 回迁房钥匙发下来，邻居站在新楼道里互相认不出门牌，旧巷子的关系要重新长出来。

这些细节能让住房线从政策说明变成家庭尊严、城市变化和基层公平的现场。

## 12. 反面写法

不要这样写：

- “领导看主角可怜，当场批了一套公租房。”
- “公租房申请只要交身份证就行。”
- “老旧小区刷了新墙，居民全都满意。”
- “拆迁户一夜暴富，所有人都开心。”
- “城中村改造后，低价租房需求自然消失。”
- “工程质量有问题，主角骂施工队，第二天全部返工。”

改成：

- 住房保障要走申请、审核、公示、轮候和配租。
- 材料链要涉及收入、住房、户籍居住、社保就业和本地政策。
- 老旧小区改造要写居民协商、管线设施、出资和后续维护。
- 拆迁安置要写权属、评估、补偿、过渡和回迁时间。
- 城中村改造要看租户、商户、低成本居住和公共服务承接。
- 工程质量要靠合同、验收、监理、审计和整改责任。

## 13. 可复用桥段

### 桥段 A：名单上的一位之差

公租房配租名单公示，主角朋友排在最后一位，另一个家庭被举报名下有车。社区大厅里，两边都不是坏人。一个怕被插队，一个怕再次搬家。主角查清车辆、收入、房产和政策口径后，让公平比同情更有力量。

### 桥段 B：六楼老人和一楼反对票

老旧小区要加装电梯，六楼老人拄拐来开会，一楼住户担心采光、噪声和房价。主角不站队喊口号，而是把方案、补偿、维护费、施工影响和安全论证摆出来，让居民从互骂进入谈判。

### 桥段 C：搬走的早餐摊

城中村改造启动，房东谈补偿，租户找房，小吃摊老板不在补偿名单里。主角发现真正的城市温度在低价空间和就业网络里，改造方案必须回答“这些人明天去哪”。

### 桥段 D：刷墙后的漏水

老旧小区改造完上了新闻，雨季一来外墙渗水。居民以为又是面子工程。主角调设计、合同、变更、验收和监理日志，发现预算调整时删掉了关键防水工序。爽点不是骂人，而是让整改有证据。
