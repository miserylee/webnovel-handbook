## 1. 核心原则：风格手册不是“漂亮文风”，而是长篇声音台账

AI agents 最容易把风格理解成“多用某类修辞”“像某个作者”“更热血/更细腻”。这会导致三个问题：

- 人物声音被口头禅替代，所有人只是在不同位置说同一种 AI 句子。
- 修订时把有用的粗糙、停顿、偏见、迟疑和社会位置全部改平。
- 续写时忘记前文已经建立的称呼、语气、专有名词和叙述距离。

项目风格手册要记录的是可执行选择：

- 叙事声音：旁白知道多少、离角色多近、能不能吐槽、什么时候克制。
- 文本温度：句子长短、段落密度、修辞强度、幽默感、情绪外放程度。
- 角色声音：每个重要人物如何选词、绕开什么、在压力下如何漏出真心。
- 名词与格式：人名、称呼、地名、术语、法宝名、组织名、标点、内心独白格式。
- 禁区：哪些表达虽然“通顺”，但会破坏本书的读者承诺或人物质感。

一句话原则：风格手册不是让 AI 复刻几句样例，而是让 AI 在新场景里做同一套叙事选择。

## 2. 什么时候建立或更新风格手册

必须建立：

- 开新书并完成项目简报后。
- 第一章或前三章试写后，已经出现可保留的叙述声音。
- 主要人物首次完成场景级登场后。
- 准备让另一个 agent 接手续写、审稿或修订前。
- 进入新卷、换地图、换视角、加入重要新角色前。

必须更新：

- 读者或编辑明确反馈“人物声音好”“这个角色好记”“这一段文风对了”。
- 改稿后保留了新的称呼、口吻、叙述距离或专有术语。
- 某个角色发生关系转折，导致他对另一个人的称呼、语气、遮掩方式改变。
- AI 审稿发现人物同声、旁白漂移、古今语混乱、术语不一致。
- 作者决定调整目标平台、频道、题材气质或叙述视角。

不必更新：

- 一次性的漂亮句子。
- 尚未被正文验证的候选口癖。
- 只在单章临时使用的气氛描写。
- 用户还没确认、可能被删掉的试写方向。

## 3. 项目风格手册的十个字段

### 3.1 类型与读者声音承诺

记录读者打开这本书时预期的文本体验。

可记录：

- 目标平台/频道：起点男频、番茄女频、晋江言情、短篇故事、轻小说感等。
- 类型气质：热血、轻松、悬疑、甜宠、爽燃、沙雕、现实、克制、疯感。
- 读者预期：快节奏、强代入、强情绪、强设定、强关系、强反转。
- 文本底色：利落、密集、口语、冷幽默、古典、烟火气、网感强。

风险：如果不记录这一层，AI 会按通用审美润色，把男频爽文改成文艺腔，把轻松文改成正剧腔，把古言改成现代职场话术。

### 3.2 叙事视角与叙述距离

记录旁白和角色的距离。

可记录：

- 视角：第一人称、紧贴第三人称、多视角、全知叙事。
- 叙述距离：紧贴角色感受 / 中距离观察 / 旁白有独立态度。
- 信息规则：旁白能不能提前透露真相；能不能替角色解释心理；哪些信息必须等角色行动后揭示。
- 叙述偏见：旁白是否会被主角误判污染；是否允许读者看出主角自欺。

执行规则：

- 第一人称和紧贴第三人称中，叙述段落是“半角色声音”，对白是“全角色声音”。
- 中距离旁白可以更清楚，但不能替人物下判词。
- 全知叙事必须写清“能透露什么、必须隐藏什么”，否则很容易剧透或谜语人。

### 3.3 文本温度与句法偏好

记录正文的基本节奏。

可记录：

- 句长：短句推进 / 中短句为主 / 可接受长句铺陈。
- 段落：手机端短段 / 情绪段可稍长 / 信息段必须拆分。
- 修辞：克制物件化 / 可用华丽比喻 / 喜剧夸张 / 悬疑冷感。
- 情绪外放：内敛、直给、疯批、热血、甜腻、毒舌。
- 节奏禁区：连续解释、排比口号、段尾总结、全员震惊、万能转折词。

检查问题：把本书任意一段删掉人名后，读者还能感到这本书的类型温度吗？

### 3.4 词汇层与时代/地域/职业边界

记录哪些词属于本书，哪些词会破坏沉浸。

可记录：

- 常用词层：现代口语、古风词、行业术语、江湖黑话、校园话术、平台网感。
- 禁用词层：不合时代的现代词、不合身份的高级词、翻译腔、论文腔、短视频营销腔。
- 地域/时代词：称谓、货币、单位、时间说法、官职、宗门/公司/家族称呼。
- 职业词：医生、律师、警察、修士、主播、编辑、网约车司机等各自会自然使用的词。

风险：角色声音常常不是“说话方式”漂了，而是词汇层漂了。一个古代侍女突然说“情绪价值”，一个低阶弟子突然说“资源配置效率”，都会让读者出戏。

### 3.5 对话系统

记录本书如何写对白。

可记录：

- 对白密度：对话驱动 / 动作对白混合 / 旁白推动为主。
- 对白长度：一句多短；什么时候允许长台词。
- 对白标签：多用动作拍 / 多用“他说” / 尽量少用花哨说话动词。
- 潜台词强度：直说、试探、讽刺、遮掩、谈条件、抢解释权。
- 群戏规则：谁先反应，谁补刀，谁沉默，谁转移话题。

硬约束：对白不是现实聊天复刻，而是压缩后的有目的说话。每一句对白至少承担以下一种功能：推进目标、制造阻力、暴露关系、遮掩信息、改变压力、引出选择、制造回报。

### 3.6 角色声音矩阵

每个重要人物记录一张声音卡。声音卡不是人物卡的重复，而是“这个人如何把自己说出来”。

字段：

```markdown
## 人物名：声音卡

- 角色位置：主角 / 对手 / 关系核心 / 搞笑担当 / 信息源 / 阻力节点
- 声音基调：冷、快、软、绕、硬、疯、懒、客气、攻击性、业务化等
- 第一关注点：利益 / 规则 / 风险 / 感情 / 面子 / 证据 / 秩序 / 生存 / 效率
- 词汇来源：职业、地域、阶层、年龄、教育、宗门/公司/家族环境
- 句子形状：短句、长句、反问、半句、命令句、比喻、数据、脏话、敬语
- 社交策略：直说、装傻、讨价还价、先夸后刺、先道歉后逼问、讲笑话卸压
- 压力漏点：紧张时变礼貌 / 变尖刻 / 话变少 / 话变多 / 开始算账 / 开始护物件
- 回避内容：不谈亏欠、不承认害怕、不说喜欢、不说没把握、不承认嫉妒
- 关系变体：对主角、上级、下属、敌人、亲密对象分别怎么说话
- 禁止写法：不能说哪些词；不能被写成哪类通用人设
- 已验证样例：1-3 句短样例，只作校准，不作复制
```

角色声音要能回答三个问题：

- 这个人看见同一件事，第一眼会注意什么？
- 他想达成目的时，习惯用什么说话策略？
- 压力变大时，他的语言会怎么失控或变形？

### 3.7 专有名词与样式表

记录需要长期一致的文字选择。

可记录：

- 人名、别名、外号、尊称、蔑称、错称。
- 地名、组织名、职位名、法宝名、系统名、技能名、副本名。
- 术语拼写、繁简、大小写、连字符、编号格式。
- 章节名规则、卷名规则、时间单位、货币单位、计量单位。
- 内心独白、传讯、弹幕、系统提示、聊天记录、公告、账册等特殊格式。

用法：专有名词一旦进入正文，必须同步进样式表。后续 agent 不得擅自改同义词、改称呼或改格式；如果确实要改，必须写入决策记录或项目台账。

### 3.8 关系称呼与称谓变化

中文网文里，称呼本身就是关系进度条。

可记录：

- 初识称呼、公开称呼、私下称呼、冲突时称呼、亲密后称呼。
- 上下级、师徒、宗门、家族、职场、宫廷、江湖、校园的称谓规则。
- 误称、旧称、故意不用称呼、突然改称呼的剧情含义。

检查问题：如果一个人物突然改称呼，正文是否知道这是关系变化，而不是 AI 随手换词？

### 3.9 可保留的粗糙与缺陷

商业级文本不等于每句话都光滑。某些不完美恰好是声音。

可保留：

- 人物紧张时重复半句。
- 地域、职业、阶层带来的不标准表达。
- 主角视角中的偏见、误判、幼稚、自我辩解。
- 喜剧角色的过度认真、错比喻、坏修辞。
- 情绪场景中的停顿、词穷、绕开正题。

风险：AI 修订最容易把这些都改成“清晰、完整、正确”的句子，结果人物变成说明书。

### 3.10 禁区与不可模仿项

记录本书不该出现的表达。

可记录：

- 禁用句式：频繁“不是 X，而是 Y”、段尾口号、万能顿悟、解释性总结。
- 禁用词气：霸总油腻腔、短视频标题腔、翻译腔、论文腔、古偶空话、职场 PPT 腔。
- 禁用人物处理：所有角色都聪明毒舌、所有人都准确理解主角、所有人都同步震惊。
- 禁用风格模仿：不要复制竞品句子，不要套某作者标志性表达，不要把样例当素材库。

## 4. 角色声音七轴

AI agents 给人物写台词前，先用七轴定位，不要先想口头禅。

| 轴 | 要问的问题 | 错误写法 | 可执行写法 |
| --- | --- | --- | --- |
| 信息优先级 | 他第一眼看什么？ | “他说话冷静” | 先问证据来源、权限、代价或谁在场 |
| 词汇来源 | 他的词从哪里来？ | “古风一点” | 账房说账、医者说脉、修士说因果、主播说节奏 |
| 句子形状 | 他如何组织句子？ | “短句酷” | 紧张时只给结论；撒谎时补很多限定条件 |
| 社交策略 | 他怎么达成目的？ | “会怼人” | 先装糊涂，让对方把关键话说出口 |
| 情绪漏点 | 压力下哪里破功？ | “生气地说” | 突然改敬称、开始算旧账、把话说得过分客气 |
| 回避内容 | 他不愿承认什么？ | “嘴硬” | 不说害怕，只说“流程还没走完” |
| 关系变体 | 他对不同人是否不同？ | “统一人设” | 对上级用规矩，对主角用刺，对下属用命令 |

七轴的目标不是让每句话都花哨，而是让人物在普通信息句里也有自己的判断方式。

## 5. 样例句的使用边界

风格样例有用，但危险。AI 很容易把样例变成复读。

样例使用规则：

- 每个角色只保留 1-3 句短样例。
- 每句样例必须标注功能：威胁、安抚、撒谎、求助、嘴硬、表白、压价、遮掩。
- 样例只校准声音，不允许直接复制进新章节。
- 如果样例来自已发布正文，后续改写不能破坏其事实和关系语境。
- 如果样例来自竞品或外部作品，只能抽象成规则，不得保留可识别句式。

错误用法：

```text
这个角色说话像样例 1，请多写这种句子。
```

正确用法：

```text
这个角色在压力下会变得过度礼貌，先确认流程和见证人，再把对方逼到公开表态。新台词必须保留“礼貌逼迫”的策略，但不能复用原句。
```

## 6. 旁白声音与角色声音要分层

很多 AI 文的问题不是人物台词不对，而是旁白抢了人物的声音。

### 第一人称

旁白就是角色的一部分。记录：

- 这个“我”愿意承认什么，不愿承认什么。
- 这个“我”如何误判别人。
- 这个“我”会用什么词描述世界。
- 这个“我”是否会自嘲、毒舌、嘴硬、装轻松。

风险：第一人称如果过于客观，就像上帝借主角嘴写设定。

### 紧贴第三人称

旁白应被视角人物轻度染色。

执行：

- 描写优先选视角人物会注意的东西。
- 信息解释不能超过视角人物此刻能理解的范围。
- 情绪不直接判词，尽量通过他看见什么、漏掉什么、误读什么呈现。

### 多视角

每个视角人物都需要叙述过滤器。

记录：

- A 视角注意权力和秩序。
- B 视角注意人情和安全。
- C 视角注意钱、效率和漏洞。

如果换视角后只换人名不换观察逻辑，说明视角声音失败。

### 全知或中距离旁白

必须记录旁白规则。

可记录：

- 旁白是否允许幽默。
- 旁白是否允许提前总结人物命运。
- 旁白何时可以解释设定，何时必须让场景证明。
- 旁白是否可以评价人物，还是只陈列后果。

## 7. 对话不是聊天，是带目的的行动

对话要像动作一样改变场面。尤其是移动端阅读里，水对话会快速消耗耐心。

每句对白至少承担一种功能：

- 推进目标：让人物更接近或远离目标。
- 制造阻力：改变对方选择成本。
- 暴露关系：显示谁欠谁、怕谁、瞧不起谁、想保护谁。
- 遮掩信息：说一半，转移焦点，让读者察觉背后有东西。
- 逼迫表态：让对方必须公开选择。
- 改变压力：把私人问题变成公开问题，把小事变成不可退让。
- 提供回报：给读者爽点、笑点、糖点、线索或情绪落点。

删除规则：

- 如果一句对白删掉后，目标、关系、信息、压力都不变，优先删。
- 如果两个人连续三轮只是在确认已知信息，合并或改成行动。
- 如果所有角色都能互换这句台词，回到角色声音七轴重写。

## 8. 多 agent 协作时的声音交接

多个 AI agents 接力写长篇时，风格手册是交接文件，不是装饰文档。

交接前必须提供：

- 当前项目风格手册。
- 当前卷或近 5 章的文本温度变化。
- 主要角色声音卡。
- 已确认称谓、术语和特殊格式。
- 最近新增但尚未完全稳定的候选声音规则。
- 禁止改平的有效表达。

接手 agent 必须做：

- 先读风格手册，再读上一章。
- 如果上一章和风格手册冲突，标出冲突来源，不直接覆盖。
- 起草后输出“风格手册更新项”：新增称呼、术语、声音变化、需要废弃的候选规则。
- 修订时保留角色有效缺陷，不把所有句子修成同一种清楚。

## 9. 声音漂移诊断

出现以下现象时，优先检查风格手册，而不是继续润色句子。

### 人物同声

表现：

- 所有人都说同长度句子。
- 所有人都会精准总结局势。
- 所有人都有同一种幽默感。
- 所有人都使用同一批连接词和抽象词。

修法：给每个角色重新标注信息优先级、词汇来源、社交策略和压力漏点。

### 旁白漂移

表现：

- 古代背景出现现代运营、心理学或职场管理腔。
- 轻松文突然变成严肃评判。
- 爽文高潮被文艺化总结削弱。
- 悬疑文旁白提前解释谜底。

修法：回到叙事距离和词汇层，限制旁白能说什么、不能说什么。

### 口癖泛滥

表现：

- 每个角色都被一个固定词替代。
- 口头禅出现频率太高，变成标签而不是人物。
- 角色只要出场就重复招牌动作。

修法：把口癖改成“策略”。例如不是每次都说“有意思”，而是每次先找到规则漏洞再轻描淡写地试探。

### 修订改平

表现：

- 原本有迟疑、误会、抢话，改后全变成完整说明句。
- 原本有职业/地域/阶层词，改后全是标准普通话书面语。
- 原本有情绪停顿，改后被解释成心理总结。

修法：在风格手册里标注“可保留粗糙”，让修订只解决歧义和无效重复，不消灭声音。

## 10. 审稿时的声音检查

审稿报告中，如果涉及人物或文本执行，必须回答：

- 不看名字，关键人物的台词能否区分？
- 不看章节标题，文本温度是否符合目标平台和题材？
- 旁白有没有泄露角色不该知道的信息？
- 对话是否推动剧情，还是只是聊天和解释？
- 称呼、术语、格式是否前后一致？
- 是否有被 AI 润色抹掉的有效缺陷？
- 新角色是否已经有可记录的声音轴，还是只是功能人？

## 12. 给 AI 的硬约束

- 不能把角色声音简化成口头禅。
- 不能为了通顺抹掉人物的阶层、职业、地域、年龄和压力痕迹。
- 不能让所有角色都用同一套解释性句式。
- 不能复制外部作品或竞品的可识别句子来建立声音。
- 不能把候选风格写成已确认规则。
- 不能在修订时擅自改变称呼、术语和特殊格式。
- 不能让旁白知道角色不该知道的信息，除非项目明确采用全知叙事。
- 不能把风格样例当素材库，只能抽象成声音规则。

## 13. 快速自查问题

- 这本书的整体声音能用 3-5 个可执行词描述吗？
- 旁白能说什么、不能说什么，是否写清？
- 每个重要人物是否有“第一关注点”和“压力漏点”？
- 样例句是否标注了功能，而不是只贴漂亮句子？
- 称呼和术语是否进入样式表？
- 角色声音是否会随关系变化而变化？
- 修订是否保留了有效粗糙，而不是全部改平？
- 新 agent 接手时，能否凭本文续写出同一套叙事选择？

## 14. 来源与迁移说明

- Writing Excuses 的 Character Voice 讨论把角色声音拆成词流、顺序、感觉、词汇、背景、叙述者规则和对白/叙述层级差异；本文迁移为“角色声音七轴”“旁白声音分层”和“样例句使用边界”。
- Writer's Digest 关于角色声音和对白的建议强调人物不能同声、对白是有目的的压缩语言；本文迁移为“对话不是聊天，是带目的的行动”。
- Reedsy 对话指南强调对白要推动故事、呈现人物动态，并区分角色声音；本文迁移为“每句对白至少承担一种功能”和“可删除水对白”。
- SFWA 关于 style sheet 的文章强调样式表记录语法约定、人物、地点、专有词和词形选择，并帮助长篇与系列保持一致；本文迁移为“专有名词与样式表”和“多 agent 声音交接”。
- Mailchimp Voice and Tone 指南区分稳定 voice 与随情境变化的 tone；本文迁移为“项目整体声音稳定，场景/人物/关系语气可变”。
- 起点社区人物与对话经验强调人物要通过语言、动作、心理等实现区分，对话要推动剧情、避免水对话；本文迁移为“角色声音矩阵”和“对白功能删除规则”。

## 与 69 号专题的衔接：视角规则独立调用

本文件负责项目风格手册和角色声音沉淀；凡涉及人称选择、视角人物、叙述距离、旁白权限、信息焦点、多视角结构和切视角规则时，必须同步调用 `docs/core-writing/69-pov-narrative-distance-information-focus.md`。

风格手册中的“叙事视角与距离”只记录项目级结论；69 号专题负责执行细则。AI 起草或续写时，不得只写“紧贴第三人称”就结束，还要补充：当前视角人物能知道什么、读者当前知道什么、本章禁止泄露什么、是否允许切视角以及切换收益。
