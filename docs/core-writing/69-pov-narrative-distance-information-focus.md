# 69. 叙事视角、叙述距离与信息焦点管理专项

状态：已确认  
适用对象：AI 写作代理、审稿代理、续写代理、风格维护代理  
核心目标：把“谁在看、能知道什么、离人物多近、读者此刻知道什么”做成可检查的叙事契约，避免视角混乱、旁白泄密、人物不活、悬疑不公平和 AI 式远景说明。

## 1. 使用场景

当任务出现以下任一情况时，必须调用本专题：

- 新书设定人称、主视角、叙述距离、多视角结构。
- 起草或续写章节，需要判断本章由谁承载信息和情绪。
- 审稿发现“上帝视角”“头脑跳跃”“旁白替人物判断”“读者不知道谁在看”。
- 多角色同场、群像、大场面、追逃战、悬疑调查、误会局、反转局。
- 用户反馈“代入感弱”“人物不活”“剧情像说明书”“情绪淡”“AI 味重”。
- 需要和 `docs/storycraft/26-suspense-reversal-information-control.md` 的信息差、`docs/core-writing/37-project-style-bible-character-voice.md` 的风格手册、`docs/workflows/38-chapter-production-pipeline-agent-handoff.md` 的交接流程联动。

## 2. 核心原则：视角是信息契约，不是镜头乱切

网文读者可以接受快节奏、强钩子、清晰解释，但不能接受作者为了方便推进剧情而随意进入每个人大脑。视角的本质是一个持续契约：本章由谁过滤世界，读者就通过谁获得信息、情绪和误判。

AI 写作前先填写这一行：

```text
视角人物 -> 可感知信息 -> 可解释范围 -> 叙述距离 -> 读者当前知道 -> 本章禁止知道
```

如果这一行填不清，章节容易出现三类问题：

- 信息乱：角色不该知道的秘密被旁白说出来，悬疑和反转失效。
- 情绪薄：所有角色都被外部描述，读者没有贴住任何一个人的身体和欲望。
- 人物假：旁白用作者判断代替人物感知，人物没有自己的偏见、误读和措辞。

## 3. 基础概念

| 概念 | 定义 | AI 写作时要回答的问题 |
| --- | --- | --- |
| 人称 | 第一人称、第三人称、第二人称等语法位置。 | 这本书默认用“我”还是“他/她/他们”？ |
| 视角人物 | 当前读者主要通过谁接收场景。 | 这一段是谁在看、听、怕、判断？ |
| 叙述距离 | 旁白离人物内心的远近。 | 读者是贴着心跳，还是在远处看局势？ |
| 信息焦点 | 当前段落最重要的可见信息。 | 这一段让读者注意什么？ |
| 旁白权限 | 旁白允许知道和解释的范围。 | 旁白能不能说幕后真相、他人心理、未来结果？ |
| 视角过滤器 | 视角人物的注意力、价值观、误读和词汇习惯。 | 同一件事，经这个人看会变成什么味道？ |
| 读者知识状态 | 读者此刻知道、猜到、误会、期待的内容。 | 读者比主角多知道什么，少知道什么？ |

## 4. 常见视角类型与网文适配

| 类型 | 适合什么 | 优势 | 高危点 |
| --- | --- | --- | --- |
| 第一人称 | 强代入、强吐槽、成长线、悬疑主观体验、恋爱自述。 | 亲密、声音强、读者容易贴住欲望。 | 容易过度解释自己；信息范围受限；若声音弱会显幼稚。 |
| 紧贴第三人称 | 多数商业网文主线、爽文、情绪戏、升级线。 | 既能贴人物，又能保持叙事效率。 | 容易被 AI 写成“第三人称全知”，偷偷泄露他人心理。 |
| 中距离第三人称 | 世界观、权谋、群像过渡、局势解释。 | 清楚、节奏快、便于压缩信息。 | 情绪可能变薄，人物像棋子。 |
| 多视角有限 | 群像、战争、权谋、双主角、追逃双方、误会与命运交叉。 | 扩大信息面，制造戏剧反差。 | 每个视角若没有独特收益，会变成混乱换镜头。 |
| 全知视角 | 史诗感、寓言感、强作者声音、宏观讽刺。 | 可跨人物、跨时间、跨空间。 | 最容易破坏悬疑和代入；必须有稳定旁白人格。 |
| 旁观者/记录者视角 | 神秘主角、传奇人物、侦探型配角、传闻感。 | 可以保留核心人物谜面，制造仰望感。 | 主角可能缺席，情绪需要靠旁观者反应补足。 |

## 5. 新书视角决策流程

立项时不要只问“用第一人称还是第三人称”，要按商业目标做选择：

- 目标读者是否更需要代入爽感，还是更需要看局和看群像。
- 类型承诺是否依赖主角隐瞒、误判、吐槽、金手指信息差。
- 主角的欲望是否足够强，能不能长期承载单一视角。
- 反派、女主、队友、家族、组织是否有必须独立展示的信息。
- 悬疑和反转是否需要读者比主角多知道或少知道。
- 世界观是否复杂到必须偶尔拉远解释。
- 平台节奏是否要求每章都有明确情绪锚点，而不是长期宏观旁白。

输出结论必须写成“视角规则”，而不是一句偏好：

```text
本书默认采用紧贴第三人称，以主角为核心视角。每章最多一个主要视角人物；若切换到反派或配角，必须在小节断点后进行，并带来主角无法获得的新压力或误会。旁白不得提前确认幕后真相，只能呈现主角可感知线索和读者可推理信息。
```

## 6. 默认规则：一场戏，一个主动意识

除非项目风格手册明确选择全知视角，否则商业网文起草时默认采用“一场戏，一个主动意识”。

执行方式：

- 每场戏先指定一个视角锚点。
- 读者只直接进入这个人的感受、判断、记忆、欲望、恐惧。
- 其他人的情绪通过表情、动作、语气、停顿、物件、行为反常来呈现。
- 同一段落内不要先写主角心里一沉，又写反派暗自得意，再写女主其实早已看穿。
- 如果必须进入另一个人的心理，先换小节、换场景、换章节，或明确采用全知规则。

错误示例抽象：

```text
陆沉心头一紧，知道自己被盯上了。秦霜表面冷淡，心里却担心他受伤。躲在暗处的黑衣人暗暗冷笑，觉得这两人已经逃不掉。
```

问题：一段进入三个人心理，信息权限全破。

修正方向：

```text
陆沉心头一紧。
秦霜没有看他，只把剑鞘往前推了半寸。她指节绷得发白，连呼吸都压得极轻。
巷口的雨声忽然断了一拍。
有人在那里。
```

修正后，读者仍在陆沉视角内，通过秦霜动作和环境异常感到压力。

## 7. 叙述距离刻度

AI 起草时可以用 D0-D5 控制远近：

| 刻度 | 距离 | 用法 | 风险 |
| --- | --- | --- | --- |
| D0 身体贴近 | 呼吸、痛感、肌肉、温度、心跳。 | 打斗、恐惧、暧昧、羞耻、濒死。 | 过多会腻，节奏变慢。 |
| D1 内心即时反应 | 念头、冲动、压住的话、误判。 | 情绪转折、选择、被冒犯、爽点前一秒。 | 容易解释太满。 |
| D2 角色过滤观察 | 经人物价值观筛过的场景。 | 日常、对话、调查、人物第一次登场。 | 过滤器弱就变普通描写。 |
| D3 场景概述 | 清楚交代局势、位置、行动结果。 | 转场、赶路、压缩低价值过程。 | 情绪距离变远。 |
| D4 叙述者评述 | 作者式判断、时代背景、规则说明。 | 世界观、历史、门派规则、权力结构。 | 容易说教、AI 味重。 |
| D5 作者总结 | 抽象主题、宏大判断、未来回望。 | 章尾余味、史诗感、讽刺感。 | 破坏当下紧张感，慎用。 |

推荐搭配：

- 动作戏：D0-D2 为主，少量 D3 交代空间。
- 情绪戏：D0-D2 为主，避免 D4 替人物总结感情。
- 调查戏：D1-D3 为主，让线索经过角色误读。
- 世界观说明：D2-D4 结合，先由人物需要触发，再短解释。
- 转场：D3 快速压缩，不要用 D0 写无价值过程。

## 8. 切视角规则

切视角不是为了“让读者知道更多”，而是为了产生新的叙事收益。

允许切视角的条件：

- 当前视角无法自然获得关键信息。
- 新视角能制造更强的压力、误会、期待、讽刺或情绪反差。
- 新视角有独立声音，不只是把主角不知道的事告诉读者。
- 切换发生在章节、小节、场景断点，而不是同一段中间。
- 切换后至少完成一个明确输出：新线索、新威胁、新代价、新误判、新情绪。

禁止切视角的情况：

- 只是为了替作者解释设定。
- 只是为了让反派宣布计划。
- 只是为了补一句“其实她也喜欢他”。
- 只是因为当前段落写不下去了。
- 切到配角后没有改变读者对主线的期待。

## 9. 信息权限管理

每章至少维护四个知识层：

```text
作者知道：完整真相。
视角人物知道：他能感知、记忆、推理、误解的内容。
读者知道：文本已经交付、暗示、误导的内容。
对手/幕后知道：能制造压力但不必全部公开的内容。
```

写作限制：

- 紧贴视角不能说“他不知道的是”，除非项目允许全知旁白。
- 视角人物不能准确描述自己没看到、没听到、没推理出的内容。
- 旁白不能提前确认秘密动机，只能呈现可见异常。
- 读者可以比主角多知道一点，用于戏剧反讽，但不能让主角显得愚蠢。
- 主角可以比读者多藏一张牌，用于爽点揭示，但必须提前埋公平线索。
- 悬疑不能靠“视角人物明明知道却故意不告诉读者”硬藏。

## 10. 视角与悬疑、反转、信息差

和 `docs/storycraft/26-suspense-reversal-information-control.md` 联动时，先判断本章采用哪一种信息差：

- 读者少于主角：适合隐藏底牌、反杀、装弱、打脸。
- 读者等于主角：适合调查、闯关、探索、求生。
- 读者多于主角：适合危机逼近、误会、双线交叉、反派倒计时。
- 读者少于反派：适合压迫感和未知恐惧。
- 读者多于反派：适合主角布局后的期待爽感。

公平原则：

- 可以隐藏答案，不要隐藏规则。
- 可以让角色误判，不要让旁白撒谎，除非是不可靠叙述者且已经建立规则。
- 可以延迟解释，不要把必要因果藏到反转后才凭空补出。
- 可以切到反派视角制造压力，但不要把反派计划讲完导致主角胜利无惊喜。

## 11. 视角过滤器：让人物变活的关键

视角过滤器决定“同一世界在不同人物眼里长什么样”。它比单纯换人称更重要。

每个长期视角人物都应有以下字段：

| 字段 | 问题 |
| --- | --- |
| 第一注意物 | 这个人进屋最先看权力、危险、钱、脸、出口、礼数还是食物？ |
| 误读倾向 | 他最容易把别人行为误解成什么？轻视、背叛、诱惑、挑衅、怜悯？ |
| 词汇习惯 | 他会用江湖话、职场话、学术话、军事话、吐槽话还是市井话理解世界？ |
| 身体焦点 | 紧张时注意呼吸、手、胃、背、嗓子、眼神还是距离？ |
| 回避内容 | 他不愿承认什么欲望、羞耻、恐惧或亏欠？ |
| 价值排序 | 他优先保命、赚钱、尊严、亲情、胜利、自由还是秩序？ |
| 关系偏见 | 面对特定人物时，他会自动信任、提防、讨好、攻击还是沉默？ |

应用方法：

- 写场景前，不先写“客厅很大”，先问视角人物为什么会注意客厅。
- 写人物登场时，不先列外貌，先写视角人物被什么细节刺到。
- 写情绪时，不先说“他很难过”，先写这个人会怎样压住、误认或转移难过。
- 写对话时，让旁白词汇和人物身份有轻微同色，而不是所有章节都像同一个 AI 解说员。

## 12. 段落焦点与移动端阅读

和 `docs/core-writing/32-chapter-paragraph-mobile-reading-experience.md` 联动时，遵守以下规则：

- 视角焦点变化时，优先换段。
- 同一段只承载一个主要观察、一个主要动作或一个主要心理转折。
- 从外部动作切到内心反应时，可以换短段制造停顿。
- 从主角视线切到环境危险时，要让视线移动有路径。
- 不要在一个手机屏里连续切换多人心理，否则读者会失去锚点。

## 13. 多代理协作交接字段

续写代理接手前，交接包必须写清：

```text
当前视角人物：
当前人称与视角类型：
当前叙述距离：
视角人物已知：
视角人物误判：
读者已知：
读者应怀疑但未确认：
本章禁止泄露：
下一次允许切视角的位置：
切视角必须带来的收益：
视角过滤器关键词：
```

缺少这些字段时，续写最容易把悬疑、人物声音和情绪连续性写散。

## 14. 常见失败模式与修复

| 失败模式 | 读者体感 | 修复动作 |
| --- | --- | --- |
| 头脑跳跃 | 不知道该贴谁，像监控加读心术。 | 回到一个视角人物，其他心理改成可见行为。 |
| 上帝视角泄密 | 悬念塌了，反转不公平。 | 删除角色不该知道的信息，改成线索或异常。 |
| 旁白替人物判断 | 人物像被作者操纵。 | 改成角色感知、误判、嘴硬、动作选择。 |
| 叙述距离一直很远 | 情绪淡，像剧情梗概。 | 在关键节点下沉到 D0-D2。 |
| 第一人称过度客观 | “我”没有自我，只是在报告。 | 加入偏见、欲望、回避和即时反应。 |
| 多视角同声 | 换人名不换灵魂。 | 给每个视角配置不同过滤器。 |
| 切视角无增量 | 读者觉得拖沓。 | 每次切换必须带来新压力或新误会。 |
| 悬疑靠不说 | 读者觉得作者耍赖。 | 明确信息权限，提前埋可回看的公平线索。 |
| 配角视角抢主线 | 主角吸引力下降。 | 配角视角必须服务主线压力或主角认知变化。 |
| 镜头化远景过多 | 像影视分镜，不像小说体验。 | 把镜头描述改为人物身体和判断。 |

## 16. 审稿检查问题

审稿代理逐章检查：

- 本章第一屏是否能明确“谁在看”。
- 每个心理判断是否属于当前视角人物。
- 旁白有没有说出当前视角人物不可能知道的信息。
- 其他角色的心理是否可以改成动作、话术、沉默或反常细节。
- 本章叙述距离是否服务场景功能，而不是全程同一档。
- 关键情绪点是否足够贴近身体和即时念头。
- 世界观说明是否由角色需求触发。
- 切视角是否发生在清晰断点，并产生新信息或新压力。
- 多视角人物是否有不同过滤器，而不是同一种 AI 口吻。
- 反转前是否给过公平线索，反转后是否没有靠旁白补丁硬圆。

## 17. 最小执行规则

时间紧时，AI 至少执行以下 6 条：

- 每章开写前指定当前视角人物。
- 同一场戏默认只进入一个人的内心。
- 其他人的心理用可见行为表达。
- 关键情绪点下沉到 D0-D2。
- 所有秘密按“作者/视角人物/读者/对手”分层。
- 切视角必须在断点后，并带来新压力、新误会或新情绪。
