# 96. 旁白人格、叙述态度与叙事声音专项

状态：已确认

适用范围：新书风格定调、第一章试写、续写交接、多 agent 协作、章节起草、修订后变平、AI 味诊断、旁白漂移、题材味不足、幽默失效、情绪温度不稳、读者反馈为“像讲解员”“不像这个题材”“作者一直在解释”“旁白太端着”“修完没味道”。

核心结论：叙事声音不是给句子加口癖，而是决定“谁在讲这个故事、离角色多近、对世界持什么态度、允许怎样判断和怎样沉默”。AI agent 写长篇时，必须把旁白人格当作连续性资产管理，避免每章都变成同一个中性万能讲解员。

最小链路：类型承诺 -> 叙述者位置 -> 信息权限 -> 情绪温度 -> 价值态度 -> 语言质地 -> 与角色声音距离 -> 场景语气变体 -> 禁用词和禁用姿态 -> 样例句。

---

## 1. 这个专题解决什么问题

AI 写作常见的声音问题，不只是“人物台词同声”，还包括旁白同声。

常见表现：

- 玄幻、都市、甜宠、悬疑、喜剧都像同一个旁白在写。
- 旁白总是解释人物动机和情绪，像万能讲解员。
- 修订后句子更顺，但原来的锋芒、幽默、粗粝、暧昧或冷感消失。
- 第一章有声音，后面续写变成普通说明文。
- 旁白时而贴着主角吐槽，时而突然上帝视角评判，信息权限漂移。
- 情绪戏里旁白过度抒情，动作戏里旁白过度总结，喜剧戏里旁白解释笑点。
- 角色声音和旁白声音混在一起，导致读者不知道这是角色判断还是作者判断。

本专题提供“旁白人格和叙述态度”的设计、审稿和续写规则。

## 4. 基础概念

### 4.1 作者声音

作者长期作品中反复出现的表达倾向、价值关注、节奏偏好、幽默方式和观察世界的角度。

知识库层面不要求 AI 伪装某个现实作者，但要求每本书有稳定、可识别的项目声音。

### 4.2 旁白声音

正文叙述部分呈现出来的声音。它负责选择信息、判断距离、组织节奏、决定什么可以吐槽、什么要克制、什么该冷眼旁观、什么该贴近角色心跳。

旁白声音可以接近角色，也可以像独立说书人，但不能漂移无规则。

### 4.3 角色声音

角色说话、思考、误判和行动中的语言习惯。角色声音可以影响贴身叙述，但不能和项目旁白完全混成一团。

### 4.4 语气 Tone

同一个项目声音，在不同场景中的情绪调节。项目 voice 应稳定，tone 可以随场景变：战斗紧、情绪戏软、喜剧轻、终局庄重。

### 4.5 叙述态度

旁白对人物、世界和事件的态度：热血、冷峻、讽刺、温柔、荒诞、克制、锋利、悲悯、轻佻、庄严。

叙述态度是商业题材味的重要来源。

## 5. 旁白人格十轴

AI agent 设计或续写一本文时，至少填写以下十轴。

| 轴 | 需要回答的问题 |
| --- | --- |
| 信息权限 | 旁白知道多少？只跟随主角，还是允许宏观补充？ |
| 叙述距离 | 贴身、近景、中景、远景、说书人式，哪种为主？ |
| 情绪温度 | 热血、冷静、温柔、压抑、荒诞、锋利，哪种为基准？ |
| 价值态度 | 旁白是否评判人物？是站队、旁观、讽刺，还是交给读者判断？ |
| 幽默权限 | 能不能吐槽？吐槽来自角色、旁白，还是情境反差？ |
| 语言质地 | 短促、华丽、硬朗、口语、古雅、轻快、冷峻、黏稠？ |
| 信息投放 | 喜欢先给结果、先给动作、先给气氛，还是先给规则？ |
| 解释阈值 | 读者能自行理解时，旁白是否克制？哪些内容必须说明？ |
| 角色污染度 | 贴身第三人称中，旁白被角色词汇和偏见污染到什么程度？ |
| 禁用姿态 | 禁止上价值、禁止网络热词、禁止万能总结、禁止卖惨，还是禁止端着？ |

这十轴比“文风爽一点/高级一点/像网文一点”更可执行。

## 6. 常见旁白模式

### 6.1 贴身主角型

旁白大多被主角的注意力、词汇、偏见和欲望污染。

适合：第一人称、贴身第三人称、爽文、甜宠、悬疑主观体验、成长流。

风险：主角声音弱时，全书会变平；主角过度解释时，会显幼稚或啰嗦。

### 6.2 冷眼记录型

旁白克制，少评判，让动作、证据和对话自己产生力量。

适合：悬疑、职场、权谋、黑色幽默、现实向、硬核职业文。

风险：如果缺情绪锚，读者会觉得冷、干、不代入。

### 6.3 说书人型

旁白有明显叙述人格，能压缩时间、做宏观判断、制造传奇感。

适合：仙侠、玄幻、历史传奇、群像、年代感强的作品。

风险：容易变成作者讲道理或剧透；必须有稳定语调和信息边界。

### 6.4 讽刺吐槽型

旁白允许轻微嘲讽、反差、黑色幽默或荒诞感。

适合：喜剧、反套路、都市讽刺、娱乐圈、社畜文、轻小说。

风险：吐槽过多会削弱情绪和危险；笑点不能靠旁白解释。

### 6.5 热血鼓动型

旁白节奏快、动词强、价值立场鲜明，强调尊严、反击、升级和燃点。

适合：男频升级、竞技、复仇、团队战、热血群像。

风险：容易喊口号；燃点必须靠代价和行动，不靠旁白鸡血。

### 6.6 温柔陪伴型

旁白更重情绪温度、生活细节、关系微变化和治愈感。

适合：甜宠、治愈、生活流、女性成长、亲情友情。

风险：容易软、慢、水；必须有小目标和微变化。

### 6.7 尖锐利刃型

旁白短促、判断锋利，强调羞耻、权力、代价、真实利益。

适合：权谋、商战、黑莲花、反英雄、复仇。

风险：如果过度冷酷，读者可能不站队；需要保留可理解欲望和情绪债。

## 7. 旁白声音与 POV 的关系

### 7.1 第一人称

旁白基本就是角色声音。所有叙述都应受角色年龄、身份、知识、偏见、欲望和盲区限制。

高危：第一人称突然说出作者级洞察，或像项目说明书。

### 7.2 贴身第三人称

旁白不是角色直接说话，但会被角色的注意力和词汇污染。

有效写法：他没看那枚玉佩的成色，只看见佩绳旧得发白。她还留着。

无效写法：这枚玉佩象征着他们过往关系中尚未清算的情感债务。

### 7.3 中距离第三人称

旁白可以比角色更清楚地组织信息，但不能随便进入所有人的内心。

适合：商业网文多数题材。需要清晰、快、稳。

### 7.4 全知或说书人

旁白可以超出单个角色，但必须有明确人格和规则。

高危：为了方便解释，随时泄露任何信息；为了制造悬念，又突然装不知道。

### 7.5 多 POV

每个 POV 的贴身叙述可以有不同污染度，但项目总体 voice 仍要稳定。

规则：切视角时，先换注意对象、风险模型和词汇习惯，不只是换名字。

## 8. 叙事声音不是口癖

很多 AI 会把声音理解为加几个固定词：啧、呵、该死、罢了、离谱、啧啧。这样很快会变成模板。

真正的声音来自：

- 选择什么细节。
- 用什么顺序讲。
- 哪些地方沉默。
- 对人物持什么态度。
- 对痛苦、胜利、羞耻、荒诞、危险的反应方式。
- 句子长短和节奏如何配合场景。
- 哪些词永远不用。
- 哪些判断留给读者。

声音是滤镜，不是贴纸。

## 9. AI 常见坏味道与修复

| 坏味道 | 读者体感 | 修复方式 |
| --- | --- | --- |
| 万能讲解员 | 像 AI 说明书 | 删除旁白总结，改成视角人物看到、误判和选择 |
| 旁白态度漂移 | 像换作者 | 填旁白十轴，固定基准 voice，只让 tone 随场景变 |
| 角色和旁白混乱 | 不知道谁在判断 | 标注这是角色认知、旁白叙述，还是作者说明，并清除作者说明 |
| 吐槽过量 | 危险和情绪变轻 | 只让吐槽服务角色防御、节奏释放或反差回收 |
| 抒情过量 | 腻、慢、像作文 | 把抒情落到物件、动作、代价和沉默 |
| 古今语混乱 | 出戏 | 建项目词库和禁用词，按时代/世界替换 |
| 修订改平 | 原文本有味道，修后标准答案 | 修订前列声音资产，保留毛刺、偏见和节奏特征 |
| 题材味不足 | 不像玄幻/都市/甜宠/悬疑 | 根据类型承诺调整旁白温度、信息投放和词汇密度 |
| 段尾总上价值 | 作者替读者下结论 | 删除总结，把意义转成下一步行动或后果 |
| 所有 POV 同声 | 群像扁平 | 每个 POV 设置不同注意对象、风险模型、词汇和情绪温度 |

## 10. 类型文旁白口径

### 10.1 男频升级流

旁白应强调目标、代价、资源、尊严、反击和阶层压力。节奏偏硬，解释阈值低，爽点靠结果和旁观者反应落袋。

慎用：过度抒情、长篇心理自怜、漂亮但无功能的比喻。

### 10.2 女频情感流

旁白应强调关系位置、情绪细节、边界、试探、误会和选择。可以更贴身，但不能只写感受。

慎用：替角色宣告“她终于明白自己爱他”，而不写互动证据。

### 10.3 悬疑推理

旁白应克制、精准、控制信息顺序。异常细节要可见但不过度解释。

慎用：为了保密让旁白含糊其辞，或为了说明真相提前泄密。

### 10.4 喜剧轻小说

旁白可更灵活、反差更强，但要让笑点来自人物和处境。

慎用：解释笑点、连续吐槽、现代梗破坏世界。

### 10.5 玄幻仙侠

旁白可有庄重、传奇、宏阔或古雅感，但必须和角色近景行动衔接。

慎用：动不动上价值、堆空泛大词、现代行政词乱入修真世界。

### 10.6 都市职场

旁白应更清楚利益、流程、身份、话术和现实压力。可带轻微讽刺或冷幽默。

慎用：行业设定像百科，旁白替角色说教。

## 11. 声音连续性维护

多 agent 续写时，旁白声音必须像设定一样交接。

至少记录：

```text
项目叙事声音三词：
基准叙述距离：贴身 / 中距 / 远景 / 说书人
情绪温度：冷 / 热 / 温柔 / 锋利 / 荒诞 / 克制
幽默权限：无 / 轻微 / 中等 / 强吐槽；来源：角色 / 旁白 / 情境
旁白能否评判人物：能 / 不能 / 只在特定场景
常用句式：
禁用句式：
常用词汇：
禁用词汇：
示范段落：
不能误伤的声音资产：
```

如果没有这些字段，续写 agent 容易把文本改成通用 AI 声音。

## 12. 修订流程

修订旁白声音时，不要先做同义词替换。按顺序处理：

1. 确认本书类型承诺和目标读者期待。
2. 确认本章 POV 和叙述距离。
3. 标注旁白当前态度：热、冷、讽刺、温柔、庄重、荒诞。
4. 找出漂移段：突然解释、突然上价值、突然吐槽、突然现代词、突然远景。
5. 判断这些漂移是否有场景理由。
6. 删除无必要作者判断。
7. 把必要信息改成角色可感知的细节、动作、对话或后果。
8. 保留有效声音资产，不把所有毛刺修平。
9. 最后才做句子节奏和用词终磨。

## 14. 审稿检查清单

审稿时逐条检查：

- 这章的旁白像谁在讲故事？能用三个词描述吗？
- 旁白的信息权限是否稳定？
- 叙述距离是否与 POV 一致？
- 旁白是在替角色感受，还是在让角色自己反应？
- 是否有突然现代词、作者词、论文词、营销词破坏世界？
- 是否有段尾总结、上价值、解释意义的 AI 腔？
- 吐槽、抒情、冷眼、热血是否服务场景，而不是到处均匀撒？
- 多 POV 是否只有名字不同，叙述声音完全一样？
- 修订是否误伤了原文的人物毛刺、节奏和题材味？
- 本章 tone 有变化时，项目 voice 是否仍稳定？
- 旁白是否符合类型承诺和目标读者期待？

通过标准：读者能感到这本书“有自己的说话方式”，但不会被旁白抢走角色、场景和情绪。

## 15. 最小可执行规则

AI 写章节时至少遵守以下规则：

- 每本书必须有稳定旁白人格，不能每章换讲解员。
- voice 稳定，tone 可随场景变化。
- 旁白声音、角色声音和作者说明必须区分。
- 贴身叙述应受 POV 角色的注意力、词汇、偏见和欲望影响。
- 全知或说书人旁白必须有稳定人格和信息规则。
- 声音来自选择细节、组织顺序、判断态度和沉默边界，不来自口癖堆叠。
- 修订时先保护声音资产，再删解释和调节奏。
- 禁止用万能旁白替读者总结意义；意义优先落到选择、后果和余波。
