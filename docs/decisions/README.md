# 决策归档

状态：已确认  
用途：保存历史决策长文，避免 `docs/99-decision-log.md` 继续膨胀。

## 读取规则

- 默认只读 `docs/99-decision-log.md`。
- 需要查旧决策时，先在本目录按月份定位，再用关键词检索归档文件。
- 不要把月度归档整篇塞进写稿、审稿或项目初始化上下文。
- 归档只回答“当时为什么这么定、影响哪些文件、执行规则是什么”，不替代当前专题文档。
- 已经形成的历史长归档默认冻结，不再继续追加正文。后续新决策如果需要长说明，写成单条短文件并由 `docs/99-decision-log.md` 链接。
- 活跃文档保持短小；如果内容膨胀，优先拆分和增加索引层级，而不是继续扩写入口文件。

## 归档文件

- `2026-06.md`：2026 年 6 月历史决策长文，包含开源治理、skill 分发、去身份化、写稿流程、对白训练、封面标准、渐进式索引、动态大纲等决策。

## 新增决策规则

- 短决策只写入 `docs/99-decision-log.md` 的短索引。
- 需要详细说明时，新建 `active/YYYY-MM-DD-short-title.md`。
- 单条决策文件也要保持短小；如果继续膨胀，拆成背景、执行规则、影响范围等更小文件。
- 同步在 `docs/99-decision-log.md` 增加短索引。
