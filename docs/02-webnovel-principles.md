# 网文核心原则

本文整理商业连载网文的通用写作原则。它不是文学理论摘要，而是后续 AI 起草和修订时可以直接执行的工作标准。

## 1. 开篇先给读者一个“正在发生的问题”

开篇的首要任务不是讲完整世界观，而是让读者尽快知道：

- 谁正在场上。
- 他/她在哪里。
- 当下有什么压力。
- 这个压力为什么马上要处理。
- 读者读下去会获得什么类型体验。

可执行写法：

- 第一场尽量用具体场景开局，例如愿池现场、公司会议、事故现场、婚礼门口、考核台前、菜市场冲突，而不是主角履历和世界设定。
- 第一章前半段就要出现可见问题：被逼承愿、被夺功劳、系统故障、客户发难、家族逼婚、债主上门、案件出现破口。
- 设定信息跟着冲突释放。读者不需要先知道规则大全，只需要知道此刻“不这样做会有什么后果”。
- 类型承诺要早出现。修仙重生要让读者看见反杀或代价；都市脑洞要让读者看见荒诞错位；技术爽文要让读者看见烂摊子和专业压制空间；感情线要让读者看见关系张力。

常见风险：

- 开篇先讲世界历史、门派制度、公司背景、主角前世履历，读者还没来得及在乎。
- 第一章只是“主角很惨”或“主角很强”的说明，没有一个当下必须解决的问题。
- 第一章末只是总结局势，没有留下下一章的具体动作压力。

自检问题：

- 第一章第一个场景是否能用一句话说清“主角此刻遇到了什么麻烦”？
- 读者是否在第一章就看见了本书最主要的爽感或情绪承诺？
- 设定是否被压进场景、选择、代价和后果里，而不是单独讲说明书？

## 2. 每章至少完成一个叙事工作

网文连载不要求每章都大爆点，但每章都要改变局面。有效章节通常至少完成以下一种工作：

- 冲突：主角和对手、规则、环境、关系发生直接碰撞。
- 反转：读者或场内人物原本以为的情况被推翻。
- 回收：前文埋下的证据、承诺、伤口、误会或资源发生兑现。
- 铺垫：给后续更大冲突放下可见材料。
- 关系变化：人物之间信任、敌意、依赖、亏欠、控制权发生改变。
- 线索或揭示：读者得到新信息，但信息必须带来新压力。
- 升级：原问题没有结束，而是变成更高层级、更公开、更难处理的问题。

弱章节常见形态：

- 只解释设定，没有行动。
- 只重复人物立场，没有局面变化。
- 只让众人震惊，没有实质收益或新问题。
- 只把上一章结尾软着陆，没有新的追读压力。

自检问题：

- 本章结束时，和本章开始相比，谁的处境、关系、资源、名声、权限或信息发生了变化？
- 这个变化是否读者能看见，而不只是旁白说“局势不同了”？
- 下一章有什么马上要处理的动作或选择？

## 3. 爽点要有阻力、代价和落袋感

爽点不是单纯“主角赢”。读者通常需要看到：

- 主角主动做了什么。
- 对手或环境为什么难缠。
- 证据、资源、时机、规则为什么不是白送。
- 主角赢下后拿到了什么实际结果。

落袋感可以是：

- 对手失去权限、名声、资源、庇护、职位、话语权。
- 主角拿到钱、证据、职位、控制权、公开记录、盟友、行动资格。
- 原本模糊的规则被确认，后续可以继续使用。
- 关系发生变化，例如旁观者开始怀疑旧秩序，盟友获得站队理由。

风险案例来自 `许愿池修仙`：

- 规则反杀前几章成立，但如果连续依赖愿池显字、问愿册翻页、水影回放，就会出现“规则自动替主角打脸”的疲劳。
- 读者反馈要求证据链增加阻力：旧账可以残缺、被遮住、被误读；对手可以局部维稳成功；主角必须抓住破绽推动下一步。
- “开旧账三页”这种钩子一旦抛出，后续必须兑现具体杀伤力，不能继续围绕“能不能开”拖延。

自检问题：

- 主角是否亲自推动了胜利，而不是等系统、账册、男主或旁白替她完成？
- 对手是否至少有一次合理自保、遮掩、误导或反击？
- 爽点兑现后是否有可见收益，而不只是对方丢脸？

## 4. 章末钩子要落在具体未完成动作上

好的章末钩子通常不是“风暴将至”这种总结，而是一个具体、未完成、读者能马上想象后果的动作或信号。

可用钩子：

- 屏幕弹出一条异常提示。
- 某个关键人物推门进来。
- 账册翻到一个不该出现的名字。
- 电话接通，对面第一句话暴露新问题。
- 主角刚要签字，发现落款已经有人替他写好。
- 某件证据被抢走、裂开、失效或突然开始记录。

慎用钩子：

- 列出三四条压力：“某人在等、某地要关、某物要坏。”
- 用旁白预告下一章任务。
- 用抽象情绪收束，例如“她知道真正的战争开始了。”
- 每章都以整齐短句或口号式判断结尾。

自检问题：

- 章末是否有一个可见对象、声音、动作、消息或选择？
- 读者是否能自然产生“下一秒会怎样”的问题？
- 结尾有没有替读者总结本章意义？

## 5. 信息投放要降低认知负担

读者愿意接受复杂设定，但前提是信息和当前问题有关。信息投放应遵循：

- 先给冲突，再给规则。
- 先给后果，再给解释。
- 先给一个能用的局部规则，再逐步扩大边界。
- 让人物因为目标、恐惧、利益而讲信息，不要让所有人像解说员。

操作方法：

- 如果必须解释规则，最好让解释发生在争夺中：谁不承认、谁想遮掩、谁要借规则压人、谁要找漏洞。
- 抽象设定要落到物件和行为：账册、契约、设备、价格、余额、证据、权限、伤口、裂痕、签名、记录。
- 不确定读者是否能懂时，优先补一个动作或后果，不要加长概念解释。

自检问题：

- 这段设定是否影响当前选择？
- 读者不懂这个设定，会不会看不懂本场冲突？如果不会，可以延后。
- 设定有没有通过对象、动作、代价或收益落地？

## 6. 降低 AI 味是基础要求

AI 味不是“用了 AI”本身，而是文本呈现出均质、空泛、过顺、过会总结、缺少真实取舍的读感。

高风险表现：

- 每段都在概括人物处境、主题意义和读者应如何理解。
- 人物说话都像同一个冷静解说员。
- 情节推进太顺，线索、电话、批准、证据总在最需要时刚好出现。
- 情绪只靠标签，如愤怒、恐惧、震惊、释然，没有动作、停顿、声音、眼神或选择。
- 章尾喜欢写判断句、格言句、整齐对仗句。
- 场景像清单扫视：桌上有什么、墙上有什么、人群如何站，但这些细节不影响压力。

降噪方法：

- 用一个具体动作替代抽象判断。
- 给证据链加一点延迟、误读、权限卡点、对手抢话或临场自保。
- 让人物按自己的利益反应，不要全部服务作者论点。
- 允许不完美：误会、迟疑、话说半截、反应慢半拍、装没听懂。
- 删除已经被场景证明的旁白总结。

自检问题：

- 是否有段落可以套进很多小说而不违和？
- 是否有角色只负责替作者解释？
- 是否所有人都在同一时间理解、震惊或站队？
- 章末是否像在写本章读后总结？
