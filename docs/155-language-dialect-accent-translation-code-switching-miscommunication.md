# 155. 语言方言、口音腔调、翻译传译、暗语行话、跨文化误读与沟通权限

状态：已确认

适用对象：AI agents、网文作者、审稿 agent、连续性维护 agent、角色声音 agent、世界观 agent、跨地域/异族/国际/星际/玄幻/古言/职场题材 agent

核心问题：AI 写长篇时常默认所有人说同一种标准普通话：边疆人、异族、古代人、外国人、妖族、宗门弟子、商队、官吏、黑市、职业圈层都能无障碍沟通；方言只靠几个错别字，口音被写成笑料，翻译没有成本，古文/外文/密语/行业术语一出现读者就自动懂。商业级长篇需要把语言当成行动权限、身份信号、信息边界、关系距离和误解来源来管理。

---

## 1. 一句话原则

已确认：语言不是台词皮肤，而是“谁能听懂、谁能说服、谁能隐瞒、谁被排除、谁必须依赖翻译”的沟通权限系统。

AI agent 只要写到跨地域、跨阶层、跨物种、跨时代、跨国、古今文本、方言口音、翻译、暗语、咒语、专业术语、密信、误会或装听不懂，都必须检查：

1. 角色实际会哪些语言/方言/文字/行话。
2. 当前场景谁听懂、谁半懂、谁误解、谁需要翻译。
3. 语言差异是否改变信息、关系、权力、风险或台账状态。

## 3. 语言权限十线卡

AI agent 设计跨语言、方言、口音、异族、古今、翻译或暗语场景前，先填这张卡。

```text
场景：
主要语言/方言/文字：
通用语/官话/标准语：谁会，谁不会，在哪些场合必须用？
角色语言能力：母语、熟练语、半懂、只会听、只会读、不会写、装不懂。
书写能力：识字、文盲、能读不能写、会古文/符文/专业文书/代码/咒文。
翻译渠道：人类译者、同传、字幕、翻译器、法术、系统、词典、手势、图像。
翻译成本：延迟、删改、误译、立场、保密、费用、技术限制、文化不可译。
语言权力：谁因会标准语/官话/外语/古文/术语获得资格或压迫他人？
语言秘密：暗语、黑话、行话、代号、双关、谐音、禁词、咒语、真名。
误解风险：听错、词义不同、敬语失礼、翻译偏向、文化背景不同、假装听不懂。
台账回写：语言能力、译名、术语版本、暗语规则、已发生误译和关系后果。
```

## 4. 语言差异的六种叙事功能

### 4.1 身份信号

语言能暴露出身、教育、职业、阶层、地区、时代、组织和伪装破绽。

写法：不要只靠“俺”“咱”“嘞”来贴方言标签。更好的是让角色关注不同词、称谓、礼貌方式、专业词和表达习惯。

### 4.2 权力门槛

谁会官方语言、法律文字、宗门古文、外语、术语或代码，谁就能进入制度和资源。

写法：语言能力可以决定能否读契约、看懂榜文、通过考试、谈判、接任务、翻译密信或破解阵法。

### 4.3 信息差

有人听懂了别人以为他听不懂的话；有人误解了关键称呼；有人依赖译者而被删改信息。

写法：语言信息差要有后果，不要只用来制造小笑话。

### 4.4 关系距离

切换语言、称谓、敬语、昵称、暗号，常常意味着关系变化。

写法：从正式称谓到私密方言，从标准语到家乡话，从翻译转为直接对话，都可以作为关系落袋。

### 4.5 喜剧与尴尬

语言错位可以搞笑，但笑点应来自人物目标和场景错位，而不是嘲笑某种方言、口音、族群或语言能力。

### 4.6 世界规则

多语言、禁语、真名、咒文、符文、古语、系统语言和异族语言可以成为世界规则，但必须有学习成本、使用限制和读者理解入口。

## 5. 方言与口音写法

### 5.1 推荐写法

优先使用以下方式，而不是大量拼音化、错字化或难读口音：

- 词汇选择：少量地区词、职业词、时代词。
- 句式节奏：短句、倒装、重复、委婉、直给。
- 称谓方式：叔、婶、老板、师兄、先生、老大、您、咱们。
- 文化关注点：先问饭、路、价钱、面子、规矩、天气、亲戚。
- 旁白轻标注：他说话带一点南方软音；她在紧张时乡音会冒出来。
- 场景触发：平时用标准语，愤怒、撒娇、害怕、回乡、见亲人时切回方言。

### 5.2 高风险写法

- 大量错别字模拟口音，影响可读性。
- 把方言口音写成蠢、土、坏、可笑的证明。
- 用真实族群、地区、阶层的语言特征做歧视性笑点。
- 所有同地区人物说同一种夸张口音。
- 为了“真实”让读者看不懂关键台词。

### 5.3 审稿原则

方言口音服务角色和剧情，不服务作者炫技。读者可以感觉到差异，但不能被迫翻译整段台词。

## 6. 翻译与传译不是透明玻璃

翻译会改变信息。

必须考虑：

- 译者立场：忠诚谁，害怕谁，想隐瞒什么。
- 译者能力：专业、临时、儿童、机器、系统、法术、半吊子。
- 时间延迟：同传、交传、书面翻译、密信破译、字幕滞后。
- 文化不可译：礼貌、讽刺、双关、称谓、誓言、侮辱、禁忌。
- 保密风险：多一个译者就多一个知情人。
- 权力差：不会语言的一方必须依赖别人。

译者场景卡：

```text
译者是谁：
忠诚对象：
能力边界：
是否完整翻译：
删改/润色/误译位置：
被翻译者是否知道：
旁听者是否听懂原文：
翻译后关系变化：
```

## 7. 读写能力与书写系统

语言不只包括说话，还包括读写。

可写压力：

- 角色能说不会写。
- 能读日常字，不懂法律文书、古文、符文、术语。
- 读得懂字面，不懂隐喻、典故、行话。
- 文盲依赖别人读契约，风险被转移。
- 书写系统不同：表音、表意、符文、阵纹、系统代码、神文。
- 译名不一致导致追踪、身份、契约、名单或通缉出错。

风险：不要把不识字写成单纯愚蠢。读写能力常与教育、阶层、时代、性别、地域、残障、资源和制度有关。

## 8. 暗语、行话、术语与圈层语言

暗语和行话能制造圈层感，但也会制造读者负担。

使用条件：

- 当前场景真的需要保密、识别同伙、提高效率或显示专业身份。
- 第一次出现时绑定动作、风险或后果。
- 不要一次抛出大量黑话。
- 重要暗语必须进入样式表或台账。

常见场景：

- 黑市切口、江湖暗号、宗门密语、军中口令、公司黑话、医法金融术语、游戏队内语音、粉圈暗语、直播间梗、规则副本禁词。

坏写法：角色用术语炫专业，读者却不知道这影响什么。

好写法：一个暗号让守门人放行，或者一个术语暴露说话人不是本行。

## 9. 跨文化误读

语言误解不只是听错词，还包括场合、礼貌、禁忌、沉默和身份。

可写误读：

- 称呼过近或过远，导致关系错判。
- 直白被当成无礼，委婉被当成同意。
- 沉默在一方是拒绝，在另一方是尊重。
- 谦辞被当成真实贬低。
- 玩笑、讽刺、双关翻译后消失。
- 对方听懂字面，却不懂背后的规矩。

规则：误读要来自可理解的文化、利益、压力或语言边界，不能让角色为了误会而降智。

## 10. 多语言写作的正文呈现

### 10.1 少量原文

可以用少量真实外语、古文、方言或自造语，但必须保证读者能从上下文理解功能。

推荐：关键字原文 + 角色反应 + 简短解释。

不推荐：整段外语/方言没有翻译，读者被排除在场景外。

### 10.2 译文标记

如果整段对话其实是外语，可用叙述说明一次，然后用中文呈现。

示例规则：

```text
以下对话使用北境语，但正文用中文转写；只有不会北境语的角色听不懂。
```

### 10.3 翻译腔控制

写“翻译腔”要克制。可以用略硬的句式、词序或称谓表现陌生感，但不能让读者阅读困难。

### 10.4 自造语

自造语最少要有：用途、使用者、语音/文字风格、学习成本、限制、重复出现的少量核心词。不要造整套词典塞正文。

## 11. 语言魔法、真名、咒语与禁语

玄幻/仙侠/奇幻中，语言可以成为力量系统，但必须有规则。

要回答：

- 谁能学，为什么能学。
- 发音、书写、理解、血脉、境界、媒介哪个是关键。
- 误读、误念、缺字、翻译是否会改变效果。
- 是否有禁词、真名、誓言、咒文、契约语言。
- 读者如何知道规则，而不是作者想让咒语怎样就怎样。

风险：语言魔法万能会让谈判、契约、信息差和战斗变得随意。越强的语言规则，越要有代价、失败方式和反制。

## 12. 题材用法

### 12.1 古言/历史/权谋

重点：官话、地方话、文书语言、礼貌称谓、诏令、契约、族谱、科举文体。

规则：不要把所有古代角色写成现代白话换称谓。真实朝代、地域、官职和文体必须查证；架空也要保持稳定语言层级。

### 12.2 玄幻/仙侠/异族

重点：宗门术语、古语、妖族/兽族/魔族语言、咒语、符文、真名、传承文字。

规则：异族语言不能只靠“嘶嘶”“嗷嗷”卖萌。要有感知方式、交流边界和翻译成本。

### 12.3 都市/职场/行业文

重点：外语能力、行业术语、公司黑话、会议翻译、合同语言、留学/跨国沟通。

规则：外语能力会影响职位、谈判和信息获取，但现实语言、法律合同和专业术语需要查证。

### 12.4 星际/科幻

重点：通用语、机器翻译、延迟、协议、外星语、非人感知、符号系统。

规则：万能翻译器必须有死角：噪声、文化不可译、隐喻、加密、技术故障、非语言信号。

### 12.5 校园/青春/地域生活

重点：普通话、家乡话、校园黑话、网络语、同学外号、家庭语言切换。

规则：方言和网络语要轻用。过量会很快过时或影响阅读。

## 13. 常见失败模式

- 全员普通话：跨地域、跨国、异族、古今都无障碍沟通。
- 方言错字化：用大量错别字模拟口音，读者看不懂，还可能冒犯。
- 口音歧视：把口音当愚蠢、低等、滑稽或反派标志。
- 翻译透明化：译者没有立场、延迟、误差和保密成本。
- 术语炫技：行业、宗门、魔法、科技术语只增加门槛，不改变行动。
- 暗语无规则：暗号随便说，对手永远听不懂，读者也不知道为什么。
- 古今语感混乱：古代、现代、异世界角色语言层级不稳。
- 自造语过量：正文像词典，读者被排除。
- 读写能力消失：所有人都能读契约、古文、符文、榜文、代码。
- 误会硬拖：明明一句解释能解决，却靠语言差异拖几十章。

## 14. 审稿检查问题

AI 审稿或修订时，逐条检查：

- 当前场景所有人真的会同一种语言吗？如果不是，谁听懂、谁半懂、谁误解？
- 是否有角色依赖翻译？译者是否有立场、能力边界和保密风险？
- 方言/口音是否可读、克制、无歧视，并服务人物或情节？
- 外语、古语、自造语、符文、术语是否让读者仍能理解当前任务？
- 暗语、行话、称谓、咒语是否有规则和后果，而不只是酷？
- 读写能力是否符合人物教育、阶层、时代、物种和职业？
- 语言误会是否来自真实边界，而非角色降智或作者强拖？
- 语言切换是否改变关系、权力、情绪或信息？
- 多语言名字、译名、称谓和术语是否进入样式表？
- 涉及真实方言、民族语言、外语、宗教用语、历史文体或专业翻译时，是否需要查证和避免刻板化？

## 15. AI 写作提示词片段

### 15.1 建立语言权限表

```text
请为这个项目建立语言权限表。
要求：列出主要角色会哪些语言、方言、文字、术语和暗语；谁能听懂谁；谁需要翻译；哪些语言能力会影响行动资格、身份伪装、关系边界或信息差。未知项标为“待确认”。
```

### 15.2 修订方言口音

```text
请修订这段方言/口音描写。
要求：减少错字化和拼音化，保留少量词汇、句式、称谓、节奏和场景触发；确保可读、无嘲弄、不把口音当低等标志，并让语言差异改变人物、关系或信息。
```

### 15.3 写翻译场景

```text
请写一段需要翻译/传译的谈判场景。
要求：译者有立场和能力边界；翻译存在延迟、删改或文化不可译；至少有一个旁听者听懂原文或一个人误解译文；结尾写清翻译改变了什么局面。
```

### 15.4 设计语言魔法/真名规则

```text
请设计一个语言魔法/真名/咒语系统。
要求：写清谁能学、发音/文字/理解哪个关键、误读失败方式、使用代价、反制手段、禁词边界和读者如何通过场景理解规则。不要只堆自造词。
```

## 17. 安全与真实性边界

- 本专题只用于小说创作，不提供真实法律、医疗、移民、外交、法庭翻译或专业翻译服务建议。
- 涉及真实民族语言、少数民族语言、方言、宗教用语、残障语言、手语、外语口音、移民身份、历史文体和现实群体时，必须查证并避免刻板化、嘲笑和歧视。
- 不使用真实群体的语言特征作为低智、野蛮、邪恶、滑稽或低等标记。
- 不为了“真实”大段复制受版权保护文本、歌词、经文、诗歌、翻译作品或现实密语资料。
- 涉及合规、仇恨、歧视、隐私、未成年人和敏感身份时，联动第 20、105 篇。
- 架空语言可以自设，但必须以读者理解和剧情功能为先，不把正文变成词典。

## 18. 来源与吸收方式

- Encyclopaedia Britannica 关于 language、dialect 和 translation 的资料：吸收语言作为人类交流系统、方言/标准语关系和翻译转换意义的基础视角。
- Linguistic Society of America 关于语言、方言、语言变体和多语现象的资料：吸收“方言不是错误语言，语言差异有社会和身份意义”的视角，用于避免口音歧视和错字化。
- UNESCO 关于 linguistic diversity / multilingualism 的资料：吸收语言多样性与文化、教育、身份和社会参与相关的视角，用于多语言世界构建。
- PlainLanguage.gov 与可读性资料：吸收文本应尽量让目标读者理解、避免不必要行话的原则，用于外语、术语、自造语和古文投放。
- American Translators Association 等翻译/传译资料：吸收翻译不是逐字替换，而是意义、语境、专业能力和伦理边界的转换过程，用于译者角色、误译和保密成本。
- Writer's Digest、Reedsy、NowNovel 等关于 dialect、dialogue 和 voice 的写作资料：吸收方言口音要克制、可读、避免刻板化，并以人物和场景为核心的写作原则。
- 本知识库第 37、75、78、89、92、100、106、111、148 篇：整合角色声音、潜台词、信息密度、内心过滤、信息差、正典事实、命名术语、查证和消息链，形成“语言能力 -> 听懂范围 -> 翻译成本 -> 误读后果 -> 样式表回写”的执行链。

## 19. 最小执行规则

已确认：AI agent 写任何语言差异、方言口音、翻译、暗语、术语或跨文化沟通，最少必须检查“三件事”：

1. 谁会说、谁听得懂、谁需要翻译。
2. 语言差异是否改变信息、关系、权力或风险。
3. 译名、称谓、术语、语言能力和误译后果是否需要进样式表或台账。

如果三件事都没有，语言差异就是装饰口癖，应压缩、删除或重构。
