# 114. 力量体系、等级曲线、资源经济与战力通胀控制专项

状态：已确认

适用范围：当 AI agent 需要设计或审稿玄幻、仙侠、修仙、异能、系统、游戏、无限流、规则怪谈、末世、星际、机甲、卡牌、御兽、领主经营等含有能力、等级、资源、技能、装备、积分、境界、地图层级和敌人升级的小说时，使用本文件。

核心结论：力量体系不是设定表，而是长篇爽点的经济系统。每一次升级、奖励、突破、装备、地图开放和敌人增强，都必须同时回答“得到什么、花掉什么、限制什么、暴露什么、改变什么、以后还值不值钱”。如果只让数值上涨，不让选择、代价、反制和资源稀缺同步升级，故事会迅速战力膨胀、爽点变轻、读者信任坍塌。

---

## 1. 本专题解决什么问题

中文网文的长篇升级类题材最常见的崩坏，不是“主角变强”，而是“强得没有价格、没有边界、没有对手、没有后果”。典型症状：

- 境界名越来越大，但读者不知道差异在哪里。
- 主角突破太快，旧敌人、旧资源、旧地图瞬间失效。
- 系统奖励越给越多，读者反而不觉得爽。
- 战斗只剩招式名、光效、众人震惊和碾压。
- 对手升级只靠头衔，不靠策略、资源和反制。
- 金手指没有冷却、消耗、误差、暴露或道德代价。
- 新地图把旧关系、旧代价、旧资产全部清零。
- 作者临时加新规则，读者感觉被作弊。

本专题把能力、等级、资源、奖励、代价、敌人和地图视为同一个“长篇平衡系统”，帮助 agent 稳定设计可持续的变强快感。

最小链路：

能力规则 -> 等级差异 -> 资源入口 -> 资源出口 -> 使用限制 -> 失败代价 -> 对手反制 -> 地图涨价 -> 回报落袋 -> 台账回写 -> 通胀复盘

## 3. 来源吸收结论

### 3.1 魔法系统三原则

Brandon Sanderson 的魔法系统法则强调：读者越理解系统，作者越能用它解决问题；限制比能力本身更有趣；扩展已有能力通常比不断添加新能力更强。迁移到中文网文：主角可以开挂，但读者必须知道挂的规则、限制、代价、误用风险和升级边界。越强的能力越需要更清楚的账单。

### 3.2 Writing Excuses 的成本、规则和后果

Writing Excuses 关于魔法系统、规则、代价和世界后果的讨论，可迁移为：能力不是孤立技能，而会改变社会、经济、战斗、职业、身份和关系。若某能力真的强，世界应该围绕它形成限制、垄断、反制、恐惧或市场。

### 3.3 中文网文升级经验

起点社区和中文作者经验中，玄幻、仙侠、系统文常强调升级体系、爽点、资源争夺、阶段目标和地图扩张。迁移规则：升级不是换境界名，而是读者能感到行动资格变化、资源价格变化、敌人层级变化和主角选择空间变化。

### 3.4 游戏平衡与 power creep

游戏设计中的 power creep、反馈循环、资源入口/出口、经济水龙头/水槽等概念，可迁移为网文长篇平衡：如果只增加奖励入口，不增加消耗、稀缺、维护成本、反制和新目标，旧资源会贬值，旧敌人会失效，读者也会对新奖励麻木。

### 3.5 平台和读者语境

在免费阅读、付费订阅和长篇连载中，读者需要持续感到“变强有证明、胜利有价格、奖励能改变后文”。这不是慢，而是让爽点有重量。无代价升级会短期爽，长期会让追读变成流水账。

## 4. 核心概念

### 4.1 力量体系

定义：作品中能力如何获得、使用、升级、失效、被反制并改变世界反馈的规则集合。

最小字段：

- 能力来源。
- 使用条件。
- 输出效果。
- 边界限制。
- 消耗成本。
- 失败方式。
- 反制方式。
- 升级路径。
- 社会后果。
- 台账字段。

### 4.2 等级曲线

定义：角色、资源、敌人、地图和回报随章节推进的阶段性价格表。

不是境界名列表，而是每一档的可见差异：

- 能做什么新事。
- 不能再靠什么旧招。
- 需要什么新资源。
- 会招来什么新敌人。
- 赢了能拿什么新资格。
- 输了会失去什么。

### 4.3 资源经济

定义：故事中所有可消耗、可争夺、可囤积、可交换、可垄断、可贬值的东西。

包括：灵石、积分、寿命、体力、法力、装备、药材、信息、名声、权限、人情、土地、门票、职位、队友信任、系统点数、读者耐心。

### 4.4 战力通胀

定义：为了维持刺激，作者不断抬高数值、境界、敌人和奖励，却没有同步抬高限制、代价、策略和情感重量，导致旧内容快速贬值。

### 4.5 资源水龙头与水槽

定义：资源入口和资源出口。

- 水龙头：任务奖励、打怪掉落、宗门补贴、系统签到、交易收益、地图资源、读者打赏式虚构设定。
- 水槽：修炼消耗、装备维护、技能冷却、疗伤、情报购买、人情偿还、身份税、风险押金、阵法材料、时间成本。

只有水龙头没有水槽，资源会通胀；只有水槽没有水龙头，读者会憋屈。

### 4.6 反制生态

定义：世界和对手会根据主角能力学习、适应和反制。

反制不等于简单更强，而是改变问题类型：封锁资源、干扰触发条件、制造信息噪音、逼迫暴露、改写战场、拖延冷却、诱导误用、利用道德代价。

## 5. 总原则

### 5.1 限制先于特效原则

原则：能力的限制比特效更能支撑长篇。

用法：每个新能力先写限制、成本、失败方式和反制方式，再写炫酷效果。

风险：只写特效会让主角越来越像万能钥匙。

检查问题：如果主角不能用这招，他还能怎么解决？如果只能靠这招，限制够不够清楚？

### 5.2 升级必须改变行动资格原则

原则：升级不是数值上涨，而是行动资格变化。

用法：每次突破都要明确新增权限、解锁场景、旧招失效、新敌人出现或新资源价格。

风险：境界名变大但打法不变，读者会觉得水。

检查问题：这次升级让主角能做哪件之前不能做的事？同时失去或暴露了什么？

### 5.3 奖励必须有后续用处原则

原则：奖励不是本章烟花，而是后文资产。

用法：每个奖励记录可用性、限制、维护成本和下一次使用场景。

风险：奖励只在获得时爽，后文不用，会让读者不再关心奖励。

检查问题：这个奖励在后 3-10 章内会如何改变选择？

### 5.4 对手必须学习原则

原则：主角旧招连续成功后，对手和世界必须产生反馈。

用法：对手从误判、试探、封锁、针对、诱导、复制、污染、公开化逐步升级。

风险：对手原地等待，主角赢得廉价。

检查问题：对手看过主角上一轮胜利后，这一轮具体改了什么？

### 5.5 旧资产不能瞬间清零原则

原则：新地图、新境界、新系统版本不能让旧人物、旧资源、旧代价全部报废。

用法：新阶段至少保留两个旧资产继续有用，同时让一个旧资产重新定价。

风险：每次换地图都像重新开书，读者投资感断裂。

检查问题：上一卷的能力、关系、资源或代价，在新卷第一场怎么影响行动？

### 5.6 通胀要靠价格变化，不靠无限加零原则

原则：刺激升级应来自价格、限制、策略和价值变化，而不只是数字变大。

用法：同样是“赢”，后期可以让主角赢得更贵、更公开、更危险、更道德两难，而不是只打更高境界。

风险：数值越大，读者越麻木。

检查问题：这一轮更难，是因为敌人数字更高，还是因为选择更贵、限制更狠、后果更重？

- 主角可反制的抓手：
- 这轮失败或胜利后对手再学到什么：

## 7. 升级曲线设计

### 7.1 早期：规则证明期

目标：让读者相信能力有用、有边界、有代价。

应写：

- 第一轮低成本验证。
- 第一次误用或半成功。
- 第一次资源不足。
- 第一次暴露风险。
- 第一次对手误判。

避免：

- 第一章就全能。
- 系统替主角判断和执行。
- 奖励多到读者记不住。

### 7.2 中期：反制与组合期

目标：让旧能力遇到新问题，主角必须组合资源、换策略、付更高代价。

应写：

- 旧招被针对。
- 新资源变贵。
- 能力组合产生新解法。
- 对手开始研究主角。
- 升级带来社会后果。

避免：

- 每个敌人只是血量更厚。
- 每次卡关都靠新奖励。
- 旧队友、旧资源失效。

### 7.3 后期：规则改写期

目标：主角不只是变强，而是理解、利用或改写世界规则。

应写：

- 终局级选择和代价。
- 规则漏洞的长期铺垫。
- 资源经济或权力秩序变化。
- 旧资产和旧关系参与终局。
- 胜利后的新世界成本。

避免：

- 临时宣布新境界。
- 终局靠隐藏能力空降。
- 赢了之后世界没有变化。

## 8. 资源经济设计

### 8.1 资源入口

常见入口：

- 任务奖励。
- 训练产出。
- 副本掉落。
- 商业经营。
- 宗门供给。
- 势力分配。
- 抢夺战利品。
- 读者见证式公开奖励。
- 人情交换。
- 信息套利。

入口设计要问：谁生产？谁控制？谁分配？谁因此不满？

### 8.2 资源出口

常见出口：

- 修炼消耗。
- 装备维护。
- 技能冷却。
- 疗伤恢复。
- 情报购买。
- 队伍分赃。
- 人情偿还。
- 失败赔偿。
- 身份税或组织抽成。
- 时间机会成本。

出口设计要问：这笔消耗是否会影响后文选择？如果不会，就只是数字装饰。

### 8.3 稀缺与垄断

资源之所以有戏，是因为它有稀缺性或控制者。

可用稀缺类型：

- 数量稀缺。
- 时间窗口稀缺。
- 地点稀缺。
- 权限稀缺。
- 信息稀缺。
- 信任稀缺。
- 道德成本稀缺。
- 使用资格稀缺。

### 8.4 通胀控制

每次资源大量增加后，至少引入一种平衡机制：

- 新水槽：更高级技能、疗伤、维护、税费、队伍成本。
- 新风险：资源暴露、被盯上、引发争夺。
- 新价格：更高等级资源不可用低级资源替代。
- 新限制：携带、存储、转换、时间、权限。
- 新选择：资源只能投向 A 或 B，不能全都要。

## 9. 战力通胀诊断

### 9.1 轻度通胀

症状：数字变大，但还不影响读者追读。

修复：补可见差异和消耗出口。

### 9.2 中度通胀

症状：旧敌人、旧资源、旧地图快速失效，读者开始说水。

修复：重新定价旧资产，让旧资源在新限制下仍有局部用途；对手开始针对旧招。

### 9.3 重度通胀

症状：主角连续白拿奖励，敌人只能靠更大头衔，读者不再相信危险。

修复：冻结新增能力，强制用旧能力组合解决新问题；提高使用代价和暴露风险；安排一次合理局部失败。

### 9.4 信任崩塌

症状：作者临时加新规则或隐藏能力，高潮靠空降解决，读者觉得被骗。

修复：回到第 72 篇反作弊闸门，重铺规则、限制、代价和前文材料；必要时更换解法。

## 11. 审稿检查清单

当审稿发现“战力崩、升级太快、系统太强、奖励太多、敌人太弱、设定乱、旧地图清零、爽点变淡”时，agent 按顺序检查：

1. 当前核心能力的限制、代价、失败方式和反制是否明确？
2. 本次升级是否改变行动资格，而不只是换名字或加数字？
3. 本次奖励是否有后续用途、维护成本或使用限制？
4. 资源是否有入口也有出口？
5. 主角是否连续白拿收益，没有消耗、暴露、风险或关系代价？
6. 对手是否根据主角旧招做出学习和反制？
7. 新地图是否保留旧资产、旧关系和旧代价？
8. 旧能力是否仍有局部价值，还是被新能力完全取代？
9. 战斗胜负是否来自策略、限制和选择，而不只是数值高低？
10. 高潮是否靠前文已知规则组合解决，而不是临时加设定？
11. 能力台账、等级台账和资源台账是否回写？
12. 读者是否能用一句话说清“主角这次变强贵在哪里”？

## 13. 常见失败模式与修复

### 13.1 境界名堆叠

症状：筑基、金丹、元婴、化神不断出现，但读者不知道实际差异。

修复：为每档写行动资格、资源门槛、社会地位、失败代价和典型对手。

### 13.2 奖励白送

症状：系统每天发技能、装备、积分，主角没有选择和代价。

修复：增加资源出口、维护成本、使用限制和互斥选择。

### 13.3 旧招万能

症状：主角靠同一能力解决所有问题。

修复：让对手干扰触发条件，改变战场，制造旧招使用代价。

### 13.4 新招空降

症状：高潮突然解锁新能力，前文没有铺垫。

修复：把解法改成旧能力组合、旧资源再定价或旧伏笔回收。

### 13.5 地图清零

症状：换地图后旧人物、旧关系、旧资源全部没用。

修复：新卷第一场带入至少两个旧资产，并让一个旧代价继续限制行动。

### 13.6 敌人只会变强

症状：敌人只从“长老”变“大长老”“老祖”，策略不变。

修复：让敌人学习主角旧招，封锁资源、切断信息、制造暴露或道德压力。

### 13.7 战斗只剩数值

症状：谁境界高谁赢，战斗没有策略和选择。

修复：增加地形、资源、冷却、目标差、时间压力、保护对象和失败代价。

## 14. 最小执行规则

如果 agent 时间很少，至少执行这 8 条：

1. 每个核心能力必须有触发条件、限制、成本、失败方式和反制。
2. 每次升级必须改变行动资格，并带来新限制或新价格。
3. 每个奖励必须在后文有用途、维护成本或使用风险。
4. 资源必须同时有入口和出口，不能只发不花。
5. 对手必须学习主角旧招，不能原地等主角变强。
6. 新地图必须保留旧资产和旧代价，不能全部清零。
7. 高潮解法必须来自前文已知规则组合，不能临时加设定。
8. 发现通胀时，优先冻结新能力、重定价旧资产、补反制和补代价。

## 15. 推荐调用顺序

### 15.1 设计升级类新书

1. 读 `docs/market-serialization/13-genre-patterns.md` 确认题材承诺。
2. 读 `docs/storycraft/14-mechanism-and-cheat-design.md` 设计机制。
3. 读本文件建立能力、等级和资源台账。
4. 读 `docs/storycraft/21-antagonist-opposition-system.md` 设计对手反制。
5. 读 `docs/workflows/43-commercial-project-greenlight-scorecard.md` 做立项评分。

### 15.2 写训练或突破

1. 读 `docs/workflows/67-training-growth-anti-padding.md`。
2. 读本文件确认突破后的行动资格和新代价。
3. 读 `docs/advanced-craft/86-payoff-currency-reward-asset-continuity.md` 确认奖励后文可用。

### 15.3 审稿发现战力崩

1. 读本文件诊断通胀等级。
2. 读 `docs/advanced-craft/72-reader-trust-plausibility-anti-cheat.md` 检查规则信任。
3. 读 `docs/storycraft/68-protagonist-failure-setback-cost-management.md` 设计合理代价胜或局部失败。
4. 回写 `docs/core-writing/100-canon-uncertainty-conflict-hallucination-control.md` 所要求的 canon 台账。

### 15.4 换地图或开新卷

1. 读本文件确认旧资产如何重新定价。
2. 读 `docs/advanced-craft/56-volume-transition-new-stage-launch.md`。
3. 读 `docs/core-writing/60-antagonist-world-synchronous-escalation.md`。
4. 读 `docs/market-serialization/113-reader-journey-conversion-lifecycle-funnel.md` 确认新阶段不会像重新开书。
