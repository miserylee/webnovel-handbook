# 116. 恐惧曲线、惊吓节拍、规则恐怖与安全阈值专项

状态：已确认

适用范围：当 AI agent 需要设计、起草、审稿或修订恐怖、惊悚、悬疑、规则怪谈、无限流副本、民俗怪谈、心理恐怖、末世生存、诡异复苏、黑暗副本、校园怪谈、都市传说、怪物追逐、精神控制、血腥暴力或高压生存场景时，使用本文件。

核心结论：恐怖不是血腥、怪物、尖叫和突发惊吓的堆叠，而是让读者在可理解的规则、可感知的危险和角色有限选择之间持续感到“不安全”。商业网文中的恐惧必须服务追读、规则、线索、人物选择和代价后果；不能用猎奇、低俗、创伤消费、可模仿危险行为或随机改规则替代真正的恐怖。

---

## 1. 本专题解决什么问题

AI 写恐怖/规则怪谈时常见问题：

1. 只堆血腥：画面很重，但读者不怕，只觉得脏、腻或低俗。
2. 只靠跳吓：门突然开、脸突然贴、声音突然响，短期有效，长期疲劳。
3. 规则随机：为了吓人临时改规则，读者无法推理，只觉得作者作弊。
4. 人物工具化：角色只负责尖叫、送死、解释规则，没有欲望和选择。
5. 氛围空转：阴冷、漆黑、诡异写很多，但没有目标、线索和进展。
6. 安全越界：自伤、犯罪、酷刑、未成年人、精神控制、创伤细节被写成猎奇刺激。
7. 追读断裂：恐怖场景吓完就完，没有答案利息、规则变化或下一步目标。

本专题的作用：把恐惧拆成 AI 可执行链路：

安全感建立 -> 违和信号 -> 规则缺口 -> 危险靠近 -> 选择受限 -> 短暂释放 -> 代价落地 -> 规则再定价 -> 新恐惧牵引

## 3. 来源吸收结论

### 3.1 恐怖写作：未知、脆弱和后果

Reedsy、Writer's Digest、Now Novel 等恐怖写作资料都强调，恐怖的关键不只是怪物，而是未知、脆弱、氛围、悬念、代价和人物关切。迁移到中文网文：agent 不能只写“有鬼”“很吓人”，必须让读者知道角色在乎什么、失去什么、还能做什么、做错会怎样。

### 3.2 恐惧曲线：不安、畏惧、恐惧、惊骇

Trajectory of Fear 等恐怖设计资料将恐惧拆成从 unease 到 dread、terror、horror 的递进。迁移到网文：先给违和，让读者意识到“不对”；再让危险逐步靠近；再让角色被迫选择；最后让后果落地。直接上最大刺激会让读者麻木。

### 3.3 规则怪谈与公平性

悬疑和规则怪谈需要读者相信规则可被观察、试探、误读和验证。迁移规则：规则可以残酷，但不能随机；可以误导，但必须给可回看的材料；可以有例外，但例外也要成为更深规则的一部分。

### 3.4 内容安全与创伤知情

SAMHSA、RAINN、NetGalley 等安全和内容提示资料提醒：涉及创伤、自伤、性暴力、未成年人、家庭暴力和高强度血腥时，应避免猎奇、教程化和二次伤害。迁移到恐怖写作：恐惧可以强，但不能把现实伤害当廉价刺激。

## 4. 核心概念

### 4.1 恐惧曲线

定义：读者从“觉得不对”到“确认危险”再到“看见代价”的情绪递进。

最小曲线：

违和 -> 不安 -> 预兆 -> 接近 -> 限制 -> 爆发 -> 后果 -> 余波 -> 新规则

### 4.2 违和信号

定义：正常世界中出现的一点不正常。

例子：

- 同一句广播重复，但少了一个字。
- 楼道灯每次闪烁后，门牌号都少一个。
- 熟人说话的称呼没错，但停顿位置不对。
- 手机时间正常，聊天记录却多出明天的回复。

违和信号要小、准、有后续意义，不是随机怪。

### 4.3 危险规则

定义：读者和角色能观察到的生存限制。

好规则：可观察、可试探、可误读、可付代价、可回收。

坏规则：作者想让谁死谁死，事后再解释。

### 4.4 恐惧货币

定义：恐怖场景用来购买读者注意力的压力类型。

常见货币：未知、孤立、失控、被注视、身体脆弱、身份被替换、熟悉物变陌生、时间倒计时、规则惩罚、信任背叛、空间封闭、声音异常、群体同化、记忆污染、代价不可逆。

### 4.5 安全阈值

定义：目标读者、平台边界和题材承诺能承受的内容强度。

安全阈值不等于“不吓人”，而是把恐惧控制在服务故事的范围内，不越过猎奇、低俗、教程化、创伤消费和平台风险。

## 5. 总原则

### 5.1 先不安，后惊吓原则

原则：惊吓前要有违和和预感。

用法：先让读者发现“某处不对”，再让危险靠近，最后释放惊吓或后果。

风险：直接跳吓会疲劳，读者只记得作者吓人方式廉价。

检查问题：惊吓前，读者是否已经知道哪里不安全？

### 5.2 规则公平原则

原则：恐怖可以残酷，但规则不能随机。

用法：每条规则都要有首次信号、试探机会、误读空间、惩罚方式和回收方式。

风险：规则随机会让读者放弃推理，只等作者宣判。

检查问题：读者回看时能否承认“前面确实给过材料”？

### 5.3 角色在乎原则

原则：恐怖必须伤到角色在乎的东西。

用法：在恐怖爆发前先建立角色目标、关系、秘密、尊严、身体安全或回家愿望。

风险：角色只是尖叫工具，读者不会怕，只会旁观。

检查问题：如果这个危险成真，角色具体失去什么？

### 5.4 细节克制原则

原则：恐怖细节要服务感知和后果，不服务猎奇。

用法：优先写声音、空间、节奏、身体反应、选择受限和后果；血腥细节只在必要时短、准、低频使用。

风险：血腥越堆，恐惧越钝，平台和读者风险越高。

检查问题：删掉血腥细节后，恐惧是否还成立？

### 5.5 释放与再定价原则

原则：高压后必须给短暂释放和新信息，让读者既喘气又想追。

用法：一次恐怖爆发后，至少给一个答案利息：规则更清楚、代价更具体、安全区更小、人物关系变了。

风险：一直高压会疲劳；只吓不解释会麻木。

检查问题：这场恐怖结束后，读者多知道了什么？下一步更怕什么？

## 6. 恐惧曲线设计

### 6.1 安全基准

先建立“正常”：时间、地点、人物关系、日常规则、空间边界。

没有正常，就没有异常。AI 不要一上来就“整个世界都诡异”，那样读者没有参照物。

### 6.2 违和信号

给一个小异常，让读者和角色都可以注意到，但暂时无法解释。

要求：

- 可感知。
- 可复现或可升级。
- 与后文规则有关。
- 不直接解释。

### 6.3 预兆累积

让异常变成模式，而不是随机现象。

可用方式：重复、变体、倒计时、旁人反应、物件变化、空间变窄、声音接近、旧规则失效。

### 6.4 危险靠近

让角色选择空间变小。

可用限制：门打不开、灯不能灭、不能回头、不能说名字、不能睡着、不能离队、不能看见某物、必须在时间内完成动作。

### 6.5 爆发或揭露

爆发要回答一个小问题，同时打开更大的问题。

例：黑影不是要杀人，而是在替换记忆；安全区不是安全，而是筛选区。

### 6.6 后果落地

恐怖必须改变状态：有人受伤、规则更清楚、资源减少、关系破裂、身份暴露、时间缩短、空间失守。

### 6.7 短暂释放

给读者喘息：整理线索、处理伤口、关系反应、重新制定规则、低压对话。

释放不是水。释放必须让下一轮恐惧更具体。

## 7. 恐惧货币表

| 恐惧货币 | 用法 | 高危错误 |
| --- | --- | --- |
| 未知 | 留出可推但未解的缺口 | 永远不给材料，变谜语人 |
| 孤立 | 切断求助、通信、队友信任 | 为孤立而孤立，无现实逻辑 |
| 失控 | 规则、身体、记忆或空间不受角色控制 | 角色完全无能，只能等死 |
| 被注视 | 让角色意识到自己暴露 | 监控万能，主角无选择 |
| 熟悉物陌生化 | 家、学校、医院、地铁变危险 | 只贴阴间滤镜，不改变规则 |
| 身体脆弱 | 伤、病、疲劳、饥饿、感官受限 | 细节猎奇或教程化 |
| 身份替换 | 熟人像熟人但不对 | 全靠“他不是他”，无证据链 |
| 时间压力 | 倒计时、循环、夜晚、班次 | 倒计时不影响选择 |
| 规则惩罚 | 违规有代价 | 惩罚随机或不公平 |
| 信任背叛 | 队友隐瞒、系统误导 | 为反转牺牲人物逻辑 |

## 8. 题材适配

### 8.1 规则怪谈

重点：规则可观察、可试探、可误读、可回收。

最低要求：

- 每条规则有来源或呈现方式。
- 至少一次低成本试探。
- 至少一次误读代价。
- 例外不是漏洞，而是更深规则。

### 8.2 无限流副本

重点：生存目标、队伍关系、资源消耗和副本规则同步推进。

最低要求：

- 副本目标明确。
- 队友功能不同。
- 线索、道具、体力和信任都可消耗。
- 每轮危险改变队伍结构或认知。

### 8.3 心理恐怖

重点：感知不可靠、关系压力、记忆缺口和自我怀疑。

最低要求：

- 不可靠感知必须有规则或触发。
- 不能用心理疾病标签替代恐怖机制。
- 涉及心理危机时联动第 105 篇。

### 8.4 民俗怪谈

重点：禁忌、仪式、地方秩序、口耳相传和现实生活入口。

最低要求：

- 民俗不能猎奇化真实群体。
- 禁忌要改变行动，不是装饰。
- 地域/文化资料要联动第 31 和第 111 篇。

### 8.5 血腥/身体恐怖

重点：身体边界、不可逆后果和角色恐惧。

最低要求：

- 不靠细节量取胜。
- 不写成可模仿伤害教程。
- 需要内容提示或降级处理时联动第 105 篇。

## 9. 安全阈值与内容分级

### 9.1 低强度恐怖

特征：违和、轻微惊吓、氛围不安、规则疑云。

适合：开篇、轻悬疑、都市怪谈、非重口目标读者。

### 9.2 中强度恐怖

特征：明确危险、追逐、轻度伤害、死亡暗示、规则惩罚。

适合：规则怪谈、无限副本、悬疑惊悚主线。

### 9.3 高强度恐怖

特征：暴力、血腥、精神控制、身体破坏、强创伤主题。

处理：必须联动第 105 篇；检查平台边界、目标读者、必要性、细节尺度、后果承接和内容提示。

### 9.4 禁止或重构区

包括：可模仿自伤/犯罪步骤、未成年人性化或猎奇化、性暴力猎奇、酷刑教程化、现实群体污名化、把创伤当爽点。

处理：不写或重构为后果、选择、保护、线索和情绪余波。

## 11. 审稿检查清单

当审稿发现“恐怖不吓人、规则怪谈像乱编、血腥太多、氛围水、惊吓疲劳、读者看不懂、内容太不适”时，按顺序检查：

1. 本场先建立正常了吗？
2. 违和信号是否具体、可感知、有后续意义？
3. 危险规则是否可观察、可试探、可误读、可回收？
4. 角色是否有目标和在乎的东西？
5. 恐惧是否改变角色选择，而不只是吓一跳？
6. 惊吓前是否有不安和预兆？
7. 血腥细节是否必要、克制、非猎奇？
8. 高强度内容是否联动第 105 篇安全闸门？
9. 本场结束后是否有后果落地？
10. 读者是否获得了答案利息或新规则？
11. 下一场恐惧是否更具体，而不是更随机？
12. 是否存在平台风险、目标读者错配或内容提示需求？

## 12. 常见失败模式与修复

### 12.1 血腥替代恐惧

症状：尸体、伤口、器官、惨叫很多，但没有规则、选择和后果。

修复：降低血腥密度，补违和信号、规则、角色目标和代价。

### 12.2 跳吓疲劳

症状：每章都突然出现黑影、鬼脸、尖叫。

修复：增加不安积累、预兆变体和后果变化，减少重复惊吓。

### 12.3 规则随机

症状：规则为了杀人临时变化。

修复：把变化改成更深规则；前文补信号和低成本试探。

### 12.4 氛围空转

症状：阴冷、压抑、诡异写很多，但没有推进。

修复：每个氛围细节必须绑定线索、限制、误读、代价或下一步行动。

### 12.5 主角只会害怕

症状：主角一直尖叫、逃跑、被解释。

修复：给主角观察、试探、选择、错误判断和付代价后的调整。

### 12.6 创伤猎奇

症状：用现实伤害、未成年人痛苦、性暴力、精神控制或酷刑制造刺激。

修复：删除或降级细节，写后果、保护、选择、支持和恢复；联动第 105 篇。

## 13. 最小执行规则

如果 agent 时间很少，至少执行这 8 条：

1. 恐怖场景先建立正常，再写异常。
2. 每个恐惧点必须有违和、规则、选择或后果，不能只是吓人画面。
3. 规则怪谈必须公平：可观察、可试探、可误读、可回收。
4. 惊吓不能连续重复使用，必须和悬念、释放、后果交替。
5. 血腥和暴力细节必须克制，不能替代恐惧机制。
6. 高风险内容必须联动第 105 篇，不写可模仿教程或猎奇创伤。
7. 每场恐怖结束后要给答案利息、代价落地或新规则牵引。
8. 恐惧要服务追读、人物选择和规则理解，不服务随机刺激。

## 14. 推荐调用顺序

### 14.1 规则怪谈/无限流副本

1. 读 `docs/storycraft/26-suspense-reversal-information-control.md`。

1. 读本文件判断恐惧目标和安全阈值。

### 14.3 恐怖场景审稿

1. 读本文件检查正常、违和、预兆、规则、选择、后果。
