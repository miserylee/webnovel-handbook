# 149. 出行旅程、地图路线、车马舟船、客栈驿站、关卡通行、传送阵与路程成本

状态：已确认

适用对象：AI agents、网文作者、审稿 agent、连续性维护 agent

核心问题：很多 AI 写作会把人物移动写成“他们赶了几日路，终于到了”，导致地图不可信、资源无限、节奏水、危险随机、到达后状态没有变化。商业级长篇里，旅程不是 A 点到 B 点的空白过渡，而是路线、交通、时间、补给、通行、住宿、风险、信息、关系和到达状态差额共同组成的叙事压力系统。

---

## 1. 一句话原则

已确认：出行旅程必须回答“为什么现在必须走、怎么走、代价是什么、路上改变了什么、到达时少了什么或多了什么”。

如果这五个问题没有答案，就不要写成一章，只做压缩过渡。

## 3. 判断：这段旅程该展开还是压缩

### 3.1 必须展开的情况

出现以下任一条件，旅程可以写成场景或章节：

- 旅程中发生资源消耗：钱、粮、水、药、灵石、票、燃料、马力、船期、手机电量、证件有效期。
- 旅程中改变关系：同行者争执、照顾、互相试探、建立暗号、分工守夜、救援、背叛、发现秘密。
- 旅程中发生信息交换：客栈传闻、车站广播、船上闲谈、商队消息、关卡告示、地图更新、导航失灵。
- 旅程中有制度压力：城门盘查、路引、过所、关税、宗门令牌、航班安检、边检、宵禁、禁飞区。
- 旅程中有地形压力：山路、水路、荒漠、雪线、沼泽、魔兽领地、塌方、洪水、封路、海雾。
- 旅程中有时间压力：赴约、开考、婚礼、开庭、秘境开启、船期、列车班次、日落闭城、传送阵轮次。
- 旅程本身就是类型承诺：游历、逃荒、镖局、商队、赶考、修仙历练、远征、末世迁徙、公路文、综艺旅行。

### 3.2 应该压缩的情况

出现以下情况，应只写 1 到 4 句压缩，不要扩成水章：

- 路上没有新冲突、新信息、新关系、新资源变化。
- 读者已经知道路线，且本次移动只是把人物送到下一场戏。
- 交通条件稳定，没有阻碍、风险或选择。
- 上一章已经结束在强钩子上，读者期待的是抵达后的冲突。
- 连续多章都在赶路，但没有阶段性成果或代价。

压缩公式：

```text
路线一句：他们从哪里出发，沿哪条路/哪种交通到哪里。
成本一句：用了多久，花掉/损耗/错过了什么。
变化一句：关系、身体、信息或身份发生了什么变化。
落点一句：刚到新地点，立刻撞上新压力。
```

示例骨架：

```text
三人沿官道北上，白日混在粮车后，夜里借破庙避风，五日后才看见青州城墙。
盘缠少了大半，马也瘦了一圈，只有客栈里听来的“城门查盐引”让他们一路没敢走正门。
进城前，排队的人群忽然散开，一张新贴的缉捕告示正钉在城门口。
```

## 4. 旅程十线卡

AI agent 写任何长距离移动前，先填这张卡。没有填满不一定不能写，但缺哪一线，就要有意识地压缩或留白。

```text
出发点：
目的地：
旅程目标：为什么现在必须移动？
必达期限：晚到会失去什么？
路线节点：起点、中转、住宿点、关卡、岔路、终点。
备选路线：更快、更安全、更便宜、更隐蔽的路线分别是什么？
交通方式：步行、骑马、车、船、飞舟、传送阵、高铁、航班、地铁、私家车等。
路程成本：时间、钱、粮、水、燃料、灵石、票、体力、马力、信誉、人情。
通行凭证：路引、令牌、邀请函、车票、护照、工作证、通关文牒、宗门腰牌。
补给计划：食物、水、药、住宿、维修、换马、充电、换乘、导航、地图。
外部风险：天气、地形、治安、怪物、检查、封路、延误、诈骗、盗窃、疾病。
同行关系：谁负责钱、谁识路、谁守夜、谁拖后腿、谁隐藏身份、谁有私心。
路上信息：传闻、告示、地图、新闻、闲谈、信件、系统提示、敌方动向。
到达差额：抵达时身体、钱、物、人际、情绪、信息、身份比出发时发生了什么变化？
台账回写：时间线、地点线、金钱线、伤病线、道具线、消息线需要更新什么？
```

## 5. 路线不是直线：四种选择压力

### 5.1 快路

快路通常有更高代价：费用高、暴露身份、需要凭证、排队、受制度监管、容易被追踪。

适合制造：主角必须暴露、借人情、赌运气、抢时间、错过休息。

### 5.2 安全路

安全路通常更慢：官道、商队、镖局、正规航班、驿站路线、宗门飞舟固定班次。

适合制造：信息传播、社会关系、制度摩擦、公共场合的压力。

### 5.3 隐蔽路

隐蔽路通常更危险：小路、山道、黑船、走私线、废弃传送阵、无人区。

适合制造：迷路、补给不足、怪物、背叛、身份伪装、道德选择。

### 5.4 省钱路

省钱路通常更消耗身体和时间：步行、拼车、搭货船、睡车站、借宿、用低阶灵舟。

适合制造：阶层差、窘迫感、现实质感、人物互相照顾。

## 6. 交通方式的叙事用法

### 6.1 步行

关键词：慢、累、距离感最强、身体代价最明显。

用法：适合写底层、逃亡、朝圣、修行、荒野求生、末世迁徙。步行不能只写“走了三日”，要写脚、鞋、饥饿、水、背包、夜宿、安全感。

风险：如果步行没有疲劳和补给压力，会显得人物像游戏单位瞬移。

### 6.2 骑马、马车、牛车

关键词：马力、饲料、换马、路况、身份。

用法：马车适合带出阶层、礼制、速度差和道路条件。骑马快，但需要会骑、马要休息、山路和雨雪会降速。

风险：马不是无限续航工具。长途赶路要考虑换马、马料、马病、惊马、车轴损坏、夜间安全。

### 6.3 舟船、水路、渡口、码头

关键词：船期、天气、水势、货舱、拥挤、隔绝。

用法：船上空间封闭，适合密室式关系推进、偷听、误会、投毒、暴风、落水、货物纠纷。渡口和码头适合展示阶层、商贸、治安、等待和消息汇聚。

风险：水路不应只当“更快的路”。它受季节、风向、水位、船家、官税、港口秩序影响。

### 6.4 客栈、旅店、驿站、服务区

关键词：中转、补给、信息、公共空间、陌生人。

用法：客栈不是吃饭睡觉的背景板，而是社会传感器。这里可以出现告示、江湖传闻、商队议价、房间不足、身份登记、黑店、店小二口风、同路人、官差查房。

风险：如果每次客栈都只点菜、听八卦、打架，会变成模板化水文。每个中转点必须服务一个具体变化。

### 6.5 商队、镖局、护送队

关键词：群体移动、信用、货物、路线经验、集体风险。

用法：适合把主角接入陌生地区的社会网络。商队能提供保护，也会带来慢速、规矩、分成、队内等级、货物秘密和路线选择。

风险：不要让商队只当免费交通工具。它至少要索取钱、人情、劳动、保密、武力或风险承担。

### 6.6 现代交通：公交、地铁、高铁、航班、网约车、自驾

关键词：班次、票、延误、安检、导航、手机、证件、行李、人群。

用法：都市、悬疑、娱乐圈、现言、商战中，现代交通可以制造非常真实的微压力：赶不上末班车、航班取消、车站人流、手机没电、证件遗失、行李被拿错、堵车、定位暴露。

风险：现实交通、边检、安检、证件和法律规则如果影响剧情，必须查最新真实资料。不要凭空写现实世界可执行的违规教程。

### 6.7 仙侠/玄幻交通：御剑、飞舟、灵兽、传送阵、空间门

关键词：成本、权限、冷却、容量、坐标、风险、阶层。

用法：高能交通必须有制度和代价，否则会摧毁距离感。传送阵可以按“站点制、预约制、灵石成本、权限等级、阵法维护、坐标偏差、追踪记录、禁飞区、战时管制”来设计。

风险：传送阵万能会让追逃、赴约、远征、边疆、秘境距离全部失效。越强的交通，越要绑定更强的限制。

## 7. 节点写法：让路程变成剧情

### 7.1 城门/关卡/边检节点

城门不是“进城”按钮，而是身份与秩序的碰撞点。

可写压力：

- 凭证是否齐全。
- 名单、画像、通缉、税契、邀请函是否匹配。
- 队伍排队导致时间损失。
- 守卫认人、索贿、刁难、放行标准不一致。
- 主角必须选择暴露实力、花钱、绕路、借身份或放弃行程。

关卡模板：

```text
关卡名称：
掌控势力：
检查对象：人、货、车、船、灵兽、储物袋、法器、证件。
放行条件：
禁止事项：
常规等待时间：
主角缺口：缺钱、缺证、身份危险、货物敏感、同行者被认出。
冲突触发：
通过代价：
后续后果：被登记、被跟踪、欠人情、错过期限、失去伪装。
```

### 7.2 客栈/旅店/驿站节点

客栈节点最好同时承担三项功能：补给、信息、关系。

```text
地点类型：客栈/驿站/旅店/服务区/车站旅馆/宗门驿馆
主角需求：住宿、吃饭、换马、洗澡、治伤、打听消息、躲避追踪。
店方规则：价钱、身份登记、宵禁、房间、押金、禁武、禁灵兽。
公共信息：告示、闲谈、新闻、传闻、商队路线、悬赏。
关系动作：同住、分房、守夜、争吵、照顾、偷听、试探。
危险：黑店、查房、偷窃、下药、火灾、暴露身份、错认。
离开时变化：钱少了多少，知道了什么，谁怀疑谁，下一站怎么变。
```

### 7.3 岔路/绕路节点

岔路的核心不是地图说明，而是价值选择。

- 走官道：安全但会被登记。
- 走山路：隐蔽但补给少。
- 走水路：快但看船期和天气。
- 走商队：有保护但要服从队伍。
- 走传送阵：快但昂贵、可追踪、需要权限。

好的岔路会让读者感到：人物不是被作者搬运，而是在约束中做选择。

## 8. 到达后必须有“状态差额”

旅程结束时，AI agent 必须明确写出到达差额。至少选择 2 项：

- 身体差额：疲惫、伤病、饥饿、晕船、冻伤、灵力消耗、精神紧绷。
- 资源差额：钱少了、票没了、马病了、药用完、法器损坏、手机没电、油量不足。
- 时间差额：提前、准时、迟到、错过船期、赶上最后一刻。
- 信息差额：路上听到新消息、旧情报失效、地图错误、敌人先到。
- 关系差额：更信任、更疏远、出现秘密、欠人情、队伍分裂。
- 身份差额：被登记、被跟踪、伪装暴露、获得通行资格、被新势力注意。
- 情绪差额：期待变警惕、轻松变焦躁、陌生感、归乡感、压迫感。

如果没有差额，旅程就不该占据叙事篇幅。

## 9. 类型化用法

### 9.1 玄幻/仙侠

重点：距离感、宗门势力边界、灵石成本、飞行权限、传送阵等级。

可用规则：

- 凡人步行与修士飞遁形成阶层差。
- 宗门飞舟按班次或任务调度，不是私人出租车。
- 传送阵只连重要城池、宗门、边关、秘境入口，偏远处仍需赶路。
- 禁飞区、护城阵、妖兽领地、灵脉紊乱可以限制高阶移动。
- 坐骑要吃、要休息、会受伤，也可能有情绪和忠诚问题。

### 9.2 古言/权谋/官场

重点：路引、驿站、官道、船运、门禁、驿递、阶层礼制。

可用规则：

- 谁能住驿站、谁只能住客栈，能体现身份差。
- 赴任、押送、赶考、嫁娶、吊丧都天然有期限与礼制压力。
- 水路、陆路、官道、私路会带出地方势力和治理水平。
- 城门盘查、宵禁、税卡、渡口纠纷可以让制度进入剧情。

### 9.3 都市/现言/悬疑

重点：导航、监控、票据、手机、班次、堵车、定位、公共空间。

可用规则：

- 交通票据和定位记录能制造证据链，也能制造暴露风险。
- 延误、末班车、堵车、暴雨、手机没电能制造低成本高代入感的压力。
- 机场、车站、酒店、服务区是人群密集的信息和误认场。

### 9.4 末世/逃荒/战争

重点：路线安全、补给、队伍秩序、伤病、迁徙速度、车辆维护。

可用规则：

- 车队速度取决于最慢的人和最差的车。
- 每次绕路都消耗燃料、粮食和士气。
- 桥、隧道、渡口、加油站、医院、仓库是战略节点。
- 弱者、伤员、老人、小孩、牲畜会真实改变行程。

## 10. 常见失败模式

- 赶路水章：写了大量吃饭住宿闲聊，但没有新信息、新关系、新代价。
- 地图瞬移：人物跨越大距离，却没有时间、成本、疲劳、凭证、风险。
- 无限补给：钱粮水药永远够，车马灵舟永远不坏。
- 传送阵万能：需要紧张时不用，不需要紧张时突然能用。
- 客栈模板化：每到客栈都是点菜、听八卦、遇恶霸。
- 关卡无逻辑：守卫只在作者需要时严查，平时毫无制度。
- 路线选择无代价：快路、安全路、隐蔽路、省钱路没有差异。
- 到达无差额：出发时什么状态，到达时还是什么状态。
- 新地图灌设定：刚到一个地方就大段介绍历史地理，没有先给当前压力。

## 11. 审稿检查问题

AI 审稿或修订时，逐条检查：

- 这段出行是否有明确目标和期限？
- 读者能不能在脑中知道大致从哪里到哪里？
- 交通方式是否符合人物身份、财力、时代、世界观？
- 路程有没有消耗至少一种资源？
- 是否存在至少一个路线选择，而不是作者直接搬运人物？
- 客栈、驿站、车站、码头、关卡是否承担了补给、信息、关系或制度压力？
- 如果使用传送阵、飞舟、航班、高铁等快速交通，它的限制是否清楚？
- 到达后人物状态是否发生变化？
- 如果这一段删掉，剧情是否只少了字数、不少信息？如果是，应压缩。
- 现实交通、边检、安检、法律、票务等内容是否需要查最新资料？

## 12. AI 写作提示词片段

### 12.1 展开旅程

```text
请根据以下旅程十线卡写一个网文章节片段。
要求：不要把赶路写成说明文；至少写出一个路线选择、一个资源消耗、一个路上信息、一个同行关系变化；结尾必须让人物带着新的状态差额抵达下一节点。
```

### 12.2 压缩旅程

```text
请把这段长距离移动压缩成 3 到 5 句。
要求：交代路线、耗时、主要消耗、关系或信息变化，并用最后一句接入抵达后的新冲突。不要展开无效吃饭住宿闲聊。
```

### 12.3 修订水章

```text
请检查这章的赶路段落是否水文。
如果没有新冲突、新信息、新关系、新消耗，请压缩。
如果需要保留，请为旅程补上路线选择、通行压力、补给代价和到达状态差额。
```

## 13. 安全与真实性边界

- 现实世界交通、边境、安检、证件、票务、执法、逃避监管等内容，不写成可执行规避教程。
- 如果剧情依赖真实法律、政策、交通规则或安检流程，必须另行查最新官方资料。
- 历史交通、驿站、关卡、商队、丝路资料可以作为叙事灵感，但不要把不同朝代、地区制度混为同一套规则。
- 幻想世界可以自设规则，但规则一旦建立，就要在后续台账中持续执行。

## 14. 来源与吸收方式

- Reedsy, “Story Setting” 与 “Worldbuilding Guide”：吸收其“设定应服务人物、情节和读者体验”的思路，用于约束旅程不要变成纯地图百科。
- Encyclopaedia Britannica, “Road”：吸收道路作为人、动物、车辆移动通道的基础定义，以及道路建设会改变运输能力的历史视角。
- Encyclopaedia Britannica, “History of Transportation”：吸收官方道路、驿站、邮递网络与中央治理能力相关的观点，用于古代官道、驿站、关卡设计。
- Encyclopaedia Britannica, “Silk Road” 与 “Caravan”：吸收长距离贸易路线、商队、海路/陆路变化、交通方式替代对社会网络的影响。
- UNESCO Silk Roads Programme：吸收丝路是路线网络而非单一路径、沿途城市/港口/驿站/旅店/商队旅舍促进贸易和文化交换的观点。

## 15. 最小执行规则

已确认：AI agent 写人物远距离移动时，最少必须保留“三件事”：

1. 走哪条路。
2. 付出什么成本。
3. 到达后哪里变了。

如果三件事都没有，这段旅程应删除或压缩。
