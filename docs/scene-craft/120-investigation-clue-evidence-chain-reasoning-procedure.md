# 调查线索、证据链、推理场景与侦查流程专项

状态：已确认  
适用对象：AI agents 在设计、起草、审稿或修订悬疑、刑侦、探案、规则怪谈、副本解谜、宫斗查案、职场调查、宗门翻案、商战取证、娱乐圈黑料、历史断案、律师/记者/侦探线、追凶线和任何“角色需要把未知变成可行动结论”的剧情时使用。  
核心目标：让调查剧情不再靠作者临时塞线索、主角突然顿悟、万能监控、匿名爆料或最终自白，而是形成可追踪、可误读、可验证、可付出代价的线索链、证据链和推理链。

---

## 1. 本专题解决什么问题

AI 写调查、推理、查案、解谜时常见问题：

- 线索突然出现：关键证据只在揭晓前一章冒出来，读者回看时找不到早期痕迹。
- 证据过度顺滑：监控刚好拍到，录音刚好完整，证人刚好愿意说，报告刚好当天出。
- 推理跳步：主角从一个细节直接跳到真相，中间没有排除、验证、对照和失败尝试。
- 红鲱鱼廉价：假线索只负责骗读者，最后不揭示人物秘密、不改变局面，也不承担后果。
- 专业悬浮：刑侦、法医、律师、记者、黑客、审计、宫廷查案、宗门执法都像同一套万能查资料。
- 角色无成本：调查没有权限、风险、时间、关系、名誉、伦理或安全代价。
- 读者不公平：作者掌握所有答案，读者只能等主角宣布结论，缺少“我也可以拼出来”的参与感。
- 多 agent 断线：前文线索、证词、疑点、排除项和待验证假设没有台账，后续 agent 容易遗忘或编造。

本专题的原则：调查不是“找到真相”的摘要，而是角色在有限信息、错误解释、现实阻力和对手干扰中，把模糊信号一步步变成可行动结论的过程。

## 3. 来源依据与可迁移结论

### 3.1 写作资料

Writing Excuses 的 mystery plotting 资料强调：好的 mystery 往往从答案反推线索，线索要分批释放，并把“看到信息”“意识到它是线索”“知道它指向谁”拆开。迁移到网文：AI agent 不能把发现、解释和定罪写在同一场景里，否则读者没有参与空间。

Reedsy 和 Writer's Digest 关于 suspense、mystery、red herrings 的资料强调：线索必须能回看成立，红鲱鱼应有故事功能，不能只有误导功能。迁移到网文：假线索也要揭示秘密、改变嫌疑、制造代价、暴露关系或推进下一场。

The Alexandrian 的 Three Clue Rule 与节点式调查设计来自互动叙事，不是网文硬规则，但可迁移为 AI agent 的稳健性原则：关键结论不要只依赖单一线索；如果读者或角色错过一条，剧情不应整体瘫痪。

### 3.2 程序与证据资料

National Institute of Justice 的 crime scene investigation guide 把现场调查拆成到场、初步记录与评估、处理现场、收尾记录和设备等步骤。迁移到小说：证据不是“被看见就能用”，需要现场边界、记录、保护、收集、保存、移交和报告；这些步骤可以转化为剧情阻力。

NIST 的 evidence management 和 digital evidence 资料强调证据在处理、保存、分析过程中要防止污染、破坏或链条断裂。迁移到小说：证据强度不仅取决于“有没有”，还取决于来源、完整性、取得方式、保管过程、能否复核和是否会被对手攻击。

### 3.3 中文网文语境

番茄小说作家专区的悬疑推理资料强调悬疑来自未知引发的怀疑心理，推理探案需要严谨逻辑，也可以和灵异、风水、盗墓、冒险、刑侦、寻宝等类型融合。迁移到网文：调查结构不只适用于传统刑侦，也适用于规则怪谈、无限流副本、宫斗宅斗、宗门执法、职场审计和现实题材。

### 3.4 AI agent 语境

DetectiveQA 等长上下文推理评测显示，侦探小说式问题需要模型从长文本中抽取关键证据并据此推理。迁移到本知识库：多 agent 写作不能依赖模型记忆，必须把线索、假设、证据状态、排除项和待验证问题写进台账或交接包。

## 4. 核心概念

### 4.1 线索

线索是能改变问题理解的信息。它不一定能直接证明真相，但必须让某个问题更清楚、更复杂或更紧迫。

线索至少要回答：

- 它来自哪里？
- 谁能看到或获得？
- 表面含义是什么？
- 真实含义是什么？
- 还有什么合理误读？
- 它改变哪个嫌疑、假设、路线或选择？
- 它什么时候被发现，什么时候被理解，什么时候被回收？

### 4.2 证据

证据是能支持、削弱或验证某个判断的信息或物件。小说里的证据不必完全等同现实法庭证据，但必须有可信的来源、保存状态和使用边界。

证据至少要回答：

- 取得方式是否合法、合规或合世界观规则？
- 是否可能被污染、伪造、剪辑、误读、调包或失效？
- 是否需要专业解释、权限、时间或二次验证？
- 对手能从哪里攻击它？
- 它能证明什么，不能证明什么？

### 4.3 推理

推理是把线索和证据转成可行动结论的过程，不是主角聪明的旁白声明。

推理至少要包含一种动作：

- 对照：把两个信息放在一起，看矛盾或一致。
- 排除：说明某个解释为什么不成立。
- 复核：用另一来源验证同一结论。
- 重构：按时间、空间、动机、机会或行动路线重排事件。
- 反证：主动寻找能推翻自己假设的材料。
- 试探：用小行动逼对手、证人或环境反馈。
- 代价判断：判断继续查下去会损失什么、触怒谁、暴露什么。

### 4.4 调查链

调查链 = 初始问题 -> 第一线索 -> 候选解释 -> 查证行动 -> 新线索/冲突 -> 假设修正 -> 关键证据 -> 风险选择 -> 结论落地 -> 后果。

如果某一环被省略，读者容易觉得主角开挂、作者作弊或剧情水。

## 5. 调查剧情的四条链

### 5.1 事实链：到底发生了什么

用途：还原事件真相。  
常用字段：时间、地点、参与者、动作、物件、结果、异常点、未知点。

风险：事实链如果只在作者脑中存在，正文会出现“读者没看见，主角却知道”的不公平感。

### 5.2 取证链：角色凭什么知道

用途：说明信息来源和可用性。  
常用字段：谁发现、在哪里发现、如何取得、是否被记录、是否被保存、是否会被污染、谁能否认。

风险：没有取证链，证据就像作者手里变出来的道具。

### 5.3 推理链：为什么能推出下一步

用途：展示角色如何从信息过渡到判断。  
常用字段：观察、候选解释、排除项、对照材料、仍不确定处、下一步验证。

风险：推理链跳步会让读者觉得主角不是聪明，而是看过剧本。

### 5.4 行动链：知道以后做什么

用途：让调查结果改变剧情。  
常用字段：新目标、风险、试探对象、公开/隐藏、回报、代价、对手反制。

风险：线索如果不改变行动，只是气氛装饰，会变成水文。

## 6. 线索类型库

| 类型 | 用法 | 风险 | 检查问题 |
| --- | --- | --- | --- |
| 物理线索 | 物件、痕迹、伤口、衣物、药物、灰尘、脚印、残片 | 容易写成万能物证 | 它能证明什么，不能证明什么？ |
| 数字线索 | 监控、聊天记录、转账、定位、门禁、日志、照片、弹幕 | 容易“刚好完整” | 是否有权限、延迟、篡改、缺失或解释风险？ |
| 证词线索 | 证人、嫌疑人、路人、同事、家属、弟子、宫人陈述 | 容易全员配合主角 | 说话者怕什么、藏什么、误记什么？ |
| 行为线索 | 避开某人、过度解释、习惯改变、异常冷静、抢答 | 容易读心 | 行为是否有其他合理解释？ |
| 缺席线索 | 没有脚印、没有记录、某人没出现、某物不在原位 | 容易太隐 | 读者是否知道“正常情况下应该存在”？ |
| 时间线索 | 不在场证明、交通时间、消息时间、伤口时间、修炼/任务进度 | 容易算错 | 时间差是否能被读者跟上？ |
| 空间线索 | 视线、听见范围、门窗路径、密室、摄像头死角、阵法边界 | 容易悬浮 | 是否联动第 88 篇空间站位？ |
| 动机线索 | 受益人、旧怨、利益冲突、情感债、身份秘密 | 容易把动机当证据 | 动机只能提示方向，不能单独定论。 |
| 规则线索 | 副本规则、系统限制、宗门戒律、宫规、合同条款、法律程序 | 容易说明书 | 规则是否在行动中被验证？ |
| 情绪线索 | 愧疚、恐惧、怨恨、羞耻、保护欲、创伤反应 | 容易替读者下结论 | 是否有动作或选择承载，而非旁白判词？ |
| 对手反制线索 | 伪造证据、嫁祸、删记录、操纵舆论、灭口、诱导主角 | 容易强行降智 | 对手为什么这样做，付出什么风险？ |

## 7. 线索设计原则

### 7.1 从答案反推，但从疑问正写

设计时先知道真相，再反推现场会留下什么痕迹、谁会知道什么、谁会撒谎、哪些证据会被破坏。正文中则从主角和读者的疑问开始，一步步接近答案。

写作动作：

1. 写出真相简表。
2. 写出真相必然留下的 3-7 个痕迹。
3. 标注每个痕迹会被谁看见、误读或遮掩。
4. 把最直接的线索拆成发现、误读、复核、回收四步。

### 7.2 关键结论不要只靠一条线索

关键结论至少要有两类支持：例如物证 + 证词，时间线 + 行为异常，规则验证 + 对手反应，数字记录 + 现场痕迹。

写作动作：

- 如果一条线索被删掉，结论是否完全崩塌？
- 如果对手否认这条证据，主角还有什么次级证明？
- 如果证据暂时不能公开，主角能否先采取小行动验证？

### 7.3 发现、识别、指向要分开

低级写法：主角看到扣子，立刻知道凶手是谁。  
更稳写法：主角先看到扣子，以为是普通遗落物；后面发现某制服缺扣；再发现嫌疑人当天换过衣服；最后通过线头、时间和证词联动，扣子才指向真相。

这能给读者参与空间，也能制造追读。

### 7.4 红鲱鱼必须有副产物

红鲱鱼不能只是“骗读者”。它至少要完成一个功能：

- 暴露另一个秘密。
- 排除一个错误解释。
- 揭示人物动机或关系。
- 让主角付出调查代价。
- 迫使对手提前行动。
- 改变读者对世界规则的理解。
- 为后文另一条线索埋钩。

无副产物的红鲱鱼是噪音。

### 7.5 线索要可误读，但不能只靠作者遮掩

好的误导来自角色立场、信息不完整、社会偏见、利益冲突、专业盲区、情绪干扰和对手主动布局。坏误导来自作者故意不写主角看到的关键内容，或用模糊代词藏事实。

自检问题：如果把线索完整写出来，读者是否仍可能误判？如果答案是否定，说明误导过度依赖作者作弊。

## 8. 调查场景流程

### 8.1 异常触发

功能：让角色必须查。  
要素：异常、损失、限制、时间压力、调查资格。

例：尸体出现、账本缺页、系统任务矛盾、弟子失踪、合同数据异常、直播画面不对、旧案证词重现。

### 8.2 现场或资料边界

功能：告诉读者能查什么、不能查什么。  
要素：地点、权限、可接触对象、危险、被污染风险、旁观者或监督者。

如果没有边界，主角就像随便翻全世界数据库。

### 8.3 初始假设

功能：给调查方向，也给后文推翻空间。  
要素：最明显解释、主角个人偏见、对手希望主角相信的解释、读者可能相信的解释。

初始假设不必正确，但必须合理。

### 8.4 第一组线索

功能：让问题从“完全未知”变成“有几个可能”。  
要素：一条真线索、一条可误读材料、一条行动钩子。

注意：第一组线索不应直接给答案，而应打开下一场。

### 8.5 查证行动

功能：让角色主动推进，而不是等线索上门。  
动作：询问、试探、复核、潜入、查账、模拟、重走路线、对照旧记录、诱敌、求助专业者、交易信息。

每次查证都应有成本：时间、暴露、关系、名誉、资源、风险或道德压力。

### 8.6 错路与反证

功能：制造公平误导和角色压力。  
要素：错误解释为什么合理、它如何被推翻、推翻后留下什么有效资产。

错误路线不能只浪费篇幅。它应该带来新证词、新风险、新人物秘密或新对手动作。

### 8.7 关键矛盾

功能：把调查从收集信息推入推理。  
常见形式：时间对不上、证词互斥、物件位置不可能、行为和动机不匹配、规则无法解释某结果、某人知道不该知道的信息。

关键矛盾最好能用一句话表达：如果 A 为真，B 就不该发生。

### 8.8 结论前的代价选择

功能：让真相不是免费获得。  
选择：是否公开、是否冒险取证、是否保护某人、是否牺牲名声、是否违背规则、是否提前打草惊蛇。

调查戏的爽点往往不只是“知道真相”，而是“为了证明真相，主角敢付出什么并承担后果”。

### 8.9 揭示与重构

功能：让读者看到旧线索如何重新拼合。  
原则：重构要短、准、有回看感；不要长篇讲座式复盘。

推荐顺序：关键矛盾 -> 被误读的线索 -> 真正解释 -> 对手漏洞 -> 主角行动证据 -> 当前后果。

### 8.10 余波与台账回写

功能：让调查改变世界。  
回写：已确认事实、新嫌疑、已排除对象、证据状态、被污染证据、公开信息、仍未回答的问题、对手反制、关系变化。

没有余波，查案就变成单元谜题，不会变成长篇资产。

## 9. 非刑侦题材的迁移方式

### 9.1 规则怪谈 / 无限流副本

调查对象：规则、禁忌、怪物行为、死亡条件、通关条件。  
证据形式：死亡案例、幸存者记录、环境变化、提示语、怪物反应、任务面板矛盾。  
重点：规则必须可验证，主角不能只靠直觉知道哪条规则是假。

### 9.2 宫斗 / 宅斗 / 古言查案

调查对象：药物、账目、宫人证词、礼制漏洞、物件流转、时间和出入记录。  
证据形式：香料、帕子、账册、赏赐、口供、规矩、站队反应。  
重点：公开证明前要考虑身份等级、证人安全、主角名声和对方反咬。

### 9.3 宗门 / 玄幻 / 仙侠执法

调查对象：秘境事故、功法反噬、资源失踪、弟子死亡、阵法篡改、魔修嫁祸。  
证据形式：灵力残留、阵纹、魂灯、任务令、宗门记录、见证者、规则誓约。  
重点：超自然证据也要有规则边界，不能想验什么就验什么。

### 9.4 都市 / 职场 / 商战

调查对象：项目事故、财务异常、合同漏洞、黑稿来源、职场甩锅、商业泄密。  
证据形式：邮件、会议记录、转账、系统日志、聊天记录、门禁、客户反馈、流程节点。  
重点：数字记录需要权限、时间戳、完整性和解释权，不等于自动获胜。

### 9.5 娱乐圈 / 直播 / 舆论黑料

调查对象：爆料真伪、剪辑恶意、时间线、利益链、账号关系、偷拍视频来源。  
证据形式：原片、后台数据、合同、行程、账号登录、现场证人、平台规则。  
重点：取证和公开是两件事。本专题管查证，第 61 篇管公开反击。

### 9.6 悬疑言情 / 悬疑群像

调查对象：旧案、身份、记忆、关系秘密、亲属隐瞒、失踪真相。  
证据形式：旧物、称呼、照片、病历、通信、习惯、反应、共同记忆缺口。  
重点：情绪线索不能替代事实证据，但可以决定角色愿不愿意继续查。

## 10. 证据可信度分级

| 等级 | 状态 | 可用性 | 写作处理 |
| --- | --- | --- | --- |
| 已确认 | 多来源互证，来源清楚，未被合理推翻 | 可以支撑关键行动 | 写入台账，后续不能随意改写 |
| 强候选 | 来源可信，但仍缺一环 | 可以支撑试探或下一步调查 | 不能直接定罪或公开结案 |
| 弱候选 | 单一来源、含主观判断或有利益偏向 | 只能作为方向 | 需要二次验证 |
| 待验证 | 有迹象但未取得材料 | 只能保留问题 | 不得当事实写进正文结论 |
| 冲突 | 与其他材料矛盾 | 推动调查重点 | 需要解释冲突来源 |
| 误导 | 看似相关但指向错误解释 | 可用于红鲱鱼 | 必须产生副产物 |
| 废弃 | 已被推翻或污染失效 | 不再支撑结论 | 仍可作为代价或对手攻击点 |

AI agent 禁止把“弱候选”“待验证”“读者猜测”“角色感觉”写成已确认事实。

## 11. 调查章节的节奏控制

### 11.1 每章至少改变一个调查状态

可改变状态包括：

- 新增一个可查问题。
- 排除一个错误解释。
- 发现一条线索但暂时误读。
- 旧线索被重新解释。
- 一个证人或嫌疑人的可信度变化。
- 主角取得下一场调查权限。
- 对手察觉调查并开始反制。
- 关键证据被污染、丢失、夺回或验证。
- 真相代价变高。

如果调查章节结束时问题、嫌疑、证据、行动资格和风险都没变，就是水。

### 11.2 调查不能连续三章只有收集

连续收集线索会疲劳。每 2-3 个调查场景要出现一次：

- 小结论。
- 错路推翻。
- 对手反击。
- 关系冲突。
- 行动试探。
- 公开压力。
- 阶段回报。

### 11.3 章末钩子不要只写“又发现一个线索”

更强的调查钩子：

- 线索证明主角刚才判断错了。
- 证据指向不该有嫌疑的人。
- 关键证人突然改口或失踪。
- 对手提前知道主角查到了哪里。
- 证据能救人，但公开会毁掉另一人。
- 规则被验证为假，真正危险刚出现。
- 读者意识到旧细节其实另有含义。

## 12. 多 agent 线索台账

调查、悬疑和推理任务必须维护线索台账。最小字段：

| 字段 | 说明 |
| --- | --- |
| 线索 ID | 便于跨章节引用 |
| 首次出现章节 | 读者第一次看到或可回看的位置 |
| 表面形态 | 正文中读者看到的样子 |
| 真实含义 | 作者/agent 知道的含义 |
| 当前状态 | 已确认 / 强候选 / 弱候选 / 待验证 / 冲突 / 误导 / 废弃 |
| 获得者 | 主角、配角、对手、公众、读者分别知道什么 |
| 误读方向 | 它会让谁误判成什么 |
| 验证方式 | 下一步怎么确认或推翻 |
| 代价/风险 | 查它会损失什么或触怒谁 |
| 回收计划 | 第几章或哪个节点解释、反转、公开或废弃 |
| 关联专题 | 是否联动第 26、61、82、90、92、100、111 篇 |

多 agent 交接时，不能只写“主角发现线索”。必须写清“线索证明什么、不证明什么、谁知道、谁误读、下一步怎么查”。

## 14. 审稿清单

### 14.1 线索公平感

- 关键答案是否至少有早期痕迹？
- 读者是否能回看发现“原来那是线索”？
- 关键线索是否被藏在作者省略里，而不是场景自然信息里？
- 线索是否有表面含义和真实含义？
- 红鲱鱼是否产生副产物，而不是只骗读者？

### 14.2 证据可信度

- 证据来源是否清楚？
- 取得过程是否有权限、时间、风险或代价？
- 证据是否可能被污染、伪造、误读或攻击？
- 证据能证明什么、不能证明什么是否明确？
- 数字证据、法证、专业报告是否避免万能化？

### 14.3 推理链

- 主角是否展示了至少一个推理动作？
- 是否有候选解释和排除过程？
- 是否把“发现信息”“意识到线索”“指向嫌疑”分开？
- 是否避免从单点信息直接跳到最终真相？
- 是否保留了不确定性和下一步验证？

### 14.4 连载追读

- 本章调查结束后，问题状态是否发生变化？
- 是否有阶段小回报，而不是只有延迟答案？
- 是否有对手反应或外部压力？
- 是否让读者更想知道下一条线索、下一次验证或下一场公开对峙？
- 调查结果是否改变人物关系、资源、名声、风险或行动资格？

### 14.5 AI 味与悬浮感

- 是否出现“主角忽然想到”“直觉告诉他”“一切都解释得通了”却没有过程？
- 是否出现全员配合、证人主动全说、记录随便可查、机构无阻力？
- 是否用长篇旁白复盘代替场景化查证？
- 是否把专业术语当真实感，而没有写限制和后果？
- 是否忘记回写线索台账，导致后续 agent 无法接续？

## 15. 常见失败模式与修法

### 15.1 最终自白解决一切

症状：反派到最后主动解释全部阴谋。  
风险：读者觉得前面调查无意义。  
修法：把自白降级为补充确认，核心结论必须在自白前已由线索链推出。

### 15.2 万能监控 / 万能录音 / 万能系统日志

症状：所有关键事实都靠一个完整记录解决。  
风险：调查没有难度，主角没有智慧。  
修法：记录只能证明一部分；加入缺失片段、权限阻力、解释歧义、保存风险或对手篡改。

### 15.3 红鲱鱼无后果

症状：误导读者后被扔掉。  
风险：读者觉得被耍。  
修法：让红鲱鱼揭示另一秘密、排除一人、暴露关系、制造代价或触发对手反击。

### 15.4 线索太密但无层级

症状：一章塞十几个细节，读者不知道哪个重要。  
风险：认知负荷过高，追读变疲劳。  
修法：每场只设 1 条主线索、1 条辅助线索、1 条误导或气氛材料；其余压缩。

### 15.5 主角过度神探化

症状：主角什么都能看穿，证人一说谎就知道。  
风险：角色失去挣扎，悬疑失去张力。  
修法：让主角也有误判、盲区、证据不足、权限不足和代价选择。

### 15.6 调查只服务谜题，不服务人物

症状：案件很巧，但人物不变。  
风险：商业网文黏性不足。  
修法：每个线索至少触碰一类人物资产：恐惧、欲望、关系、尊严、旧伤、身份、利益或信任。

### 15.7 程序真实但不好看

症状：大量写流程、报告、术语和手续。  
风险：像资料摘抄，缺少故事压力。  
修法：程序细节只保留能制造限制、冲突、延迟、误读、风险或选择的部分。

## 16. 最小可用规则

如果时间很紧，AI agent 至少执行以下 8 条：

1. 先写真相，再反推线索。
2. 每个关键结论至少准备两类支持。
3. 把发现、识别、指向拆开写。
4. 每条线索标注来源、状态、误读和回收。
5. 每场调查至少改变一个状态变量。
6. 证据必须有取得方式和可攻击点。
7. 红鲱鱼必须产生副产物。
8. 写后回写线索台账，禁止把待验证当已确认。
