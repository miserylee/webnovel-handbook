## 终局承接入口

当某一卷已经承担“全书最后一卷”或“主线收束卷”的功能时，必须从本文件切换联动 `docs/storycraft/24-ending-completion-risk-control.md`。中段卷末可以留下下一卷牵引，但终局卷末必须解决主冲突、兑现主承诺、回收关键台账并给最终情绪。

## 中段悬疑线入口

中段用谜团、阴谋、规则真相或反转维持追读时，必须联动 `docs/storycraft/26-suspense-reversal-information-control.md`。中段可以新增谜团，但不能连续加坑不推进；每个卷级悬念都要有阶段答案、线索回收和卷末局部真相。

## 1. 核心原则：中段不是延长篇幅，而是持续改变局面

长篇网文的中段最容易崩，因为开篇承诺已经证明，终局又还不能到来，AI 会本能地用更多对话、更多解释、更多路人挑衅和更多地图名填充篇幅。读者真正需要的不是“还有很多事”，而是“每一段都让局面发生变化”。

判断一章是不是水，不看它是否有打斗、吵架或信息量，而看它是否改变了至少一种状态。

```text
人物状态 / 关系状态 / 资源状态 / 目标状态 / 对手状态 / 世界状态 / 信息状态
```

如果一章删掉后，前后章节几乎能无缝相接，这章大概率是水。

### 用法

- 每章至少产生一个状态变化。
- 每 3-8 章形成一个小单元回报。
- 每 20-40 章形成一个阶段问题的明显转折或结算。
- 每一卷要有自己的目标、阻力、高潮和余波，不能只是“主角换地方继续打脸”。
- 支线必须服务主线、人物、机制、世界或情绪回报，不能只负责拖字数。

### 风险

- 中段只靠升级数字，读者会觉得机械。
- 中段只靠新地图，读者会觉得换皮。
- 中段只靠新角色，读者会觉得人物膨胀。
- 中段只靠大伏笔不兑现，读者会觉得作者画饼。
- 中段只靠日常和过渡，读者会觉得作者在水。

## 2. 水文不是慢，而是没有增量

慢节奏不等于水。治愈、种田、群像、感情线和悬疑都可能有慢章节，但只要状态在变化、关系在推进、信息在更新、情绪在积累，就不是水。

### 有效慢章节

| 类型 | 为什么不水 | 必须有的增量 |
| --- | --- | --- |
| 休整章 | 让阶段高潮后的代价和余波落地 | 关系变化、资源清点、伤势/损失、下一目标 |
| 日常章 | 强化人物关系和世界质感 | 新梗、新关系动作、新生活压力 |
| 训练章 | 让能力成长有过程 | 新限制、新失败、新技巧、新代价 |
| 赶路章 | 连接地图并制造路上问题 | 遭遇、信息、选择、环境阻力 |
| 会议章 | 分配目标和暴露利益差异 | 决策、分歧、权责、风险承担 |
| 情绪章 | 处理人物在乎的事 | 误会变化、信任变化、决定变化 |

### 真水章节

- 只是复述读者已经知道的信息。
- 人物反复表达同一种态度，关系没有变化。
- 对手用同一种方式挑衅，主角用同一种方式打脸。
- 训练、赶路、吃饭、开会没有新选择和新后果。
- 新角色登场后不改变目标、阻力或关系。
- 章节结尾的钩子和本章内容无关，只是硬吊胃口。
- 一整章都是解释设定，但下一章不使用这些设定。

## 3. 卷结构：每卷都要是一个完整故事

长篇不能只靠一个终局大目标吊着。一个终局目标可以提供方向，但不足以支撑几十万字。中段需要“卷级故事”：每一卷都有自己的起点、阶段目标、阻力升级、高潮结算和余波。

### 卷设计八件套

```text
【卷名/阶段名】
【本卷承诺】这一卷读者主要期待什么回报
【本卷目标】主角要拿到、救下、证明、打破或改变什么
【本卷阻力】阶段对手、组织、制度、环境或内在阻力
【本卷机制】金手指/规则/资源在本卷如何升级或受限
【本卷关系】哪条关系会推进、破裂、确认或反转
【本卷中点】本卷中段最大转折或失败
【本卷高潮】公开结算、资源落袋、关系变化和新问题
【下一卷牵引】还有什么余波或更大问题没有解决
```

### 好卷的体感

- 读者知道这一卷在追什么。
- 每隔几章都有小回报，不用等到卷末才爽。
- 卷中有一次方向变化，不是直线跑到底。
- 卷末解决阶段问题，同时抛出更高层问题。
- 主角的资源、关系、地位或认知至少改变一个维度。

## 4. 故事套娃：小故事承担大故事职能

适合长篇网文的中段结构，不是“开头铺垫十万字、过程铺垫二十万字、最后一次爆发”，而是把大故事拆成多个可读的小故事。小故事本身要有起因、阻力、转折和回报，同时承担大故事里的铺垫、证明、升级或转场职能。

### 套娃结构

```text
全书主线
  -> 卷级故事
    -> 单元故事
      -> 章节目标
        -> 场景冲突
```

每一层都要有自己的问题和回报，但下层不能脱离上层。

### AI 执行规则

- 单元故事不能只是随机事件，必须服务本卷目标。
- 章节目标不能只是“写一场打斗/对话/赶路”，必须服务单元变化。
- 场景冲突不能只是热闹，必须造成角色选择、信息更新或资源变化。
- 大伏笔不能只藏在旁白里，要通过小故事中的可见行动留下痕迹。

## 5. 中段增量清单：每章至少命中一项

| 增量类型 | 说明 | 例子 |
| --- | --- | --- |
| 目标增量 | 主角目标更清楚、更困难或被迫改变 | 从“进宗门”变成“进内门前拿到资格” |
| 资源增量 | 得到、失去、消耗或转换资源 | 钱、丹药、线索、人情、权限、名声 |
| 关系增量 | 信任、误会、欠账、敌意或暧昧改变 | 搭档不再旁观，第一次替主角担责 |
| 对手增量 | 对手换策略、升级资源或暴露弱点 | 从嘲讽变成卡规则、买通证人 |
| 机制增量 | 金手指边界、代价、升级或反制出现 | 系统奖励变少，必须主动创造条件 |
| 信息增量 | 读者或主角知道新的有效事实 | 真相碎片、规则漏洞、敌方动机 |
| 情绪增量 | 读者获得爽、甜、燃、笑、虐或治愈的有效回报 | 旧账被公开清算，关系糖落地 |
| 世界增量 | 地图、行业、组织或制度更具体 | 新规则让旧套路失效 |

AI 写中段时，每章开写前先选 1-3 个增量；写完后检查是否真的改变后续局面。

## 6. 支线不是岔路，是压力分流器

支线的作用不是拖长篇幅，而是给主线提供额外压力、信息、关系和情绪变化。支线越多，AI 越容易散，所以必须给每条支线登记功能和退出条件。

### 支线功能表

| 支线功能 | 可保留条件 | 必须避免 |
| --- | --- | --- |
| 主线压力 | 支线会改变主线资源、敌我或目标 | 支线结束后主线毫无变化 |
| 人物成长 | 迫使主角或配角做出新选择 | 只是展示人设标签 |
| 关系推进 | 改变信任、欠账、误会或亲密度 | 关系原地拉扯 |
| 机制训练 | 展示能力限制、代价或新用法 | 金手指自动赢，没有新理解 |
| 世界展开 | 让平台、宗门、行业、家族或副本规则具体 | 只堆名词和地图 |
| 情绪调节 | 在高压后提供糖、笑、治愈或余韵 | 变成无后果日常 |

### 支线硬约束

- 同时活跃支线不宜过多；AI 默认最多管理 2-3 条活跃支线。
- 每条支线要有进入点、阶段回报和退出点。
- 支线不能抢走主线承诺，除非用户明确要群像或多线叙事。
- 支线角色必须影响主角选择，不能只负责讲设定。
- 支线回收时要有资源、信息、关系或情绪落袋。

## 7. 中点转折：让卷从“推进”变成“变局”

卷中段如果只是顺着目标往前走，容易疲软。一个有效中点转折，会改变主角对目标、对手、规则或代价的理解。

### 中点转折类型

- 目标变了：原来要拿资格，现在发现资格本身是陷阱。
- 对手变了：表面敌人只是代理人，真正阻力是组织规则。
- 代价变了：赢不是免费，继续推进会伤害关系或消耗底牌。
- 规则变了：旧办法失效，主角必须换策略。
- 关系变了：同盟暴露私心，敌人出现可合作部分。
- 信息变了：读者和主角重新理解前面事件。

### 检查问题

- 中点之后，主角还能不能按原计划直线胜利？如果可以，中点太弱。
- 中点是否让前半卷的小故事有新意义？
- 中点是否给后半卷提供更高压力，而不是只多一个敌人？

## 8. 卷末结算：既要爽，也要留下余波

卷末不能只把当前敌人打倒。它要证明本卷承诺已经兑现，同时让读者愿意进入下一卷。

### 卷末四步

1. 公开结算：关键人物或群体看见结果。
2. 回报落袋：资源、地位、关系、真相、能力或情绪回报到手。
3. 代价显影：暴露新的敌意、损失、伤痕、关系裂缝或制度压力。
4. 下一卷牵引：新地图、新规则、新敌人、新关系问题或更大真相出现。

### 常见失败

- 敌人倒了，但主角没有实际收益。
- 主角赢了，但世界没有反应。
- 卷末只爆更大设定，不处理本卷承诺。
- 为了下一卷强行留坑，导致本卷读者不满足。
- 卷末把所有问题都解决，下一卷没有牵引。

## 9. 防重复升级：换问题，不只换数值

中段常见假升级是：敌人更强、地图更大、奖励更多，但问题本质不变。真升级应该改变读者关注的问题。

| 假升级 | 真升级 |
| --- | --- |
| 敌人从炼气变筑基，但还是嘲讽送脸 | 敌人开始卡规则、动用组织、攻击关系 |
| 地图从小城换皇城，但流程仍是被看不起再打脸 | 皇城规则让旧爽点失效，主角必须建立新身份 |
| 系统奖励数字变大 | 系统出现代价、限制、反制或选择分歧 |
| 新女主/新搭档登场 | 新关系改变主角选择和旧关系结构 |
| 新副本怪更强 | 新副本规则改变解题方法 |

AI 设计升级时，必须同时升级至少一个维度：规则、对手策略、主角代价、关系压力、信息差、世界反应。

## 10. 不同题材的中段防水重点

| 题材 | 中段最容易水在哪里 | 防水抓手 |
| --- | --- | --- |
| 玄幻/仙侠 | 重复修炼、比斗、秘境捡宝 | 卷级目标、资源争夺、宗门/家族规则变化 |
| 都市脑洞/神豪 | 重复消费、打脸、炫富 | 身份升级、行业项目、舆论和关系成本 |
| 现言/古言 | 误会反复、关系原地拉扯 | 阶段关系确认、家族/职场/婚恋压力变形 |
| 悬疑/规则怪谈 | 线索拖延、谜语人过多 | 每案阶段答案、规则试错代价、真相层级推进 |
| 无限/副本 | 副本换皮、怪物升级 | 每个副本不同规则、队友博弈、长期机制变化 |
| 种田/经营 | 日常流水账、生产过程重复 | 市场、灾害、政策、邻里关系和生产链升级 |
| 群像喜剧 | 梗重复、角色标签化 | 新组合、新场景、新误会结构和情绪余味 |

## 11. AI 中段写作硬约束

- 不得连续两个单元使用同一种冲突结构。
- 不得用新地图代替新问题。
- 不得用新角色代替关系变化。
- 不得用设定解释代替行动后果。
- 不得让主角连续多章只旁观局势。
- 不得让支线超过 10 章仍没有阶段回报。
- 不得让卷末只有铺垫而没有本卷结算。
- 每 5 章必须更新或检查一次连续性与伏笔台账，联动 `docs/core-writing/07-continuity-ledger.md`。
- 如果中段数据下滑，诊断必须联动 `docs/feedback-revision/18-reader-data-feedback-diagnosis.md`，不能直接判定“读者不懂”。

## 12. 中段防水与卷结构设计卡

```text
【作品名/当前卷】
【当前章节范围】
【本卷承诺】
【本卷目标】主角要拿到、证明、救下、改变或打破什么
【本卷阶段对手/阻力】
【本卷中点转折】目标 / 对手 / 代价 / 规则 / 关系 / 信息 如何改变
【本卷高潮】公开结算、落袋回报、代价显影
【下一卷牵引】
【活跃支线】每条支线的功能、进入点、阶段回报、退出点
【近 5 章增量】目标 / 资源 / 关系 / 对手 / 机制 / 信息 / 情绪 / 世界
【重复风险】是否重复打脸、重复赶路、重复修炼、重复误会、重复开会
【防水结论】保留 / 压缩 / 合并 / 改成场景 / 移入台账 / 删除
```

## 13. 中段审稿流程

1. 先问本卷承诺：这一卷读者到底在等什么。
2. 再看近 5 章：每章是否有状态增量。
3. 再看单元：是否 3-8 章有一次小回报。
4. 再看中点：是否改变目标、规则、代价或关系。
5. 再看支线：每条支线是否有功能和退出点。
6. 再看对手：是否升级策略，而不是只升级战力。
7. 再看卷末：是否有公开结算、回报落袋、代价显影和下一卷牵引。
8. 最后看读者体感：读者是期待下一个变化，还是觉得作者在拖。

## 14. 快速检查问题

- 这 5 章删掉任意一章，后续是否会失去重要变化？
- 本卷目标能否用一句话说清？
- 本卷中点之后，主角是否被迫换策略？
- 支线是否改变了主线资源、关系、信息或情绪？
- 卷末是否兑现了本卷承诺，而不是只开新坑？
- 新地图、新敌人、新能力是否真的改变问题结构？
- 读者在中段得到的是持续回报，还是只被要求等待大高潮？

## 15. 来源与迁移说明

本专题吸收并重组了以下资料：番茄小说作家专区关于卡文、逻辑断层、情节失去张力、人物成长不足和大纲回顾的公开课程；起点社区关于一级/二级/三级结构、故事套娃、卷级单元、持续爽点和水文背叛读者期待的作者经验；Writing Excuses 关于 saggy middle、try-fail cycles、M.I.C.E. 线程交织、章节断点与 pacing 的写作建议。

落地时不要把任何章节数、字数或卷长当成绝对规则。中段防水的核心不是固定长度，而是每个章节、单元和卷都持续改变状态、兑现阶段承诺，并把变化接回长线主线。

## 长篇训练、修炼与成长段落防水入口

当章节或小单元涉及训练、修炼、闭关、学习、备赛、技能升级、职业成长、系统任务或团队磨合时，需要调用 `docs/workflows/58-training-growth-anti-padding.md`。该专题负责判断训练段是否应该展开、压缩或跳过，并要求训练段具备目标、瓶颈、方法、代价、反馈、验证和下一阶段限制，防止训练流水账、闭关后直接无敌、数值升级无证明和对手原地等待。

## 反派、竞争对手与外部世界同步升级入口

当主角胜利、训练突破、暴露新能力、夺取资源、改变规则、经过时间跳跃、进入新卷或开启新地图后，需要调用 `docs/core-writing/59-antagonist-world-synchronous-escalation.md`。该专题负责判断外部世界是否同步变化：谁看见了主角变化、谁产生误判、对手换了什么策略、组织调了什么资源、规则或舆论怎样反馈、下一轮压力如何针对主角旧优势，防止世界原地等待、对手重复送、新敌人空降和升级失真。

## 高潮战、大场面与群体冲突调度入口

当章节或卷级单元涉及高潮战、决战、群战、团队战、公开审判、朝堂对峙、比赛决胜、灾难救援、发布会、直播爆点、群体谈判，或任何三条以上行动线同时推进的大场面时，需要调用 `docs/advanced-craft/60-climax-battle-large-scene-coordination.md`。该专题负责确认目标、胜负条件、空间锚点、视角锚点、群体角色分工、关键转折、动作压缩、战后结算和台账回写，防止只堆人数、招式、爆炸、喊话和震惊。

## 日常章、过渡章与低强度章节防水入口

当章节涉及日常、过渡、休整、赶路、聊天、吃饭、生活流、低冲突推进、战后余波、关系缓冲、情绪沉淀、信息整理、准备章、轻松章或主线暂时低烈度推进时，需要调用 `docs/storycraft/62-everyday-transition-low-intensity-anti-padding.md`。该专题负责判断低强度章节是否值得独立成章，是否承接旧结果、产生微变化、交付小回报、保留人味、生成下一步并回写台账，防止日常流水账、过渡无增量、休整清零、赶路只写到了和聊天无目的。
