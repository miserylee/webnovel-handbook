## 1. 核心原则：设定必须被剧情使用

世界观不是给作者展示知识量的地方，而是给故事制造限制、机会、冲突、代价和承诺的工具。

AI 写作时按这条链路处理设定：

```text
读者当前需要 -> 场景压力 -> 必要设定 -> 角色选择 -> 后果验证 -> 台账固化
```

如果一个设定不能进入这条链路，先放入后台资料，不要写进正文。

## 2. 六类概念不要混用

| 名称 | 作用 | 可以写进正文的条件 | 常见 AI 误用 |
|---|---|---|---|
| 世界观 | 作品世界如何运行 | 当前场景需要读者理解世界差异 | 开篇三段讲历史、地理、势力 |
| 设定 | 具体规则、等级、物品、职业、组织 | 会影响角色选择或冲突结果 | 只给酷名词，不给限制和用途 |
| 规则 | 设定的因果边界 | 一旦出现就会在后文被验证 | 为了救场随时改规则 |
| 背景 | 人物或世界的过去 | 当前情绪、动机或悬念需要 | 角色一出场就自传式回忆 |
| 氛围 | 读者对世界的感官印象 | 能强化题材承诺和情绪基调 | 用堆砌形容词替代行动 |
| 资料库 | 作者/AI 自己查阅的后台材料 | 通常不直接进正文 | 把资料库原样倒进章节 |

## 3. 三层设定：当场必懂、近期会用、后台暂存

### 3.1 当场必懂

读者不懂就看不明白眼前压力、选择或爽点的信息。

写法：短、准、贴动作。通常一两句即可。

例：主角进门前先摸袖口里的身份牌，因为没有身份牌的人会被阵法当场绞碎。读者立刻知道身份牌重要、阵法危险、规则会影响行动。

### 3.2 近期会用

读者现在不必全懂，但需要先挂一个钩子，等后文回收。

写法：让角色看见异常、听见禁忌、遭遇限制，但暂不解释完整原理。

例：城里所有人都避开西钟楼。主角只注意到卖糖人的老人听见钟声后手抖了一下。解释留到后文事件。

### 3.3 后台暂存

作者/AI 需要知道，但读者暂时不需要知道的信息。

写法：写入设定卡、台账或大纲，不进入正文。

例：王朝三百年税制演变、七大宗门完整谱系、远古神战详细年表，除非当前剧情正在使用，否则不要在章节中展开。

## 4. 设定投放五问

每次想写设定前，AI 先回答：

1. 读者现在不知道这条信息，会不会看不懂当前情节？
2. 角色为什么会在这个时刻接触、想起或说出它？
3. 这条信息会造成什么限制、代价、机会或冲突？
4. 读者知道后，会改变什么期待或判断？
5. 它将在第几章、哪个场景被验证、反转或升级？

五问答不出三问以上，这条设定先不写正文。

## 5. 开篇设定投放规则

开篇不是世界观说明会。开篇只负责让读者明白四件事：主角是谁、正在承受什么压力、作品给什么题材承诺、第一件必须解决的问题是什么。

### 5.1 第一章

只投放当场必懂设定。

优先顺序：主角处境 > 眼前压力 > 关键规则 > 爽点/危机验证。

禁止：王朝史、修炼等级大全、地图介绍、组织图谱、金手指说明书、十个陌生名词连续出现。

### 5.2 第 3 到 6 章

重复验证核心规则。

例如：系统不是只弹窗，而是用任务、惩罚、奖励、误导来改变主角选择；修炼等级不是只报名称，而是让低阶角色因规则限制吃亏或翻盘。

### 5.3 第 10 章前

完成一次设定回收。

读者前面见过的规则、禁忌、物品、身份或地理限制，至少有一项要在第 10 章前变成爽点、危机解决、反转或新目标。

### 5.4 第 30 章前

建立可连载的设定引擎。

读者要能看出：主角以后会如何持续遇到问题、利用规则、升级资源、扩大地图或触发新冲突。

## 6. 设定必须提供四种功能

### 6.1 限制

限制让故事有难度。

示例：秘境每三日只开一次；系统奖励不能直接兑换寿命；女主的身份不能公开；某个城市禁止使用灵力。

### 6.2 代价

代价让爽点有重量。

示例：越级使用法宝会损伤经脉；预知剧情会改变别人命运；每次复活都会丢失一段记忆。

### 6.3 机会

机会让主角能主动行动。

示例：别人看不懂旧地图，主角能识别废弃标记；低级材料在新规则下反而有奇效。

### 6.4 冲突

冲突让设定进入剧情。

示例：两个门派对同一条规则解释不同；主角的外挂奖励与家族禁令冲突；官方法律与民间习俗冲突。

## 7. 规则一致性：写进去就要进台账

任何会影响后续剧情的设定，写入正文后必须登记到设定台账。台账至少记录：首次出现章节、原文功能、限制、代价、已触发后果、后续可回收点。

推荐字段：

```text
【设定名称】
【类别】世界规则 / 等级体系 / 金手指 / 组织制度 / 地理限制 / 物品 / 习俗 / 法律 / 历史
【首次出现】第 X 章，场景：
【读者已知】
【角色已知】
【真实规则】
【限制】
【代价】
【例外】
【已造成后果】
【后续回收】
【风险】是否过强、是否破坏悬念、是否与旧章矛盾、是否有合规风险
```

硬规则：不要为了让主角脱困临时新增从未铺垫的例外。可以升级规则，但要让读者觉得“原来如此”，而不是“作者开后门”。

## 8. 世界观资料库的九个模块

AI 初始化新书时，可以先做资料库，但不要把资料库直接变正文。

| 模块 | 最少需要想清楚的问题 | 进入正文的触发条件 |
|---|---|---|
| 地理与空间 | 主角去哪、为什么难去、距离造成什么限制 | 赶路、封锁、资源分布、逃亡 |
| 历史与旧账 | 哪件旧事还在影响现在 | 谜团、仇恨、制度来源、身份反转 |
| 阶层与权力 | 谁能命令谁，谁怕谁 | 压迫、晋升、打脸、谈判 |
| 经济与资源 | 稀缺物是什么，谁控制它 | 赚钱、交易、抢夺、升级材料 |
| 法律与习俗 | 什么事被允许或禁止 | 冲突、误会、破局、身份暴露 |
| 技术/魔法/修炼 | 能做到什么，不能做到什么 | 爽点、战斗、解谜、限制主角 |
| 组织与派系 | 组织目标、利益冲突、内部规矩 | 阵营选择、背叛、合作、追杀 |
| 日常生活 | 普通人如何吃住行、娱乐、传闻 | 氛围、代入感、喜剧、信息传播 |
| 语言与命名 | 读者能否快速识别身份、地域、等级 | 需要制造辨识度，但不能增加理解负担 |

## 9. 冰山原则：作者知道十成，读者先看一成

作者/AI 可以准备大量设定，但正文只露出当前场景需要的那一小部分。

好的世界观像冰山：读者看到的是行动、选择和后果；水下资料让作者写得稳定，但不抢镜。

检查标准：删掉某段设定说明后，如果当前场景仍能看懂，而且情绪更顺，这段说明就应进入后台资料，而不是正文。

## 10. 十种自然投放方式

1. 行动投放：角色因为规则改变动作。
2. 冲突投放：两方因同一设定产生争执、交易或对抗。
3. 后果投放：读者先看到违反规则的代价，再理解规则。
4. 选择投放：主角在多个方案中受设定限制而取舍。
5. 对话投放：角色不是互相背课本，而是在争、骗、求、威胁、试探。
6. 物件投放：身份牌、契约、地图、账本、符箓、药瓶承载信息。
7. 日常投放：习俗、价格、交通、称谓自然显示社会结构。
8. 旁观投放：主角看见别人犯错或受罚。
9. 传闻投放：茶馆、弹幕、榜单、新闻、宗门公告给出不完整信息。
10. 异常投放：本该正常的东西突然不正常，引出读者提问。

## 11. 名词密度规则

读者不是为了背词典而来。陌生名词越多，理解成本越高。

### 11.1 第一章

最多保留 1 到 3 个必须记住的新名词。每个新名词都要绑定动作或后果。

### 11.2 同一段落

不要连续堆叠势力名、地名、等级名、法宝名。新名词之间要插入角色动作、情绪、判断或冲突。

### 11.3 创新方向

优先创新冲突、爽点机制、主角处境和题材组合。不要只把“练气、筑基、金丹”换成一串陌生词就以为有新意。

## 12. 不同题材的设定重点

| 题材 | 设定重点 | 最容易出错 |
|---|---|---|
| 玄幻/仙侠 | 等级、资源、宗门、秘境、因果代价 | 等级表太长、战力膨胀无约束 |
| 系统/金手指 | 触发条件、奖励边界、惩罚、误导、升级路线 | 系统万能，主角不需要选择 |
| 都市脑洞 | 异常如何嵌入现实秩序 | 现实逻辑崩坏、合规风险被忽略 |
| 历史/古言 | 礼法、身份、家族、朝堂、财产关系 | 现代观念硬塞，制度只当背景板 |
| 悬疑/规则怪谈 | 规则真伪、信息差、禁忌、违反后果 | 规则随意变，反转靠骗读者 |
| 无限流/副本 | 副本规则、通关条件、资源继承、队伍关系 | 每个副本像孤岛，主线不推进 |
| 种田/经营 | 资源、技术、交易、组织协作、长期收益 | 只记流水账，没有目标压力 |
| 喜剧/轻松文 | 世界规则如何制造误会、反差和包袱 | 为搞笑破坏人物智商和主线规则 |

## 13. 设定与人物绑定

设定要压到人物身上，读者才会在乎。

AI 起草时不要写“这个世界有三大禁忌”，而要写“主角现在必须违反第一条禁忌，否则妹妹今晚会被带走”。

绑定方式：

1. 身份绑定：设定决定角色能不能进入某个空间。
2. 资源绑定：设定决定角色缺什么、抢什么、换什么。
3. 情感绑定：设定伤害或保护角色重视的人。
4. 目标绑定：设定决定主角短期目标和长期目标。
5. 道德绑定：设定让角色在正确与代价之间选择。

## 14. 设定与悬念绑定

有些设定不需要立刻解释，而是可以先形成问题。

可延迟解释的信息：规则来源、例外条件、历史真相、隐藏身份、组织真正目的。

不可延迟的信息：当前行动的基本因果、主角正在怕什么、读者判断爽点或危险所需的底线规则。

底线：可以让读者好奇，不能让读者迷路。

## 15. AI 写作硬性禁令

1. 禁止第一章用世界观百科开头。
2. 禁止连续三段纯设定说明。
3. 禁止设置后文不用的酷名词。
4. 禁止让角色对已知常识进行“如你所知”式背诵。
5. 禁止为救场临时新增无铺垫规则。
6. 禁止把设定创新等同于生造名词。
7. 禁止让世界观抢走主角目标、压力和选择。
8. 禁止正文写完后不更新设定台账。

## 16. 世界观与设定设计卡

```text
【作品/世界名】
【题材/平台】
【核心世界承诺】这本书的世界给读者什么新鲜体验？
【主角当前处境】主角为什么必须进入或面对这套规则？
【当场必懂设定】第 1 章必须知道的 1-3 条。
【近期会用设定】前 10 章会回收的设定。
【后台暂不投放设定】作者知道但暂不写正文的信息。
【核心规则】这个世界最重要的因果规则。
【限制】规则不能做什么，主角受什么约束。
【代价】使用能力、越过边界或获得资源要付出什么。
【资源/机会】主角可以利用什么差异打开局面。
【冲突来源】人物、组织、制度、资源、情感中的冲突点。
【第一次验证场景】读者第一次看见规则产生后果的场景。
【后续回收/升级】第几章回收，第几章升级。
【命名密度】第一章新名词数量；是否每个名词都绑定动作或后果。
【风险检查】设定堆砌 / 规则漂移 / 名词过载 / 现实文化风险 / 合规风险
```

## 17. 设定投放审稿流程

审稿时按以下顺序删改：

1. 标出所有纯设定段落。
2. 判断每段是否服务当前压力、选择或后果。
3. 没有服务的，移入后台资料或台账。
4. 必须保留的，改成动作、冲突、后果、对话或物件投放。
5. 检查新名词密度，保留最少必要名词。
6. 检查已出现规则是否进入台账。
7. 检查前 10 章是否有至少一次设定回收。

## 18. 快速自查问题

- 这一章最重要的设定，是否让主角做出了不同选择？
- 如果删掉某段设定说明，读者是否反而读得更顺？
- 新名词是否都能在本章或近期章节被使用？
- 规则是否有边界、代价和例外？
- 读者是在看角色解决问题，还是在看作者讲课？
- 后文有没有回收本章抛出的规则、物品、禁忌或异常？

## 19. 来源与迁移说明

- 起点社区《怎么写设定及大纲》强调设定与大纲相互影响，基础设定要先有，细节设定可随正文完善，但写进正文后要立即归档；还强调无谓名词创新会增加说明成本，剧情要靠冲突推动。本文将其迁移为“设定台账”“名词密度”“设定必须制造冲突”的规则。
- 起点社区《网络小说作者入门篇》强调网文读者重视新奇、有趣、精彩情节与开篇抓眼球，也提示开篇少用设定介绍性文字充数。本文将其迁移为“开篇只投放当场必懂设定”。
- Reedsy 的 worldbuilding guide 将世界观拆为历史、地理、政治、文化、社会结构、技术/魔法、经济和权力等模块，并提醒世界不能替代人物。本文将其迁移为“资料库九模块”和“设定与人物绑定”。
- Writer's Digest 中 Orson Scott Card 关于科幻奇幻说明的文章强调信息投放需要平衡，陌生术语可以通过暗示和延迟解释建立读者钩子，但承诺必须真实且重要。本文将其迁移为“三层设定”“近期会用设定”和“命名密度”。
- Writing Excuses 14.9 讨论如何在不造成枯燥信息倾倒的情况下传递大量背景，重点包括让角色与信息互动、给信息情绪重量、用争执和片段材料承载设定。本文将其迁移为“十种自然投放方式”。

## 反派、竞争对手与外部世界同步升级入口

当主角胜利、训练突破、暴露新能力、夺取资源、改变规则、经过时间跳跃、进入新卷或开启新地图后，需要调用 `docs/core-writing/60-antagonist-world-synchronous-escalation.md`。该专题负责判断外部世界是否同步变化：谁看见了主角变化、谁产生误判、对手换了什么策略、组织调了什么资源、规则或舆论怎样反馈、下一轮压力如何针对主角旧优势，防止世界原地等待、对手重复送、新敌人空降和升级失真。
