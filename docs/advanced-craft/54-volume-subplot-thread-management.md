## 1. 核心判断：支线不是越多越厚，而是越有功能越厚

30 章后，AI 写长篇最容易犯的错不是“故事不够多”，而是“线太多但没有管理”。

常见问题：

- 新人物不断登场，但没有退出。
- 支线看似丰富，实际不推进卷目标。
- 伏笔只埋不回收。
- 关系线只刷好感，不改变选择。
- 悬念线只加谜团，不给阶段答案。
- 机制线只升级，不产生代价。
- 每章都热闹，但卷末无法清账。

一句话规则：

> 每条支线都必须服务主线、改变状态、定期交叉、阶段回收；不能说明功能、交叉点和退出点的支线，不要开。

---

## 2. 什么时候需要本专题

出现以下情况时，必须读取本专题：

- 30 章后开始设计卷级故事。
- 一个卷里同时有 3 条以上支线。
- 人物越来越多，读者可能记不住。
- 悬念、伏笔、关系、机制、地图线同时推进。
- 审稿时发现“水”“散”“支线太多”“主线没进”。
- 卷末需要清点已结算、活跃、延期和删除的线。
- 多 agent 分工时，有人负责人物线，有人负责悬念线，有人负责主线。

---

## 3. 支线的六种合法功能

支线不能只因为“有趣”而存在。它至少要承担一种功能，最好承担两种。

| 功能 | 作用 | 合格表现 | 不合格表现 |
| --- | --- | --- | --- |
| 推进主线 | 让卷目标更接近或更困难 | 支线结果改变主线行动 | 支线结束后主线无变化 |
| 增加代价 | 让主角付出资源、关系、时间、秘密或情绪成本 | 支线让主角选择更难 | 只制造麻烦但不留后果 |
| 加深人物 | 让人物欲望、弱点、选择方式更清楚 | 支线改变人物关系或行动 | 只给配角加背景 |
| 展开规则 | 展示世界、机制、行业、组织或地图规则 | 规则影响行动选择 | 只灌设定 |
| 回收伏笔 | 给旧问题阶段答案或误导修正 | 支线让旧伏笔变贵 | 只埋新坑 |
| 转换节奏 | 在高压主线中提供轻重变化，同时仍有状态增量 | 轻松或情绪单元也推进关系/信息 | 纯插曲 |

开支线前，AI agent 必须写出：

```text
这条支线服务哪个卷目标？
它会改变什么状态？
它预计在哪个节点交叉、回报、退出或并入主线？
```

---

## 4. 线程类型：先分类，再管理

本专题把长篇中的线分为七类。

| 线程类型 | 核心问题 | 典型回报 | 高危风险 |
| --- | --- | --- | --- |
| 主线 | 本卷到底要解决什么？ | 卷目标推进、卷末结算 | 被支线稀释 |
| 人物线 | 这个人物为什么重要？ | 选择变化、身份变化、能力变化 | 只补背景 |
| 关系线 | 人与人之间怎样互相改变？ | 信任、裂痕、亏欠、站队、共同承担 | 只撒糖或只误会 |
| 悬念线 | 读者在追哪个问题？ | 阶段答案、反转、真相半开 | 只加谜语 |
| 机制线 | 能力、系统、规则如何变复杂？ | 新用途、新限制、新代价 | 只升级不反噬 |
| 世界线 | 地图、组织、行业、制度如何展开？ | 规则变化、资源分布、权力结构 | 设定讲解 |
| 情绪线 | 读者该为什么在乎？ | 痛点、补偿、爽点、共情、遗憾 | 情绪口号 |

每条线都要有状态，不要只写“进行中”。

推荐状态标签：

- 已确认。
- 候选。
- 待确认。
- 活跃。
- 待回收。
- 已回收。
- 延期。
- 删除。

---

## 5. 支线生命周期

一条合格支线应该有生命周期。

```text
引入 -> 挂钩 -> 升级 -> 交叉 -> 回报/代价 -> 退出/并入/延期
```

### 5.1 引入

引入支线时，读者至少要知道：

- 谁被卷入。
- 为什么现在出现。
- 它和卷目标有什么关系。
- 它会给主角带来机会还是麻烦。

不合格引入：

- 新人物突然出现，只为讲背景。
- 新地图突然出现，只为换场景。
- 新谜团突然出现，和旧问题无关。

### 5.2 挂钩

挂钩是支线和主线的连接点。

挂钩方式：

- 同一个资源。
- 同一个敌人。
- 同一个关系债。
- 同一个规则漏洞。
- 同一个秘密。
- 同一个时间限制。
- 同一个情绪伤口。

没有挂钩的支线，会像另一部小说。

### 5.3 升级

支线不能只保持原状。它至少要升级一次。

升级可以是：

- 目标变难。
- 代价变贵。
- 关系变复杂。
- 信息被重释。
- 对手开始利用支线。
- 主角必须在主线和支线之间做选择。

### 5.4 交叉

交叉是多线程管理的关键。

支线必须定期和主线或其他支线发生交叉：

- 支线提供主线所需资源。
- 支线消耗主线所需时间。
- 关系线改变主线选择。
- 悬念线重释人物线。
- 机制线暴露世界线规则。
- 对手利用支线压迫主角。

如果一条支线超过一个小单元没有交叉，就要考虑压缩、并入或删除。

### 5.5 回报或代价

支线必须给结果。结果可以小，但不能没有。

可用结果：

- 小胜。
- 小败。
- 代价胜。
- 真相半开。
- 关系变化。
- 资源变化。
- 风险升级。
- 伏笔回收。

### 5.6 退出、并入或延期

支线不能永远活跃。

退出：该线完成阶段功能，暂时离场。

并入：该线成为主线、对手线、关系线或卷末结算的一部分。

延期：该线暂不解决，但必须有下一次回收窗口。

删除：该线没有功能、没有交叉、没有回报，应从规划中移除。

## 6. 多线程交叉矩阵

设计卷级故事时，先列出主线和支线，再设计交叉点。

| 交叉组合 | 有效用法 | 坏例 |
| --- | --- | --- |
| 主线 x 人物线 | 人物弱点影响主线选择 | 只插人物回忆 |
| 主线 x 关系线 | 关系债改变主角行动成本 | 只撒糖或吵架 |
| 主线 x 悬念线 | 线索改变目标判断 | 只加谜语 |
| 主线 x 机制线 | 能力限制影响主线打法 | 系统直接给答案 |
| 主线 x 世界线 | 规则或组织结构限制行动 | 设定百科 |
| 对手线 x 支线 | 对手利用支线压迫主角 | 对手只口头威胁 |
| 关系线 x 悬念线 | 人物隐瞒或误解改变关系 | 为误会而误会 |
| 机制线 x 世界线 | 机制规则反映世界规则 | 机制升级像外挂 |
| 情绪线 x 主线 | 主线选择触发痛点、补偿或遗憾 | 旁白喊情绪 |

每 5 章至少检查一次：

- 哪些线发生了交叉？
- 哪些线没有交叉？
- 哪些线交叉后改变了状态？

## 7. 主线优先级：所有支线都要回到卷目标

卷级故事的主线优先级最高。

支线可以抢镜，但不能抢方向。

判断支线是否抢方向：

- 读者更关心支线，但支线不影响卷目标。
- 主角为支线忙了很多章，卷目标无进展。
- 支线人物比主角更主动解决问题。
- 支线结算后没有资源、关系、信息、风险变化。

处理方法：

- 并入：让支线结果成为主线下一步条件。
- 压缩：把支线从 5 章压到 2 章。
- 转功能：让支线承担伏笔回收、关系变化或代价集中。
- 延期：把暂时不服务本卷的支线挪到下一卷。
- 删除：无法说明功能时直接删。

## 8. 伏笔与悬念线的清账规则

伏笔和悬念是最容易堆烂的线程。

每条伏笔必须记录：

- 伏笔内容。
- 首次出现位置。
- 读者表面理解。
- 真实含义。
- 下一次回响位置。
- 阶段回收位置。
- 最终回收位置。
- 当前状态。

伏笔状态：

| 状态 | 含义 | 处理 |
| --- | --- | --- |
| 新埋 | 刚出现，读者未必注意 | 在 3-10 章内给一次回响 |
| 活跃 | 读者知道这是问题 | 必须有阶段进度 |
| 待回收 | 已经接近答案 | 安排具体回收章 |
| 已回收 | 阶段答案落地 | 记录余波 |
| 延期 | 暂时不在本卷解决 | 写明下一卷窗口 |
| 删除 | 无法服务主线或人物 | 不再继续暗示 |

红线：

- 不允许连续埋伏笔但不给任何回响。
- 不允许用“以后会解释”替代阶段答案。
- 不允许反转时才临时补伏笔。
- 不允许伏笔回收后没有人物或情节后果。

## 9. 人物线和关系线的收束

人物线不是给配角写履历。

人物线要回答：

- 这个人物在本卷想要什么？
- 他和主角的目标是同向、冲突还是交易？
- 他在本卷至少做出什么选择？
- 他的选择如何影响卷目标？
- 卷末他的位置如何变化？

关系线要回答：

- 两人关系起点是什么？
- 本卷有什么共同压力？
- 哪个事件改变信任、误会、亏欠、站队或边界？
- 关系变化如何反过来影响主线选择？
- 卷末关系状态是什么？

关系线阶段结果：

- 信任增加。
- 信任破裂。
- 亏欠形成。
- 边界改变。
- 共同秘密。
- 公开站队。
- 暂时分离。
- 冲突升级。

不合格关系线：

- 只靠误会拖。
- 只靠撒糖刷存在。
- 只靠吵架制造热闹。
- 关系变化不影响任何行动。

---

## 10. 机制线和世界线的收束

机制线和世界线很容易变成设定膨胀。

机制线必须回答：

- 这个能力或规则本卷的新用途是什么？
- 新限制是什么？
- 新代价是什么？
- 谁能利用或反制它？
- 卷末主角对机制的理解如何变化？

世界线必须回答：

- 本卷展示哪个规则、组织、地图或行业层级？
- 这个规则如何限制主角行动？
- 这个规则给主角什么机会？
- 卷中转折是否重释该规则？
- 卷末是否进入更高层级？

坏例：

- 机制只升级，不反噬。
- 世界观只解释，不影响行动。
- 新组织只挂名，不提供规则和资源。
- 地图只换皮，冲突逻辑一样。

## 11. 单元级线程管理

不要试图让所有线每章都推进。长篇需要轮换焦点。

推荐小单元线量：

- 主线：必须推进。
- 核心支线：1-2 条。
- 轻量回响：1 条。
- 暂停线：只保留一句状态或不出现。

每个 3-8 章小单元结束时，必须做一次线程小结：

```markdown
小单元线程小结：

- 本单元主线进度：
- 本单元活跃支线：
- 已交叉线：
- 已回收伏笔：
- 新增伏笔：
- 延期线程：
- 删除线程：
- 下个单元必须处理：
```

如果一个小单元同时推进 5 条以上重线，读者很容易感觉乱。

---

## 12. 卷末清账

卷末不是只结算主线，还要清账。

卷末必须把所有活跃线程分为四类：

| 分类 | 含义 | 处理 |
| --- | --- | --- |
| 已结算 | 本卷功能完成 | 记录结果和余波 |
| 并入主线 | 下卷成为主线条件 | 写入下一卷目标 |
| 延期回收 | 暂不解决但有窗口 | 标明下一次回响位置 |
| 删除/冻结 | 不再服务故事 | 停止提示，避免继续占注意力 |

卷末清账问题：

- 哪条支线推进了卷目标？
- 哪条支线只占篇幅？
- 哪个伏笔回收了？
- 哪个伏笔必须下卷回响？
- 哪个人物位置变了？
- 哪段关系必须影响下一卷？
- 哪个机制或世界规则进入新阶段？

---

## 13. 多 agent 协作中的线程交接

多 agent 写长篇时，支线最容易断。

每次交接必须包含：

- 当前卷目标。
- 活跃线程清单。
- 每条线程当前状态。
- 本章或本单元必须推进的线程。
- 本章不能触碰或不能解决的线程。
- 本章可回收的伏笔。
- 本章新增线程的许可。
- 本章完成后要回写哪些台账。

新增线程许可规则：

- 没有服务卷目标，不新增。
- 没有预计回收窗口，不新增。
- 没有状态变化，不新增。
- 只是为了热闹，不新增。

---

## 14. 线程数量控制

不同作品能承载的线程数量不同。AI agent 默认要保守。

| 作品类型 | 同时活跃重线建议 | 说明 |
| --- | --- | --- |
| 短平快爽文 | 主线 + 1-2 条支线 | 避免影响爽点密度 |
| 常规长篇 | 主线 + 2-3 条支线 | 每个小单元轮换焦点 |
| 悬疑权谋 | 主线 + 3-4 条线 | 必须用真相台账管理 |
| 群像文 | 主线 + 多人物线 | 每个人物线都要有卷内选择和退出点 |
| 女频强关系 | 主线 + 情绪线 + 1-2 条现实压力线 | 关系变化必须影响行动 |

超过建议数量时，必须写线程优先级：

- A 级：本卷必须结算。
- B 级：本卷必须推进，但可不结算。
- C 级：只需回响一次。
- D 级：延期或冻结。

## 15. 常见失败模式

### 15.1 支线只开不收

表现：

- 每卷新增人物和问题。
- 旧支线长期不出现。
- 卷末没有清账。

修法：

- 给每条支线补状态和回收窗口。
- 卷末按已结算、并入、延期、删除分类。
- 删除没有功能的支线。

### 15.2 多线互不交叉

表现：

- 人物线、悬念线、机制线各写各的。
- 主线没有被支线改变。
- 读者像在看几本书拼一起。

修法：

- 每个小单元设计至少一个交叉点。
- 让支线结果改变主线行动条件。
- 让对手利用支线制造压迫。

### 15.3 伏笔越堆越假

表现：

- 每章都有神秘人、黑影、谜语。
- 没有阶段答案。
- 回收时读者觉得空降。

修法：

- 给活跃伏笔安排回响和阶段回收。
- 删除没有真实含义的神秘提示。
- 每次反转前至少有可回看线索。

### 15.4 人物线抢主角

表现：

- 配角比主角更主动。
- 主角旁观支线解决。
- 主角目标停滞。

修法：

- 让人物线改变主角选择，而不是替主角选择。
- 把配角行动结果变成主角下一步条件。
- 限制支线视角和篇幅。

### 15.5 关系线只消耗情绪

表现：

- 误会反复。
- 撒糖反复。
- 吵架反复。
- 关系没有阶段状态。

修法：

- 每个关系单元必须有状态变化。
- 关系变化必须影响主线行动。
- 卷末写清关系状态。

---

## 16. 质量分级

### 高质量

- 每条活跃支线都有功能、挂钩、升级、交叉、回报和退出计划。
- 支线定期改变主线条件。
- 伏笔有回响和阶段回收。
- 人物线和关系线有选择成本。
- 机制线和世界线影响行动。
- 每个小单元有线程小结。
- 卷末完成线程清账。

### 可用但需加强

- 支线功能清楚，但交叉点偏少。
- 伏笔有回收计划，但阶段回响不足。
- 人物线有变化，但对主线影响偏弱。
- 卷末清账存在，但延期线程过多。

### 不合格

- 支线只开不收。
- 多线互不交叉。
- 伏笔没有台账。
- 人物线抢主线。
- 关系线只制造情绪消耗。
- 卷末不知道哪些线已结、哪些线还活跃。

不合格时，不要继续批量起草下一卷。先做线程清账和支线删并。

## 17. 卷级线程闸门

每卷结束前，必须做线程闸门。

允许进入下一卷的条件：

- A 级线程全部结算或并入下一卷主线。
- B 级线程有阶段进展和下一卷窗口。
- C 级线程至少回响一次或冻结。
- 没有无状态支线。
- 伏笔台账可读。
- 人物关系状态可交接。
- 下一卷活跃线程不超过作品承载量。

暂停修订的信号：

- 不知道哪些线还活着。
- 不知道某伏笔要在哪里回收。
- 支线结算不影响主线。
- 下一卷需要靠新线救场。
- 读者可能记不住关键人物和问题。

闸门结论模板：

```markdown
卷级线程闸门：

- 本卷 A 级线程：
- 本卷 B 级线程：
- 本卷 C 级线程：
- 已结算线程：
- 并入下一卷线程：
- 延期线程与回收窗口：
- 删除/冻结线程：
- 已回收伏笔：
- 活跃伏笔：
- 人物关系状态变化：
- 机制/世界规则变化：
- 下一卷活跃线程上限：
- 结论：可进入下一卷 / 先清 A 级线程 / 先补伏笔回响 / 先收束人物线 / 先冻结低价值支线 / 重排本卷线程
```

## 20. 一页检查清单

设计、起草或审稿卷级支线时，逐项检查：

- 每条支线是否服务卷目标？
- 每条支线是否有功能，不只是热闹？
- 每条支线是否有交叉点？
- 每条支线是否有回报或代价？
- 每条支线是否有退出、并入或延期节点？
- 是否存在超过一个小单元没有交叉的支线？
- 活跃伏笔是否有下一次回响？
- 待回收伏笔是否有具体回收章？
- 人物线是否改变主角或卷目标？
- 关系线是否影响行动？
- 机制线和世界线是否影响行动？
- 小单元结束后是否做线程小结？
- 卷末是否做清账？
- 下一卷活跃线程是否超载？
- 是否删除或冻结低价值支线？

## 21. 来源与迁移说明

本专题吸收的是通用叙事资料、多线叙事资料和网文连载经验，经中文网文连载场景重组后使用。

- Writing Excuses 关于 M.I.C.E. threads、middles 和 conflicts 的资料：中段多线要围绕不同问题类型运行，并通过尝试、失败和后果维持推进，迁移为线程类型、交叉矩阵和小单元线程小结。
- Helping Writers Become Authors 关于 multiple plotlines 和 subplots 的资料：多条情节线必须互相关联、服务整体结构，并在合适节点收束，迁移为支线生命周期、主线优先级和退出/并入/延期规则。
- Writers.com 关于 subplot 的资料：支线应有意图地支持主线、深化人物或主题，而不是抢走主线，迁移为支线六种合法功能。
- Jane Friedman 关于多故事线和中段疲软的资料：多线叙事需要清楚的因果、张力和管理，否则会造成松散和遗忘，迁移为线程数量控制、卷末清账和暂停修订信号。
- Story Grid 关于 scene 和 value shift 的资料：每个场景和线程推进都应产生价值变化，迁移为支线必须改变状态。
- 起点社区关于矛盾冲突、大高潮小高潮和节奏的讨论：小悬念、小高潮要持续喂养大悬念，迁移为支线交叉、阶段回收和卷末清账。

迁移到单本小说项目时，不要整段复制本专题。应复制“卷级支线与多线程管理卡”，并填入该项目本卷目标、活跃线程、伏笔清单、小单元线程计划、交叉矩阵和卷末清账结论。


## 与 63 号专题的衔接：卷级线头要管理回响频率

卷级支线、伏笔、旧账和隐藏关系跨越多个小单元时，必须调用 `docs/storycraft/63-foreshadowing-callback-reader-memory-management.md` 管理读者记忆温度。卷级台账中建议为每条线补充：最近回响、记忆温度、下一次提醒、阶段回收、最终回收。

卷中不需要每条线都强推进，但不能长期无声消失。低强度章、转场章、战后清点章、关系缓冲章都可以承担 C1 轻回响或 C3 后果提醒，确保卷末回收时不显得空降。
