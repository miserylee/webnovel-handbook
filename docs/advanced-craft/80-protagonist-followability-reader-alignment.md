# 80. 主角可追随性、读者站队与代入入口专项

状态：已确认

适用场景：开篇设计、主角人设设计、反英雄/黑暗主角设计、审稿、改稿、判断“主角不讨喜”“读者不站队”“代入感弱”“人物不活”。

一句话规则：读者不一定要喜欢主角，但必须知道为什么要继续追随主角。

## 1. 本专题解决的问题

很多 AI 写作会把“主角可追随性”误解成“主角要善良、完美、讨喜”。这会产生两种常见失败：

- 主角太正确、太干净，像功能性道德样板，没有欲望、破绽和选择压力。
- 主角很酷、很疯、很惨、很强，但读者不知道为什么要站在他/她这边。
- 主角只被剧情推着走，读者看到的是设定和事件，不是一个正在挣扎并行动的人。
- 主角的缺陷没有代价，恶行没有叙事后果，读者会觉得作者在强行要求自己喜欢 TA。
- 开篇先堆身世、世界观、系统、仇人、金手指，却没有给出当前欲望、当前压力和当前动作。

本专题把“喜欢主角”拆成更稳定、更适合 AI 执行的“可追随性”。可追随性不是道德评价，而是读者投资理由：读者愿意把注意力、情绪和期待押在这个主角身上。

## 2. 来源吸收

本专题综合了以下外部写作资料，并转化为中文网文可执行规则：

- Writer's Digest《5 Tips to Hook Readers From Chapter One》强调开篇要有切身赌注、好奇心、少解释、让读者有角色可追随，并用章末悬念留下继续阅读理由。
- Writer's Digest《How to Craft Compelling Characters》强调有吸引力的人物通常具备强欲望、秘密、矛盾和脆弱性。
- Reedsy《Protagonist vs. Antagonist》强调主角需要动机、缺陷与变化，而不是完美模板。
- Reedsy Write-In《Characters》强调角色必须渴望某物；没有目标的角色会被动、不变化。
- Helping Writers Become Authors《Conflict + Choices = Character Agency》强调冲突与选择共同构成角色能动性。
- Helping Writers Become Authors《6 Bits of Common Writing Advice You're Misusing》区分“讨喜”和“完美”，提醒读者喜欢有缺陷但有投资价值的人物。

注意：以上资料提供的是通用叙事原理。本知识库已经将其改写为网文语境下的 AI agent 执行规则，不要求后续 agent 逐条引用原文。

## 3. 核心链条

生成或审稿时，主角可追随性应按以下链条检查：

```text
主角处境 -> 可理解欲望 -> 当前压力 -> 主动选择 -> 可见能力/缺陷 -> 价值底线 -> 情绪入口 -> 读者站队理由 -> 后续回报承诺
```

如果这条链断裂，读者常见体感是：

- “我不知道他想干嘛。”
- “她好像只是被剧情推着走。”
- “这个人很惨，但我不心疼。”
- “这个人很强，但我不想跟。”
- “作者想让我爽，但我觉得主角不配赢。”
- “设定很大，但人不抓我。”

## 4. 讨喜、共情、追随的区别

### 4.1 讨喜

讨喜指读者觉得角色可爱、善良、有趣、舒服。它有用，但不是唯一入口。

风险：如果 AI 只追求讨喜，主角会变成低冲突、低欲望、低攻击性的“好人纸片”。

### 4.2 共情

共情指读者能理解角色为什么这样感受、这样行动。读者不一定赞同主角，但能看懂其内在逻辑。

风险：如果只有共情，没有行动和回报，主角会变成“值得同情但不值得追”的受害者。

### 4.3 可追随

可追随指读者愿意继续看主角做选择、付代价、拿回报、改变命运。可追随性可以来自善良，也可以来自能力、欲望、反差、脆弱、幽默、狠劲、正义、秘密、成长潜力或危险魅力。

执行原则：不要问“主角讨不讨喜”，先问“读者追随 TA 的理由是什么”。

## 5. 主角可追随性的八个支柱

### 5.1 欲望清晰

原则：读者必须知道主角现在想要什么。

用法：第一章至少给出一个当前可见欲望，不要只写宏大终极目标。当前欲望可以很小，例如保住饭碗、逃过追杀、拿回一笔钱、赢一次尊严、确认某个人是否还活着。

风险：没有当前欲望，主角会像剧情摄像机，读者只能看事件，不能追人。

检查问题：如果删掉旁白解释，只看主角动作，读者能否判断 TA 想要什么？

### 5.2 处境有压

原则：欲望必须遇到压力，压力必须发生在当前，而不是只存在于背景故事。

用法：给主角一个正在逼近的限制，例如时间、资源、身份、关系、敌人、规则、名誉、身体状态、情绪崩溃点。

风险：只有悲惨身世，没有当前压力，读者会觉得作者在索取同情。

检查问题：这一章里有什么东西正在逼主角现在就做决定？

### 5.3 主动动作

原则：主角必须在压力中做事，而不是只等系统、师父、霸总、重生记忆、朋友或反派推动。

用法：即使主角很弱，也要让 TA 做一个小动作：藏证据、套话、忍住冲动、换路线、先救一个人、反咬一句、把失败变成信息。

风险：主角被动会削弱爽点，因为读者会觉得回报不是主角挣来的。

检查问题：如果去掉外部救援，主角这一章是否仍然做过影响局面的选择？

### 5.4 可见能力或潜力

原则：读者需要看到主角有某种值得下注的东西。

用法：能力不一定是战斗力，可以是观察力、忍耐、学习速度、情绪控制、谈判、审美、手艺、道德勇气、记忆、幽默、奇怪的直觉、底层经验。

风险：只写“他其实很强”“她以后会成长”不够，读者需要现场证据。

检查问题：这一章有没有一个动作能证明主角不是空壳？

### 5.5 缺陷可理解且有代价

原则：缺陷不是装饰，也不是作者豁免卡。缺陷要有来源、表现和代价。

用法：让主角因为冲动、骄傲、敏感、冷漠、贪婪、怕输、过度保护、低自尊等付出小代价，再让 TA 逐步调整。

风险：缺陷没有代价，读者会觉得作者偏袒主角；缺陷只有解释没有行为，读者会觉得人设是贴标签。

检查问题：主角的缺陷在本段剧情中造成了什么麻烦？

### 5.6 价值底线

原则：读者需要知道主角不是任意行为机器。

用法：通过小事展示主角“不做什么”或“做了会付什么代价”。反英雄也需要规则，例如不伤无辜、不背叛某个人、不拿孩子当筹码、报复只对准施害者、越线后承受反噬。

风险：没有底线的主角不是危险迷人，而是叙事不可信。

检查问题：主角有没有一条能被读者记住的边界？如果越界，叙事是否承认了成本？

### 5.7 情绪入口

原则：读者追随主角，往往先从一个具体情绪入口进入。

用法：情绪入口可以是被误解、被轻视、被亏欠、羞耻、饥饿、失去、孤独、不甘、压抑的怒气、终于被看见、终于敢拒绝。

风险：只有抽象情绪词，没有具体处境和动作，读者不会真的被带进去。

检查问题：读者能否用一句人话说出“我为什么替 TA 不爽/心疼/紧张/期待”？

### 5.8 回报承诺

原则：可追随性还包括类型承诺：继续看这个主角，会得到什么爽感或情绪回报。

用法：男频升级文给尊严和收益承诺；女频情感文给选择权、边界和被珍视承诺；悬疑文给解谜能力承诺；喜剧文给声音和反应承诺；反英雄文给复仇、智斗或黑色幽默承诺。

风险：主角本身有趣，但类型回报不清，读者会不知道该用什么期待方式阅读。

检查问题：读者追随主角三章后，最期待看到什么被兑现？

## 6. 读者站队的七种入口

### 6.1 投射型

读者想成为主角。适合升级、事业、权谋、经营、爽文。

执行：强化能力、资源增长、身份跃迁、压迫反击、尊严修复。

### 6.2 心疼型

读者想看主角被理解、被补偿、被治愈。适合情感、救赎、成长、虐转甜。

执行：具体化伤口，不滥用惨；让主角在伤口中仍然保留选择。

### 6.3 崇拜型

读者想看主角解决问题。适合天才流、医生、律师、刑侦、技术、强者重生。

执行：用现场判断和方法展示能力，不只用旁人惊呼。

### 6.4 共谋型

读者享受主角的狡黠、危险、报复或黑色幽默。适合反英雄、复仇、权谋、黑莲花。

执行：明确目标、规则、代价和叙事态度，让读者知道自己在享受什么。

### 6.5 反差型

读者被外冷内热、强者脆弱、疯批守序、毒舌护短等反差吸引。

执行：反差要落在行为上，而不是只靠标签。

### 6.6 成长型

读者想见证主角从不成熟走向成熟。适合长篇连载、群像、修仙、校园、职业成长。

执行：早期必须给出可成长的种子，不要只给缺点。

### 6.7 喜剧型

读者喜欢主角的声音、反应和活法。适合轻小说、沙雕、日常、吐槽流。

执行：让幽默来自角色独特应对方式，不要让主角只会说段子。

## 7. 反英雄、黑莲花、疯批主角的安全规则

反英雄不等于“可以随便恶”。黑暗主角尤其需要更清晰的读者契约。

可执行规则：

- 目标清楚：TA 的攻击对象是谁，为什么是这个对象。
- 伤害有边界：TA 不是什么都做，或者做了会承受后果。
- 能力有证据：危险魅力来自策略、执行、洞察或意志，不是只靠作者说“疯”。
- 内在可理解：读者可以理解 TA 的逻辑，即使不赞同。
- 叙事不洗白：越线可以被理解，但不能被轻飘飘抹掉。
- 回报明确：读者知道自己看的是复仇、智斗、清算、讽刺、黑色幽默还是道德坠落。

失败信号：主角伤害弱者、背叛无辜、靠身份碾压、靠作者偏袒逃避代价，却仍被所有角色称赞。这会迅速破坏读者信任。

## 8. 开篇首屏检查

第一章前 1000 到 2000 字，最好让读者至少看到以下七项中的四项：

- 主角是谁。
- 主角当前被什么压力逼着。
- 主角现在想要什么。
- 主角做了什么主动动作。
- 主角有什么能力、缺陷或独特声音。
- 读者为什么替 TA 不爽、心疼、紧张、好奇或期待。
- 下一页有什么明确问题等着被回答。

首屏不要优先写：完整世界观、完整身世、完整等级体系、全部仇人名单、系统说明书、长篇心理总结。

更好的首屏策略：让读者先站到主角身边，再逐步告诉读者世界有多大。

## 9. 类型适配

### 9.1 男频升级与爽文

优先级：压迫 -> 主动选择 -> 能力证据 -> 硬回报 -> 尊严修复。

常见风险：主角只靠金手指自动赢，读者觉得收益没有挣来。

修正：在金手指生效前，让主角先做一个判断、冒一次险、忍一次屈辱、抓住一次机会。

### 9.2 女频情感与成长

优先级：情绪入口 -> 选择权 -> 边界 -> 被看见 -> 关系回报。

常见风险：主角只善良、只隐忍、只被爱，缺少欲望和决定。

修正：让主角在关系压力中做选择，并承担选择后果。

### 9.3 悬疑、刑侦与无限流

优先级：问题意识 -> 观察能力 -> 恐惧/好奇 -> 行动策略 -> 递进谜题。

常见风险：主角只是跟着线索走，没有自己的判断。

修正：让主角先提出一个错误但合理的假设，再通过事件修正。

### 9.4 喜剧与轻松向

优先级：独特声音 -> 反应方式 -> 小目标 -> 连续误差 -> 情绪松弛。

常见风险：主角像段子容器，剧情和情感都不推进。

修正：每个笑点都让主角处境发生一点变化。

### 9.5 反英雄与黑暗爽文

优先级：目标 -> 规则 -> 能力 -> 代价 -> 清算回报。

常见风险：把“恶”写成“酷”，没有读者契约。

修正：明确被清算对象的压迫性，明确主角越线后的反噬或孤独成本。

### 9.6 群像文

优先级：先确认读者锚点，再扩展群像。

常见风险：每个人都有设定，但没有一个人让读者先追。

修正：开篇给一个主锚点角色，其他人物围绕这个锚点制造选择、冲突和镜像。

## 10. AI agent 生成流程

### 10.1 起草前

先回答以下问题：

- 主角当前最想要什么？
- 如果 TA 得不到，会立刻失去什么？
- TA 会主动做哪件小事？
- TA 的能力或潜力用哪个动作证明？
- TA 的缺陷会带来什么麻烦？
- TA 的底线是什么？
- 读者追随 TA 的主要入口是哪一种？
- 三章内承诺兑现什么回报？

### 10.2 起草中

写作时优先展示，不要解释：

- 用动作展示欲望。
- 用压力展示处境。
- 用选择展示能动性。
- 用代价展示缺陷。
- 用小边界展示价值。
- 用反应展示情绪入口。
- 用章末问题展示后续回报。

### 10.3 修订时

如果读者反馈“主角不讨喜”“不想看他/她赢”“代入不进去”，不要先润色句子。先补链条：

```text
欲望是否可见？
压力是否当前？
动作是否主动？
能力是否有证据？
缺陷是否有代价？
底线是否存在？
情绪入口是否具体？
回报承诺是否明确？
```

## 11. 快速修复方案

### 11.1 主角太被动

修复：增加一个微选择。哪怕主角仍然失败，也要让失败来自 TA 的行动，而不是纯粹被砸中。

### 11.2 主角太完美

修复：增加一个会影响局面的缺陷，并让缺陷带来短期损失。

### 11.3 主角太惨但不抓人

修复：把抽象惨改成当前压力，再让主角做一个保住尊严或求生的小动作。

### 11.4 主角太坏

修复：明确目标、边界和代价；如果主角越线，叙事要承认这件事的重量。

### 11.5 主角太强但无聊

修复：增加限制、误判、情感牵挂或必须付出的成本。

### 11.6 主角人设像标签

修复：每个标签替换成一个行为证据。例如“冷静”改成“所有人都在吵时，他先把门锁好，再问谁碰过杯子”。

## 12. 审稿清单

审稿时逐项打标：已确认、候选、待确认。

- 主角当前欲望：已确认/候选/待确认。
- 当前压力：已确认/候选/待确认。
- 主动动作：已确认/候选/待确认。
- 能力或潜力证据：已确认/候选/待确认。
- 缺陷及代价：已确认/候选/待确认。
- 价值底线：已确认/候选/待确认。
- 情绪入口：已确认/候选/待确认。
- 读者站队理由：已确认/候选/待确认。
- 类型回报承诺：已确认/候选/待确认。
- 三章内兑现路径：已确认/候选/待确认。

如果有三项以上是“待确认”，优先重写主角入口，而不是继续扩写设定。

## 14. 最小执行规则

- 第一章必须让读者知道主角现在想要什么。
- 主角必须主动做事，不能只是被剧情推着走。
- 主角可以不完美，但缺陷必须有代价。
