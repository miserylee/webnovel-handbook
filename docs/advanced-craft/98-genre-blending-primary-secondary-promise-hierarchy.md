# 98. 类型混搭、主副卖点与承诺优先级专项

状态：已确认

适用范围：新书立项、题材定位、书名简介、开篇设计、融合题材、跨频道项目、类型漂移诊断、续写跑偏、支线抢主线、读者反馈为“不是这个味”“不知道看点”“四不像”“感情线抢戏”“悬疑线拖爽点”“系统设定抢人物”“标签党”“前面像 A 后面像 B”。

核心结论：类型混搭可以成立，但读者承诺不能平均分配。AI agent 必须先确定主类型/主承诺，再规定副类型负责提供新鲜变量、阻力、节奏释放或情绪补充；副卖点不能抢走主卖点的第一轮证明和阶段回报。

最小链路：目标读者 -> 主类型入口 -> 主承诺 -> 主回报币种 -> 副类型功能 -> 副承诺边界 -> 类型底线 -> 开篇证明 -> 前 10 章回报 -> 长线融合规则。

---

## 1. 为什么需要独立专题

已有第 13 篇覆盖常见题材模式，第 29 篇覆盖读者预期、类型承诺和卖点验证。但 AI 在融合题材时仍容易出问题：

- 把“有很多元素”误当成“卖点丰富”。
- 开篇同时承诺爽文、恋爱、悬疑、经营、喜剧、群像，导致读者不知道按什么期待读。
- 副题材连续多章推进，主承诺停滞。
- 包装写的是甜宠，正文先写三章悬疑设定。
- 读者为升级爽感而来，却被长篇感情误会拖住。
- 读者为悬疑真相而来，却被日常经营冲淡问题链。
- 系统、重生、穿书、快穿等工具型题材抢走人物和主线。

本专题负责把混搭从“元素堆叠”改成“承诺层级”。

## 4. 基础概念

### 4.1 主类型入口

读者第一眼把这本书归到哪里。它通常由平台频道、书名、简介、标签、第一章场景和主角处境共同决定。

主类型入口决定读者最先要什么。

### 4.2 主承诺

本书最先欠读者的阅读合同。它不是题材名，而是体验句。

错误：这是一本系统修仙甜宠悬疑经营文。

正确：读者将看到女主用前世账本和系统漏洞，在宗门债务局一步步翻盘。

### 4.3 副类型

副类型是辅助维度。它可以提供：

- 新鲜设定。
- 阻力来源。
- 情绪调剂。
- 关系张力。
- 节奏释放。
- 单元任务。
- 公开见证。
- 长线谜团。

副类型不是第二个主角，不应连续抢走主回报。

### 4.4 回报币种

不同类型给读者的回报不同：升级给资源和阶层变化；甜宠给关系确认和情绪糖；悬疑给答案和重释；经营给改善和资产；喜剧给错位和笑点；权谋给解释权、站队和局势变化。

混搭时，必须先定义主回报币种。

### 4.5 类型底线

某些类型有不能违约的底线。比如甜宠不能长期只有虐无补偿；悬疑不能最后全靠空降答案；爽文不能长期不给主角阶段回报；经营文不能只有口号没有可见改善；喜剧不能为了笑点长期破坏人物逻辑。

## 5. 主副承诺层级

建议用四层管理混搭：

| 层级 | 功能 | 示例 |
| --- | --- | --- |
| P0 不可动核心 | 主角底盘、目标读者、主承诺、主回报币种 | “主角用职业能力翻盘” |
| P1 主类型证明 | 第一章、前三章、前十章必须证明的体验 | 第一次反制、第一次落袋、第一次公开结果 |
| P2 副类型功能 | 提供变量、压力、关系、梗点或谜团 | 恋爱线提供情绪代价，悬疑线提供旧案压力 |
| P3 装饰元素 | 氛围、趣味、轻设定、短梗 | 宠物、直播、学院、饭馆、论坛等 |

执行原则：P2 必须服务 P0/P1；P3 可以删，不影响主承诺。

## 6. 副类型的七种合法功能

### 6.1 提供新鲜变量

例：升级流里加入直播，直播不是主线，而是让打脸有公开见证和舆论回报。

### 6.2 提供阻力

例：甜宠里加入豪门规则，豪门不是百科设定，而是制造关系边界和选择代价。

### 6.3 提供情绪补充

例：悬疑主线中加入亲情线，让真相不只是答案，还会改变人物关系。

### 6.4 提供节奏释放

例：权谋主线中加入喜剧配角，用低强度错位缓冲高压，但不能打断关键博弈。

### 6.5 提供单元结构

例：经营主线中加入副本任务，每个副本提供一次经营资源或客户关系变化。

### 6.6 提供公开见证

例：职场文加入论坛/直播/热搜，让结果被更多受众看见，放大名声资产。

### 6.7 提供长线谜团

例：爽文主线中加入身份谜，但身份谜应服务翻盘和资源重定价，而不是长期拖住爽点。

## 7. 常见混搭公式与风险

### 7.1 升级 + 系统

主承诺通常是升级、反压、资源和阶层变化。系统负责解释成长速度、制造任务、提供限制和回报凭证。

风险：系统面板抢走人物选择；奖励太万能，主角没有代价。

### 7.2 重生 + 复仇

主承诺通常是改命、清算、预知信息带来的反击。重生信息负责提供先手，但必须允许变量变化。

风险：前世记忆万能，后文没有不确定性。

### 7.3 甜宠 + 悬疑

需要先判断主承诺是关系情绪还是真相推进。

如果主承诺是甜宠，悬疑负责制造误会、保护、共同调查和关系升级。

如果主承诺是悬疑，感情线负责增加代价和人物软肋。

风险：恋爱拖掉真相，或真相压扁关系糖点。

### 7.4 经营 + 古言/种田

主承诺通常是生活改善、资源增长、身份翻盘。古代规则负责制造限制，种田/经营负责可见资产。

风险：生活细节很满，但没有经营改善；感情线连续抢走经营回报。

### 7.5 快穿 + 打脸/救赎/恋爱

快穿负责位面新鲜感和单元结构。主承诺必须另定：打脸、救赎、攻略、修复剧情、经营任务或情绪补偿。

风险：每个位面开头都像新书，主角长期主线停滞。

### 7.6 悬疑 + 无限流/规则怪谈

主承诺通常是真相推进、规则破解、生存压力。无限流提供单元压力和规则容器。

风险：规则随机、只开新副本不回答旧谜。

### 7.7 都市 + 神豪 + 直播

主承诺通常是财富回报、公共见证、阶层反压和社交反馈。直播负责放大见证者和传播结果。

风险：金额变大但局面不变，直播观众只会震惊。

### 7.8 群像 + 主线题材

群像不是主承诺，除非本书卖点就是多人物共同承压。否则群像应服务主线多角度推进。

风险：配角线长期抢主角和主承诺，读者站队分裂。

## 8. 类型混搭的一票否决

以下情况应暂停正文生产，回到项目定位：

- 不能一句话说清主承诺。
- 第一章无法证明主卖点，只能展示设定名词。
- 主回报币种不清：读者不知道等爽、甜、谜、燃、经营改善还是喜剧。
- 副类型连续三章推进，主承诺无状态变化。
- 包装承诺和正文前 3 章体验不是同一本书。
- 两个类型底线互相冲突，且没有明确优先级。
- 目标读者分裂成两批，分别想读不同主承诺。
- 编辑或读者建议要求作品改成另一个主承诺，agent 没有做小样验证就直接改。

## 9. 开篇证明规则

融合题材开篇尤其不能平均铺陈。

### 9.1 第一章

第一章只需要证明一个主入口：主角、困境、主承诺影子、第一压力或第一反压。

副类型可以出现，但只能作为压力、诱因或钩子，不应抢走主承诺证明。

### 9.2 前三章

前三章必须让读者知道：这本书主要提供什么体验。副类型可以形成第二层期待，但不能让读者误以为主类型换了。

### 9.3 前十章

前十章至少完成一次主回报落袋，并让一个副类型功能真正服务主回报。

例：甜宠+悬疑里，悬疑线不仅制造谜团，还让主角和对象关系位置发生变化。

---

## 10. 章节级承诺分配

每章起草前都应标注：

```text
本章推进主承诺：是 / 否
本章推进哪条副承诺：
副承诺如何服务主承诺：
本章回报币种：资源 / 情绪 / 关系 / 答案 / 名声 / 权限 / 笑点 / 经营改善 / 其他
如果本章不直接推进主承诺，它提供什么必要铺垫或节奏释放：
连续几章未推进主承诺：
风险：主承诺停滞 / 副线抢戏 / 回报币种错位 / 类型底线违约
```

硬规则：连续三章只推进副承诺，要么合并压缩，要么让副承诺转化为主承诺回报。

## 11. 包装与正文的混搭顺序

包装最多突出一个主承诺和两个副承诺。

### 11.1 书名

书名优先传达主承诺或最强可点击卖点。副类型词不要堆到看不清主入口。

### 11.2 简介

第一段证明主承诺，第二段展示副变量和钩子。

### 11.3 标签

标签可以多，但主标签必须和正文前 3 章体验一致。

### 11.4 第一章

第一章必须兑现包装最醒目的那条承诺，至少给出影子、压力或可见证明。

## 12. AI 常见坏味道与修复

| 坏味道 | 读者体感 | 修复方式 |
| --- | --- | --- |
| 卖点全塞 | 看点多但不知看什么 | 只保留一个主承诺，其他降级为功能 |
| 副线抢主线 | 跑题、水、不是这个味 | 标注连续未推进主承诺章数，合并或转化副线 |
| 包装正文错配 | 标题党、被骗点击 | 重写书名简介或重写前 3 章证明 |
| 回报币种乱跳 | 期待错位 | 固定主回报币种，副回报服务主回报 |
| 类型底线互撞 | 两边读者都不满意 | 选择主类型底线，另一类型降级为元素 |
| 工具题材抢人 | 系统/快穿/重生比人物重要 | 让工具改变选择、代价和关系，不替人物做决定 |
| 融合只停留在设定 | 新鲜但不推进 | 让副类型承担阻力、回报或公开见证 |
| 读者群分裂 | 评论要求完全相反 | 回到目标读者和主承诺，不按票数大改 |
| 中段类型漂移 | 前后像两本书 | 用卷目标和回报币种重新校准 |
| 为创新破坏底线 | 不是反套路，是违约 | 保留类型必备回报，再做变体 |

## 14. 审稿检查清单

- 主承诺能否一句话说清？
- 主类型入口是否和平台、频道、书名、简介一致？
- 读者第一章能否看见主承诺影子？
- 副类型是否承担明确功能，而不是单纯增加元素？
- 副承诺是否连续多章抢走主承诺？
- 主回报币种是否稳定？
- 第一轮主回报是否在前 10 章落袋？
- 类型底线是否被创新破坏？
- 包装最醒目的卖点是否在正文前三章出现？
- 读者反馈“不是这个味”时，问题是包装错、开篇错，还是中段漂移？
- 如果删掉某个副类型，本书主承诺是否仍成立？如果不成立，它可能不是副类型而是主类型。

通过标准：混搭元素彼此加成，读者仍能清楚知道自己主要在等什么回报。

---

## 15. 最小可执行规则

AI 处理融合题材时至少遵守以下规则：

- 先定一个主类型入口和一个主承诺。
- 副类型最多两个，且必须服务主承诺。
- 主回报币种必须明确，前十章至少落袋一次。
- 包装最多突出一个主承诺和两个副承诺。
- 副线连续三章抢主承诺，必须合并、压缩或转化为主回报。
- 类型底线不能互相牺牲；冲突时按主类型底线处理。
- 工具型题材不能替人物选择，只能制造变量、限制、任务或代价。
- 读者反馈要求换主承诺时，不直接采纳，先回到目标读者和项目定位。
