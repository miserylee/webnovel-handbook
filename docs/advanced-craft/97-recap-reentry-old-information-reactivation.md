# 97. 前情重提、读者再入场与旧信息再激活专项

状态：已确认

适用范围：长篇连载、断更恢复、跨卷启动、付费章节、复杂设定回收、伏笔回收前、旧角色回归、旧关系/旧债/旧规则重启、多 agent 续写、章节开头承接、审稿反馈为“前面忘了”“回顾太多”“重复解释”“像前情提要”“读者接不上”“断更回来读不进”“旧信息突然有用但想不起来”。

核心结论：前情重提不是复述过去，而是把旧信息通过当前压力重新激活。AI agent 不能用整段回顾替代推进；旧信息每次再出现，都必须改变当前判断、选择、关系、风险、情绪或行动。

最小链路：读者可能遗忘的信息 -> 当前场景为什么需要它 -> 最小提醒方式 -> 角色如何重新碰到它 -> 它改变什么判断/选择 -> 后果进入本章推进。

## 2. 来源与吸收方式

本专题吸收以下资料与已有知识库规则：

- arXiv 论文 `Previously on the Stories: Recap Snippet Identification for Story Reading`：吸收“类似影视 previously-on 的片段可以帮助读者回忆重要元素以理解后续情节”的思路，迁移为连载正文的再入场策略。
- Jane Friedman 关于 summary and spacers 的资料：吸收 summary 可以移动时间、提供定位、连接场景，但应像 mortar 一样服务叙事结构，而不是替代场景。
- Jane Friedman 关于 backstory/exposition 的资料：吸收过量前情和背景会打断当前故事，旧信息需要围绕现在的情境和理解需求投放。
- Savannah Gilbo 关于 backstory without info-dumping 的资料：吸收背景信息应在读者需要它理解当前行动时出现，且不能让当下故事停摆。
- Laterpress 关于 serial fiction 的资料：吸收连载单元需要让每一集有自身推进、结尾牵引和持续可追性。
- `Reader Memory Decay` 资料：吸收长篇和连载中读者工作记忆会衰减，提醒应通过后果而不是机械 recap 进入正文。

吸收原则：不照搬外部资料原文，只转化为中文商业网文和 AI agent 可执行的连载重提规则。

## 3. 为什么长篇需要“再入场”

连载读者不是每天都以完美记忆进入下一章。常见情况包括：

- 今天更新和上次阅读之间隔了几小时、几天或更久。
- 读者同时追多本书，角色名、旧线索和支线容易混淆。
- 跨卷、跨副本、跨地图后，旧资产和旧代价需要重新定价。
- 复杂设定、规则、组织、时间线和身份真相需要阶段性唤醒。
- 断更恢复或爆更后，读者阅读节奏变化。
- 多 agent 续写时，agent 本身也可能只记得台账，不知道读者记忆温度。

但再入场不能等于“前情提要塞正文”。正文的任务仍是推进当前章节。

## 4. 可提醒信息与不可提醒信息

### 4.1 应该提醒

以下旧信息在重新影响当前剧情前，应给读者一个抓手：

- 旧伏笔即将回收。
- 旧角色隔很久后回归。
- 旧规则、旧道具、旧技能、旧关系债将改变当前选择。
- 上一章章末钩子需要在本章承接。
- 断更恢复后第一个关键冲突。
- 跨卷后旧卷资产、代价、伤势、关系、名声仍在影响新卷。
- 复杂时间线、身份真相、案件线索在本章要重新使用。

### 4.2 不应该提醒

以下信息不应重复：

- 读者刚刚在上一屏或上一章读过，且本章没有新用途。
- 只是作者担心读者忘记，但当前场景不需要。
- 只是为了凑字数复述前情。
- 只是让角色互相讲双方都知道的事。
- 只是把台账内容搬进正文。
- 只是说明设定“很重要”，但没有立刻改变行动。

判断标准：如果旧信息重提后，本章的判断、选择、风险、关系或情绪没有变化，就不要提醒。

## 5. 六种重提方式

### 5.1 后果提醒

用旧事留下的后果提醒读者。

示例：不是“他还记得三章前被刺伤”，而是“他举剑时，左肩先塌了一寸”。

适合：伤势、债务、名声、资源损耗、关系裂痕、失败代价。

### 5.2 物件提醒

让旧信息附着在物件上重新出现。

示例：那枚裂了边的玉牌还在桌角，偏偏新来的执事一眼认出了它。

适合：伏笔、关系小物、身份凭证、旧账本、钥匙、武器、信物。

### 5.3 对话冲突提醒

让旧信息在争执、试探、威胁、谈判或误会中出现。

有效：你上次也是这么说的。然后她把那份没签完的契约推回去。

无效：你还记得吗？上次我们在青石巷遇到了那个人，他给了我们这份契约。

### 5.4 选择压力提醒

旧信息不被解释，而是在新选择中成为限制。

示例：他可以冲进去，但上次就是因为冲得太快，才让阿宁少了一只手。

适合：失败教训、旧策略、代价胜、关系边界、职业规则。

### 5.5 环境或公共反馈提醒

让世界记得旧事：传言、公告、伤亡名单、价格变化、通缉令、舆论、组织态度。

适合：公开打脸、商战、权谋、宗门/公司/学院组织线、世界同步升级。

### 5.6 极短摘要提醒

在跨卷、长间隔、复杂回收前，可以用一句到三句摘要。

要求：必须短、具体、带当前用途。

有效：三个月前，林照用这条密道逃过一次命。今天它只剩半截，另一半被新城墙压在了下面。

无效：三个月前，他们经历了很多事情，包括密道逃生、城墙改建和敌人追杀，这些都让局势变得复杂。

## 6. 重提强度分级

| 等级 | 方式 | 适用情况 | 风险 |
| --- | --- | --- | --- |
| R0 不提醒 | 直接推进 | 信息很近、读者应记得、当前用途清楚 | 读者断更后可能接不上 |
| R1 微提醒 | 一个词、一个动作、一个物件 | 上章或近 3 章信息 | 过少会漏，过多会水 |
| R2 场景化提醒 | 通过冲突、选择、后果唤醒 | 跨 3-10 章旧信息 | 需要占用场景空间 |
| R3 摘要桥 | 一到三句短摘要 | 跨卷、断更、复杂设定回收前 | 容易变前情提要 |
| R4 正文外 recap | 作者话、卷首前情、作品页资料 | 长断更、系列新书、番外/新卷入口 | 不能替代正文推进 |

原则：能用 R1/R2 解决，就不要用 R3/R4。正文内优先场景化再激活，正文外 recap 只用于读者服务，不用于替正文解释。

## 7. 章节开头的再入场

章节开头常需要帮读者重新进入状态，但第一屏不能变成前情提要。

### 7.1 第一屏要回答

- 上一章留下的核心问题是否被承接？
- 当前人物在哪里？
- 当前目标或压力是什么？
- 旧信息里哪一项现在正在影响他？
- 读者为什么要继续看这一章？

### 7.2 第一屏禁止

- 复述上一章完整剧情。
- 连续解释多个旧线索。
- 大段回顾人物经历。
- 用“上回说”“还记得”式旁白进入正文。
- 让角色互相背诵双方都知道的信息。

### 7.3 推荐句式

```text
天还没亮，昨夜那封信已经传遍了半座城。
```

这一句没有复述信的全文，但告诉读者：上一章的信已经产生公共后果。

## 8. 断更恢复与跨卷再入场

断更恢复、长假后更新、跨卷开头，可以比普通章节给更多提醒，但仍要保护正文推进。

### 8.1 正文内

- 用当前场景重启旧压力。
- 第一场让旧代价、旧资产或旧敌人发生作用。
- 不把恢复章写成“前情提要章”。
- 前 25% 承接断更前核心钩子或卷末余波。

### 8.2 正文外

可以在作者话或作品页提供简短非剧透 recap，但不能用来补正文缺口。

正文外 recap 适合：

- 很长时间断更。
- 新卷开始。
- 系列第二部。
- 复杂支线重新启动。

正文外 recap 不适合：

- 替主线解释关键答案。
- 替付费章兑现回报。
- 替正文补人物动机。

## 9. 付费章节的重提规则

付费读者对重复前情更敏感，因为他们支付的是新内容。

首付章和关键付费章应遵守：

- 前 25% 先承接并兑现一个旧期待。
- 不用大段回顾免费期替代新回报。
- 必要提醒控制在最小可理解范围。
- 旧信息重提后必须立刻产生新动作、新风险或新收益。
- 付费章可以唤醒旧爽点，但必须给新落袋。

如果付费章前半段全是回顾、解释、赶路、复盘，读者会觉得被水。

---

## 10. 多 agent 续写的旧信息重提

多 agent 续写时，不能把台账原样搬进正文。

正确流程：

1. 从台账读取上一章结果、活跃伏笔、人物状态、规则状态。
2. 判断本章真正需要激活哪 1-3 项旧信息。
3. 为每项旧信息选择 R0-R4 重提强度。
4. 把旧信息挂到当前场景的目标、阻力、选择或后果上。
5. 写后只回写变化，不复述整章。

错误流程：

- 起草前读台账，正文开头把台账摘要复述一遍。
- 角色见面后互相解释前几章发生了什么。
- 每次旧人登场都重新介绍完整人物卡。

## 11. 类型文专项用法

### 11.1 重生文

前世信息不能无限重复。每次重提前世，应当说明当前变量如何改变旧答案。

有效：旧信息被新变量挑战。

无效：主角每次行动前都复述“前世这里发生过什么”。

### 11.2 系统文和升级文

面板、等级、技能冷却、资源数量可以短提醒，但必须和当前选择绑定。

有效：他只剩三点灵力，刚好够开一次门，不能再出剑。

无效：重复展示完整面板。

### 11.3 悬疑推理

旧线索重提要改变假设，而不是重新列清单。

有效：旧证词被新时间线推翻。

无效：侦探把已知线索重新念一遍。

### 11.4 甜宠和关系文

旧关系小动作适合轻回响，但必须有新意义。

有效：他这次没有替她拉椅子，而是把椅子推远了半寸。

无效：反复回忆第一次拉椅子的细节。

### 11.5 权谋商战

旧合同、旧承诺、旧舆论、旧债务重提时，要进入当前利益交换。

有效：旧合同不只是被提起，而是让对手今天不能退场。

### 11.6 喜剧

旧梗可以重提，但必须变奏升级。

有效：第一次怕狗，第二次他带了防狗喷雾，结果遇见的是机械狗。

无效：每次都重复“他怕狗”。

## 12. 常见坏味道与修复

| 坏味道 | 读者体感 | 修复方式 |
| --- | --- | --- |
| 章节开头复述上一章 | 水、拖、像提要 | 改成上一章结果的当前后果 |
| 角色互背前情 | 假、尬 | 改成争执、威胁、误会或交易中的信息碰撞 |
| 台账搬进正文 | 像工作说明 | 只选本章需要激活的 1-3 项旧信息 |
| 旧信息突然有用 | 回想不起来、空降 | 提前 R1/R2 提醒，或在使用前场景化再激活 |
| 提醒太频繁 | 重复解释、消耗耐心 | 降低重提强度，改为物件或后果提醒 |
| 回顾不产生行动 | 纯水 | 让旧信息改变选择、风险或关系位置 |
| 跨卷旧资产断线 | 像重开 | 新卷第一单元让旧资产重新定价 |
| 断更恢复全是 recap | 恢复章不值 | 作者话可简短 recap，正文先推进旧钩子 |
| 付费章回顾过多 | 被骗订阅 | 前 25% 兑现旧期待，提醒只服务新回报 |
| 旧梗重复无变奏 | 疲劳 | 每次重复必须升级、反转或产生新后果 |

## 14. 审稿检查清单

审稿时逐条检查：

- 本章提醒了哪些旧信息？每条是否有当前用途？
- 旧信息是否改变了角色判断、选择、风险、关系或情绪？
- 是否有大段复述上一章或上一卷？
- 是否让角色互相说双方已经知道的事？
- 是否把台账摘要搬进正文？
- 旧角色回归时，是否给了可识别动作或当前功能，而不是重读人物卡？
- 旧规则重启时，是否展示它如何限制当前行动？
- 跨 5 章以上的旧信息是否联动第 63 篇管理记忆温度？
- 断更恢复或跨卷开头是否先推进旧钩子，而不是先写前情提要？
- 付费章是否把回顾压缩到最小，并交付新回报？

通过标准：读者能重新想起必要旧信息，同时感觉故事仍在向前走。

## 15. 最小可执行规则

AI 写章节时至少遵守以下规则：

- 旧信息重提必须服务当前选择、风险、关系、情绪或后果。
- 优先用后果、物件、对话冲突、选择压力和环境反馈提醒，而不是整段 recap。
- 章节开头可以帮读者再入场，但不能复述上一章完整剧情。
- 跨 5 章以上旧信息要联动第 63 篇；跨卷或断更可考虑正文外 recap。
- 台账只供 agent 读取，不能原样搬进正文。
- 付费章节的前情提醒必须短，并迅速交付新内容。
- 旧梗、旧物、旧称呼、旧规则每次重提都要有变奏或新后果。
- 如果旧信息重提后本章没有状态变化，就删掉或降级。
