# 93. 场景化、摘要化、说明化与 Show/Tell 选择专项

状态：已确认

适用范围：章节起草、续写、改稿、压缩水章、扩写关键场景、设定投放、情绪戏、动作戏、开篇留存、低强度过渡、审稿反馈为“像大纲摘要”“解释太多”“流水账”“该写的没写出来”“小事写太长”“场景不在现场”“全是展示导致拖沓”。

核心结论：Show/Tell 不是“永远展示、不要说明”，而是叙事变焦选择。商业网文需要把关键选择、代价、爽点、情绪债和关系变化写成场景，把重复过程、低价值移动、背景事实和已知状态压缩为摘要或说明。

最小链路：本段功能 -> 读者需要体验还是只需知道 -> 是否有选择/代价/状态变化 -> 选择场景/摘要/说明/省略 -> 给出可见证据 -> 进入下一步行动。

---

## 1. 这个专题解决什么问题

AI 写小说时常见两个极端：

- 过度说明：人物、世界、情绪、动机、爽点都靠旁白解释，读者像在看设定报告。
- 过度展示：赶路、吃饭、等待、重复训练、普通寒暄都被写成完整场景，读者觉得水。

本专题不是重复“展示优于说明”，而是提供一个选择器：什么时候必须让读者在现场体验，什么时候只要告诉读者结果，什么时候要用摘要桥，什么时候应该直接删掉。

## 3. 来源与吸收方式

本专题吸收以下外部资料与已有知识库规则：

- Reedsy 关于 Show, Don't Tell 的资料：吸收用动作、感官、语言和情绪表达让读者体验故事，而不是只靠作者解释。
- MasterClass 关于 effective exposition 的资料：吸收有效说明应通过行动、感官、人物语言和情绪表达，而非单独灌输。
- Writer's Digest 关于 Showing vs. Telling 的资料：吸收 showing 和 telling 都是叙事工具，关键在于平衡和使用位置。
- NowNovel 关于 Showing vs Telling 的资料：吸收“telling 可用于开头、转场、时间压缩和不重要过程”的原则。
- Jane Friedman 关于 scene、summary 和 moving between scenes 的资料：吸收场景与摘要共同构成长篇节奏，摘要可以移动时间、压缩背景、连接场景。
- The Center for Fiction 关于 Scenes & Summary 的资料：吸收长篇小说需要用 scene 和 summary 配合处理时间、节奏和结构。
- Story Grid 关于 Show vs Tell 的资料：吸收展示可以通过语气、动作、沉默、潜台词和身体语言产生推断空间。
- 本知识库已有文档：`28-worldbuilding-setting-exposition`、`57-long-timeskip-state-compression`、`71-embodied-sensory-detail-grounding`、`73-scene-value-shift-purpose-gate`、`76-sentence-rhythm-narrative-breath-ai-flavor`、`78-information-density-cognitive-load-term-management`、`91-scene-transition-bridge-continuity-momentum`。

## 4. 四种叙事模式

### 4.1 场景化 Show as Scene

让读者在现场经历事件。通常包含人物目标、阻力、互动、选择、代价和状态变化。

适合：关键冲突、选择、爽点、情绪爆发、关系质变、线索发现、规则验证、失败代价、反派反制。

读者体感：我亲眼看见这件事发生，所以相信它、记得它、愿意为它付费。

### 4.2 摘要化 Summary

概括一段过程，只保留对后文有用的状态差额。

适合：赶路、等待、重复训练、低价值调查、日常消耗、时间流逝、背景铺垫、已经没有悬念的后续处理。

读者体感：我知道中间发生了什么重要变化，但不用陪角色走完每一步。

### 4.3 说明化 Tell/Exposition

直接告诉读者一个必要事实、规则、关系、时间、背景或判断。

适合：读者必须立刻知道但不需要体验的事实；不值得戏剧化的常识；场景开头定位；复杂规则的最小解释；避免误读的必要信息。

读者体感：我获得了必要信息，能继续看正文，不被说明拖住。

### 4.4 省略 Omission

不写。让读者从前后状态自行补足。

适合：无新信息、无选择、无代价、无情绪增量、无回报的重复过程。

读者体感：故事干净，节奏快，没有废动作。

## 5. 决策总表

| 内容类型 | 推荐模式 | 原因 |
| --- | --- | --- |
| 关键选择 | 场景化 | 读者必须看见压力、代价和主动性 |
| 爽点兑现 | 场景化 | 回报需要现场证据和情绪释放 |
| 情绪债兑现 | 场景化 | 不能只说“他很感动/她很痛苦” |
| 关系质变 | 场景化 | 关系变化需要动作、台词、沉默和后果 |
| 重复训练 | 摘要化，偶尔场景化关键节点 | 重复过程易水，突破点才值得展示 |
| 赶路移动 | 摘要化或省略 | 只保留状态差额、危险、关系变化 |
| 复杂规则首次使用 | 场景化加少量说明 | 让规则通过限制和后果被理解 |
| 已知规则复用 | 说明化或省略 | 不要每次都重复规则说明 |
| 世界背景 | 说明化或嵌入冲突 | 背景只有影响选择时才展开 |
| 开场定位 | 短说明 | 让读者快速进入时间、地点、视角和目标 |
| 人物旧伤 | 场景化当前影响，少量摘要过去 | 过去解释不能替代当下选择 |
| 战后清点 | 摘要化加余波证据 | 结果入账即可，除非有新冲突 |
| 悬疑线索发现 | 场景化 | 线索必须有获得过程和误读空间 |
| 低价值寒暄 | 省略或一句带过 | 不承担关系变化就不要展开 |

## 6. 必须场景化的内容

以下内容如果只用摘要或说明，读者会觉得“没写出来”：

- 主角做出会改变后文的选择。
- 角色付出代价、失去东西、背负后果。
- 爽点、打脸、反杀、公开结果落袋。
- 情绪债兑现：道歉、崩溃、承认、放手、保护、决裂。
- 关系位置发生变化：从敌到友、从怀疑到信任、从暧昧到确认、从依附到独立。
- 新规则第一次真正改变行动。
- 关键线索被发现、误读、争夺或公开。
- 反派第一次展示真实威胁。
- 主角能力、职业爽点或金手指第一次证明价值。
- 读者等待很久的答案、回报或名场面。

原则：承诺越大，越要场景化兑现。

## 7. 适合摘要化的内容

摘要不是偷懒，而是节奏控制。

适合摘要的内容：

- 同类动作重复三次以上。
- 已经没有悬念的执行过程。
- 角色从 A 到 B 的移动过程。
- 一段时间里多件小事共同改变状态。
- 训练、调查、筹备、经营、恢复、等待的常规过程。
- 战斗或争执后的后勤处理。
- 角色已经决定，后续只是执行。

有效摘要必须包含状态差额，而不是流水账。

低效摘要：他们调查了很多地方，问了很多人，最后终于找到了线索。

有效摘要：天黑前，三条街的铺子都关了门，只有卖纸钱的老头承认，昨夜子时后还听见过婴儿哭。

有效摘要有三个特点：时间推进、筛选结果、后文抓手。

## 8. 适合说明化的内容

直接告诉读者并不总是坏事。说明化适合以下情况：

- 读者需要最小背景才能理解场景。
- 信息本身不值得戏剧化。
- 解释能避免误读，且不拖慢当前冲突。
- 这是场景开头的时间、地点、身份、规则定位。
- 这是读者已经见过的规则复用。
- 这是连接两场的状态差额。
- 这是专业、制度、地图或机制的必要边界。

说明化的底线：短、准、贴近当前目标。

坏说明：长篇介绍宗门制度的历史沿革。

好说明：外门弟子无权进内库，所以他必须在天亮前偷到执事腰牌。

好说明不是百科，而是限制、目标和代价。

## 9. 应该省略的内容

以下内容默认不写，除非它承担新功能：

- 起床、洗漱、穿衣、出门，除非其中有危险、情绪或线索。
- 从一个地点正常移动到另一个地点，除非路上发生变化。
- 普通吃饭聊天，除非关系、信息、梗点或情绪发生变化。
- 已经知道结果的重复训练和重复失败。
- 没有对手、没有目标、没有新信息的会议。
- 角色互相解释读者已经知道的事实。
- 只是为了显得真实而补的流程细节。

判断标准：删掉后，如果只需要一句“他们到了/准备好了/休息了一夜”就能接上，那就不要展开。

## 10. 叙事变焦：近景、中景、远景

AI 可以把同一段内容切成三种距离。

### 10.1 近景

读者在现场。看动作、听台词、感受停顿和压力。

适合：选择、冲突、情绪、爽点、线索、名场面。

### 10.2 中景

读者知道过程轮廓，但不进入每个动作。

适合：调查推进、训练进度、关系缓慢变化、经营过程。

### 10.3 远景

读者只知道结果和状态差额。

适合：时间跳过、低价值流程、已知结局的后续处理。

变焦原则：关键时刻拉近，重复过程拉远，状态切换用中景。

## 11. Show/Tell 的商业网文口径

商业网文的目标不是把每个情绪都写得文学化，而是让读者快速、稳定、持续地获得体验。

因此：

- 情绪标签可以作为导航，但不能替代证据。
- 规则说明可以存在，但必须服务当前选择。
- 摘要可以很有力，只要它带来状态差额。
- 场景化不是堆细节，而是让选择和后果可见。
- 说明不是罪，错误的说明才是罪：太长、太早、太晚、太抽象、太脱离冲突。

网文读者讨厌的不是“告诉”，而是“告诉了也没有回报”。

## 12. AI 常见坏味道与修复

| 坏味道 | 读者体感 | 修复方式 |
| --- | --- | --- |
| 摘要高潮 | 该爽的地方没爽出来 | 把目标、阻力、动作、见证者、结果写成场景 |
| 展示过量 | 小事拖三页 | 压缩成状态差额，删除无功能细节 |
| 说明书开篇 | 像设定集，不像小说 | 只保留理解当前压力必需的信息 |
| 情绪标签替代证据 | 知道人物难过，但不共情 | 改成动作、停顿、物件、台词和选择 |
| 对话灌设定 | 人物像客服 | 让信息成为争夺、隐瞒、威胁或交易 |
| 摘要无抓手 | 像大纲 | 给一个具体结果、代价、线索或状态变化 |
| 场景无价值转变 | 热闹但无用 | 补选择、代价、信息或关系变化，否则压缩 |
| 说明太晚 | 读者误解前文 | 在需要理解行动前给最小说明 |
| 说明太早 | 读者还不关心 | 等目标、危险或问题出现后再解释 |
| show 变成表演 | 到处握拳、发抖、眼神一暗 | 用角色特有动作和真实选择替代通用身体词 |

## 13. 类型文专项用法

### 13.1 升级流

必须场景化：第一次能力证明、突破代价、公开打脸、资源落袋。

适合摘要：重复刷怪、常规修炼、后勤清点。

说明化重点：能力限制、资源消耗、规则边界。

### 13.2 系统文

必须场景化：系统规则第一次改变选择、奖励第一次改变局面、惩罚第一次落地。

适合摘要：日常任务重复执行、已知面板刷新。

说明化重点：只解释本章用到的系统规则。

### 13.3 甜宠与情感文

必须场景化：关系误会、靠近/退缩、承认在乎、边界变化。

适合摘要：日常相处的长时间累积。

说明化重点：少量交代关系背景，但情绪变化必须落在互动里。

### 13.4 悬疑探案

必须场景化：线索发现、证词矛盾、误导形成、假设被推翻。

适合摘要：排除大量无效地点和重复问询。

说明化重点：时间线、空间关系、已知事实边界。

### 13.5 权谋商战

必须场景化：谈判、公开对峙、利益交换、背叛、证据亮相。

适合摘要：流程性会议、文件流转、舆论发酵过程。

说明化重点：规则、利益结构、职位权限，但必须绑定当前争点。

### 13.6 喜剧

必须场景化：包袱触发、错位升级、角色反应。

适合摘要：笑点之间的普通移动和准备。

说明化重点：解释太多会杀死笑点，喜剧优先让结果自己荒唐。

## 14. AI 起草流程

起草前，把每段内容先分类。

```text
本段内容：
本段功能：选择 / 代价 / 爽点 / 情绪 / 关系 / 信息 / 过渡 / 背景 / 重复过程 / 定位
读者需要：现场体验 / 知道结果 / 知道必要事实 / 不需要知道
是否有不可替代的选择：有 / 无
是否改变后文状态：人物 / 关系 / 资源 / 信息 / 风险 / 情绪 / 无
推荐模式：场景化 / 摘要化 / 说明化 / 省略
如果场景化：目标、阻力、转折、选择、后果是什么？
如果摘要化：状态差额和后文抓手是什么？
如果说明化：最小必要事实是什么？能否嵌入目标、限制或代价？
如果省略：前后两段如何自然衔接？
风险检查：解释过满 / 展示过量 / 摘要高潮 / 说明书化 / 无状态变化
```

## 15. 审稿检查清单

审稿时逐条检查：

- 本段是让读者体验，还是只让读者知道？
- 本段是否包含关键选择、代价、爽点、情绪债或关系质变？如果有，是否场景化？
- 本段是否只是移动、等待、准备、重复执行？如果是，是否压缩？
- 摘要是否包含状态差额，而不是流水账？
- 说明是否短、准，并绑定当前目标、限制或代价？
- 是否有高潮、反转、告白、失败、奖励被一句话带过？
- 是否有低价值过程被写成完整场景？
- 是否连续多屏都是解释，没有动作、台词、选择或后果？
- 是否为了 show 而堆通用身体反应，反而显得套路？
- 读者已经能从场景看懂的内容，旁白是否又解释了一遍？
- 如果删掉这一段，后文是否真的受影响？

通过标准：重要内容有现场感，低价值过程不拖沓，必要说明不喧宾夺主，整体节奏既清楚又有冲击。

## 16. 最小可执行规则

AI 写章节时至少遵守以下规则：

- 关键选择、代价、爽点、情绪债和关系质变必须优先场景化。
- 重复过程、低价值移动和已知执行结果应摘要化或省略。
- 说明只保留当前场景理解和下一步行动必需的信息。
- 摘要必须给状态差额：人物、关系、资源、信息、风险或情绪至少变一项。
- 不要用情绪标签替代动作、台词、沉默、物件和选择。
- 不要为了 show 把小事写成长镜头；展示必须服务回报。
- 读者能从场景看懂的内容，不要再解释一遍。
- 设定、规则、背景进入正文时，必须绑定目标、限制、代价或后果。
- 场景化、摘要化、说明化和省略都是工具，选择标准是读者体验和商业回报。
