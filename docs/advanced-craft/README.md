## 先读结论

如果某类内容持续膨胀，应拆到更明确的目录；本目录不是长期堆放杂项的仓库。

## 推荐链路

### 开篇 30 章工程

```text
../workflows/45 第一章任务单 -> 46-52 前 30 章专项 -> ../storycraft/22 开篇留存 -> ../core-writing/07 连续性台账
```

适合：开书、前三章、前十章、前 30 章追读不稳时。先判断承诺链和回报链，不要只改单章句子。

### 卷级推进和中段防水

```text
53 第一卷结构 -> 54 支线管理 -> 55 卷末结算 -> 56 卷间启动 -> ../storycraft/23 中段卷结构
```

适合：30 章后支线散、重复推进、卷末没满足感或新卷像重开。

### 降 AI 味和场景功能修复

```text
73 场景价值 -> 70 行动反应 -> 71 具身细节 -> 75 对白潜台词 -> 76 句子节奏
```

适合：正文像提纲展开、台词不自然、成片短句、漂亮但没功能、人物说话没压力。

### 读者反馈定位

```text
95 反馈症状路由 -> ../feedback-revision/43 优先级队列 -> 命中专题 -> ../workflows/58 修订闭环
```

适合：评论、试读、编辑意见互相冲突，或者用户只给出“水、假、没意思、看不懂”等症状词。

## 维护规则

- 本目录只保留跨目录、高颗粒度或暂时过渡的高级专题。
- 如果某组文件已经形成稳定大类，应迁移到 `core-writing/`、`storycraft/`、`market-serialization/`、`feedback-revision/` 或 `scene-craft/`，并保留索引跳转。
- 新增文件必须同时更新本 README、`../00-index.md` 或相关目录 README 中至少一个入口。
- 不要把普通写作原则继续追加到本目录；能落到已有专题的，优先并入已有专题。
