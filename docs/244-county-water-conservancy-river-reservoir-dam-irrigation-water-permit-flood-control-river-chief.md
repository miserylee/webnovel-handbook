# 244 县域水利、水资源、河湖长制、水库大坝、河道堤防、灌区取水与防洪抗旱工程专项

> 状态：已确认  
> 用途：供 AI agent 在写县域水利局、河湖长、河道水库、堤防水闸泵站、灌区渠道、农田水利、取水许可、地下水、抗旱调水、防洪工程、水土保持、河道采砂、项目占河占岸、群众用水纠纷和水利工程运维时使用。  
> 核心目标：把“水”从背景资源升级为水权、水量、水位、工程、调度、许可、巡查、证据、责任和公共利益的制度链，避免把水库开闸、河道施工、灌溉分水和取水用水写成领导一句话或村里随便决定。

---

## 1. 为什么需要这一篇

县域现实题材里，水不是单纯自然景色。它决定村庄能不能种田，县城会不会内涝，工厂能不能投产，学校有没有安全饮水，景区能不能开漂流，水库下游村民能不能睡安稳觉。

水利题材的戏剧价值在于：

- 空间压力：上游下游、左岸右岸、库区移民、灌区末端、低洼圩区、河道弯道和险工险段。
- 时间压力：汛前检查、抢修窗口、抗旱轮灌、洪峰过境、错峰调度、施工度汛。
- 利益压力：生活饮水、农业灌溉、工业取水、生态流量、景观用水、养殖用水、发电和防洪库容互相挤压。
- 责任压力：水利部门、河湖长、乡镇、村委、工程管理单位、用水户、企业和施工方都可能有职责。
- 证据压力：水位、雨量、闸门启闭、取水计量、巡查记录、工程验收、河长巡河、投诉工单、行政处罚都能成为线索。

已有文档分工：

- 第 207 篇负责河道污染、生态环境投诉和环保执法。
- 第 208 篇负责城乡供水、水电气通信和民生服务连续性。
- 第 237 篇负责暴雨洪涝、干旱等自然灾害应急和救助。
- 本篇负责水利和水资源本身：河湖库渠闸泵、取水许可、用水分配、防洪工程、河湖长制、灌区运维和水利证据。

## 2. 资料来源与写作边界

本专题根据以下官方资料整理：

- 《中华人民共和国水法》：支撑水资源所有权、开发利用、节约保护、水资源配置、取水许可、水工程、水事纠纷和法律责任。
- 《中华人民共和国防洪法》：支撑防洪规划、防洪工程、河道湖泊治理、防汛抗洪、蓄滞洪区和保障措施。
- 《中华人民共和国水土保持法》：支撑坡地开发、工程建设、取土采石、弃渣、植被破坏、水土流失预防治理。
- 《中华人民共和国河道管理条例》：支撑河道管理范围、堤防护岸、河道清障、建设项目、采砂、阻水障碍和行洪安全。
- 《水库大坝安全管理条例》：支撑水库大坝安全管理、运行调度、检查维修、险情报告和责任边界。
- 《取水许可和水资源费征收管理条例》及取水许可公开资料：支撑企业、农业、工程、地下水等取水许可、计量、审批和监督。
- 《地下水管理条例》：支撑地下水超采、取用地下水、地下水保护和管控区剧情。
- 《农田水利条例》：支撑灌区、农田水利工程、农民用水合作组织、工程运行维护和农业用水。
- 《关于全面推行河长制的意见》《关于在湖泊实施湖长制的指导意见》：支撑河湖长巡河、问题交办、跨部门协同和长效治理。

安全边界：

- 不提供破坏堤防、水闸、水库大坝、泵站、供水设施、监测设施的方法。
- 不提供私自开闸放水、盗水、非法取水、非法采砂、规避取水计量或伪造水利审批材料的方法。
- 不提供危险涉水、私闯泄洪区、围观洪水、下河救人、违规施工度汛等可模仿危险行为教程。
- 不把水库调度、防洪抗旱和灌溉分水写成个人凭胆量或关系拍板；必须写专业研判和责任链。
- 涉及污染、水质、供水、灾害救助、事故伤亡时，分别联动第 207、208、237、236 等专题。

## 3. 一句话总链路

> 写水利剧情，先判断对象是水资源、河道、湖泊、水库、堤防、闸泵、灌区、地下水、取水口、排水沟还是水土保持工程，再核权属和管理范围、许可审批、工程运维、监测调度、巡查记录和用水优先级；冲突最后应落到水量水位、工程安全、许可责任、上下游利益、公共安全、生态保护和长期管护上。

AI 写作最少问 10 个问题：

1. 这件事发生在河道、水库、灌区、泵站、水闸、堤防、地下水井、农田沟渠，还是供水管网？
2. 水的用途是什么：生活饮水、灌溉、工业、养殖、景观、生态、发电、防洪，还是排涝？
3. 管理主体是谁：水利局、河湖长办、工程管理单位、乡镇、村委、灌区管理处、用水合作组织，还是企业？
4. 是否需要取水许可、建设项目涉河审批、使用水工程、河道采砂许可、水土保持方案或防洪影响论证？
5. 有没有水位、流量、雨量、闸门启闭、取水计量、巡查、调度和工单记录？
6. 上下游、左右岸、干渠支渠、库区下游、城乡之间有什么利益冲突？
7. 是否涉及汛期、枯水期、抗旱期、施工期、灌溉期、鱼汛、旅游旺季等时间压力？
8. 是否影响弱势对象：老人饮水、学校供水、末端农田、低洼村庄、库区移民、残障者出行？
9. 争议出口是什么：协商分水、行政检查、责令整改、工程抢修、补偿救助、行政处罚、公益诉讼，还是事故调查？
10. 有没有把私开水闸、偷采河砂、偷接水、破坏堤防、危险围观洪水写成爽点？如果有，必须改成风险或违法线索。

## 4. 水利对象分层

| 对象 | 关键制度点 | 适合剧情 | 常见误写 |
|---|---|---|---|
| 水资源 | 取水许可、用水总量、节约用水、地下水保护 | 工厂投产、灌溉争水、抗旱调度 | 井一打就能用，河水随便抽 |
| 河道湖泊 | 管理范围、行洪安全、岸线、清障、采砂、涉河建设 | 修桥、建栈道、违建、采砂、河长巡河 | 河滩荒着没人管 |
| 水库大坝 | 安全责任、调度、汛限水位、巡查维修、险情报告 | 水库泄洪、养鱼、旅游、下游转移 | 水库管理员随手开闸 |
| 堤防护岸 | 防洪标准、险工险段、巡堤查险、抢险 | 管涌、滑坡、堤脚取土、违建 | 堤坝只是拍照背景 |
| 水闸泵站 | 启闭调度、排涝灌溉、设备运维、值班 | 城区内涝、农田干旱、停电影响 | 开关闸像开家里水龙头 |
| 灌区渠道 | 干渠支渠斗渠、轮灌、维护、末端用水 | 上游截水、末端没水、清淤分摊 | 村里吵完就能改水路 |
| 农田水利 | 工程建设、运行维护、受益户分摊 | 合作社、粮食生产、项目验收 | 修完渠就永远好用 |
| 水土保持 | 坡地、弃渣、施工扰动、植被恢复 | 修路、采石、果园开发、矿山 | 下雨冲泥只是自然现象 |
| 河湖长制 | 巡河、发现问题、交办督办、跨部门协同 | 河道垃圾、岸线占用、群众投诉 | 河长牌只是摆设 |
| 水文监测 | 水位、流量、雨量、泥沙、资料证明 | 洪水、工期、理赔、事故争议 | 主角凭肉眼判断洪峰 |

## 6. 高频场景写法

### 6.1 干旱时的灌溉分水

低质写法：村干部一拍桌子，上游村把水让给下游村。

高质写法：

- 先确认水源：水库、河道、机井、泵站、灌区渠道还是临时送水。
- 再确认分水规则：轮灌计划、用水指标、渠道能力、作物生长期、末端损耗、用水户协商。
- 上游不一定是恶人，下游也不一定只会哭；上游可能刚插秧，下游可能果树要保命，泵站可能没电，渠道可能漏水。
- 冲突的解法可以是轮灌、错峰、临时泵站、渠道抢修、限额用水、抗旱补助、保险和长期工程改造。

可写桥段：

- 上游村夜里私自堵渠，末端合作社一夜没水。主角从水尺、泵站运行记录和巡渠照片查清事实。
- 灌区管理人员被两边村民围住，真正难点是水量不够，谁先保苗、谁减产、谁补偿。

### 6.2 水库汛限水位和下游恐慌

低质写法：水库管理员为救下游，自己决定开闸泄洪。

高质写法：

- 水库调度应看雨情、水情、库水位、汛限水位、入库流量、下游河道承载、调度指令和预案。
- 开闸泄洪不是个人英雄动作，涉及会商、指令、预警、下游通知和人员转移。
- 下游群众的恐惧是真实的：鱼塘、低洼房、老人转移、牲畜、车辆和农田都在水位变化里。
- 争议证据包括水位记录、调度令、闸门启闭记录、通知记录、监控和转移台账。

可写桥段：

- 暴雨夜，水库水位逼近汛限，管理员连续上报。村民群里传“马上垮坝”，乡镇干部一边辟谣一边组织下游低洼户预警转移。
- 养鱼承包户反对提前泄洪，水利人员解释防洪库容优先于个人收益。

### 6.3 河道里建了“临时”停车场

低质写法：河滩空着浪费，村里铺石子当停车场。

高质写法：

- 河道管理范围内的建设、堆放、种植、采砂、便道、停车场、民宿露营设施，都要看是否影响行洪和河势稳定。
- “平时没水”不代表可以占用。汛期行洪空间被占，后果可能波及上下游。
- 河湖长巡河、群众举报、无人机影像、水利执法和整改复查可以构成完整链条。

可写桥段：

- 景区为了节假日客流，把河滩当临时停车场。暴雨预警后车辆来不及撤，河水上涨，游客投诉、河长办和文旅部门同时被推上前台。

### 6.4 企业取水和地下水超采

低质写法：工厂在厂区打井，解决生产用水。

高质写法：

- 企业取用水资源通常涉及取水许可、取水计量、用途、取水量、节水措施和水资源费/税费等问题。
- 地下水还涉及超采区、管控区、取水井、回补和替代水源。
- 冲突常见于工业园区招商、农户井水下降、村民怀疑工厂抢水、企业成本和许可审批之间。
- 证据包括取水许可证、取水计量、井位、用水台账、地下水监测和群众投诉记录。

可写桥段：

- 工厂投产后周边农户机井水位下降，企业说自己手续齐全。主角不靠吵架，而是查取水许可、实际计量、监测井数据和区域地下水管控要求。

### 6.5 河道采砂和夜间运输

低质写法：反派开船挖砂，主角当场跳船打人。

高质写法：

- 河道采砂涉及许可、禁采区、禁采期、采砂总量、船舶机具、堤防安全和河势稳定。
- 非法采砂常与夜间运输、洗砂场、砂石票据、工程项目用料和保护伞传闻相连。
- 证据链可以是无人机影像、船舶轨迹、砂场堆料、运输单据、过磅记录、河床变化和群众举报。
- 处置应由水利、公安、交通、自然资源、生态环境等依法协同，不写私斗教程。

可写桥段：

- 村民夜里听到河道机械声，第二天发现堤脚被掏。河长巡查记录显示此前已有苗头但未整改，案件牵出砂石生意和工程用料链。

### 6.6 农田水利工程修完就坏

低质写法：上级拨款修渠，主角一来就发现贪污，抓人后工程好了。

高质写法：

- 农田水利工程不只看建设，还看设计、招投标、施工质量、验收、移交、管护主体、维修资金和受益户参与。
- 渠道修好后没人清淤、泵站没电费、闸门没人管，都会让工程“账面完成、实际不好用”。
- 审计、财政、农业农村、水利、村集体和用水户都可能进入剧情。

可写桥段：

- 村里新修的渠第一年能用，第二年淤满。项目验收照片很漂亮，农户却说末端没水。主角查验收单、管护协议、清淤记录和受益户缴费，发现问题不是单一贪腐，而是建管脱节。

### 6.7 水土保持和泥水下山

低质写法：雨大，山坡泥水冲进村，是天灾。

高质写法：

- 修路、采石、果园开垦、矿山、弃渣场、民宿施工等扰动地表，可能需要水土保持方案和防治措施。
- 暴雨只是触发条件，前期弃渣、裸露边坡、排水沟缺失、植被破坏才是放大器。
- 证据包括施工范围、弃渣位置、排水设施、雨量资料、水土保持方案、监管检查和现场照片。

可写桥段：

- 山上民宿施工后，暴雨把泥水冲进村小学操场。老板说是百年一遇，水利和自然资源部门查出弃渣场没有按要求防护。

## 7. 证据链与材料清单

| 材料 | 能证明什么 | 剧情用途 |
|---|---|---|
| 水位、雨量、流量资料 | 水情变化和调度依据 | 洪水、干旱、工程争议 |
| 水库运行日志 | 库水位、巡查、闸门、险情 | 水库责任和调度链 |
| 调度令/会商纪要 | 谁决定何时调水泄洪 | 避免领导随口开闸 |
| 闸门启闭记录 | 实际开关时间和幅度 | 泄洪、灌溉、排涝争议 |
| 取水许可证 | 取水主体、地点、用途、数量 | 企业和农户争水 |
| 取水计量和用水台账 | 实际取水量 | 超许可取水、节水整改 |
| 河道管理范围图 | 是否占河、阻水、影响行洪 | 违建和清障依据 |
| 涉河建设审批材料 | 桥梁、栈道、管线、便道是否合法 | 项目合规与事故责任 |
| 河湖长巡查记录 | 是否发现并交办问题 | 墙牌是否变成行动 |
| 投诉举报和 12345 工单 | 群众诉求进入制度渠道 | 分流、督办、回访 |
| 工程设计、验收和移交资料 | 水利工程是否建成并交管 | 农田水利、泵站、渠道质量 |
| 维修养护记录 | 工程是否长期可用 | “修完就坏”的证据 |
| 水土保持方案和验收资料 | 工程扰动是否有防治措施 | 泥水下山、弃渣冲沟 |
| 采砂许可和运输票据 | 砂石来源是否合法 | 非法采砂和工程材料链 |
| 监控、无人机、卫星和现场照片 | 现场状态和变化 | 线索，但需结合专业资料 |
| 处罚、整改和复查文书 | 责任后果 | 让治理闭环 |

写法提示：水利剧情特别适合用“数值”增加压迫感，但不要堆术语。比如水位差几厘米、泵站停几小时、末端渠少一半水、闸门迟开二十分钟，都能让冲突有抓手。

## 10. 常见 AI 误区

### 10.1 把水库开闸写成个人决定

水库调度涉及雨水情、库水位、预案、会商、调度令、下游通知和转移。不能让管理员或县长凭情绪随手开闸。

### 10.2 把河滩当闲置土地

河道管理范围、行洪空间和岸线都有公共安全功能。平时没水不代表能建停车场、露营地、仓库和砂场。

### 10.3 把取水写成谁有泵谁有水

取水涉及许可、计量、用途、总量控制和地下水保护。企业打井、合作社抽河水、景区造湖，都不能只靠设备解决。

### 10.4 把河湖长制写成墙上牌子

河湖长制的叙事价值在巡查、发现、交办、督办、整改和跨部门协同。只写牌子会很空。

### 10.5 把水利工程写成一次性建设

泵站、渠道、水闸、堤防和小型水库最怕“建成后没人管”。运维、维修、清淤、值班、资金和责任才是长线戏。

### 10.6 把非法采砂写成普通生意

河道采砂影响河势、堤防、桥梁和生态，背后常有运输、工程用料、票据和保护伞线索。不能把偷采写成赚钱智慧。

### 10.7 忘记上下游关系

水利冲突很少只发生在一个点。上游多取一点，下游就少一点；水库多蓄一点，下游风险和上游收益都变。上下游是天然剧情引擎。

## 11. 审稿检查表

看到水利、水库、河道、堤防、水闸、泵站、灌区、取水、抗旱、防洪、排涝、河湖长、采砂和水土保持时，至少检查：

- 是否明确水利对象和空间位置：河道、水库、堤防、泵站、灌区、取水口、地下水井、低洼区？
- 是否明确用途：生活饮水、农业灌溉、工业取水、养殖、生态、景观、防洪、排涝？
- 是否明确管理主体：水利局、河湖长办、水库管理单位、灌区管理处、乡镇、村委、企业？
- 是否处理取水许可、涉河建设审批、河道管理范围、水土保持、防洪影响或工程验收？
- 是否写出水位、雨量、流量、闸门启闭、取水计量、巡查记录、调度令等证据？
- 是否让预警、调度、巡查和整改产生具体行动？
- 是否把私开水闸、非法取水、非法采砂、破坏堤防、伪造许可、危险涉水写成风险和后果，而不是爽点？
- 如果涉及污染、供水、灾害、事故、工程审计或刑事案件，是否联动对应专题？

## 12. 可复用桥段

### 桥段 A：末端渠没水

连续干旱，灌区上游稻田刚灌完，末端果园却开始落果。合作社要求马上开闸，上游村说自己也没多拿。主角不靠吵架，而是查轮灌表、闸门启闭记录、水尺和渠道漏损，发现中间一段年久失修漏水严重，真正矛盾从“村斗村”变成“工程运维和补偿分配”。

用途：农业、合作社、基层治理、抗旱。

### 桥段 B：水库夜里涨水

暴雨夜，小型水库水位快速上涨。管理员连续巡坝记录异常，村民群里却传“要垮坝”。乡镇一边根据调度指令通知下游低洼户转移，一边辟谣。最终险情解除，但调查发现平时维修资金迟迟不到位，风险并非只来自一场雨。

用途：防汛、基层干部群像、水利工程运维。

### 桥段 C：河滩停车场

节假日景区停车位不够，运营方把河滩临时铺成停车场。上游暴雨后水位上涨，游客车辆被困。河长巡查记录显示早已要求整改，但景区为了客流一拖再拖。文旅、水利、应急和市场投诉同时入场。

用途：文旅开发、河道管理、公共安全。

### 桥段 D：工厂的井

招商项目投产后，周边农户发现机井水位下降。企业拿出营业执照说自己合法，村民不买账。主角追到取水许可证、取水计量、地下水监测和区域水资源论证，发现问题不是有没有执照，而是许可水量、实际取水和地下水管控不一致。

用途：招商、营商环境、资源冲突。

### 桥段 E：河长牌下的垃圾

村民拍视频嘲讽河长牌下漂满垃圾。低质写法是河长赶来捡垃圾。高质写法是河湖长办交办问题，乡镇查明上游市场雨天冲入、保洁公司漏巡、拦污设施损坏，整改从清漂、截污、巡查和合同考核一起推进。

用途：基层治理、公共舆论、长效机制。
