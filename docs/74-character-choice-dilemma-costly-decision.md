# 74. 角色选择、两难压力与昂贵决定专项

状态：已确认

适用对象：AI 写作 agent、审稿 agent、人物 agent、大纲 agent、章节流水线 agent、情绪线 agent、爽点设计 agent。

适用任务：章节起草、场景设计、主角主动性修复、人物弧线推进、关系线推进、爽点兑现、失败与代价设计、反转设计、低强度章节增量、读者反馈为“主角没用”“被剧情推着走”“选择太假”“人物不活”“剧情生硬”的稿件修订。

## 1. 本文解决什么问题

AI 写小说时，经常能安排事件，却不会让人物真正选择。结果就是剧情看似推进，人物却像被作者推着走。

典型问题：

- 主角只是遇到事件、接受任务、获得奖励，没有主动决定。
- 选择看似困难，但其实只有一个明显正确答案。
- 角色做了选择，却没有付出代价，也没有后续后果。
- 两难是作者硬造的，角色明明可以沟通、求助或绕开，却被迫装傻。
- 女频情绪线里，一方持续被推着走，失去选择权和边界。
- 爽点里主角只是开挂，没在压力下暴露价值排序。
- 反派或配角为了剧情需要突然背叛、原谅、送资源，没有自身利益和选择链。
- 章节结尾有“他决定了”，但下一章行动并没有因此改变。

本文把 crisis choice、best bad choice、irreconcilable goods、character agency、scene-sequel decision、character motivation 等资料，转化为中文网文 AI agent 可执行的“昂贵选择”设计规则。

## 2. 核心原则：人物靠选择活起来，剧情靠后果长出来

最小链条：

```text
目标/欲望 -> 外部压力 -> 互斥选项 -> 每项代价 -> 角色选择 -> 可见行动 -> 即时后果 -> 后续债务
```

有效选择必须满足：

```text
他不是随便选，而是在压力下暴露自己更看重什么。
```

读者追人物，不是因为人物拥有完美答案，而是因为人物在不能全都要的时候，仍然做出符合自身欲望、缺陷、底线和成长方向的选择。

## 3. 有意义选择的六个条件

### 3.1 选择者有主体性

角色必须能影响结果。即使环境强压、资源不足、信息不全，他也要有一个属于自己的动作。

不合格：

```text
系统替他选了任务，贵人替他解围，反派自己犯蠢，主角只是旁观。
```

合格方向：

```text
系统只给限制信息，贵人只提供窗口，反派有自保动作，主角必须判断、试错、付代价并亲自行动。
```

### 3.2 选项互斥

真正的两难不是“好选项”和“坏选项”，而是不能同时拥有。

常见互斥：

- 现在赢，还是保留底牌。
- 救一个人，还是保住任务。
- 说出真相，还是保护关系。
- 维护规则，还是救下例外。
- 公开反击，还是继续潜伏。
- 保住尊严，还是拿到资源。
- 相信旧经验，还是承认眼前的人变了。

如果两个选项可以轻松兼得，选择就没有压力。

### 3.3 每个选项都有收益和代价

有意义选择不是“选对就全赚，选错就全亏”。更好的选择是两边都有诱惑，也两边都要付账。

```text
选 A：拿到短期收益，但暴露底牌。
选 B：保住底牌，但失去当下机会。
```

### 3.4 选择发生在不完全信息下

人物不应像读完大纲一样做完美判断。信息不足、时间压力、情绪盲区和利益冲突，会让选择更像真人。

但不完全信息不是降智。角色必须基于当时能获得的信息，做出可以理解的判断。

### 3.5 选择暴露价值排序

选择的核心不是答案，而是排序：角色更看重安全、尊严、亲情、权力、真相、自由、承诺、成长，还是胜利。

同一个事件，不同人物应选得不一样：

- 权谋角色可能先保棋局。
- 热血主角可能先救眼前人。
- 创伤角色可能先自保。
- 职业角色可能先守流程。
- 恋爱脑角色可能先保关系，但后文要为代价付账。

### 3.6 选择必须改变后文

选择如果没有后果，就只是姿态。

有效选择至少改变一种状态：

- 资源减少或增加。
- 关系亲近、破裂、欠债或误会加深。
- 对手获得信息或产生误判。
- 世界规则开始针对主角。
- 主角暴露底牌、留下伤、失去机会。
- 下一章目标被迫改变。

## 4. 两大核心两难类型

### 4.1 最好的坏选择

角色必须在两个负面选项之间选一个。

```text
救人会暴露身份。
不救人会保住潜伏，但失去自我认同。
```

适用场景：主角失败、逃亡、权谋、悬疑、战斗、付费节点、高压感情戏。

作用：制造焦虑、代价感和读者替角色捏一把汗的体验。

### 4.2 不可兼得的好选择

角色必须在两个正向价值之间放弃一个。

```text
留下能保护爱人。
离开能完成更大的责任。
```

适用场景：人物弧线、女频情绪线、家国冲突、师徒/亲情、团队关系、终局选择。

作用：暴露人物真正的价值排序，让选择成为人物高光或关系转折。

## 5. 网文常用选择轴

| 选择轴 | 可用问题 | 常见风险 |
|---|---|---|
| 胜利 vs 底牌 | 现在赢，还是保留后手？ | 主角无脑全开，后续失去压力 |
| 尊严 vs 资源 | 要面子，还是拿到活路？ | 为虐而羞辱，缺少反弹抓手 |
| 安全 vs 真相 | 退出保命，还是继续查？ | 悬疑靠作死推进 |
| 关系 vs 目标 | 保住关系，还是完成任务？ | 感情线工具化或恋爱脑降智 |
| 规则 vs 例外 | 守制度，还是救具体的人？ | 价值口号化，没有现实代价 |
| 公开 vs 潜伏 | 当众反击，还是继续隐藏？ | 反转空降，前文无痕迹 |
| 短期爽 vs 长期局 | 立刻打脸，还是放长线？ | 一直忍导致憋屈，立刻赢导致变轻 |
| 自保 vs 承担 | 先保护自己，还是背起责任？ | 道德绑架主角或牺牲无回报 |
| 旧认知 vs 新证据 | 相信过去，还是承认变化？ | 误会硬拖、角色装瞎 |
| 个人愿望 vs 类型承诺 | 人物想逃，故事要求推进 | 主角长期抗拒主线，读者失去追随理由 |

## 6. 选择压力的制造工具

### 6.1 时间压力

倒计时、截止时间、对手先手、伤势恶化、消息即将公开，都会让选择不能无限拖。

### 6.2 资源限制

钱、灵力、名额、证据、人情、冷却次数、交通工具、队友体力都可以让角色不能全都要。

### 6.3 信息不完全

角色知道一部分真相，但不知道全貌。选择应基于当前信息成立，后文可以让其付出误判代价。

### 6.4 关系牵扯

角色的选择会伤害、保护、背叛、绑定或欠下某个人。关系代价比抽象代价更容易让读者有感。

### 6.5 见证者压力

公开场合、直播、朝堂、宗门、公司会议、家族饭局、评论区风向，会让选择同时改变名声和站队。

### 6.6 规则限制

系统冷却、宗门规矩、合同条款、法律程序、比赛规则、世界禁忌，可以逼出“想做但不能直接做”的选择。

### 6.7 情绪盲区

创伤、羞耻、旧爱、亏欠、嫉妒、执念、恐惧，会让角色选择不完全理性，但必须可理解。

## 7. 选择规模分级

| 等级 | 类型 | 用途 | 例子 |
|---|---|---|---|
| C0 微选择 | 小动作、小回应、小隐瞒 | 显示人物性格 | 要不要接那句话 |
| C1 战术选择 | 当前场景的方法选择 | 推动节拍 | 是硬刚、绕开、试探还是示弱 |
| C2 关系选择 | 影响信任、边界、亲疏 | 推进情绪线 | 是说实话，还是继续保护对方 |
| C3 阶段选择 | 改变接下来数章目标 | 推进小单元 | 是追线索，还是先救人 |
| C4 不可逆选择 | 带来长期代价 | 卷中/卷末转折 | 暴露身份、背叛组织、毁掉资源 |
| C5 身份选择 | 决定人物成为什么人 | 高潮/终局 | 是做旧秩序的赢家，还是打碎旧秩序 |

不是每章都需要 C4/C5，但每章至少应有 C1/C2 级别的主动选择，否则主角容易像被事件推着走。

## 8. 不同题材中的选择设计

### 8.1 男频升级爽文

高价值选择：现在打脸还是保留底牌；抢资源还是换长期资格；救人还是抢机缘；服从规则还是规则反杀。

注意：主角可以强，但必须有选择成本。强不是没有代价，而是能承担别人承担不了的代价。

### 8.2 都市、职场和经营

高价值选择：保项目还是保人；走流程还是越级；公开证据还是继续钓鱼；短期利润还是长期信誉。

注意：现实题材不能让主角靠一句话解决制度问题。选择要进入流程、证据、人情和后续风险。

### 8.3 女频情绪线和感情线

高价值选择：原谅还是设边界；留下还是离开；相信对方还是保护自己；公开关系还是保住事业。

注意：情绪线的重点不是“谁更爱谁”，而是谁在关系里保有选择权，谁为旧伤付账，谁真正改变行动。

### 8.4 悬疑、探案、规则怪谈

高价值选择：公开线索还是藏线索；追真相还是保安全；相信证据还是相信人；破坏规则还是试探规则。

注意：选择不能建立在读者没有机会知道的隐藏规则上。角色可误判，作者不能耍赖。

### 8.5 喜剧和轻松文

高价值选择：保面子还是解决问题；装傻还是承认；小利还是大局；嘴硬还是补救。

注意：喜剧选择可以小，但不能让人物持续降智。笑点最好来自人物价值排序错位。

### 8.6 群像和团队文

高价值选择：谁代表团队做决定；少数服从多数还是保护少数；队长承担还是分摊代价；个人目标是否让位于团队目标。

注意：群像选择要有分歧过程，不能所有人自动同意主角。

## 9. 假选择识别与修复

| 假选择表现 | 读者体感 | 修复方法 |
|---|---|---|
| 只有一个明显正确答案 | 没压力 | 让正确答案付出代价，错误答案有诱惑 |
| 选择后没有后果 | 姿态化 | 补资源损失、关系变化、对手反应或下一章压力 |
| 别人替主角选择 | 主角没用 | 让外力只提供窗口，关键动作由主角完成 |
| 选择被系统奖励抹平 | 金手指廉价 | 让系统奖励带来新限制、新风险或暴露 |
| 两难靠误会硬拖 | 角色降智 | 补信息污染、情绪盲区、说清代价或关系风险 |
| 选择只靠口号 | 假大空 | 把价值冲突落到具体人、资源、伤害和后果 |
| 选择与人物不符 | 剧情生硬 | 回到欲望、恐惧、利益和旧伤重建动机 |
| 选择太小却大写特写 | 矫情 | 降级为微选择，压缩处理 |
| 选择太大却轻轻带过 | 没冲击 | 展开反应、代价、见证者和后文余波 |
| 选择后清零 | 长篇无连续性 | 写入台账，让代价至少影响一次后续选择 |

## 10. AI 起草流程

起草前：

1. 写出本场角色目标。
2. 写出阻力如何让目标无法直接完成。
3. 设计两个互斥选项。
4. 为每个选项写收益和代价。
5. 判断这次选择暴露角色什么价值排序。
6. 选择一个可见动作承载决定。
7. 写出即时后果和后续债务。

起草中：

1. 不要用旁白说“他终于做出了艰难决定”，要写他做了什么。
2. 不要让角色在没有压力时长篇思考选择。
3. 不要让外力直接替选择买单。
4. 不要让选择和人物旧设定脱节。
5. 不要把所有代价都推迟到很久以后，至少给一个即时反馈。

起草后：

1. 检查选择是否改变了目标、关系、信息、资源、风险或情绪状态。
2. 检查选择代价是否进入台账或下一章任务。
3. 检查读者能否复述“他为什么这么选”。
4. 检查这次选择是否让人物更清楚，而不是更模板。

## 11. 审稿检查清单

逐场检查：

- 本场是谁在选择？
- 他想要什么？
- 他为什么不能直接得到？
- 选项是否互斥？
- 每个选项是否都有收益和代价？
- 选择是否发生在压力下，而不是慢悠悠做题？
- 选择是否符合角色当时的信息、利益、情绪和旧伤？
- 选择是否暴露了角色价值排序？
- 选择是否被可见动作承载？
- 选择之后是否有即时反馈？
- 选择是否改变下一场目标或关系？
- 代价是否进入后文，而不是本章清零？

## 13. 最小执行规则

1. 关键场景必须有角色主动选择，不要只让事件发生。
2. 有意义选择必须有互斥选项。
3. 每个选项都要有收益和代价。
4. 选择要发生在压力、限制或不完全信息下。
5. 选择必须暴露角色价值排序。
6. 选择要用可见行动承载，不只用旁白宣布。
7. 选择之后必须有即时反馈和后续债务。
8. 女频情绪线必须保护关键人物的选择权和边界。
9. 爽点选择必须让主角承担成本，而不是免费开挂。
10. 选择代价不能清零，应进入台账或至少影响一次后续选择。

## 14. 来源与吸收方式

外部来源主要提供四类启发：

- Story Grid 关于 crisis、climax、resolution 和 scenes 的资料：场景中的危机是互斥选择，高潮是角色把选择落实为行动，结局展示选择后果。
- Helping Writers Become Authors 关于 character agency 与 scene-sequel decision 的资料：冲突引出选择，选择带来后果；sequel 中的 decision 应从 dilemma 中长出并导向下一场目标。
- Reedsy 关于 character motivation 与 conflict 的资料：人物动机是行动原因，冲突要阻碍目标并迫使人物做出有后果的决定。
- Purdue OWL 关于 compelling characters 的写作资料：人物目标、动机和主要冲突需要连接，否则角色缺少行动理由。

本文未照搬来源原文，而是将这些原则改写为中文商业网文 AI agent 可执行的“昂贵选择”系统：目标受阻，选项互斥，代价落地，行动可见，后果进入后文。
