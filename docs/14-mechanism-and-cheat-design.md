## 相关入口：世界观、设定与信息投放

涉及世界观、等级、组织、地图、金手指规则、历史背景、制度习俗或信息投放时，先调用 [28. 世界观、设定与信息投放](28-worldbuilding-setting-exposition.md)。该专题负责判断哪些设定进正文、哪些留后台、哪些必须进台账，以及如何避免开篇设定堆砌、规则漂移和名词过载。


本文用于帮助 AI agents 设计系统、重生、穿书、空间、读心、弹幕、规则反杀、数据面板、特殊道具等“机制型卖点”。机制不是万能外挂，而是让主角更主动、让冲突更清晰、让连载更可持续的故事引擎。

## 1. 机制设计的核心原则

好机制要回答六个问题：

- 它解决什么读者承诺：升级、打脸、情绪代偿、真相推进、经营改善，还是喜剧错位。
- 它为什么只适合这个主角：性格、处境、旧伤、技能、关系或选择必须和机制相互咬合。
- 它能做什么：能力范围要清楚。
- 它不能做什么：边界比能力更重要。
- 它要付出什么：代价可以是资源、风险、关系、伦理、时间、信息误差或暴露。
- 它如何升级：每一轮升级要改变冲突方式，而不是只把数字变大。

原则：

- 机制要制造选择，不要替主角做选择。
- 机制要加压，不要消灭压力。
- 机制要提供可重复回报，但每次回报都要有变体。
- 机制要和题材承诺一致。玄幻系统服务升级，现言读心服务关系张力，悬疑规则服务真相推进，种田空间服务资源循环。
- 机制越强，限制、代价和对手反制越要清楚。

高危信号：

- 系统自动给答案、自动打脸、自动结算，主角只负责点确认。
- 重生主角记得所有细节，永远提前预判，世界没有蝴蝶效应。
- 穿书主角掌握剧情后，原书人物像 NPC 一样按剧本走。
- 读心、弹幕、天眼让主角无限获取真实信息，没有误读、噪音、代价或伦理压力。
- 空间、商城、签到、抽奖只负责发资源，不制造冲突。

## 2. 机制设计卡

设计任何金手指前，先填这张卡。

| 项目 | 要写清 |
| --- | --- |
| 机制名称 | 系统、重生、穿书、空间、读心、弹幕、规则账册、数据面板等 |
| 核心承诺 | 这个机制给读者什么期待 |
| 触发条件 | 什么情况下能用，什么情况下不能用 |
| 输出结果 | 它能给信息、资源、能力、身份、证据还是机会 |
| 限制边界 | 范围、次数、冷却、权限、对象、环境、时间线、信息质量 |
| 使用代价 | 消耗、暴露、误读、关系损伤、身体代价、道德代价、规则反噬 |
| 失败方式 | 用错会怎样，敌人如何利用，主角会损失什么 |
| 对手反制 | 谁能遮掩、误导、封锁、抢解释权、污染信息或绕开规则 |
| 升级路径 | 升级后改变什么冲突方式 |
| 台账字段 | 需要记录哪些规则、次数、代价、奖励和已暴露信息 |

AI 起草前必须能用一句话说明：

```text
主角可以在【触发条件】下使用【机制】，获得【输出】，但会受到【限制/代价】，如果误用会导致【失败方式】，对手可以通过【反制】制造新冲突。
```

## 3. 限制的八种类型

限制不是削弱爽点，而是让爽点更可信、更可持续。

- 范围限制：只能影响某类人、某个地点、某种规则、某个时间段。
- 触发限制：必须满足条件，例如签字、接触、完成任务、进入副本、被公开指认。
- 次数限制：每日、每章、每阶段、每个对象只能使用有限次数。
- 冷却限制：使用后需要等待、恢复、补资源或承受副作用。
- 资源限制：消耗钱、寿命、积分、灵力、名声、人情、材料、信任。
- 信息限制：只能得到片段、概率、情绪、弹幕猜测、错误历史，不给完整答案。
- 社会限制：用了会暴露秘密、引发怀疑、破坏关系、触犯制度或被抢定义权。
- 对手反制：对手可以遮掩证据、制造伪信息、切断权限、利用规则漏洞、转移战场。

检查问题：

- 这次使用机制前，读者是否知道它有什么边界？
- 这次使用机制后，是否带来新问题，而不是只消灭问题？
- 对手是否有合理反制空间？

## 4. 系统/任务/签到/商城

系统类机制的核心不是“发奖励”，而是把主角推入有选择、有代价、有回报的行动链。

可用设计：

- 任务型：系统给目标，但完成方式由主角选择。
- 签到型：签到奖励提供资源，但资源如何使用才是冲突。
- 商城型：商城提供选项，但积分有限，选择 A 就放弃 B。
- 惩罚型：惩罚不应只是电击或死亡威胁，应影响关系、资格、时间、信息或代价。
- 反转型：系统规则逐渐暴露来源、漏洞和真正目的。

用法：

- 第一章只给一个局部规则和一次可验证收益，不要把系统说明书全贴出来。
- 奖励要改变下一章行动资格：进场资格、证据、技能、资金、身份、治疗、逃生路线。
- 任务要迫使主角取舍：救人还是保密、赚钱还是暴露、赢比赛还是破坏关系。
- 系统不能替主角判断最优解，最多给条件、提示、惩罚和反馈。

风险：

- 系统任务像日常清单，主线被打散。
- 奖励随机堆砌，读者无法预期成长方向。
- 系统一开口解释所有设定，人物都不用思考。
- 主角只执行系统命令，没有自己的欲望。

检查问题：

- 如果去掉系统任务，本章主角还有没有自己的目标？
- 奖励是否改变了局面，而不只是加数字？
- 任务是否制造了选择和代价？

## 5. 重生/预知未来

重生的爽点来自“知道旧局，但不能照抄旧局”。旧记忆是优势，也是负担。

可用限制：

- 记忆缺损：只记得关键事件，不记得细节、日期、参与者或隐藏原因。
- 蝴蝶效应：主角改变一处，后续事件开始偏离。
- 情绪负债：旧伤、愧疚、恐惧、亏欠会影响判断。
- 社会限制：主角知道真相，却没有证据、身份、资源或发言权。
- 对手成长：对手不是静态 NPC，会因主角动作提前警觉。

用法：

- 第一章用一个旧局回收证明重生价值，但不要让主角一次性全赢。
- 重生知识要被转化为行动：抢时间、找证据、避坑、救人、换路线、抓破绽。
- 每次利用记忆后，要产生偏差和新问题。
- 旧人要有新反应，不能永远按上一世剧本走。

风险：

- 主角全知全能，所有人被降智。
- 重生只用来解释开挂，没有情绪代价。
- 每个危机都靠“我上一世知道”解决，缺少现场选择。

检查问题：

- 主角知道什么，不知道什么？
- 这次改动让未来偏离了吗？
- 旧记忆是否带来情绪压力或证据压力？

## 6. 穿书/穿越剧情/原著记忆

穿书机制的核心矛盾是“剧情知识”和“活人世界”的冲突。原著不是绝对未来，而是主角进入世界前的旧版本。

可用限制：

- 原著视角有限：主角只看过主线，不知道配角真实动机和幕后交易。
- 剧情节点会漂移：主角改了人物关系，原事件换形式发生。
- 人物有自主性：原书角色不是标签，他们会因为主角变化而重新选择。
- 读者知识差：主角知道原著，当前读者未必知道，需要用场景证明，不要只自言自语。

用法：

- 第一章明确主角穿入后的当下危机，而不是长篇复述原书。
- 让主角用原著知识做一次局部选择，但立刻遇到偏差。
- 原著信息应制造两难：按剧情保命，还是改变某人的命运。
- 穿书主角要逐步从“读者视角”转为“局中人责任”。

风险：

- 大段回忆原著设定，当前场景停摆。
- 原书人物只按人设标签运行。
- 主角总能靠“我知道剧情”绕开所有冲突。

检查问题：

- 当前危机是否不依赖读者了解原著也能成立？
- 原著知识这次是优势，还是诱导主角误判？
- 被改变的人物是否出现了新选择？

## 7. 空间/随身仓库/灵泉/农场

空间机制适合种田、逃荒、经营、末世、古言和都市脑洞，但它必须进入资源循环，而不是无限仓库。

可用限制：

- 容量限制：格子、重量、体积、活物、温度、时间流速。
- 进入限制：地点、时间、精神状态、是否被人看见。
- 产出限制：灵泉、农田、加工、保鲜、材料都需要周期。
- 安全限制：空间物品来源会暴露，使用不当会引发怀疑。
- 经营限制：有资源还要解决渠道、价格、运输、信任和制度。

用法：

- 第一章展示空间如何解决一个小困境，同时暴露一个大问题。
- 空间收益要可视化：食物、药材、钱、逃生工具、证据保存、经营改善。
- 让空间制造新冲突：东西从哪来、怎么卖、谁发现、如何解释、如何扩大生产。

风险：

- 无限囤货，无外部压力。
- 空间万能，缺什么拿什么。
- 种田经营变成流水账盘点。

检查问题：

- 空间本章改善了什么可见处境？
- 空间使用是否带来暴露风险或经营压力？
- 资源循环是否能持续产章？

## 8. 读心/弹幕/评论区/观众视角

读心和弹幕类机制的核心是“信息差”，不是全知。信息必须有噪音、立场和解释成本。

可用限制：

- 信息片段化：只能听到情绪、关键词、弹幕猜测、某类人的心声。
- 信息噪音：弹幕会吵、会误导、会玩梗、会剧透不全。
- 对象限制：只能读特定人、特定距离、特定情绪、特定场景。
- 代价限制：读多了头痛、情绪污染、暴露表情、破坏信任。
- 伦理限制：知道隐私后如何选择，会影响人物关系。

用法：

- 第一章用一次信息差制造爽点或悬念，但不能给完整答案。
- 读心/弹幕最好给主角“破绽线索”，真正推进仍靠主角行动。
- 弹幕和评论区要有群体口味和误判，不要像作者外挂提示器。

风险：

- 弹幕直接告诉主角真相。
- 读心让所有谎言失效，悬念消失。
- 信息太多，正文变成弹幕吐槽合集。

检查问题：

- 主角得到的是线索还是答案？
- 信息是否可能被误读？
- 主角如何承担知道隐私后的关系后果？

## 9. 规则反杀/账册/契约/审判机制

规则反杀的爽点来自“旧秩序反噬压迫者”，但规则必须可争夺、可解释、可遮掩。自动显灵太多会削弱主角。

可用限制：

- 证据不完整：账册残页、契约缺口、记录被遮挡。
- 解释权争夺：对手抢先定义规则含义。
- 程序压力：审判、仲裁、宗门规矩、公司流程、平台规则都需要步骤。
- 权限限制：主角能调取什么，谁能封存什么。
- 见证限制：公开见证越多，收益越大，阻力也越强。

用法：

- 第一轮可以让规则显效，证明卖点。
- 第二轮开始加入误读、遮掩、权限卡点和程序反扑。
- 主角必须主动抓破绽：诱导对手补话、找见证、转移战场、保存证据。
- 每次反杀要有落袋：资源、名声、资格、旧账、外部见证、对手损失。

风险：

- 账册/系统/水影自动替主角举证。
- 对手只会嘴硬，没有合理自保。
- 爽点只停留在众人震惊，没有实际收益。

检查问题：

- 这次证据是否可争夺？
- 主角做了什么主动动作？
- 反杀后谁损失了什么，主角获得了什么？

## 10. 数据面板/属性/技能树

数据面板适合升级、游戏、体育、经营、末世和科幻题材，但数字必须代表可见变化。

可用限制：

- 测量误差：面板只显示部分属性，不显示意志、经验、关系、环境。
- 隐藏属性：某些能力需要条件触发。
- 成长成本：技能点、训练、材料、失败经验、身体负担。
- 选择代价：点攻击就暂时放弃防御，点商业就影响战斗。
- 外部差异：现实战斗、市场、关系不会完全按数值运行。

用法：

- 数值变化要落到动作结果：跑得更快、看穿破绽、谈判更稳、产量增加。
- 技能树要迫使主角选择路线，不要全加满。
- 面板信息可以误导主角，让他意识到“数据不是全部现实”。

风险：

- 连续报属性，读者看不到场景变化。
- 数值暴涨但对手和世界没有反应。
- 面板比人物更有存在感。

检查问题：

- 这个数值变化在场景里如何被看见？
- 主角选择某项能力后放弃了什么？
- 数据有没有无法解释的人心、环境或意外？

## 11. 机制升级节奏

机制升级不是“奖励越来越大”，而是冲突方式变化。

推荐节奏：

- 第一轮：证明机制存在，给小回报。
- 第二轮：暴露限制，让主角不能照抄第一次。
- 第三轮：对手开始反制，机制从优势变成争夺点。
- 第四轮：机制升级或解锁新用法，但同时带来更大代价。
- 阶段末：机制带来一次硬落袋，并打开更高层问题。

升级方向：

- 范围扩大：从个人到群体，从室内到公共场，从小地图到大势力。
- 信息升级：从提示到证据，从猜测到记录，从片段到因果。
- 代价升级：从积分到名声，从资源到关系，从身体到道德选择。
- 对手升级：从不知情到遮掩，从嘴硬到反利用规则。
- 关系升级：机制被某人发现、误解、利用或共同承担。

检查问题：

- 最近三次使用机制是否长得一样？
- 机制升级是否改变了冲突方式？
- 机制是否已经成为主角和对手争夺的对象？

## 12. AI 起草机制文的硬约束

AI 起草含金手指章节时，必须遵守：

- 正文不写机制说明书，先用场景证明机制。
- 机制第一次使用可以清晰，但不能把所有规则一次讲完。
- 主角必须做选择，不能只执行系统命令。
- 奖励必须落袋到资源、权限、关系、信息、能力或行动资格。
- 限制必须在前 3 次使用中至少显现一次。
- 对手必须逐步出现反制，不能永远被机制单方面碾压。
- 每次机制使用后都要更新台账：触发、输出、代价、暴露信息、规则变化。

AI 容易写坏的地方：

- 把金手指当作弊器，不当故事引擎。
- 把限制写在设定里，但正文从不触发。
- 把代价写成无关痛痒的扣积分。
- 把奖励写成抽象数字，不改变场景。
- 让系统比主角更像主角。

## 13. 机制审稿问题

审稿时重点问：

- 机制是否服务本书主承诺？
- 主角是否因为机制更主动，而不是更被动？
- 机制的限制、代价和失败方式是否在正文里出现？
- 最近几次使用机制是否重复？
- 对手是否理解、误判、遮掩或反制机制？
- 机制是否制造了新的关系、伦理、程序或资源压力？
- 如果删除机制，故事是否完全塌掉？如果会，它是否已经成为真正引擎，而不是外挂补丁？

结论写法：

- 有效点：机制最能制造什么爽点或追读。
- 风险点：机制是否过强、过散、过自动、过说明书。
- 修改优先级：先补限制、补代价、补对手反制，还是重写第一轮使用场景。
