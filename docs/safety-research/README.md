# safety-research

合规、安全、版权、敏感内容、事实核查、来源转化和 AI 生成内容边界。

本目录是风险闸门。只要写作、审稿、资料整理或开源维护涉及未成年人、医疗金融、违法犯罪、公共安全、隐私、版权、平台审核、AI 生成内容、深度合成或外部来源改写，先读本目录相关专题，再进入正文创作或资料沉淀。

## 2. 推荐链路

### 2.1 发布前合规审查

```text
20 合规总审 -> 105 伤害表达 -> 167 版权授权 -> 目标平台当前规则
```

适合：投稿、签约、创建作品、上传封面、发布敏感章节前。

### 2.2 外部资料入库

```text
111 来源评估 -> 102 来源转化 -> sources/01-source-inventory.md 登记 -> 目标专题抽象写入
```

适合：联网资料、课程笔记、读者反馈、小说片段、平台截图、行业资料进入开源知识库前。

### 2.3 小说情节涉及高风险现实操作

```text
20 合规总审 -> 目标 reality-reference 专题 -> 105/189/167 按风险追加 -> 只写诱因、线索、后果和人物选择
```

适合：犯罪、诈骗、公共安全事故、医疗金融、未成年人、网络攻击、深度合成等情节。

### 2.4 AI 生成内容和版权风险

```text
167 版权署名授权 -> 189 深度合成风险 -> 102 原创性转化 -> 项目/平台规则
```

适合：AI 封面、AI 配音、数字人、同人、改编、引用外部作品和开源分发。

## 3. 硬规则

- 不提供违法犯罪、诈骗、黑客攻击、武器、制毒、逃避监管、公共安全破坏等可操作教程。
- 不把受版权保护作品的连续片段、付费课程原文、未公开稿件、私密聊天或可识别反馈放进开源仓库。
- 不把“用户提供过”当作可公开发布的授权；开源前仍要做去身份化、来源边界和原创性转化。
- 不用“小说需要刺激”抵消未成年人、创伤、自伤、性暴力、心理危机和污名表达的伤害风险。
- 不把平台后台截图、尺寸、审核规则视为永久规则；正式发布前必须重新确认目标平台当前要求。

## 4. 常见误用

- 只查现实资料，不查安全边界：容易把专业流程写成危险教程。
- 只做“去名字”，没有去身份化：仍可能通过路径、群名、项目结构、时间线和细节拼出真实来源。
- 只改几个词，就把外部文章或小说片段当作原创资料：这不是知识沉淀，是近似改写。
- 只看版权，不看平台合同：签约、授权、同人、改编和 AI 生成内容还受平台规则影响。
- 只强调“真实”，忽略读者伤害：创伤、疾病、残障、未成年人和社会群体表达要兼顾叙事功能与伤害避免。
