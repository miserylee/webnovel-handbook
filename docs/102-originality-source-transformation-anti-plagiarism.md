# 102. 原创性、来源转化、防洗稿与借鉴边界专项

状态：已确认

适用范围：联网搜集写作资料、竞品拆解、题材跟风、热门梗吸收、书名简介参考、角色声音样例、风格手册、真实案例改编、历史/行业资料转化、AI 续写、章节改写、审稿发现“像某书”“融梗”“洗稿”“抄桥段”“标题仿写”“文风像某作者”时使用。

核心结论：AI agent 可以学习市场规律、类型语法、读者承诺、节奏结构和抽象功能，但不能把来源作品的具体表达、桥段组合、人物关系、专名体系、标志性梗、章节顺序、独特设定或可识别风格改名后写成原创。安全借鉴不是“换词”，而是“抽象功能、改变变量、重建因果、换人物欲望、换情绪回报、重新生成正文，并保留来源边界”。

最小链路：资料收集 -> 来源登记 -> 可迁移抽象 -> 不可迁移隔离 -> 三重变形 -> 独立生成 -> 相似风险检查 -> 项目留痕。

---

## 1. 为什么需要独立专题

已有第 20 篇覆盖内容合规、版权和平台风险，第 34 篇覆盖竞品拆解和可迁移边界。但 AI agent 在实际创作中仍容易出现三类问题：

- 把“学习同类热销文”变成复刻热门设定、桥段和标题句式。
- 把“改写来源资料”变成逐句换词、顺序不变、含义不变的洗稿。
- 把“风格参考”变成模仿某作者标志性句式、节奏和口头禅。

这会带来四种损害：

- 商业风险：读者一眼觉得像某书，评论区出现“融梗”“抄袭”“缝合怪”。
- 平台风险：版权、低质、搬运、同质化或 AI 生成风险上升。
- 创作风险：故事没有自己的因果、人物欲望和情绪资产，长篇写不下去。
- 协作风险：上游把来源桥段写进任务单，下游 agent 误以为是本书 canon。

本篇的目的不是替代法律判断，而是给 AI 写作流程一个更早、更细、更可执行的原创性护栏。

## 3. 来源吸收与迁移边界

### 3.1 美国版权局：思想、事实和表达的区分

美国版权局 FAQ 和 Circular 33 均强调，版权保护原创表达，不保护事实、想法、系统、方法、概念、原则等本身；标题、名称和短语通常也不属于版权保护对象，但可能涉及商标或其他风险。

迁移为创作规则：AI 可以学习“重生复仇”“系统升级”“真假千金”“师徒禁忌张力”这类类型思想和通用模式，但不能照搬某作品表达这些思想的具体桥段、人物关系、专名体系、句子和独特组合。

### 3.2 中国版权法公开信息：作品需要独创性和表达形式

国家版权局转载的《全国人民代表大会常务委员会关于修改〈中华人民共和国著作权法〉的决定》明确将作品表述为文学、艺术和科学领域内具有独创性并能以一定形式表现的智力成果，并提到改编、翻译、注释、整理、汇编已有作品用于出版等场景需要关注原作品权利。

迁移为创作规则：AI agent 不能把“已有作品改名重组”误当原创；涉及真实作品、影视动漫、游戏设定、同人或改编时，默认高风险，必须进入第 20 篇合规流程。

### 3.3 美国版权局 AI 报告：人类创作控制与 AI 输出边界

美国版权局关于 AI 与版权的报告强调，应区分人类作者的创造性贡献和由 AI 系统生成的材料。不同司法辖区和平台规则会变化，本知识库不把它当作中国平台发布结论，只迁移为内部流程：AI 生成的内容也要有人类或项目侧的原创设计、选择、改写和审查，不能把模型输出直接当无风险原创。

### 3.4 Purdue OWL / Harvard Guide：防止换词式挪用

Purdue OWL 和 Harvard Guide 关于引用、转述、总结与避免抄袭的写作指南强调，使用来源材料时要保留来源记录，直接用语要标注，转述也不能只是逐句换词或保留原结构。

迁移为 AI 写作规则：知识库可以吸收外部资料，但要把来源改写成原则、用法、风险、检查问题和案例启发；不能把来源段落换成近义词后写入文档或正文。

### 3.5 语言模型抄袭研究

关于语言模型是否会生成训练材料相似文本的研究提醒：AI 生成内容可能出现逐字、转述式或思想层面的相似风险，尤其在提示中放入长段来源文本或要求“仿写某作者/某作品”时风险上升。

迁移为流程规则：给 AI 的提示词不要塞入大段竞品原文；需要风格样例时，只保留抽象特征、禁用词、节奏偏好和功能说明。

## 4. 概念定义

### 4.1 可迁移规律

定义：从多个来源中抽象出的结构功能、读者期待、类型底线、节奏模式、情绪回报或商业判断。

例子：

- 开篇要快速证明主承诺。
- 失败必须留下反弹抓手。
- 反派失败后要换策略。
- 甜宠误会要有信息边界和缓解进度。
- 悬疑反转需要公平线索。

这些可以迁移，但仍要用本项目的人物、场景、因果和语言重新表达。

### 4.2 不可迁移内容

定义：来源作品中可识别、具体、独特、组合性强或被读者记住的表达和设计。

例子：

- 具体桥段顺序。
- 角色名、组织名、功法名、技能名、地名、物件名。
- 主角和配角的独特关系配置。
- 标志性台词、口头禅、句式和标题结构。
- 核心谜底、反转方式、名场面调度。
- 金手指规则的独特组合。
- 章节开局和章末钩子的连续排列。
- 读者一眼能联想到原作的梗。

不可迁移内容不能改名后使用。

### 4.3 洗稿

定义：保留原文结构、顺序、信息组织和核心表达，只替换词语、调整句子、删改局部，使文本表面不同但实质仍依附来源。

AI 风险：模型很擅长“润色”和“改写”，但这不是原创转化。只要骨架、顺序、表达重点和独特组合仍来自来源，就有洗稿风险。

### 4.4 融梗

定义：将多个作品的高识别度桥段、关系、设定或名场面拼接到一起，声称是原创。

风险：来源多不等于安全。多个来源的可识别内容拼接，仍可能形成“缝合怪”体感。

### 4.5 风格模仿

定义：要求 AI 写得像某位在世作者、某部作品或某一平台爆款的可识别声音。

处理：允许分析“轻快、讽刺、口语密、短段、强反差”等抽象特征；禁止复制可识别句式、比喻习惯、段落节奏和口头禅。

## 5. 来源材料四格分流

AI agent 收到资料后，先分四格，不得直接写正文：

| 格子 | 内容 | 允许动作 | 禁止动作 |
| --- | --- | --- | --- |
| A 通用知识 | 类型惯例、公开常识、行业基础、结构理论 | 吸收为规则，登记来源 | 声称由本项目原创发现 |
| B 可迁移功能 | 承诺方式、节奏功能、冲突功能、读者反馈规律 | 抽象后重组 | 复制桥段、顺序和人物配置 |
| C 高识别表达 | 台词、句式、名场面、专名、独特设定组合 | 仅作风险提醒 | 改名、换词、拼贴 |
| D 高风险权利内容 | 同人、影视游戏 IP、歌词、长段原文、现实人物素材、未授权设定 | 进入合规流程 | 写成原创正文或项目 canon |

只有 A 和经过抽象后的 B 可以进入创作提示。C 和 D 只能进入禁区或风险记录。

## 6. 三重变形原则

一个灵感要进入本书，至少完成三重变形。

### 6.1 功能变形

把“来源发生了什么”改成“它在叙事中起什么作用”。

例子：

```text
来源内容：主角在退婚宴上当众打脸未婚妻家族。
功能抽象：开篇用公开羞辱制造主角尊严危机，并让主角用新获得的能力完成第一次反压。
```

可用的是功能，不是退婚宴、未婚妻家族、当众打脸的具体组合。

### 6.2 变量变形

改变人物欲望、社会位置、冲突场域、胜负资源、情绪回报和后果。

```text
安全变体：
- 公开羞辱 -> 专业资格被质疑。
- 家族退婚 -> 项目组除名。
- 金手指反压 -> 旧经验 + 新证据完成局部反证。
- 爽点回报 -> 得到一次试用机会，而不是直接碾压所有人。
```

### 6.3 因果变形

重新建立本书自己的原因、限制、选择和后果。

要求：

- 冲突必须来自本书人物的欲望和压力。
- 解决方式必须来自本书已建立资源和代价。
- 后果必须影响本书后续目标、关系或风险。
- 不能只换壳保留原作因果链。

## 7. 安全借鉴的“五不一要”

### 7.1 不学单本，学样本池

不要只拆一本爆款。至少用直接竞品、相邻竞品、头部标杆、中腰部样本和反例形成样本池。

原因：只学一本，容易复刻；多样本抽象，才容易得到类型规律。

### 7.2 不记桥段，记功能

调研笔记中不保存长段桥段复述，只记录功能：

```text
错误：第 3 章男主假装穷保安，被女方亲戚嘲笑，最后拿出黑卡。
正确：早期通过身份误判制造轻压迫，随后用身份/资源反证完成短爽点。
```

### 7.3 不仿句子，仿约束

不要写“文风像某某”。改成：

```text
旁白短句多；少形容词；每段保留一个动作或判断；喜剧来自人物价值排序错位；禁止复制具体比喻和口头禅。
```

### 7.4 不搬专名，建系统

不能把“宗门名、技能名、组织名、装备名”改几个字就用。要先确定本书机制、文化、权力结构和命名规则，再重新生成专名。

### 7.5 不拼高光，造压力

名场面不能从别处搬。高光应从本书人物长期欲望、前文压力、资源积累和当前选择自然长出来。

### 7.6 要保留来源边界

每条来源进入项目时都要标注：

- 来源链接或来源名称。
- 吸收的抽象规律。
- 禁止迁移的具体内容。
- 是否存在版权、同人、真实人物或平台风险。

## 8. 原创转化工作流

### 8.1 第一步：来源登记

记录：

```text
来源名称：
链接/出处：
来源类型：官方规则 / 写作教程 / 竞品 / 评论 / 旧项目 / 学术资料 / 真实案例 / 其他
使用目的：市场定位 / 结构参考 / 角色关系 / 题材资料 / 风格样例 / 审稿风险 / 其他
可引用程度：可公开引用 / 内部参考 / 高风险不可复述
```

### 8.2 第二步：抽象功能

把来源改写成：

```text
它解决的叙事问题：
它服务的读者体验：
它依赖的前提：
它造成的风险：
它能迁移的结构功能：
它不能迁移的具体内容：
```

### 8.3 第三步：生成三种不相似方案

AI 不允许只给一个“改名版”。必须至少给三种方向：

```text
方案 A：换场域。
方案 B：换人物关系。
方案 C：换回报币种。
```

如果三种方案仍然像同一本来源，说明抽象不足，不能进入正文。

### 8.4 第四步：建立本书因果

对选定方案重建：

- 本书主角为什么会遇到这个问题。
- 本书对手为什么采取这个动作。
- 本书世界规则为什么允许这个冲突。
- 本书主角靠什么解决或局部失败。
- 本章结果怎样改变后续。

### 8.5 第五步：相似风险检查

检查：

- 是否保留来源桥段顺序？
- 是否保留来源角色关系配置？
- 是否保留来源专名或近似专名？
- 是否保留来源标志性台词、口头禅、标题句式？
- 是否保留来源独特金手指规则组合？
- 是否读者能用一句话指出“这不就是某某”？
- 是否上游提示词中仍有长段来源文本？

任一为“是”，退回重构。

## 9. 风险等级

| 等级 | 定义 | 处理 |
| --- | --- | --- |
| O0 安全抽象 | 只使用通用规律和类型功能 | 可进入规则文档或任务单 |
| O1 低风险相似 | 同题材常见模式，变量和因果已重建 | 可使用，记录来源 |
| O2 中风险靠近 | 场景功能相似，部分变量仍接近来源 | 必须再变形，审稿标注 |
| O3 高风险融梗 | 角色关系、桥段顺序、设定组合明显相似 | 不进入正文，重构 |
| O4 禁止使用 | 未授权 IP、同人改原创、长段原文、歌词、可识别名场面 | 停止生产，进入合规流程 |

O2 以上不能批量生产后文。O3 以上不能进入正式项目 canon。

## 10. 不同来源的处理规则

### 10.1 竞品小说

可以学：

- 目标读者。
- 类型承诺。
- 第一章证明方式。
- 回报节奏。
- 评论中读者真正买单的体验。
- 失败案例和避坑点。

不能学：

- 具体桥段。
- 人物关系配置。
- 专名体系。
- 章节顺序。
- 标志性反转。
- 标题句式和简介句式。

### 10.2 写作教程

可以学：原则、检查问题、结构功能。

不能做：整段搬运、换词写入知识库、把来源观点说成本项目原创。

### 10.3 真实案例

可以学：冲突类型、行业流程、情绪触发、社会压力。

必须变形：人物身份、时间地点、关键细节、结果归因、法律/隐私风险。

不能做：影射现实个人、泄露隐私、把新闻细节直接戏剧化。

### 10.4 历史和民俗

可以学：制度、风俗、物质生活、价值冲突。

必须做：资料交叉验证，避免把单一说法写成确定事实；涉及少数族群、宗教和地域文化时避免刻板化。

### 10.5 风格样例

可以学：节奏约束、叙述距离、段落长度、信息密度、语气温度。

不能做：复制句子、复制比喻、复制口头禅、要求“像某某作者”。

### 10.6 AI 生成内容

可以作为候选草稿。

不能直接视为原创安全。需要：

- 删除提示中来源长文。
- 重建人物和因果。
- 检查是否有常见爆款桥段拼贴。
- 做相似风险审稿。
- 由项目规则决定是否进入 canon。

## 11. 防洗稿检查

以下情况高度危险：

- 原文每段对应新文每段，只是换词。
- 信息顺序基本一致。
- 场景动作和情绪节拍一致。
- 人物功能一一对应。
- 原文的独特比喻被改成近似比喻。
- 原文的悬念、反转和答案顺序不变。
- 原文的标题、章节名和卖点句式太接近。
- 读者不看来源也能感觉“这像某书”。

安全改法不是继续换词，而是回到功能抽象和三重变形。

---

## 12. 防融梗检查

多个来源拼接时，必须检查：

```text
来源 A 提供了什么？
来源 B 提供了什么？
来源 C 提供了什么？
这些是否都是高识别内容？
它们是否只是被改名后拼接？
本书自己的核心问题、人物欲望和情绪回报在哪里？
如果删掉来源元素，本书还剩下什么？
```

如果答案是“只剩类型标签”，说明项目缺少原创核心，不能开书。

## 13. 原创性的五个锚点

商业网文不需要每个元素都前所未见，但必须有可识别的本书资产。

| 锚点 | 检查问题 |
| --- | --- |
| 主角欲望 | 这个主角为什么非写不可？他的缺口、执念、底层算法是什么？ |
| 冲突引擎 | 本书长期冲突如何自动产生，而不是靠搬桥段续命？ |
| 回报币种 | 读者追更时最期待哪种独特回报？爽、甜、燃、笑、谜、经营、关系确认？ |
| 关系网络 | 哪些关系只属于本书的人物压力和选择？ |
| 语言与视角 | 本书旁白、视角过滤器和人物声音是否有自己的限制和质地？ |

只要这五个锚点成立，常见类型元素也能变成自己的故事。

## 14. 项目文档中的留痕要求

新项目或知识库专题吸收外部资料时，应记录：

```text
来源：
吸收日期：
用途：
可迁移规则：
不可迁移内容：
风险等级：O0/O1/O2/O3/O4
是否进入正文：是/否
如果进入正文，已完成哪些变形：功能/变量/因果/语言/回报
复查人或复查状态：已确认/待确认
```

这不是为了把所有灵感公开给读者，而是为了多 agent 协作时知道哪些内容是规则、哪些是禁区、哪些还待确认。

## 15. 提示词安全规则

### 15.1 禁止提示

```text
照着这本书的设定写一个原创版本。
把下面这段改写成自己的话。
文风像某某作者。
把这个桥段换个背景写。
把这个爆款开头改成古代版。
```

### 15.2 推荐提示

```text
请从以下资料中只抽象“叙事功能、读者承诺、节奏作用和风险”，不要保留具体桥段、专名、句式或人物关系。输出可迁移规则和不可迁移清单。
```

```text
请基于本项目已确认的人物欲望、世界规则和主承诺，生成三个与来源桥段在场域、关系、因果和回报上均不同的原创场景方案。
```

```text
请审查以下方案是否存在洗稿、融梗、标题仿写、专名近似、角色关系复刻或风格模仿风险，并按 O0-O4 分级。
```

## 16. 审稿时的原创性诊断

当审稿反馈为“像某书”“缝合”“融梗”“标题太像”“设定太像”时，先不要辩解，按以下流程查：

1. 读者指出的是题材相似，还是具体表达相似？
2. 相似点是否属于通用类型惯例？
3. 是否有三个以上高识别元素同时接近同一来源？
4. 是否存在桥段顺序和人物功能一一对应？
5. 是否存在专名、标题、口头禅或金手指规则近似？
6. 本书是否有独立主角欲望、冲突引擎和回报币种？
7. 如果删除疑似来源元素，本书是否仍然成立？

如果 3、4、5 任一成立，按 O3 处理，暂停后续生产。

## 17. 常见错误与修法

| 错误 | 风险 | 正确做法 |
| --- | --- | --- |
| “同题材都这样” | 用类型惯例掩盖具体复刻 | 区分通用模式和独特组合 |
| “我改了名字” | 专名变了，结构没变 | 重建人物欲望和因果链 |
| “我参考了很多本” | 多来源拼贴仍可能融梗 | 只保留抽象功能，重建原创锚点 |
| “标题热词大家都用” | 可能造成同质化和标题仿写 | 保留类型识别，换本书独特承诺 |
| “AI 自己生成的” | AI 可能复现训练模式或提示来源 | 做来源隔离和相似风险检查 |
| “只是风格像” | 可识别作者声音也会引发争议 | 转成抽象约束，不仿句式 |
| “这是现实新闻” | 现实人物、隐私、名誉和改编风险 | 去身份化、交叉验证、合规审查 |

## 18. 最小规则

时间很少时，至少执行以下 10 条：

1. 不把单本竞品当模板。
2. 不把桥段改名后使用。
3. 不把来源段落换词后写入知识库或正文。
4. 不要求 AI 模仿在世作者或具体作品的可识别风格。
5. 不在提示词里长期保留竞品原文和名场面。
6. 每条来源先输出“可迁移规律”和“不可迁移内容”。
7. 灵感进入正文前至少完成“功能、变量、因果”三重变形。
8. O2 以上风险不批量生产后文。
9. O3 以上风险不进入 canon。
10. 涉及同人、影视游戏 IP、歌词、真实人物和长段原文，直接进入第 20 篇合规流程。
