## 1. 核心原则：商业网文先兑现预期，再谈惊喜

读者点开一本商业网文时，通常已经被平台、频道、分类、书名、封面、标签、简介和开篇语气建立了预期。

AI 写作时必须先回答：这本书要让哪类读者产生哪种期待，并在前几章怎样证明这种期待没有落空。

基础链路：

```text
目标读者 -> 类型入口 -> 主承诺 -> 可见卖点 -> 第一章证明 -> 前 10 章回报 -> 长线追读理由
```

如果链路断裂，作品会出现三种典型失败：

- 点击低：读者看不出为什么要点。
- 留存低：读者点进来后发现不是自己想看的。
- 追读低：读者知道这本书是什么，但看不见持续进展和回报。

## 2. 五个概念不要混用

| 名称 | 解决的问题 | AI 常见误用 |
|---|---|---|
| 目标读者 | 谁最可能为这本书付出时间和情绪 | 写成“所有喜欢小说的人” |
| 类型预期 | 读者进入该题材时默认想得到什么 | 只写题材名，不写体验 |
| 主承诺 | 本书最核心、最先兑现的阅读合同 | 同时承诺爽、虐、甜、悬疑、群像、无敌，互相打架 |
| 卖点 | 有限展示窗口里最该被读者看见的吸引物 | 把热门词当卖点，不做表达 |
| 回报 | 读者读下去后实际得到的爽、情绪、答案、成长或关系变化 | 只铺垫不结算，或回报和承诺不一致 |

一句话区分：

```text
目标读者决定写给谁；类型预期决定他们默认等什么；主承诺决定本书最先欠读者什么；卖点决定怎样让读者愿意试读；回报决定读者是否继续留下。
```

## 3. 目标读者不是人口标签，而是阅读需求

AI 不要只写“18-35 岁女性”“男频读者”“喜欢玄幻的人”。这些标签太粗，不能指导开篇和章节。

更有用的目标读者描述应包含：

```text
【读者正在寻找的体验】
【读者能接受的节奏】
【读者熟悉的套路】
【读者想看到的新鲜差异】
【读者不能忍的违约点】
【读者愿意追更的长期理由】
```

示例：

- 不是“喜欢末世的读者”，而是“想看末世生存压力、资源争夺、秩序崩坏，但同时期待主角有清晰优势和阶段性安全屋反馈的读者”。
- 不是“喜欢甜宠的读者”，而是“想看关系安全感、偏爱、暧昧拉扯和阶段确认，不接受长期误会只虐不补偿的读者”。
- 不是“喜欢系统文的读者”，而是“想看规则带来的超常回报、主角主动利用规则、奖励改变行动资格，而不是系统替主角自动赢的读者”。

## 4. 类型预期：承诺的底座

类型不是装饰标签，而是读者和作者之间的默认协议。

AI 判断类型预期时，至少拆成四层：

| 层级 | 要问的问题 | 示例 |
|---|---|---|
| 情绪层 | 读者想获得什么体感？ | 爽、甜、虐、燃、轻松、惊悚、治愈、沉浸 |
| 结构层 | 读者期待怎样推进？ | 升级、破案、恋爱推进、经营扩张、副本通关、卷级冒险 |
| 内容层 | 必备元素是什么？ | 金手指、线索、CP 化学反应、资源争夺、身份压迫、喜剧错位 |
| 回报层 | 多久给一次阶段满足？ | 第一章证明、前三章加深、前十章小结算、卷末大结算 |

违反类型预期不是绝对不行，但必须有清楚目的，并用更强的替代回报补偿。

## 5. 主承诺只能先立一个

商业连载可以多元素融合，但开篇阶段只能有一个最强主承诺。副承诺可以存在，但不能抢主承诺的第一轮证明。

主承诺句式：

```text
这本书首先承诺给【目标读者】一种【核心体验】：他们会看到【主角】在【压力/规则/关系】下，通过【机制/能力/选择】持续获得【回报】。
```

例：

- 升级流：读者会看到主角从被压制处境出发，利用独特规则持续变强，并把资源、身份和敌人层级逐步踩上去。
- 现言情绪文：读者会看到女主在关系和现实压力中夺回选择权，并在核心关系里获得被看见、被偏爱和自我确认。
- 悬疑规则文：读者会看到主角在危险规则中试探边界、识别谎言，并阶段性获得可验证真相。
- 群像喜剧：读者会看到一组稳定癖好的人物从日常小事滚出荒诞后果，同时关系网越来越厚。

## 6. 卖点不是点子，是被表达出来的吸引物

一个点子只有被读者看见、理解、期待，才算卖点。

卖点必须通过四个位置表达：

```text
包装展示 -> 第一章场景 -> 前 3-6 章重复证明 -> 前 10 章回报落袋
```

如果书名和简介承诺“末世没来”，第一章就应让读者看见主角按末世逻辑准备、付出代价、惹出麻烦，然后真的发现末世没来。这样卖点才从概念变成阅读欲望。

如果承诺“杀怪升级系统”，第一章或前三章必须看到系统如何改变主角选择、收益、风险和下一步行动资格，而不是只弹出界面。

## 7. 卖点的六种来源

| 来源 | 可用方式 | 风险 |
|---|---|---|
| 题材卖点 | 热门类型、细分流派、读者熟悉入口 | 只有题材名，没有差异 |
| 机制卖点 | 系统、重生、穿书、读心、规则、空间 | 机制替主角赢，人物失去能动性 |
| 关系卖点 | 宿敌、师徒、先婚后爱、破镜重圆、群像 | 关系标签强，正文没有阶段变化 |
| 人物卖点 | 极强人设、反差、成长、疯感、专业性 | 只靠口号，不通过选择证明 |
| 世界卖点 | 新鲜背景、规则、职业、制度、奇观 | 开篇百科化，读者看不到角色压力 |
| 叙事卖点 | 悬念、反转、喜剧、文风、结构实验 | 表达不稳定，或牺牲类型回报 |

AI 需要先选主卖点，再判断副卖点是否服务主卖点。

## 8. 微创新：熟悉入口 + 新鲜差异

读者需要熟悉感来判断“这是我想看的类型”，也需要新鲜感来判断“这本不是旧套路复读”。

推荐公式：

```text
熟悉类型入口 + 一个新鲜变量 + 可见第一回报
```

例：

- 末世 + 末世没来 + 主角提前准备反而惹出新麻烦。
- 修仙升级 + 规则反常的低阶资源 + 主角利用冷门规则反杀。
- 甜宠关系 + 双方都误以为自己在演戏 + 每次演戏都制造真实偏爱。
- 悬疑规则 + 规则看似保护人其实筛选人 + 主角第一次试探得到代价。

风险：变量太冷门、太抽象、太难解释，会让新鲜感变成理解成本。

## 9. 承诺、进展、回报模型

每一条承诺都要有进展和回报。

```text
承诺：读者相信后面会看到什么。
进展：读者每隔一段能看见主角朝这个方向移动。
回报：读者获得阶段满足，且确认前面的等待有意义。
```

AI 规划章节时，不要只问“发生了什么”，要问“这章推进了哪条承诺”。

### 9.1 承诺类型

- 情绪承诺：爽、甜、虐、燃、轻松、恐惧、治愈。
- 事件承诺：比赛、复仇、破案、通关、经营、考试、婚约、逃亡。
- 人物承诺：主角会变强、变主动、被认可、被爱、被理解、完成自我修复。
- 关系承诺：从误会到信任、从敌对到合作、从暧昧到确认、从亏欠到偿还。
- 规则承诺：系统会升级、禁忌会被验证、谜团会被揭开、设定会产生后果。

### 9.2 进展必须被读者感知

读者感到“没进展”，不一定是剧情没动，而可能是剧情动错了承诺。

例：读者是为 CP 拉扯来的，正文连续十章都在打副本，即使副本很热闹，也会觉得关系线停滞。读者是为破案来的，正文连续铺日常互动，也会觉得作者跑题。

### 9.3 回报要和承诺同币种

承诺爽，就要给压迫后的反制和落袋。

承诺甜，就要给关系状态变化、偏爱证据或安全感。

承诺悬疑，就要给新线索、阶段答案或旧认知被公平推翻。

承诺经营，就要给可见改善、资源循环或环境变化。

不要用“多写了一段漂亮文笔”来支付“读者等了一章打脸”的账。

## 10. 开篇承诺验证

### 10.1 第一章

第一章必须让读者看见主承诺的影子。

最低标准：

- 主角可见。
- 当前压力可见。
- 类型入口可见。
- 主卖点可见或被强钩子指向。
- 读者知道下一章为什么要看。

### 10.2 前 3-6 章

前三到六章要证明卖点不是一次性噱头。

检查：

- 主机制是否第二次改变主角选择？
- 主关系是否有阶段变化或新误读？
- 主悬念是否给出新材料？
- 主爽点是否出现第二轮压迫和反制？
- 主世界规则是否产生后果？

### 10.3 前 10 章

前十章至少完成一次小闭环，让读者确认“我等的东西拿到了”。

可用回报：第一次打脸落袋、第一次关系确认/误会升级、第一次任务结算、第一次线索答案、第一次经营改善、第一次等级差异验证。

## 11. 长篇卖点续航

卖点不能只在开头存在。每三到五万字，AI 需要重新做一次卖点拆解。

```text
【本阶段主卖点】
【它服务哪条主承诺】
【本阶段读者等待什么】
【每 3-5 章给什么进展】
【阶段结尾给什么回报】
【下阶段如何升级或变体】
```

续航方式：

- 升级：同一卖点进入更高层级、更大场景、更难阻力。
- 变体：同一机制在不同人物、地图、关系中产生不同后果。
- 反转：读者熟悉的卖点暴露新代价或隐藏规则。
- 组合：主卖点与副卖点结合，制造新体验。
- 结算：阶段性兑现，不让读者长期空等。

## 12. 预期错配的诊断

| 症状 | 可能原因 | 修复方向 |
|---|---|---|
| 点击低 | 包装没有清楚展示主卖点 | 重写书名、简介、标签和封面承诺 |
| 第一章弃读高 | 第一章没有兑现最显眼承诺 | 把主角压力、卖点验证和下一章钩子提前 |
| 前三章掉读 | 卖点只出现一次，没有重复证明 | 安排第二轮卖点使用和明确回报方向 |
| 前十章追读弱 | 承诺没有阶段落袋 | 加小闭环，给读者第一次结算 |
| 评论说“不是这个味” | 语气、类型或主承诺错配 | 回看包装、开篇语气、题材预期 |
| 读者说“水” | 章节在发生事情，但没有推进主承诺 | 标注每章推进哪条承诺，删掉无关热闹 |
| 读者说“套路” | 熟悉入口有了，新鲜变量不够 | 强化微创新变量和角色独特选择 |
| 读者说“看不懂” | 新鲜变量解释成本太高 | 降低名词密度，用场景证明卖点 |

## 13. 不同平台和频道的预期管理

### 13.1 番茄向

优先让读者迅速识别高概念、强钩子、第一回报和标签匹配。卖点要更外显，第一章要更快证明。

风险：只追标题党和高概念，正文留不住。

### 13.2 起点/阅文向

优先让读者相信长篇引擎、升级空间、世界纵深和持续冲突。卖点可以不全喊出来，但必须在开篇证明有可写长的结构。

风险：开篇过慢，设定和大局压过第一回报。

### 13.3 晋江向

优先管理关系、情绪、标签、文案声音和主角价值立场。承诺不一定是强事件，也可以是强情绪、强关系、强人设和稳定基调。

风险：标签承诺甜/虐/轻松/强强/无 CP，但正文阶段回报错位。

## 14. AI 写作硬性禁令

1. 禁止把“题材名”当成主承诺。
2. 禁止把“系统、重生、穿书、甜宠、规则怪谈”等热词堆叠成卖点。
3. 禁止未确定目标读者和主承诺就起正式书名、简介或第一章。
4. 禁止第一章只铺设定、不证明卖点。
5. 禁止包装承诺正文没有的 CP、爽点、虐点、无敌程度、反转密度或喜剧频率。
6. 禁止为了惊喜而破坏类型底线，除非准备了更强补偿回报。
7. 禁止连续多章推进副承诺，却让主承诺停滞。
8. 禁止把读者反馈简单归因为“读者不懂”，先查承诺是否错配。

## 15. 读者预期与卖点设计卡

```text
【目标平台/频道】
【目标读者】不要写泛人群，写阅读需求。
【类型入口】读者一眼会把它归到什么类型？
【类型底线】这个类型绝不能缺什么？
【主承诺】本书最先欠读者的阅读合同。
【副承诺】最多两个，必须服务主承诺。
【主卖点】有限展示窗口里最该被看见的吸引物。
【熟悉入口】读者熟悉的套路、题材或情绪。
【新鲜变量】这本和同类作品的差异。
【第一章证明】第一章用什么场景证明承诺？
【前 3-6 章证明】卖点如何重复出现并升级？
【前 10 章回报】第一轮小结算是什么？
【长线追读理由】读者 30 章后继续追什么？
【违约风险】错频道 / 错题材 / 标题党 / 类型底线缺失 / 卖点没表达 / 主承诺停滞
```

## 16. 审稿流程

1. 只看书名、封面、标签和简介，写出读者会期待什么。
2. 看第一章，判断主承诺是否出现或被强力指向。
3. 看前三章，判断卖点是否重复证明，而不是一次性噱头。
4. 看前十章，判断是否完成第一轮同币种回报。
5. 看章节列表或阶段大纲，判断每三到五万字是否有新卖点拆解。
6. 对照读者反馈，判断是包装错配、正文违约、回报太慢，还是新鲜变量不够。
7. 输出修改优先级：先修承诺链，再修章节句子。

## 17. 快速自查问题

- 目标读者是否具体到阅读需求，而不是泛标签？
- 主承诺能否一句话说清？
- 第一章是否证明最显眼的卖点？
- 卖点是否在前三章重复出现并升级？
- 前十章有没有同币种回报？
- 熟悉入口和新鲜变量是否同时存在？
- 副承诺是否抢走主承诺的进展？
- 包装吸引来的读者，是否正是正文能留下的读者？
- 读者说“水”“不是这个味”“看不懂”“套路”时，能否定位到承诺链的哪一环？

## 18. 来源与迁移说明

- 起点社区“卖点写作”系列强调卖点是在有限宣传窗口中最该展示给目标读者的东西，卖点需要设计、细化、包装，并在书名、简介、开篇和后续故事中持续表达。本文将其迁移为“目标读者 -> 类型入口 -> 主承诺 -> 可见卖点 -> 第一章证明 -> 前 10 章回报”的承诺链。
- Writing Excuses 10.14 将开篇视为对读者做出承诺，强调读者会根据开篇判断作品会带来什么情绪、人物弧和故事问题。本文将其迁移为“主承诺”“承诺类型”和“开篇承诺验证”。
- Writing Excuses 10.31 强调读者的进展感要和承诺匹配；如果读者等的是关系、谜团或某类体验，正文却推进另一条线，会形成节奏错配。本文将其迁移为“承诺、进展、回报模型”。
- Mary Kole 关于 genre expectations 的文章强调类型预期包括结构、人物、设定、主题和语气，创作者应在满足预期和提供新鲜变化之间取得平衡。本文将其迁移为“类型预期四层”和“微创新”。
- Writer's Digest 关于 target audience 的文章强调商业类型读者和编辑预期较明确，作者需要先判断作品类型、读者期待和包装方式。本文将其迁移为“目标读者不是人口标签，而是阅读需求”。
- Reedsy 关于 target audience 的文章强调应构建具体理想读者画像、从读者视角判断作品，并通过相似作品和读者反馈校正假设。本文将其迁移为“目标读者设计卡”和“预期错配诊断”。


## 与 64 号专题的衔接：卖点要落成场景资产

读者预期、类型承诺和卖点验证不能只停留在书名、简介、标签和大纲层。关键卖点需要在正文中落成可复述场景，详见 `docs/workflows/68-iconic-scene-character-highlight-shareable-moment.md`。

验证一个卖点是否真正进入正文，可以问：读者能否复述一个证明该卖点的具体场景？这个场景是否有人物选择、可记忆载体、状态翻转和后文余波？如果只能说“设定很强”“人设挺爽”，但说不出场景，说明卖点还没有完成文本化。
