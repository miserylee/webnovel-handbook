# 108. 番外、外传、后日谈与读者福利专项

状态：已确认

适用范围：完本番外、卷末番外、CP 糖番外、群像番外、配角外传、反派外传、前传、后日谈、if 线、小剧场、读者福利章、活动番外、周年番外、续作钩子、新书衔接、读者投票番外，以及任务中出现“番外写什么”“能不能用番外补坑”“读者想看婚后/配角/反派/if线”“完本后怎么保持余温”“番外会不会破坏结局”等问题时使用。

核心结论：番外是余味和延展工具，不是正文补丁。AI agent 必须先确认正文主承诺、主冲突、关键伏笔、人物弧和最终情绪已经在正文完成；番外只能补充新常态、关系余味、配角后续、小人物生活、世界一角、读者福利、续作轻钩子或口碑资产回响，不能替正文解决主线、解释真正结局、补主角成长、硬改 CP、洗白反派或临时续命。

最小链路：正文完整性 -> 番外目的 -> 读者期待 -> canon 状态 -> 时间位置 -> 视角选择 -> 冲突强度 -> 余味回报 -> 风险检查 -> 发布沟通 -> 台账回写。

---

## 1. 为什么需要独立专题

第 24 篇已经规定：番外只能补余味，不能替正文收主线。第 66 篇规定完本说明和番外沟通不能替正文解释剧情。但 AI agent 真正执行时，还会遇到更细的问题：

- 读者想看 CP 婚后、配角结局、反派过去、if 线，要不要写？
- 完本后发番外，是否会破坏正文最后一口气？
- 番外可以揭示新真相吗？可以接续作钩子吗？
- 番外是放在正文末尾、单独开篇，还是作为活动福利？
- 读者投票能决定番外内容吗？
- 外传和续作的边界在哪里？
- 番外能不能修复烂尾感？
- if 线会不会污染 canon？

本篇负责把番外、外传、后日谈和读者福利章从“想写就写”变成可判断、可管理、可回写的延展系统。

## 3. 来源吸收与迁移边界

### 3.1 Writer's Digest：epilogue 必须有清楚目的

Writer's Digest 关于 epilogue 的资料强调，后记/尾声需要明确理由，不能只是用来收拾正文应该完成的松散线头；它可以展示高潮后的后果、角色未来、大群像去向或系列后续暗示。

迁移规则：番外必须有独立功能。若只是替正文解释主线、补关键伏笔、证明结局成立，就不是合格番外，而是正文结局失败。

### 3.2 Writer's Digest：denouement 是读者应得的收束

Denouement 资料强调，高潮后需要展示主角最终状态、世界新局面、松散线头处理和未来方向。

迁移规则：中文网文的“终局余波”应优先放在正文结尾和最后一卷完成，不能全部挪到番外。番外可以加深新常态，但不能替代 denouement。

### 3.3 Reedsy：结尾需要平衡收束和余味

Reedsy 关于结尾和 denouement 的资料强调，过少收束会让结尾仓促，过多收束会拖慢余味；结尾应符合类型期待。

迁移规则：番外不是越多越好。正文已经给足收束时，番外要轻；正文仍欠主承诺时，先修正文，而不是加番外。

### 3.4 Writer's Digest：续作要有新问题，不只是重复成功元素

关于 sequel 的写作建议强调，续作需要新的目标、设置、主题或人物问题，不能只是复制第一本的成功模式。

迁移规则：外传和续作钩子必须提出新的冲突引擎。如果番外内容已经需要新主线、新地图、新敌人和新人物弧，它可能不再是番外，而是外传或续作。

### 3.5 平台完本与活动规则

番茄小说作家专区有完本续签、完本奖励和完本相关活动说明，且明确平台活动规则、奖励、违规和信誉要求会随时间变化。

迁移规则：平台规则只作为“完本和番外会影响商业状态”的提醒，不能把当前活动写成永久规则。正式完本、番外发布、奖励申请和续签前必须查看目标平台后台、编辑通知和最新官方公告。

## 4. 概念定义

### 4.1 番外

定义：正文主线结束或阶段结束后，补充余味、关系、配角、小人物、世界一角、读者福利或轻量后续的附加内容。

### 4.2 后日谈

定义：展示主线完成后角色和世界的新常态，重点是“变化后的生活”，不是再开一个同等强度主线。

### 4.3 外传

定义：围绕非主角、过去事件、旁支地图、其他时代或同世界独立冲突展开的延展故事。外传通常比番外更长，拥有自己的目标、冲突和结算。

### 4.4 if 线

定义：与正文 canon 不同的假设路线。必须明确标注非正典或平行可能，不能污染正篇 canon。

### 4.5 读者福利章

定义：为回应读者情绪、纪念节点、活动、完本余温或口碑资产而写的轻量内容。福利章可以甜、暖、搞笑、补小人物后续，但不能牺牲正文完整性。

---

## 5. 番外写作许可

写番外前，必须满足五个条件：

1. 正文主冲突已经解决。
2. 主角人物弧已经在正文完成可见选择。
3. 必须回收的伏笔已经在正文回收。
4. 主要情绪债已经在正文偿还。
5. 番外有明确增值功能，而不是补漏洞。

如果任一条件不满足，先回正文修结局。

## 6. 番外类型表

| 类型 | 适合写什么 | 禁止写什么 |
| --- | --- | --- |
| 糖番外 | CP 日常、婚后、并肩生活、小承诺兑现 | 替正文告白、替正文和解、临时改 CP |
| 友情/群像番外 | 队友新生活、庆功、重逢、小人物去向 | 替正文解释主线结局 |
| 配角番外 | 配角选择后果、独立小回报、未尽关系余味 | 抢走主角终局意义 |
| 反派番外 | 败局后果、过去动机补光、主题反照 | 强行洗白、推翻正文审判 |
| 世界番外 | 新时代一角、制度变化、后人视角 | 临时补世界观漏洞 |
| 搞笑番外 | 熟悉梗变奏、低风险日常、小剧场 | 破坏正篇终局情绪 |
| if 线 | 平行可能、读者趣味、非正典糖/刀 | 改写 canon、不标注状态 |
| 前传外传 | 旧事件、长辈/反派/世界源头 | 替正文解释核心谜底 |
| 续作钩子 | 新问题露头、下一代/新地图一角 | 把本书最终答案留给续作 |
| 活动福利 | 节日、周年、投票小剧场 | 为活动牺牲人物逻辑和 canon |

## 7. 番外不能做的十件事

1. 不能补主线结局。
2. 不能回收正文必须回收的关键伏笔。
3. 不能证明主角人物弧其实已经完成。
4. 不能解释“真正结局”。
5. 不能用作者话或番外替代正文情绪兑现。
6. 不能推翻正篇 CP、价值判断或角色选择，除非明确非正典 if 线。
7. 不能为了续作强行复活最大反派或重置胜利。
8. 不能把读者投票结果写成核心主线决定。
9. 不能为了福利写低质水文、重复章节或无关内容。
10. 不能破坏读者对正文结尾的最后印象。

## 8. 番外设计流程

### 8.1 先定目的

```text
这篇番外要补：糖 / 余味 / 配角 / 世界变化 / 梗点 / 主题回响 / 续作轻钩子 / 活动福利
不是为了补：主线 / 关键伏笔 / 人物弧 / 真正结局
```

### 8.2 再定时间位置

| 时间位置 | 用途 | 风险 |
| --- | --- | --- |
| 结局后数日 | 情绪余波、亲密关系确认 | 太近会像正文少一章 |
| 结局后数月/数年 | 新常态、生活落地 | 容易变成流水账 |
| 过去时间线 | 补人物动机或旧债 | 不能推翻正文理解 |
| 平行 if | 趣味和读者福利 | 必须标注非正典 |
| 下一时代/下一代 | 世界延展和续作感 | 不要让正篇主角退场变廉价 |

### 8.3 再定视角

- 主角视角：适合展示新常态和主角变化。
- CP/关系另一方视角：适合补被爱、被理解、互相选择。
- 配角视角：适合群像余味和主角影响力回声。
- 小人物视角：适合世界改变和口碑传说化。
- 未来人/后人视角：适合历史余韵，但慎防空泛总结。

### 8.4 最后定冲突强度

番外也需要小问题，但不应重开主线级危机。

推荐冲突：

- 日常小误会。
- 旧物归还。
- 婚礼/庆功/重逢中的小选择。
- 配角的新决定。
- 世界规则变化后的一个生活细节。
- 旧称呼、旧梗或旧承诺的新语境。

## 9. 番外的结构

轻量番外可用五拍：

```text
旧回响 -> 新常态 -> 小问题 -> 小选择 -> 余味落点
```

### 9.1 旧回响

用一个旧物、称呼、梗、地点、伤痕、约定或台词唤醒正篇记忆。

### 9.2 新常态

展示主线结局后的生活、关系或世界变化。

### 9.3 小问题

给番外一个轻冲突，不要只是百科式总结。

### 9.4 小选择

让角色用结局后的新状态做一次选择，证明变化真实。

### 9.5 余味落点

落在一句动作、物件、称呼、站位或情绪回响上，不要用作者总结替代。

---

## 10. 后日谈写法

后日谈的重点是“变化后的生活”。

检查：

- 主角现在如何处理过去处理不了的问题？
- 关系现在有哪些新边界和新亲密？
- 世界是否因主线结局产生可见变化？
- 旧伤、旧债、旧承诺是否以更轻的方式回响？
- 读者看完是满足，还是觉得正文少了一章？

后日谈不应变成：

- 全员履历表。
- 作者解释主题。
- 大量“后来他们都很好”。
- 新主线第一章。

## 11. 外传和续作的边界

### 11.1 还是番外

满足以下条件，仍是番外：

- 篇幅较短。
- 依赖正篇情绪和人物关系。
- 冲突强度低于主线。
- 不要求新读者独立进入。
- 不改变正篇结局意义。

### 11.2 应升级为外传

满足以下条件，应作为外传或独立项目：

- 有新主角。
- 有完整目标、阻力、高潮和结算。
- 涉及新地图、新组织、新敌人和新承诺。
- 可独立吸引读者。
- 篇幅超过轻量福利。

### 11.3 应升级为续作

满足以下条件，应作为续作：

- 新冲突会改变世界级格局。
- 原主角或新主角需要完整人物弧。
- 读者需要重新建立主承诺和长期追读理由。
- 不能只靠“正篇读者想看更多”支撑。

## 12. if 线规则

if 线必须明确标注状态：非正典 / 平行线 / 小剧场 / 读者福利。

可以写：

- 如果当年没有错过。
- 如果反派选择另一条路。
- 如果 CP 在现代校园相遇。
- 如果死亡角色拥有一次团圆梦。

不能写：

- 不标注状态，导致读者以为 canon 改了。
- 用 if 线推翻正文代价。
- 用 if 线补正文未兑现的糖或回报。
- 用 if 线引发 CP、结局和价值观混乱。

---

## 13. 读者投票和福利边界

读者可以影响：

- 番外主题优先级。
- 小剧场类型。
- 想看哪组角色日常。
- 哪个梗点回响。
- 是否写节日福利。

读者不能决定：

- 正文核心结局。
- CP 归属。
- 主角价值选择。
- 关键角色生死。
- 必须回收的伏笔答案。
- 世界规则是否推翻。

如果投票结果与 canon 冲突，优先保护 canon。

## 14. 番外发布沟通

### 14.1 推荐说明

```text
这篇是正文完结后的余味番外，只补【关系/日常/配角/世界一角】，不影响正文主线结局。
```

```text
这篇为 if 线小剧场，非正典，写给想看【某种可能】的读者。
```

### 14.2 避免说明

```text
真正结局在番外里。
```

```text
大家投票决定主角最后和谁在一起。
```

```text
正文没填的坑，后面番外慢慢补。
```

### 14.3 不承诺未确认内容

不要提前许诺番外数量、更新时间、续作和新书联动，除非已有稳定计划。

## 15. 番外审稿闸门

### 15.1 正文完整性

- 这篇番外是否在补正文缺失的主线？
- 如果不读番外，正文结局是否仍然完整？
- 番外是否解释了本应在正文解释的关键问题？

### 15.2 角色一致性

- 角色是否延续终局后的新状态？
- 是否为了福利让人物退化？
- CP 互动是否来自关系积累，而不是强行发糖？

### 15.3 情绪余味

- 读者看完是否更满足？
- 是否破坏正篇最后情绪？
- 是否过度消费名场面、梗点或角色痛苦？

### 15.4 canon 和台账

- 番外是否正典？
- 新增事实是否需要写入台账？
- if 线是否明确不污染 canon？
- 续作钩子是否不影响本书完整性？

### 15.5 商业与平台

- 是否违反平台完本、番外、活动或奖励规则？
- 是否为活动发布低质水文？
- 是否过度承诺读者福利？

## 16. 常见错误与修法

| 错误 | 读者体感 | 修法 |
| --- | --- | --- |
| 番外补主线 | 正文烂尾 | 回正文修结局 |
| 糖番外替正文告白 | 正文情绪债没还 | 把告白移回正文，番外写新常态 |
| 配角番外抢主角 | 主线意义被稀释 | 降低配角番外冲突强度 |
| if 线不标注 | canon 混乱 | 明确非正典和平行状态 |
| 续作钩子太重 | 本书不完整 | 只留轻钩子，不留主答案 |
| 番外全员履历 | 像设定表 | 写一个小问题和小选择 |
| 为活动水番外 | 消耗口碑 | 只有明确余味功能才发 |
| 承诺太多写不完 | 伤读者信任 | 只承诺已确定计划 |
| 反派番外强洗白 | 价值观反噬 | 写复杂性，但不推翻后果 |
| 福利章低俗猎奇 | 破坏品牌 | 走第 105 篇敏感内容闸门 |

## 18. 最小规则

时间很少时，至少执行：

1. 正文不完整，不写番外补洞。
2. 番外必须有明确余味功能。
3. 不读番外，正文结局也必须成立。
4. 番外不能替正文完成告白、和解、伏笔回收或人物成长。
5. if 线必须标注非正典。
6. 读者投票只能决定福利方向，不能决定核心主线。
7. 续作钩子不能扣住本书主答案。
8. 番外新增 canon 事实必须回写台账。
9. 不承诺未确定的番外数量和更新时间。
10. 发布前重新确认目标平台的完本、番外和活动规则。
