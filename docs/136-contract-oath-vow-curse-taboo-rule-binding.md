# 136. 契约誓言、禁忌诅咒、规则绑定与违约反噬

状态：已确认  
适用：契约、誓言、誓约、血誓、天道誓言、主仆契约、婚约、系统契约、神明契约、诅咒、禁忌、禁制、规则束缚、怪谈规则、违约反噬、破誓代价、解除契约、转移诅咒、规则漏洞和任何“角色被一条承诺或禁令长期改变行动资格”的剧情。  
核心结论：契约/誓言/禁忌/诅咒不是一句漂亮设定，而是长篇中的规则资产。它必须回答“谁有权绑定、绑定什么、何时触发、如何执行、违约付什么、能否解除、解除后还剩什么账”。绑定规则越强，越要保护角色选择、读者公平和后果入账。

## 1. 本专题解决什么问题

AI 写契约、誓言、禁忌、诅咒和规则绑定时，常见问题不是“设定不够酷”，而是规则没有剧情功能：

- 契约只负责让角色听话，没有谈判、执行、漏洞、违约和余波。
- 誓言只是一句热血台词，后文不验证，也不改变角色选择。
- 禁忌只是世界观装饰，没人害怕、没人绕路、没人利用、没人付代价。
- 诅咒只是惩罚按钮，随机发作，方便作者制造危机。
- 规则绑定太强，角色被规则拖着走，主角主动性消失。
- 解除太便宜，一颗丹药、一句真爱、一场顿悟就清空旧账。

本专题把这些素材统一成“绑定规则链”：

1. 来源：谁或什么有权让规则生效。
2. 条款：规则到底要求、禁止、交换或保护什么。
3. 触发：什么行为、意图、语言、期限或证据会启动它。
4. 执行：规则如何被看见、追踪、惩罚、奖励或扭曲。
5. 代价：履约、违约、钻漏洞和解除分别损失什么。
6. 选择：角色是否仍有可见选择，而不是被设定牵着走。
7. 余波：规则改变了关系、资源、名声、行动资格或世界秩序。

## 3. 来源资料的可迁移结论

### 3.1 魔法/规则系统资料

Helping Writers Become Authors、Reedsy 等写作资料强调：魔法系统和奇幻规则会因为限制不清、代价不明、万能解决问题而削弱故事。迁移到网文：契约、诅咒、誓言和禁忌都应被当成“小型规则系统”，必须有范围、限制、代价、例外和后果。

迁移规则：越强的绑定越不能无成本使用。能控制别人，就必须承担执行成本、反噬风险、伦理风险、漏洞风险或读者信任风险。

### 3.2 诅咒和被诅咒物资料

All Write Alright、The Write Life、Campfire 等资料把诅咒视为会制造压力、诱惑、代价和角色选择的叙事工具，而不是单纯的邪恶装饰。迁移到网文：诅咒要能改变行动资格、关系账、资源消耗和时间压力；解除诅咒必须比“找到解药”更有故事价值。

迁移规则：诅咒最好有“症状、触发、加重、误解、利用、代价、解除后残留”七个阶段。

### 3.3 世界观和禁忌资料

Reedsy 的 worldbuilding template、LawShun 与 Other Atlas 关于 taboos/beliefs 的资料都强调：禁忌来自社会、信仰、法律、身份和文化秩序，必须通过行为后果体现。迁移到网文：禁忌不能只写“这个世界有三大禁忌”，而要写人们如何避开、如何利用、如何惩罚、如何沉默，以及主角为什么不得不触碰。

迁移规则：禁忌不是信息，是压力。读者必须看到触犯前的绕路、触犯时的恐惧和触犯后的代价。

### 3.4 类型功能索引资料

TV Tropes 等类型索引可作为功能参考：誓言、破誓、被诅咒物、禁忌、契约漏洞、惩罚性规则都是常见类型功能。迁移时只吸收功能，不复制桥段。

迁移规则：誓言的价值不在“说出口”，而在后文让角色遇到一个更想违背它的理由。

### 3.5 中文网文语境

起点社区、番茄小说作家专区和中文网文常见题材中，契约、誓言、禁忌、诅咒常用于玄幻仙侠、御兽、规则怪谈、无限流、古言婚约、权谋盟约、系统任务、宗门门规、血脉反噬和重生改命。其商业价值通常来自：限制主角、制造公开见证、延迟回报、逼迫选择、引出漏洞、形成反杀、留下长期钩子。

迁移规则：绑定规则必须服务追读。它要么让读者担心“会不会触发”，要么期待“怎么破解”，要么等着“违约反噬什么时候回收”。

## 4. 绑定规则八线法

设计或审稿时，先填八条线。缺两条以上，本规则通常会变成作者方便工具。

### 4.1 来源线：谁有权让规则生效

常见来源：

- 法律/合同：靠制度、证据、执法、赔偿、名誉和资源冻结执行。
- 宗门/家族/组织：靠门规、族规、职位、身份、资源、惩戒和逐出执行。
- 天道/神明/命运：靠因果、劫数、功德、气运、灾厄、失格和不可见代价执行。
- 系统/副本/怪谈：靠任务、倒计时、惩罚、奖励、抹杀、污染和通关权限执行。
- 魔法/血脉/灵魂：靠印记、反噬、伤害共担、记忆、感知和能力限制执行。
- 社会禁忌/文化禁令：靠群体沉默、舆论、隔离、污名、惩罚和关系切断执行。
- 个人誓言：靠自我约束、荣誉、羞耻、关系信任和后文选择验证执行。

检查问题：这个规则如果被违反，谁会知道？谁会执行？如果没人知道、没人执行、没人承担，它就不是规则，只是气氛。

### 4.2 对象线：绑定了谁和什么

不要只写“他被契约束缚”。要明确绑定对象：

- 人身：行动、位置、身体、寿命、伤痛、自由。
- 能力：法力、技能、系统权限、召唤次数、变身条件、职业资格。
- 信息：秘密、真名、记忆、谎言权限、不能说出的真相。
- 关系：主仆、师徒、婚约、盟友、血亲、共犯、债主债务人。
- 资源：财产、领地、名声、职位、信物、血脉、命格。
- 空间：不能离开、不能进入、必须守护、必须回到某地。
- 时间：期限、倒计时、日夜规则、节气、月圆、任务周期。

检查问题：绑定之后，角色少了哪种行动资格？多了哪种风险或权限？

### 4.3 条款线：规则到底要求什么

条款要可执行，不能只写抽象口号。

弱条款：不得背叛。  
强条款：不得主动向外泄露同盟名单；不得在见证符前说出三句以上假话；不得伤害契约对象的本体；每月初一必须交回一次任务进度；一旦收下令牌，三日内必须抵达审判台。

条款类型：

- 禁止型：不能说、不能去、不能杀、不能离开、不能触碰。
- 必须型：必须保护、必须汇报、必须完成、必须偿还、必须赴约。
- 交换型：获得能力，但支付寿命、记忆、自由、秘密或关系。
- 保护型：被攻击时转移伤害、触发结界、公开证据或召唤见证。
- 标记型：暴露位置、身份、谎言、恶意、血脉或违约状态。
- 追索型：欠债、旧誓、救命恩、婚约、师门令、复仇令持续追踪。

检查问题：条款能不能被一句话转换成“如果 A，就 B，否则 C”？

### 4.4 触发线：什么条件启动它

触发条件越模糊，读者越容易觉得作者作弊。

可用触发：

- 语言触发：说出真名、发誓、否认事实、念出禁句。
- 行为触发：踏入禁地、触碰信物、攻击对象、离开范围。
- 意图触发：起杀心、主动背叛、恶意隐瞒，但这种最容易作弊，要慎用。
- 期限触发：三日、七夜、月圆、任务倒计时、成年礼、卷末审判。
- 见证触发：神像、契约石、族谱、留影、系统日志、众人面前。
- 条件触发：对方先违约、秘密被第三人知晓、血脉觉醒、信物破裂。

写法建议：首次出现时先给低成本触发，让读者看懂规则；关键高潮再用高成本触发。

### 4.5 执行线：规则如何落地

规则必须有可见执行方式。

执行方式：

- 身体反应：疼痛、印记发热、失声、失明、衰弱、伤口重开。
- 能力变化：技能封锁、法力紊乱、召唤失败、系统权限冻结。
- 信息暴露：谎言显形、名字浮现、契约字迹变红、证据公开。
- 社会惩戒：逐出族谱、取消资格、悬赏、封号、封店、失去见证人。
- 空间限制：无法越界、被传送回原地、门不开、路变长、禁地排斥。
- 关系后果：信任破裂、伤害共担、连坐、误会公开、关系降级。
- 命运/因果：好运被扣、劫数提前、旧债转移、下一次选择成本变高。

检查问题：如果把执行方式删掉，剧情是否还成立？如果成立，说明规则没有真正改变故事。

### 4.6 代价线：履约、违约和钻漏洞都要付账

只写违约代价不够。强规则应有四种代价：

- 履约代价：遵守规则会错过机会、暴露身份、消耗资源或伤害关系。
- 违约代价：触发惩罚、失去资格、被追踪、被反噬或欠下新债。
- 利用代价：主角钻漏洞也会失去信任、暴露聪明、改变规则或招来更严约束。
- 解除代价：解除后要还债、失去收益、转移风险、留下残疾、公开秘密或欠人情。

检查问题：主角如果利用规则获利，他付出的账是否足以保护爽点不廉价？

### 4.7 选择线：规则压迫下仍要有人物选择

绑定规则不是为了取消选择，而是为了逼出选择。

有效写法：

- 主角不能说真相，于是选择用行动、物件或第三方证据绕开。
- 主角不能杀仇人，于是设计公开审判，让对方自己触发旧约。
- 主角必须保护契约对象，但可以选择保护方式、优先级和牺牲对象。
- 主角被诅咒越用能力越痛，于是必须决定本章是否值得付这个代价。
- 主角触犯禁忌会失去身份，于是选择在众人面前触犯，把代价转成证词。

无效写法：

- 系统强制主角做任务，主角只能照办。
- 契约强制配角忠诚，配角从此没有欲望。
- 诅咒随机发作，角色只能等作者救。
- 禁忌一触即死，所有人只能避开，无法产生策略。

检查问题：这一条规则有没有让角色做出比无规则时更昂贵、更有辨识度的选择？

### 4.8 余波线：绑定规则必须回写台账

绑定规则改变长期状态，写完必须回写。

台账字段：

- 规则名称：契约、誓言、禁忌、诅咒、禁制、系统任务、天道誓约。
- 规则来源：谁制定、谁见证、谁执行。
- 绑定对象：角色、关系、物品、地点、能力、秘密。
- 当前条款：必须、禁止、交换、保护、标记、追索。
- 已知触发：角色知道什么，读者知道什么，反派知道什么。
- 已发生代价：身体、能力、资源、名声、关系、命运。
- 漏洞/例外：已确认、候选、待确认。
- 解除条件：明确、误读、未知、不可解除、需要代价。
- 下次回收：第几章、哪个场景、哪条关系或哪次高潮。

## 5. 类型适配

### 5.1 玄幻仙侠

常见用法：天道誓言、血誓、心魔誓、师门禁制、禁地规则、功法反噬、神魂契约、因果债。

重点：不要让天道替作者判案。天道规则要有已知边界，主角反杀要靠取证、诱导、选择和代价，而不是“天道自动知道谁坏”。

有效回报：敌人自己破誓反噬，主角因遵守旧誓保住资格，旧禁制在高潮中转成保护，违约者公开失格。

### 5.2 西幻/奇幻

常见用法：巫术诅咒、神明契约、真名束缚、恶魔交易、被诅咒物、血脉禁令、誓约骑士。

重点：魔法规则需要限制。真名、灵魂、诅咒、神约等越强，越要有反制、例外、误读和执行成本。

有效回报：角色用对方条款漏洞换取自由，但失去某段记忆或被神明标记。

### 5.3 古言、宫廷、宅斗和先婚后爱

常见用法：婚约、和离约、族规、宫规、家训、誓言、禁足、名节禁忌、祠堂见证。

重点：社会规则的惩罚往往不是魔法，而是名声、身份、资源和关系网络。不要只写“名声受损”，要写谁不再开门、谁撤回婚书、谁失去见证资格。

有效回报：主角利用族规程序逼出证词，或者主动触犯禁忌换来公开审判资格。

### 5.4 都市、商战和娱乐圈

常见用法：合同、保密协议、竞业限制、对赌、代言条款、公司制度、平台规则、舆论禁区。

重点：现实合同和法律内容只做叙事可信度参考，不写法律建议。剧情上要写执行主体、证据、违约成本和舆论后果。

有效回报：主角不靠嘴炮撕合同，而是找到对方先违约证据，把赔偿、舆论和商业资格一起落袋。

### 5.5 规则怪谈、无限流和副本

常见用法：禁忌规则、违反惩罚、通关契约、身份牌、怪物规则、污染条件、倒计时。

重点：可以让角色不知道规则全貌，但不能让读者觉得规则随作者变。关键规则要有可回看的线索、误读方向和验证成本。

有效回报：主角不是“猜对作者答案”，而是通过小样本试错、队友牺牲、线索对照和风险选择确认规则。

### 5.6 御兽、召唤和伙伴绑定

常见用法：主仆契约、平等契约、血契、灵魂链接、召唤冷却、伤害共担、解除契约。

重点：契约不能等于洗脑。非人伙伴仍要有欲望、感知、压力和选择。越是亲密绑定，越要写照护成本、战斗协同成本和解除风险。

有效回报：伙伴在契约允许范围内做出自主选择，证明关系不是控制。

### 5.7 复仇、审判和清算

常见用法：旧誓、赌约、证词誓约、公开契约、反噬条款、罪证绑定、因果清算。

重点：清算不能全靠规则自动替主角赢。规则最好作为证据容器或执行器，主角仍负责收集证据、诱导对方选择、公开见证和承担余波。

有效回报：仇人以为条款保护自己，最终因旧约漏洞或主动违约自毁。

### 5.8 救赎、和解和关系修复

常见用法：赎罪誓言、保护誓、二次合作契约、权限限制、试用期约束。

重点：誓言不能替代赎罪。一个角色说“我发誓改过”没有价值，后文必须在相似压力下做出不同选择，并接受受害者边界。

有效回报：角色遵守新誓付出具体代价，但不要求受害者立刻原谅。

## 6. 场景写法

### 6.1 首次引入：用小后果教读者规则

不要第一次出现就灌输完整规则表。更有效的是：

1. 一个具体物件：契书、红线、印记、令牌、族谱、系统面板、禁地门。
2. 一个轻微触发：有人说错半句话、越过半步、碰到信物。
3. 一个可见后果：字迹变色、声音消失、伤口裂开、门不开、旁人变脸。
4. 一个不完整解释：角色只解释当前必须知道的部分。
5. 一个待回收问题：更严重违约会怎样？谁曾经违约？怎样解除？

### 6.2 中段推进：让规则产生策略

中段不要反复写“不能违反”。要让规则变成策略资源：

- 角色试探边界。
- 对手利用规则设陷阱。
- 主角用条款漏洞争取时间。
- 配角因旧誓无法帮忙，引发关系误会。
- 诅咒症状变化，提示真相接近。
- 禁忌背后不是迷信，而是旧灾难的残留证据。

### 6.3 高潮回收：让规则成为选择和结算

高潮不是规则自动爆炸，而是角色主动把自己、对手或世界推到规则面前。

可用结构：

1. 前文建立规则。
2. 中段建立误读或漏洞。
3. 高潮让角色选择触发或不触发。
4. 触发后同时支付代价和获得回报。
5. 结果公开或被关键关系看见。
6. 后文回写状态变化。

### 6.4 解除规则：解除不是清零

解除契约、破除诅咒、绕开禁忌时，至少保留一种余波：

- 身体残留：旧伤、印记、感知变化、能力折损。
- 关系残留：信任未恢复、被保护者知道真相、债主换人。
- 资源残留：花掉珍稀资源、失去资格、欠下新债。
- 信息残留：秘密公开、敌人学会规则、组织修改条款。
- 主题残留：角色承认自己不能再用旧方式活下去。

## 7. 读者体感风险

| 问题 | 读者体感 | 修法 |
| --- | --- | --- |
| 契约万能 | 配角像工具，主角像开挂 | 写执行范围、解除条件、伙伴欲望和反制成本 |
| 誓言空喊 | 情绪假，燃不起来 | 后文安排更想违背誓言的压力，再让角色付代价守住或破誓 |
| 禁忌只装饰 | 世界观空，像设定表 | 写日常绕路、社会反应、处罚案例和主角触碰理由 |
| 诅咒随机 | 作者作弊，危机廉价 | 明确症状、触发、加重规律和可验证线索 |
| 违约无代价 | 爽点掉价，规则崩 | 回写身体、资源、关系、名声或行动资格变化 |
| 解除太便宜 | 旧压力清零，追读失望 | 保留残留账、转移账或新规则 |
| 条款太复杂 | 读者看不懂，不想追 | 每次只投放当前场景必需条款，用行动示范 |
| 意图触发作弊 | 作者随意判定人物内心 | 尽量用行为、语言、证据和见证替代纯意图 |
| 禁忌猎奇化 | 冒犯、奇观消费、失真 | 绑定文化、历史、群体经验和人物处境，避免消费现实群体 |
| 规则剥夺主体性 | 主角被任务牵着走 | 让角色选择执行方式、代价承受和漏洞利用 |

## 8. AI agent 执行流程

处理契约、誓言、禁忌、诅咒和规则绑定剧情时，按以下顺序工作：

1. 判定功能：限制、保护、交换、追索、诱惑、惩罚、见证、清算、悬疑、关系绑定。
2. 填八线：来源、对象、条款、触发、执行、代价、选择、余波。
3. 判定读者信息：读者知道哪些规则，角色知道哪些，反派知道哪些。
4. 设计首次示范：用低成本触发展示规则，不灌完整说明书。
5. 设计中段变体：试探、误读、漏洞、利用、反制、加重。
6. 设计高潮回收：角色主动选择是否触发，付出代价并获得回报。
7. 设计解除或长期保留：解除有残留，保留有下次回收点。
8. 回写台账：规则状态、已知触发、已付代价、未回收漏洞、下一次使用。

## 10. 场景审稿清单

- 这个规则有没有明确来源，而不是作者凭空判定？
- 条款是否能被读者理解成具体条件？
- 是否至少展示过一次低成本触发或后果？
- 违约、履约、钻漏洞和解除是否都有代价？
- 规则是否改变了角色行动资格，而不是只做背景气氛？
- 主角是否仍有策略选择，而不是被规则强制推进？
- 对手是否能理解、误读、利用或反制这条规则？
- 解除规则后是否保留余波，而不是清零？
- 禁忌是否嵌入社会反应、文化秩序或组织惩戒？
- 诅咒是否有症状、触发、加重、误读和解除链？
- 誓言是否在后文被更高压力测试？
- 是否需要回写连续性台账、关系台账、能力台账或世界规则台账？

## 11. 常见修法

### 11.1 “契约签完，问题解决”

症状：签约后冲突消失，角色变听话。  
修法：增加执行机制、监督者、违约诱因、条款漏洞和履约成本，让契约成为后续剧情资产。

### 11.2 “发誓很燃，后文忘了”

症状：誓言只在当章制造情绪。  
修法：设置誓言回收点。后文让角色遇到更想违背誓言的利益、恐惧或关系压力。

### 11.3 “禁忌很多，但没人真的害怕”

症状：世界观列了十条禁忌，角色照样乱闯。  
修法：写一个处罚案例、一个绕路习惯、一个利用禁忌牟利的人、一个不得不触碰禁忌的选择。

### 11.4 “诅咒像随机 debuff”

症状：需要危机时就疼，不需要时就消失。  
修法：固定触发条件和加重规律，把症状变化变成线索、倒计时或选择成本。

### 11.5 “主角钻漏洞太轻松”

症状：规则看似强，主角一眼找到漏洞。  
修法：让漏洞有证据成本、试错成本、关系成本和后续修补风险。主角赢一次后，对手或规则制定者要升级规则。

### 11.6 “解除诅咒后没有余波”

症状：旧线被一章清空。  
修法：解除后保留残留状态，把旧诅咒转成新的关系、能力、名声、欠债或敌人反应。

## 12. 可复用案例骨架

### 12.1 旧誓反杀

前文：反派逼主角在宗门石前发誓不得泄露某件事。  
中段：主角不能直接说真相，只能收集不受誓言限制的旁证。  
高潮：主角引反派自己在同一宗门石前否认关键事实，反派触发旧誓反噬。  
余波：主角没违誓，但失去某个能直接解释真相的机会，关系误会延迟回收。

### 12.2 禁忌触碰

前文：村中禁忌是夜里不能叫死者名字。  
中段：主角发现大家不是迷信，而是有人用禁忌掩盖失踪案。  
高潮：主角在众人面前叫出名字，触发旧禁制，也逼出隐藏见证。  
余波：主角被村人排斥，但获得进入祠堂的资格。

### 12.3 诅咒代价

前文：主角每使用一次血脉能力，手腕黑线延长一寸。  
中段：黑线不是单纯寿命扣除，而是追踪标记，敌人能通过它定位。  
高潮：主角故意使用能力暴露自己，引敌人进入反杀局。  
余波：追踪解除，但黑线残留让主角失去一次安全潜入资格。

### 12.4 契约关系反转

前文：灵宠契约规定灵宠必须回应召唤。  
中段：主角发现强召会让灵宠承担伤害共振。  
高潮：主角放弃强召，改用平等契约的“求援”条款，给灵宠选择权。  
余波：灵宠主动回应，关系升级，但主角失去一次强制召唤权限。

## 13. 写作禁区

- 不把现实宗教、民族、地域或弱势群体的禁忌猎奇化、妖魔化或随意拼贴。
- 不把现实合同、法律程序写成法律建议；只保留叙事层面的证据、执行和后果。
- 不把精神控制、强迫服从和剥夺意志包装成浪漫关系。
- 不让契约替角色赎罪。誓言、契约、赎罪条款只能开启验证，不能自动清账。
- 不让规则为了主角临时改口。规则可以被误读、钻漏洞、升级，但必须能回看。
- 不为了黑暗感堆叠血腥、酷刑、身体伤害或不可逆创伤细节。

## 14. 与台账和多 agent 交接的联动

多 agent 接力时，只要出现绑定规则，交接包必须包含：

```text
【绑定规则状态】
规则名称：
规则来源：
绑定对象：
条款摘要：
已知触发：
已知代价：
读者已知：
角色已知：
反派已知：
当前漏洞/误读：
下一次回收：
禁止误写：
```

禁止下游 agent 私自修改规则触发条件、代价和解除方式。需要修改时，必须在项目台账或决策记录中标为“待确认”，并说明原因。

## 15. 一句话总规则

契约让承诺有执行，誓言让价值接受测试，禁忌让世界拥有边界，诅咒让代价持续追踪。AI agent 写它们时，不要只问“这个设定酷不酷”，而要问：它让谁少了选择、谁多了风险、谁付了代价、谁获得了回报、谁在下一章不能再假装没发生过。
