# 项目索引

## 项目定位

- 项目名：学习写网文
- 项目类型：网文写作知识库 / AI 写作权威参考资料
- 当前用途：为后续 AI 起草、改稿、审稿、搭建小说项目、维护长篇连续性提供可复用标准。
- 核心要求：不只总结结论，要沉淀详细内容参考，包括方法、适用场景、风险、检查问题和案例经验。

## 当前状态

- 已初始化基础目录：`docs/`、`templates/`、`cases/`、`prompts/`、`notes/`。
- 已吸收本地 `novel` skill 的通用框架：项目流程、澄清流程、网文写作、人物塑造、连续性台账。
- 已抽取旧项目经验来源：`许愿池修仙`、`人均猛将`、`编辑部`、`读者群`。
- 已根据试写问题新增“人味增强”专题：人物鲜活、剧情自然、读者黏着、梗点和情绪浓度。
- 本项目定位为知识库，因此允许沉淀通用方法；这一点不同于单本小说项目。

## 下一步

1. 后续如果有新的试读、审稿或章节修订经验，把它追加到对应专题文档和案例笔记。
2. 如果准备让 AI 写某一本具体小说，先用 `docs/03-project-workflow.md` 创建单本小说项目，再把本知识库作为通用参考。
3. 如果某条规则被实际写作证明不适用，更新对应专题，并在 `docs/99-decision-log.md` 记录原因。

## 维护规则

- 每条新规则尽量回答：为什么、怎么用、何时不用、如何检查。
- 案例经验要写清来源和可迁移部分，避免把某本书的专属设定误当通用规律。
- 任何要给 AI 作为硬约束的内容，必须进入正式文档，不能只留在 `notes/`。
