## 维护规则

- 新增稳定结论要写进专题文档，不只留在聊天里。
- 新增来源要登记到 `docs/01-source-inventory.md`。
- 新增关键决策要记录到 `docs/99-decision-log.md`。
- 临时想法可以先放 `notes/`，但不能替代正式文档。
- 状态标签使用“已确认”“候选”“待确认”。
- 文件读写使用 UTF-8，文档默认中文。
- 不创建 `final`、`v2` 这类重复文档名；需要版本脉络时写进决策记录或案例笔记。

## 索引维护规则

- README 面向快速接续，保持任务路线和分组地图。
- `00-index.md` 面向 agent 导航，保持更详细的阅读顺序和专题分组。
- 专题新增后，必须同时更新 README、`00-index.md`、`01-source-inventory.md`、`10-templates-and-checklists.md`（如有模板）和 `99-decision-log.md`。
- 如果某专题只适合特定任务，不要塞进所有阅读路径；用“按任务读取”分流。
- 当专题数量继续增加或任务容易横向发散时，优先读取 `docs/57-knowledge-base-routing-consolidation-guide.md`，先生成最小阅读包，再决定是否需要读取细分专题。
