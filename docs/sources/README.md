# sources

状态：已确认

用途：来源登记、来源边界、引用核查和资料转化入口。

本目录只有一个大文件：`01-source-inventory.md`。不要默认整读。只有在新增来源、核查来源、处理引用边界、来源冲突、开源去身份化或资料转化时，才按关键词检索命中小节。

## 1. 什么时候读

| 任务 | 先读 | 作用 |
|---|---|---|
| 新增联网资料、平台规则、论文、官方文件 | `01-source-inventory.md` 命中主题 + `../../SOURCE_POLICY.md` | 登记来源、说明吸收点和使用边界 |
| 引入私有项目经验、编辑反馈、读者反馈 | `../../DEIDENTIFICATION.md` + `01-source-inventory.md` 匿名样本区域 | 去身份化后只保留可迁移经验 |

| 处理医疗、金融、法律、平台规则、公共安全等时效性内容 | `../safety-research/111-research-fact-checking-source-evaluation-fiction-transformation.md` + 目标来源小节 | 核查来源可靠性和时效性 |

## 2. 来源登记最小字段

新增稳定来源时，至少写清楚：

- 来源标题或匿名来源标签。
- 链接、出处或“用户提供资料/匿名样本”类型。
- 记录日期或访问日期。
- 使用目的：它解决哪个写作、审稿、项目管理或安全问题。
- 吸收方式：沉淀到了哪个专题、模板、索引或规则。
- 使用边界：哪些内容不能复制、不能扩展、需要重新核查或只作候选。

不要把来源清单写成链接仓库。每条来源都必须说明它如何帮助小说创作或 agent 工作流。

## 3. 来源类型处理

| 来源类型 | 入库方式 | 注意 |
|---|---|---|
| 官方公开资料 | 摘要为流程、边界、术语和事实核查点 | 涉及时效性时，正式使用前重新确认 |
| 平台规则和后台截图 | 抽象为当前可见字段、上传要求和风险提醒 | 不升格为永久规则 |
| 研究论文、行业报告 | 提炼可迁移方法和限制 | 不把论文结论当小说公式 |
| 公开写作文章和访谈 | 提炼方法论、检查问题和误区 | 不大段复制原文 |
| 外部小说短句 | 只做单句级功能拆解和原创练习 | 不扩展为片段库 |
| 匿名旧项目经验 | 去身份化后写成原则、用法、风险和检查卡 | 不保留项目名、路径、人物和正文 |
| 编辑/读者反馈 | 聚类成问题类型和可迁移规则 | 不提交原话、截图、昵称、群名 |
| 用户当前指令 | 可作为本轮决策依据 | 稳定规则要写入专题或决策记录 |

## 4. 禁止入库

- 未授权付费课程原文、内部资料、未公开稿件。
- 真实读者反馈原文、编辑内部意见原文、完整聊天记录。
- 本机绝对路径、私有仓库地址、群名、账号、联系方式、截图原图。
- 大段复制的文章、书籍、小说片段、平台规则或课程内容。
- 可操作违法、规避监管、侵犯隐私、诈骗、攻击、公共安全破坏或伤害他人的材料。

不确定能不能公开时，默认不入库；只保留抽象经验，或放在本地未提交材料里。

## 5. 推荐流程

### 5.1 联网资料沉淀

```text
明确任务 -> 查来源 -> 111 来源评估 -> 102 来源转化 -> 写入目标专题 -> 登记 01-source-inventory.md
```

### 5.2 私有经验沉淀

```text
识别私有来源 -> DEIDENTIFICATION 去身份化 -> 抽象为可迁移经验 -> 写入专题/模板 -> 登记匿名来源边界
```

### 5.3 来源冲突处理

```text
找到冲突来源 -> 标注时间、权威层级和适用范围 -> 查 99-decision-log -> 按最新用户决定或已确认规则处理
```

## 6. 常见误用

- 为了写正文整读来源清单；正文应先读专题规则，必要时再回源。
- 只贴链接不写吸收点；后续 agent 不知道这个来源为什么存在。
- 把平台某一时点后台截图写成所有平台永久规则。
- 把外部小说片段、课程材料或文章观点改几个词后写进知识库。
- 去掉姓名但保留路径、群名、项目结构和时间线，导致仍可被拼接识别。
