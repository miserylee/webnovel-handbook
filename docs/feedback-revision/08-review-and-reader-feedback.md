# 审稿与读者反馈

写手、试读、审稿与改稿一体化入口：

- 如果审稿不是单独交付意见，而是要在写稿流程里自动完成“试读 subagent -> 审稿 subagent -> 反馈仲裁 -> 修订 agent”，先看 `docs/workflows/58-integrated-drafting-beta-review-revision-workflow.md`。本篇继续提供审稿标准和读者反馈判断，但一体化写稿时不再要求用户手动切换读者群或匿名编辑反馈样本。

反馈冲突入口：

- 如果编辑反馈、读者反馈、后台数据、作者直觉或项目主承诺互相冲突，先看 `docs/feedback-revision/42-editor-reader-data-feedback-conflict-resolution.md`。该专题负责判断来源权重、证据同向性、主承诺保护、修改成本和观察窗口，避免简单投票或盲从单一反馈。


评论语料入口：

- 如果任务涉及读者评论、章评、书评、试读反馈、匿名读者反馈或编辑反馈的整理与取舍，先看 `docs/feedback-revision/41-reader-comment-coding-feedback-clustering.md`。该专题负责把评论原文拆成观察点、打标签、聚类、判断证据强度，并转成可观察的修改动作。


相关入口：

- 连载期数据已经出现异常、推荐期需要判断要不要改、上架前后需要设置观察窗口，先看 `docs/feedback-revision/40-serial-data-tuning-loop.md`。


## 相关入口：开篇试写与 A/B 版本评估

审稿遇到多个开篇方案、第一章重写或读者反馈分歧时，调用 `docs/feedback-revision/39-opening-variant-ab-evaluation.md`。审稿结论应基于类型承诺、第一屏钩子、主角可追随性和下一章欲望，而不是只判断文笔。

## 相关入口：章节生产流水线与多 agent 交接规范

审稿、试读、发布前质检或多 agent 对照审稿时，先调用 [38. 章节生产流水线与多 agent 交接规范](../workflows/38-chapter-production-pipeline-agent-handoff.md)。审稿 agent 必须先列有效资产，再判断问题层级和修改优先级；不能边审边乱改，也不能把多份审稿意见未经聚类就全部塞给修订 agent。

## 相关入口：AI agents 小说创作总流程

开新书、续写、起草章节、审稿、修订、数据复盘、完本或维护知识库前，先调用 [35. AI agents 小说创作总流程](../workflows/35-ai-agent-novel-creation-workflow.md)。该专题负责判断当前阶段、必读文档、退出产物和进入下一阶段门槛，避免跳过调研、读者承诺、核心资产、台账或审稿直接写正文。


## 相关入口：修订、重写与版本控制工作流

改稿、重写、审稿后执行修改、处理读者反馈或修复 bug 时，先调用 [33. 修订、重写与版本控制工作流](./33-revision-rewrite-version-workflow.md)。该专题负责判断问题层级、小修/中修/大修边界、有效资产保留、已发布章节风险和台账同步，避免只润句子、不解决结构问题，或把有风格的文本改成 AI 平滑文。


## 相关入口：章法、段落与移动端阅读体验

起草、修订、审稿或发布前检查章节时，先调用 [32. 章法、段落与移动端阅读体验](../core-writing/32-chapter-paragraph-mobile-reading-experience.md)。该专题负责检查章节主任务、场景目标阻力转折、段落功能、移动端阅读呼吸、转场压缩、章末牵引和章节名体验，避免密集段落、假节奏、无进展屏幕和只扣答案不回报。


## 相关入口：读者预期、类型承诺与卖点验证

开新书、改包装、写开篇、拆题材、做融合题材、审稿或分析读者反馈前，先调用 [29. 读者预期、类型承诺与卖点验证](../market-serialization/29-reader-expectation-genre-promise-selling-point.md)。该专题负责确认目标读者、类型底线、主承诺、主卖点、第一章证明、前 3-6 章证明和前 10 章回报，避免包装吸引错人、卖点没有表达或章节推进错承诺。


本文用于评估网文项目、章节、投稿材料和 AI 生成稿。它整合了编辑部审稿标准和匿名读者试读规则。

## 完本与烂尾审稿入口

审稿对象如果是最后一卷、大结局、番外、完本作品或被读者评价为“烂尾”的作品，必须联动 `docs/storycraft/24-ending-completion-risk-control.md`。判断重点不是作者有没有写到最后，而是开篇承诺、全书旋涡、主冲突、关键伏笔、人物弧和最终情绪是否完成。

## 情绪线审稿入口

审稿对象如果被评价为“感情线尬”“没有 CP 感”“只撒糖不心动”“误会硬拖”“女主工具化”“虐而无补偿”，必须联动 `docs/storycraft/25-emotion-line-relationship-progression.md`。判断重点是关系状态是否变化、情绪债是否成立、互动是否有特殊性，以及主线和关系线是否互相改变。

## 悬疑反转审稿入口

审稿对象如果被评价为“看不懂”“谜语人”“反转太硬”“伏笔没回”“全靠最后解释”“规则随机变”，必须联动 `docs/storycraft/26-suspense-reversal-information-control.md`。判断重点是读者是否知道自己在追什么、真线索是否公平出现、红鲱鱼是否有功能、反转是否满足铺垫/误读/重释/后果。

## 1. 审稿的三件核心事

一篇网文是否值得继续，核心看：

1. 类型承诺是否清楚：读者一眼知道会获得什么体验。
2. 连载动力是否稳定：主角欲望、外部压力和阶段目标能否持续制造追读。
3. 文本执行是否承接卖点：正文是否把设定和大纲潜力落成场景、冲突和情绪。

这三点要同时看。只有设定好，不代表能连载；只有文笔顺，不代表有追读；只有大纲热闹，不代表正文能落地。

## 2. 设定审查

重点看：

- 核心卖点是否一句话能说清。
- 世界规则是否主动制造冲突，而不是只做背景介绍。
- 金手指、职业、系统、修炼、案件、经营或感情机制是否有升级空间。
- 设定是否方便开篇快速进入。
- 差异化是否具体，而不是只说“宏大世界观”“反套路”“群像史诗”。

常见风险：

- 设定很大，但和主角当下困境无关。
- 规则复杂，却不能带来选择、代价、爽点或悬念。
- 金手指过强导致压力消失，或过弱导致期待不足。
- 卖点只能靠作者解释，正文里呈现不出来。

审稿输出要写：

- 卖点是否可转述。
- 规则如何制造冲突。
- 开篇能否直接进入。
- 最大商业风险是什么。

## 3. 大纲审查

重点看：

- 主线目标是否清楚。
- 阶段目标是否递进。
- 每个阶段是否有对手、阻碍、代价和成果。
- 爽点、悬念、反转和情绪释放是否有节奏。
- 主角成长是否通过选择和行动体现。
- 长线伏笔是否服务追读。

常见风险：

- 大纲只有事件列表，没有因果推进。
- 主角被剧情推着走，缺少主动欲望。
- 阶段结尾没有新问题或更高压力。
- 支线收益不清，拖慢连载。
- 总纲看起来过关，但卷级细纲不可执行。

可执行建议：

- 为每个阶段补“目标、阻碍、结果、新压力”。
- 为前 10 章补“场景问题、主角动作、阻力、落袋、钩子”。
- 把长线真相拆成弱线索，不要一次解释完。

## 4. 正文审查

重点看：

- 第一章是否迅速建立人物处境、目标、压力和期待。
- 场景是否有即时矛盾。
- 信息投放是否跟着行动走。
- 对白是否带目的、关系和压力。
- 章末是否有钩子。
- 语言是否顺畅、具体、有画面。

常见风险：

- 开篇从世界观、履历、身份说明开始。
- 爽点来得太晚。
- 情绪靠旁白解释，行动和选择不足。
- 每章结尾只是事情结束，没有新期待。
- 证据链太顺，像纪要，不像冲突现场。

审稿时要区分：

- 读者知道发生了什么。
- 读者想知道接下来会怎样。

后者更关键。

## 5. AI 含量判断

AI 含量指“疑似 AI 生成或重度 AI 润色痕迹”，不是绝对鉴定。

必须基于文本证据：

- 句式、段落长度、转折方式高度均质。
- 万能总结、抽象概括、“正确但不具体”的表达反复出现。
- 情绪主要靠旁白解释。
- 人物声音同质。
- 设定说明像百科条目或提示词展开。
- 情节表面顺滑但因果松散。
- 模板化连接词、排比归纳、同义反复、空泛升华过多。

等级：

- `低`：不影响判断。
- `中`：局部有痕迹，但核心场景、人物目标和叙事判断清楚。
- `高`：人物和冲突被明显削平，需要大幅改稿。
- `极高`：整体像提示词扩写，缺少作者判断和可追读场景。

审稿原则：

- 不用单个词或单句判定。
- 必须说明文本现象如何影响读者体验、签约风险或改稿成本。
- 材料不足时写“无法充分判断”，并说明需要补哪些正文样本。

## 5.1 审稿反例：这些输出本身不合格

审稿 agent 不只要判断稿件，也要避免输出“看似专业、实际不可执行”的审稿话术。以下反例用于自查审稿质量。

| 错误审稿反例 | 为什么不合格 | 应改成的审查动作 |
| --- | --- | --- |
| “整体还可以，节奏再加强一点。” | 没有指出哪一段节奏弱，也没有说明是开场慢、场景无目标、回报晚还是章末无钩 | 标出具体段落，判断它缺的是目标、阻力、转折、落袋还是下一章牵引 |
| “文笔流畅，问题不大。” | 把通顺当作合格，可能漏掉主角无主动选择、信息空降、AI 味重等结构问题 | 先查类型承诺、主角动作、因果链、人物声音，再评价语言 |
| “人物可以更鲜活。” | 只给抽象方向，修订 agent 不知道该补什么 | 指出人物在哪个场景只承担功能，并要求补自保动作、误判、关系欠账或压力反应 |
| “这里 AI 味有点重，润一下。” | 把 AI 味当词句问题，容易越改越平 | 先判断是人物同声、情绪靠旁白、场景像纪要、因果太顺还是抽象总结，再给对应改法 |
| “逻辑有点怪。” | 没有定位前后错位点，无法修复 | 写清“哪句话从哪里跳来、缺少哪个信息来源、哪个角色知道了不该知道的事” |
| “这段可以删。” | 没说明删掉后保留什么功能，可能误伤伏笔、情绪或节奏 | 说明该段是否无功能；如果有功能但位置错，给出移位或压缩方案 |
| “读者应该能懂。” | 用作者脑内补全替代冷读者体验 | 站在第一次阅读者视角，列出当前文本已经明示的信息和仍需补足的信息 |
| “反派有点降智。” | 结论正确但不够可执行 | 指出反派哪一步没有利益、信息或风险依据，并补一个合理阻力动作 |
| “爽点不够。” | 没区分是压力不足、打脸无见证、回报未落袋还是对手代价太轻 | 按压力、行动、见证、落袋、反弹五项定位缺口 |
| “这一章没问题，可以继续。” | 如果没有列有效资产和风险，就可能只是放行话术 | 至少列出有效资产、最大风险、是否有交付阻塞和下一章必须承接 |

审稿输出最低要求：

- 每个主要问题必须有证据位置。
- 每条建议必须能转化为删除、补因果、移位、改对话、补动作、补回报或暂停重构之一。
- 不允许只用“加强、优化、丰满、自然一点、再打磨”作为最终建议。
- 如果审稿 agent 找不到具体证据，应写“当前材料不足以判断”，不能硬凑结论。

## 5.2 文本问题命中反例

以下反例不是审稿话术反例，而是稿文本身的坏例。审稿 agent 看到类似文本时，应直接命中对应问题，不要只评价“略显生硬”。

### 反例 1：逻辑跳跃

```text
林照刚把账册合上，忽然明白所有人都在骗她。

她抬头看向长老，冷声道：“原来三年前那场火，也是你安排的。”
```

命中问题：

- `忽然明白` 没有推理过程。
- `三年前那场火` 没有前置信息来源。
- 角色直接知道作者想揭示的答案。

审查动作：

- 要求补出证据来源、误导线索或角色推理链。
- 如果暂时不能揭示，应改成怀疑或试探，而不是直接定罪。

### 反例 2：信息空降

```text
门外传来脚步声。

来的人正是失踪十年的国师，他手里握着能证明女主清白的玉佩。
```

命中问题：

- 国师失踪、玉佩功能、证明清白的规则同时空降。
- 新人物和关键证据一起出现，替主角解决问题。
- 读者没有参与期待、铺垫或判断。

审查动作：

- 拆分为预置线索、受阻获取、争夺证据和部分证明。
- 关键证据不能一出现就完成全部清算。

### 反例 3：角色知识越界

```text
小师妹站在人群后，脸色发白。

她知道，只要林照把第三页翻开，自己借命的秘密就藏不住了。
```

命中问题：

- 小师妹为什么知道第三页内容，没有交代。
- 如果她不该知道账册结构，这就是角色知识越界。
- 如果她知道，前文必须有偷看、参与、签字、被告知等来源。

审查动作：

- 标明角色信息来源。
- 如果没有来源，改成身体反应或误判，不直接写“她知道”。

### 反例 4：动作顺序错误

```text
她冲出大殿，反手把门闩插上。

殿内众人被关在里面，长老的脸色终于变了。
```

命中问题：

- 人已经冲出大殿，通常无法从外侧插内门闩。
- 长老在殿内的脸色是否还能被她看见不清楚。
- 空间、视线和动作顺序不成立。

审查动作：

- 检查门的位置、门闩朝向、人物站位和视线。
- 改成她先退到门边、拉下外侧铜栓，或让殿内另一个人完成关门动作。

### 反例 5：无功能漂亮句

```text
风从廊下穿过，像一只迟来的手，轻轻抚平所有沉默。

众人继续看着账册。
```

命中问题：

- 意象没有改变动作、信息、人物、情绪或节奏。
- 下一句没有承接“沉默被抚平”的结果。
- 漂亮句孤立存在，像润色器自动加氛围。

审查动作：

- 问这句是否承担转场、情绪压迫、人物反应或伏笔功能。
- 无功能则删；如果要保留，必须让风影响物件、声音、视线或人物反应。

### 反例 6：AI 味情绪旁白

```text
她终于明白，真正可怕的不是背叛，而是人心的凉薄。

这一刻，她彻底成长了。
```

命中问题：

- 抽象总结替代场景动作。
- 情绪和成长由旁白宣布，没有选择、代价或行为证据。
- “不是 X，而是 Y”式升华模板化。

审查动作：

- 改为人物动作、沉默、选择或关系后果。
- 成长必须通过下一步行为证明，不能由旁白盖章。

### 反例 7：全员同声同气

```text
众人纷纷震惊。

“怎么会这样？”大师兄说。

“怎么会这样？”二师姐说。

“怎么会这样？”执事也说。
```

命中问题：

- 所有人反应一致，没有利益差异。
- 对话只是重复读者已知情绪。
- 群像变成合唱团。

审查动作：

- 按利益分层：维稳者、自保者、抢解释权者、沉默观察者、趁机切割者。
- 每个人的第一反应应服务自己的目标。

### 反例 8：证据链像纪要

```text
系统显示，长老三年前修改记录，小师妹借走灵力，执事负责遮掩，大师兄知情不报。

众人看完，全都无话可说。
```

命中问题：

- 证据一次性给全答案。
- 对手没有解释、反咬、抢夺、销毁或转移视线。
- 冲突现场变成结果公告。

审查动作：

- 证据只能给部分信息。
- 必须安排解释权争夺、证据可信度攻击或主角逼问关键承认。

### 反例 9：假章末钩子

```text
真正的风暴，才刚刚开始。

她知道，明天会更危险。
```

命中问题：

- 只有预告，没有具体未完成动作。
- 没有新问题、新物件、新来人、新代价或新选择。
- 读者不知道下一章具体想看什么。

审查动作：

- 改成可承接动作：门被推开、名单出现、证据缺页、对手先一步出招、旧伤被点名。
- 章末要同时留下外部动作和内部情绪。

### 反例 10：主角被剧情推走

```text
林照本来不想去后山。

但剧情已经到了这里，她还是去了。
```

命中问题：

- 人物行动没有欲望、压力、代价或信息触发。
- “剧情需要”直接暴露。
- 主角主动性消失。

审查动作：

- 补一个迫使她去的理由：证据即将被毁、某人以关系相逼、她必须抢先确认、后山有只对她重要的旧物。
- 行动必须来自人物目标或压力，不来自作者调度。

### 反例 11：反派降智送证据

```text
长老冷笑一声，把真正的账册扔到她面前。

“你不是想查吗？给你。”
```

命中问题：

- 反派主动交出致命证据，没有利益理由。
- 缺少陷阱、误导、交换条件或自信依据。
- 主角不用行动就获得关键资源。

审查动作：

- 给反派一个合理动机：账册被篡改过、只能证明小罪、里面藏着反制、交出副本是为了引主角犯规。
- 让主角必须识破、争夺或承担代价。

### 反例 12：爽点无见证、回报未落袋

```text
她赢了。

那些曾经看不起她的人，终于都后悔了。
```

命中问题：

- 没有写出赢在哪里、谁见证、对手付出什么代价。
- “后悔了”是作者替读者结算。
- 回报没有入账凭证。

审查动作：

- 写清回报币种：身份恢复、资源到账、规则改变、关系站队、公开道歉、对手失去资格。
- 让见证者产生分层反应，不能只用一句话概括。

## 6. 结论分级

可用分级：

- `A`：方向明确，卖点可见，文本已有追读潜力，可重点跟进。
- `B`：有潜力，但需要针对关键问题改稿后再判断。
- `C`：局部有亮点，整体商业承诺或执行不足，谨慎跟进。
- `D`：核心问题较重，当前不适合投入编辑资源。

结论不要只给等级，要说明：

- 最大有效点。
- 最大风险点。
- 最优先的三条修改建议。
- 是否建议继续写下一章，还是先补大纲/修开篇。

## 6.1 AI 审稿评分表

评分表用于帮助 AI agents 稳定审稿，不是用数字替代判断。每一项扣分都必须绑定文本证据和读者影响。

总分 100 分：

| 维度 | 分值 | 重点看什么 | 低分信号 |
| --- | --- | --- | --- |
| 类型承诺 | 20 | 卖点是否清楚，开篇是否让读者知道会获得什么体验 | 卖点只能靠简介解释，正文看不出类型；开篇支线热闹但主承诺缺席 |
| 连载动力 | 20 | 主角欲望、外部压力、阶段目标和可重复机制是否能继续制造章节 | 只有事件列表，没有阶段目标；主角被推着走；阶段结尾没有新压力 |
| 文本执行 | 20 | 正文是否把设定、大纲和人物落成场景、冲突、行动和情绪 | 大段说明、冲突现场像纪要、人物只负责讲设定、章末没有追读动作 |
| 人物与关系 | 15 | 主角是否有主动选择，主要人物是否有差异化目标、声音和压力反应 | 人物同声同气，全员同步理解或震惊；反派/配角只做工具人 |
| AI 含量风险 | 15 | 是否存在均质表达、提示词扩写感、结构模板化、空洞水文和长线失控 | 句式重复、万能总结、提纲式正文、复述剧情、文风杂糅、前后矛盾 |
| 修改成本 | 10 | 问题是局部终磨、章节返修、前 10 章重排，还是设定/主线级返工 | 核心卖点不成立、开篇无法进入、卷级细纲不可执行、需要推倒重搭 |

分数对应建议：

- `85-100`：A 档。方向明确，可进入重点跟进或发表前终磨。
- `70-84`：B 档。有潜力，但需按优先级改关键问题后再判断。
- `55-69`：C 档。局部有亮点，但商业承诺、连载动力或执行质量不足，谨慎跟进。
- `0-54`：D 档。核心问题较重，当前不适合投入较多编辑或写作资源。

一票降级规则：

- 如果开篇材料无法看出类型承诺，最高不超过 `C`。
- 如果主角连续数章没有主动目标和选择，最高不超过 `C`。
- 如果正文大量是提纲式扩写、复述剧情、模板化拼接或空洞水文，最高不超过 `D`。
- 如果材料不足以判断，例如只有设定没有正文，不能强行打完整分；应标为“无法充分判断”，并说明需要补哪些样本。
- 如果 AI 含量风险高但核心场景和人物目标仍成立，可以不直接判死刑；先判断是局部降噪、章节返修，还是结构性重写。

AI 审稿输出必须包含：

- 总分和分档。
- 六个维度的分数。
- 每个主要扣分项的文本证据。
- 这些问题对追读、签约风险或改稿成本的影响。
- 前三条修改优先级。
- 建议继续写下一章、先修开篇、先补大纲，还是暂缓。

## 7. 匿名读者试读

读者群不是文学研讨会，重点是“读起来什么感觉”。

默认输出：

1. `群内速判`：一句话整体体感。
2. `开麦短评`：8 到 12 个读者发言。
3. `分歧点`：群里吵起来的地方。
4. `追读建议`：适合谁、不适合谁、是否值得继续。
5. `可沉淀记忆`：以后可拿来对比的点。
6. `记录回执`：是否写入记录。

体感标签：

- `能追`：有明显继续读动力。
- `口粮`：不惊艳，但能看。
- `挑读者`：优缺点明显，适合特定口味。
- `毒点重`：雷点足以劝退一批人。
- `弃了`：当前材料追读欲很低。

匿名读者反馈要保留分歧。少数派可能指出某类读者的毒点，不能被平均结论抹掉。

## 8. 试读后如何沉淀

正式试读后记录：

- 读的是哪一版、哪些章节。
- 哪些读者参与。
- 每个读者的大致体感。
- 主要共识和分歧。
- 下一版最该改什么。

再把稳定经验转入：

- 单本项目的 `docs/09-reader-feedback.md`。
- 本知识库的 `docs/feedback-revision/09-case-notes.md`，如果经验有通用价值。
- 写作指南或节奏文档，如果规则要影响后续写作。

不要让读者反馈只停留在聊天里。尤其是“下一章必须兑现什么”“某个套路已经疲劳”“某个人物存在感下降”这类信息，必须写入项目文档。

## 9. 发布后数据与评论复盘

如果作品已经发布或进入推荐、试读、活动、榜单、上架等阶段，审稿不能只看正文体感，还要联动 `docs/feedback-revision/18-reader-data-feedback-diagnosis.md`。

复盘时优先判断：

- 点击低：先看书名、封面、简介、标签和平台/频道适配。
- 第一章读完差：先看开篇是否兑现包装最强承诺。
- 前 3 章掉：先看题材、机制、关系和主冲突是否被证明。
- 前 10 章追读弱：先看第一轮回报是否落袋。
- 中期追读突然掉：先找掉点前后 3-5 章的剧情、更新和评论变化。
- 评论集中负面：先做评论聚类，不要被单条情绪化评论牵着走。
- 评论集中正面：把有效卖点沉淀为后续可重复变体。

数据复盘输出必须区分“包装问题”“开篇问题”“当前剧情问题”“更新节奏问题”“评论区互动问题”和“数据不足”。没有真实数据时，不能编造点击率、读完率、追读率、收藏转化或评论比例。


## 审稿专项：成片短句与机械断句

审稿时要把“成片短句”视为 AI 味高风险，不要因为它读起来清楚就放行。短句密集只有在动作急促、人物受惊、喜剧包袱、强转折、情绪断裂时成立；如果普通叙述也全靠短句推进，就是节奏失控。

### 命中反例

```text
他走进大厅。
灯亮着。
人很多。
他们都看着他。
他知道自己来晚了。
气氛变得尴尬。
```

审稿判断：这是分镜清单，不是小说段落。问题包括句长单一、动作没有阻力、人物没有自保、情绪靠旁白宣布、场面没有反馈链。

### 审稿问题

- 是否连续三句以上使用相同主语、相似句长和相似语气？
- 是否把动作、信息和情绪全部切开，导致段落像提纲？
- 短句是否有明确戏剧功能，还是只是默认输出习惯？
- 这一段如果改成更流动的自然段，是否能保留一个关键短句作为落点？
- 角色对话之外的旁白，是否也出现全员同声、同节奏、同强度？

### 修改要求

命中该问题时，不能只把句号换成逗号。修订 agent 必须重建动作链和心理承接：先合并无功能短句，再补现场反馈、人物遮掩、关系压力或具体物件，最后只保留真正有打击力的短句。

## 审稿专项：人物对白是否像真人在争东西

人物说话薄弱时，不要只评“台词不自然”。审稿 agent 要判断每句重要对白是否在争取目标、遮掩弱点、改变关系或制造压力。

### 高风险信号

- 所有人都准确表达自己的真实动机。
- 对话像轮流发表观点，没有抢话、回避、试探、打断和误解。
- 角色用同一种句式解释设定、总结情绪、推进剧情。
- 只有口头禅差异，没有利益差、知识差、阶层差、关系差。
- 每句台词删掉后，场景压力和对方反应都不变。

### 审稿动作

- 给每句关键对白标注功能：威胁、安抚、撒谎、试探、补刀、求助、认输、压价、转移话题。
- 找出无功能台词，要求修订 agent 改成动作、沉默、误解、物件或更有策略的说法。
- 如果对白问题来自角色声音缺失，退回 `docs/core-writing/37-project-style-bible-character-voice.md` 补声音矩阵，不要只润句子。
- 如果用户提供顶尖网文片段参考，可以保留极短摘句做功能拆解，但新稿必须原创改写。

## 审稿专项：对白训练是否有效

如果 agent 只是引用名句或仿出类似气势，审稿必须判为无效训练。有效对白训练至少要看到：原场景压力、说话动作、坏仿写、原创改写和听者反应。

检查问题：

- 这句台词有没有让对方产生新反应？
- 这句台词是否只有情绪，没有关系杠杆？
- 是否把顶尖网文的名句气势误学成中二喊话？
- 是否把职业热爱误写成获奖感言？
- 是否把暧昧试探误写成直白问答？
- 是否能删掉这句而场景不变？如果能，说明它不是有效对白。

## 审稿专项：对白自然度优先

审稿时先不要问“这句台词够不够帅”。先问：它像不像这个人在这个压力、关系和信息条件下会说出来的话。

一票返修信号：

- 角色准确说出自己的心理分析。
- 角色替作者解释设定、主题或关系。
- 角色在紧张场景中仍然每句话完整、礼貌、准确、漂亮。
- 角色知道自己不该知道的信息。
- 角色忽然说出不符合身份、年龄、职业、地域和关系距离的词。
- 台词是金句，但前后没有场景压力支撑。

审稿动作：把不自然台词先改成“人物能说出口的版本”，再决定是否需要保留锋利度、幽默感或压场感。
