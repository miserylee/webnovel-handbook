# 59. 反派、竞争对手与外部世界同步升级专项

状态：已确认

适用对象：AI agents 在长篇网文中处理主角胜利、训练突破、换地图、卷间转场、时间跳跃、阶段高潮、对手反制、组织压力、世界规则反馈、舆论变化、市场变化、派系博弈和连续追读压力时使用。

## 1. 核心原则：长篇不是主角单机升级

长篇网文最常见的“世界失真”不是设定错，而是外部世界没有学习能力：

- 主角打脸一次，对手下一次还用同一招。
- 主角暴露新能力，敌方没人研究。
- 主角拿走资源，市场、宗门、公司、家族没有反应。
- 主角闭关数月，反派、盟友、舆论、制度都停在原地。
- 主角换地图，新敌人像从空气里冒出来，旧结果不再影响剧情。
- 阶段高潮结束后，只有主角获得回报，外部世界没有任何结构性变化。

本知识库采用以下判断：

> 主角每一次有效成长，都必须改变外部世界对他的认知、资源分配、行动策略或风险等级。

这不等于每章都要上更强敌，也不等于反派必须全知全能。同步升级指的是：

- 对手根据可获得信息调整判断。
- 组织根据损失或利益重新调度资源。
- 制度根据漏洞或违规改变规则。
- 舆论根据公开结果改变站队。
- 市场根据供需变化改变价格或机会。
- 盟友根据主角表现改变信任、期待和依赖。
- 世界规则根据主角的使用方式暴露新边界。

如果主角升级后外部世界没有任何反应，读者会感觉故事像单机游戏，胜利越来越廉价。

## 2. 和反派系统的区别

`docs/21-antagonist-opposition-system.md` 解决的是“一个对手怎么设计、怎么出手、怎么反制、怎么落袋”。

本专题解决的是“主角状态变化之后，整个外部系统怎么跟着变”。

区别如下：

| 维度 | 反派与阻力系统 | 同步升级系统 |
| --- | --- | --- |
| 关注对象 | 单个反派、对手、组织、制度、环境阻力 | 对手群、组织链、舆论、市场、规则、地图、盟友、旁观者 |
| 主要问题 | 对手是否有目标、资源、战术和失败后果 | 主角行动是否改变外部状态，外部是否学习和反馈 |
| 时间跨度 | 单章、单元、阶段对手 | 多章、小单元、卷、卷间、时间跳跃 |
| 防止问题 | 反派降智、纸老虎、只会骂人 | 世界原地等待、对手重复送、空降新敌人、升级失真 |
| 输出物 | 对手设计卡、对手戏审稿 | 对手状态差、世界状态差、同步升级台账、下一轮反制 |

两个专题需要联动：

- 写单场对手戏，先看 `21`。
- 写主角连续胜利、训练、卷级推进、时间跳跃或阶段切换，必须看本专题。

## 3. 什么时候必须调用本专题

出现以下情况时，AI agent 必须调用本专题：

- 主角赢下一场公开冲突。
- 主角暴露新能力、新身份、新资源、新关系或新底牌。
- 主角打破一个组织、制度、行业或世界规则的旧平衡。
- 主角闭关、训练、备赛、养伤、赶路、调查或跳过一段时间。
- 对手失败但没有退场。
- 一个阶段、卷、小副本、比赛、案件、项目或关系单元结束。
- 新地图、新势力、新组织层级、新行业圈层即将开启。
- 主角的金手指、系统、功法、战术、话术或商业模式已经被外界看见。
- 读者反馈指出“反派降智”“对手太蠢”“世界像围着主角转”“中段水”“升级没压力”。

一句话判断：

> 凡是主角状态明显上升，就必须问外部世界因此改变了什么。

## 4. 同步升级的七类外部变化

主角成长后，外部世界至少要在以下七类中变化一类。重要节点最好变化两到三类。

### 4.1 信息变化

外部知道了新信息，或产生新的误判。

可写内容：

- 对手知道主角会某种招式。
- 组织只知道结果，不知道过程。
- 舆论误以为主角背后有大势力。
- 反派发现主角不是独行，而有人暗中支持。
- 盟友知道主角有隐藏代价，因此不再盲目让他出手。

写作重点：

- 外部信息必须有来源：目击者、战报、账本、监控、传闻、榜单、系统公告、价格波动、尸体痕迹、失败者复盘。
- 不要让敌人凭空知道作者知道的一切。

### 4.2 资源变化

资源分配因主角行动改变。

可写内容：

- 主角夺走稀缺材料，下一批材料涨价。
- 对手失去名额，必须向上级借资源。
- 组织调配更多人手盯主角。
- 盟友愿意投入资源，但附带条件。
- 市场开始模仿主角产品，利润被压缩。

写作重点：

- 资源变化要能影响下一次行动。
- 不要只写“上面震怒”，要写上面调了什么资源、冻结了什么渠道、下了什么命令。

### 4.3 策略变化

对手或组织换方法。

可写内容：

- 从正面挑衅改成舆论围堵。
- 从武力压制改成资源封锁。
- 从个人出手改成制度卡点。
- 从公开打压改成挑拨盟友。
- 从试探主角本人改成攻击主角的供应链、队友、信誉或时间表。

写作重点：

- 下一次阻力不能复制上一轮失败招式。
- 对手换策略时，要保留他自己的性格和资源边界。

### 4.4 关系变化

旁观者、盟友、中立者、亲密关系或家族/团队内部态度改变。

可写内容：

- 中立者开始观望主角，暂时不站队。
- 盟友对主角期待上升，也给他更重任务。
- 队友因主角隐藏代价而产生不信任。
- 亲密关系从保护变成并肩，或从信任变成担忧。
- 家族、宗门、公司内部出现“扶持他”与“压住他”的分歧。

写作重点：

- 关系变化不能只有夸奖或震惊。
- 要让关系变化改变任务、资源、权限、情绪债或选择成本。

### 4.5 规则变化

制度、规则、流程、赛制、门规、平台算法、系统任务或副本机制发生调整。

可写内容：

- 主角利用规则漏洞后，下一轮规则补丁出现。
- 对手推动赛制改变，限制主角优势。
- 宗门开放更高权限，但要求主角承担义务。
- 平台流量规则变化，让旧打法失效。
- 副本检测到异常打法，增加新限制。

写作重点：

- 规则变化不能纯为卡主角而空降。
- 必须来自前文结果、组织利益、公开漏洞或世界逻辑。

### 4.6 环境变化

地图、天气、局势、公共安全、供需、战争、灾害、流言、社会氛围发生变化。

可写内容：

- 主角赢下矿脉争夺后，周边散修涌入，治安变差。
- 旧案件曝光后，城中开始查户籍，主角行动空间缩小。
- 主角救人后名声上升，求助者也大量涌来。
- 经营线打开市场后，原材料短缺、竞争者跟进。
- 宗门内斗加剧，原本中立的规则执行者开始站队。

写作重点：

- 环境变化适合做长篇厚度。
- 变化要压回主角选择，不能变成背景新闻。

### 4.7 风险等级变化

主角在外部系统中的风险等级、价值等级、悬赏等级、关注等级、保护等级或可利用等级变化。

可写内容：

- 从“路人”变成“可招揽对象”。
- 从“小麻烦”变成“需要监控对象”。
- 从“潜力股”变成“必须控制的变量”。
- 从“被保护者”变成“被要求承担责任的人”。
- 从“可利用棋子”变成“可能反噬棋局的人”。

写作重点：

- 风险等级变化最好能带来下一轮行动。
- 比如派人试探、提高悬赏、给出橄榄枝、封锁情报、转移战场。

## 5. 同步升级的触发点

AI agent 可以用以下触发点检查外部是否该变化。

| 主角动作或结果 | 外部应有反应 | 如果没有反应的风险 |
| --- | --- | --- |
| 主角公开胜利 | 对手复盘，旁观者重新估值 | 打脸像一次性烟花 |
| 主角夺取资源 | 原持有者追责，市场供需变化 | 资源像白捡 |
| 主角暴露能力 | 对手测试边界，盟友调整预期 | 金手指越来越无敌 |
| 主角救人或伤人 | 名声、责任、债务、敌意变化 | 事件没有重量 |
| 主角破解规则 | 规则方修补漏洞或暗中观察 | 规则像纸糊 |
| 主角闭关训练 | 外界局势推进，对手搜集弱点 | 世界停摆 |
| 主角换地图 | 旧结果影响新地图入口 | 像重新开书 |
| 对手失败 | 损失、转暗线、牵出上级或换策略 | 对手像经验包 |
| 阶段高潮结束 | 组织、舆论、资源、关系重排 | 爆点没有余波 |
| 主角获得名声 | 新机会和新麻烦同时出现 | 只有爽，没有长篇压力 |

触发点越公开，外部变化越快。触发点越隐秘，外部变化可以表现为误判、试探和局部反应。

## 6. 外部学习链：看见、解释、试探、反制、升级

一个成熟的外部反应不是“敌人更强了”，而是一条学习链。

### 6.1 看见

外部先接触到结果或痕迹：

- 现场目击。
- 战报传回。
- 榜单变化。
- 资源价格波动。
- 失踪人员名单。
- 网络热搜。
- 副本记录。
- 监控残片。
- 对手伤势。
- 任务失败回执。

### 6.2 解释

外部会根据自身立场解释结果：

- 低估：认为主角只是运气好。
- 高估：以为主角背后有大势力。
- 误读：把机制胜利误解成修为碾压。
- 归因错误：把队友贡献算到主角头上。
- 利益化：不管真相，只借事件达成自己的目标。

解释不一定正确。错误解释可以制造下一轮冲突，但错误必须符合信息来源。

### 6.3 试探

对手不会一上来就出终极招。更自然的是试探：

- 派小人物挑衅。
- 卡一次流程。
- 抬高一次价格。
- 放出一条谣言。
- 递一个条件苛刻的合作。
- 让主角做一个暴露能力的任务。
- 用弱点不明显的对手测试主角边界。

试探让读者感觉敌人会思考，也给主角换策略的机会。

### 6.4 反制

试探确认后，对手开始针对主角优势：

- 主角速度快，就改用范围限制。
- 主角证据强，就抢解释权。
- 主角靠系统提示，就制造假线索。
- 主角有人脉，就挑拨关系。
- 主角擅长低价经营，就切供应链。
- 主角擅长规则漏洞，就推动规则补丁。

反制必须针对前文主角已经展示过的优势。

### 6.5 升级

反制失败或局势失控后，外部升级：

- 牵出更高层。
- 改变赛制或规则。
- 封锁资源。
- 公开通缉或公开招揽。
- 由个人冲突升级为组织冲突。
- 由局部矛盾升级为地图、行业、宗门、家族或舆论矛盾。

升级不是加一个更强的人，而是改变主角解决问题的方式。

## 7. 同步升级不等于反派全知

高质量同步升级必须区分“外部知道什么”和“作者知道什么”。

### 7.1 外部只能通过渠道获得信息

信息渠道包括：

- 目击者。
- 失败者复盘。
- 组织档案。
- 账本和交易记录。
- 监控、留影、阵法痕迹。
- 伤口、残留物、现场破坏。
- 舆论、榜单、热搜、传闻。
- 系统公告、副本记录、任务结果。
- 同盟泄露、内鬼传话、无意暴露。

AI 写作时必须标出至少一个信息渠道。否则对手会像偷看大纲。

### 7.2 误判比全知更好用

对手不必知道真相。很多追读来自误判：

- 对手以为主角强在武力，实际强在信息。
- 对手以为主角靠后台，实际靠旧账证据。
- 对手以为主角只能用一次底牌，实际底牌有条件可重复。
- 对手以为主角没有同盟，实际主角已换成松散合作网络。
- 对手以为主角会保护名声，实际主角愿意牺牲短期名声换长期证据。

误判能让对手合理行动，也能让主角的反制更有期待。

### 7.3 保留盲区

同步升级仍要留盲区：

- 对手知道主角有新能力，但不知道代价。
- 盟友知道主角赢了，但不知道他快撑不住。
- 组织知道规则被利用，但不知道漏洞源头。
- 舆论知道结果，但不知道真相。

盲区会产生下一轮信息差。

## 8. 外部反应的正文呈现方式

同步升级不必每次写反派视角。可以用更轻的方式呈现。

### 8.1 一行信号

适合章末或转场：

> 当晚，灵药铺把止血散的价格往上提了三成。

这说明事件影响了资源市场。

### 8.2 物件信号

用物件显示外部变化：

- 新贴出的通缉令。
- 被划掉的榜单排名。
- 临时改过的比赛规则。
- 关上的城门。
- 被收回的权限牌。
- 多出来的护卫。
- 账本里忽然断掉的供货记录。

物件比旁白“局势变了”更可信。

### 8.3 旁观者反应

旁观者不是只负责震惊，而是改变利益判断：

- 商人不再压价。
- 中立者不敢再公开嘲讽。
- 队友主动交出一部分权限。
- 老师把更难的任务交给主角。
- 竞争者开始抄作业或绕开正面竞争。

### 8.4 对手内部动作

简短写对手内部动作即可：

- 失败者被召回。
- 上级撤掉原计划。
- 组织给主角建立档案。
- 反派要求下属不要再正面冲突。
- 竞争者买下主角下一步所需资源。

不要动不动写大段反派会议。会议只有在能产生新决策、新资源、新误判或新钩子时才值得写。

### 8.5 规则或环境变化

通过行动展示规则变化：

> 第二天报名处多了一条新规：所有参赛符箓必须现场验真，违者取消资格。

这比“对手针对主角做了准备”更有压迫感。

## 9. 同步升级的章节节奏

主角每一次有效推进后，可以按以下链条安排后续：

| 节点 | 功能 | 常用写法 |
| --- | --- | --- |
| 主角行动 | 改变局面 | 打赢、救人、夺资源、破解规则、公开选择 |
| 外部记录 | 世界看见痕迹 | 战报、榜单、价格、流言、证据、任务结果 |
| 外部解释 | 形成判断或误判 | 低估、高估、归因错误、利益化解释 |
| 外部动作 | 改变资源或策略 | 派人试探、封锁渠道、改规则、拉同盟 |
| 主角撞上变化 | 下一章压力出现 | 旧招受限、资源涨价、规则变动、队友受压 |
| 新状态落账 | 进入下一循环 | 对手状态、世界状态、主角风险等级更新 |

如果主角刚赢完，下一章开头却像什么都没发生，就是同步断裂。

## 10. 不同题材的同步升级重点

### 10.1 玄幻、仙侠、武侠

外部变化重点：

- 宗门和家族对主角风险等级重新评估。
- 稀缺修炼资源价格、名额、归属变化。
- 主角功法痕迹被高层认出或误认。
- 对手研究主角招式，改用克制法器、阵法、人数或规则。
- 秘境、榜单、悬赏、宗门考核随主角表现变化。

常见误区：

- 只把敌人境界抬高，不改变策略。
- 主角拿走天材地宝后，没有任何势力追查。
- 旧对手被打脸后仍然正面硬刚。

### 10.2 系统、金手指、重生文

外部变化重点：

- 对手只能观察金手指的外部效果，不能凭空知道系统界面。
- 主角熟悉未来后，外部被改变，未来情报逐渐失效。
- 系统奖励或任务公开效果会引起旁人误判。
- 对手从主角行动模式里总结规律，逼主角打破惯性。
- 系统限制、代价或触发条件在外部反制下变得更重要。

常见误区：

- 主角永远靠信息差碾压，世界不被改写。
- 对手忽然知道系统秘密。
- 未来情报永远准确，没有蝴蝶效应。

### 10.3 都市、职场、商战

外部变化重点：

- 竞争者跟踪客户、数据、供应链、成本和舆论。
- 公司、平台、监管、行业规则会根据结果调整。
- 主角赢项目后，不只获得夸奖，还会获得更高 KPI、更大责任和更多敌意。
- 对手从公开表现判断主角优势，转攻流程、预算、人事、客户信任。

常见误区：

- 职场对手只会当面吵架。
- 主角赢一次项目后，公司结构和利益格局没有任何变化。
- 商战没有价格、渠道、合同、交付、现金流压力。

### 10.4 娱乐圈、直播、文娱

外部变化重点：

- 热度带来新资源，也带来黑料、粉圈、平台、竞品、合约压力。
- 对手不只骂人，还会抢通告、买营销、剪辑断章取义、改档期。
- 平台算法、榜单、经纪公司和广告方会根据数据调整态度。
- 主角作品出圈后，下一次必须面对更高期待。

常见误区：

- 热搜只负责证明主角红了。
- 黑粉和对家没有利益链。
- 数据变化不影响下一步资源。

### 10.5 悬疑、探案、规则怪谈

外部变化重点：

- 凶手或幕后方会根据主角调查进度销毁、伪造、转移或提前引爆证据。
- 规则怪谈会因主角试探暴露新边界，但不能随意改规则。
- 警方、媒体、受害者家属、嫌疑人之间的信息差会变化。
- 主角越接近真相，外部误导越要升级，但线索必须可追踪。

常见误区：

- 凶手坐等主角推理。
- 反转靠作者藏信息，不靠对手行动。
- 规则为吓人临时改变。

### 10.6 宅斗、宫斗、权谋

外部变化重点：

- 一次公开选择会改变长辈态度、派系站队、规矩解释和后宅/朝堂风向。
- 对手会从名分、礼法、账册、下人、姻亲、舆论切入。
- 主角赢一次不能只收获脸面，还要改变权力、资源、信任或风险。
- 谣言传播速度、证据保存方式和礼法边界要成为压力。

常见误区：

- 反派只会下毒和尖叫。
- 规矩只在卡主角时存在。
- 一个人道歉就解决组织性压迫。

### 10.7 种田、经营、基建

外部变化重点：

- 新技术、新产品、新产量会改变价格、供应链、邻里关系和政策关注。
- 对手可能不直接攻击主角，而是抢原料、压运输、仿冒、造谣、拉走人手。
- 自然环境、市场周期、税费、人情债会同步给压力。
- 主角扩大生产后，管理复杂度上升。

常见误区：

- 经营线只记流水账。
- 产品卖爆后没有竞争者、仿冒者、供应限制或监管。
- 种田世界像独立小院，外部市场不存在。

### 10.8 女频成长、感情与关系线

外部变化重点：

- 主角边界感增强后，旧关系会重新试探。
- 男主、家人、朋友、前任、上司、竞争者会根据主角新选择改变策略。
- 情绪成长要改变关系权力结构，不只是主角内心独白。
- 主角事业或身份上升后，亲密关系中的期待、压力、嫉妒、保护欲或控制欲会变化。

常见误区：

- 主角成长只有旁白宣布，周围关系不变。
- 反派女配只会嫉妒，不能改变局面。
- 男主只负责保护或欣赏，不受主角选择影响。

### 10.9 无限流、副本、群像团队

外部变化重点：

- 其他队伍会研究主角打法。
- 副本规则、评分、资源继承、队伍位置会因上一次结果变化。
- 队友会根据主角表现改变分工。
- 主角越强，团队内部依赖、嫉妒、责任和牺牲分配越复杂。

常见误区：

- 每个副本像孤岛。
- 队友只负责跟随主角。
- 对手队伍不复盘、不换战术。

## 11. 同步升级和时间跳跃

时间跳跃最容易抹掉外部变化。凡是出现“几天后、数月后、闭关一年、三年后、比赛前夕、项目推进期”时，必须补同步检查。

跳跃前记录：

- 主角暴露了什么？
- 对手损失了什么？
- 组织知道了什么？
- 资源、舆论、规则、关系有哪些未结变化？

跳跃后回答：

- 对手这段时间做了什么？
- 世界局势哪些地方变了？
- 旧优势是否被研究、模仿、克制或制度化？
- 旧代价是否扩大、转移或被人利用？
- 哪个外部变化会在跳后第一场景压住主角？

时间跳跃后第一场景应至少展示一项外部差异：

- 原本便宜的材料涨价。
- 城门多了盘查。
- 旧对手改了招式。
- 队友接了新任务。
- 舆论把主角推到更高位置。
- 上级给主角安排更危险职责。

如果跳后只有主角变强，外界没变，读者会感觉漏掉了一本书。

## 12. 同步升级和训练成长

训练段结束后不能只验证主角变强，也要验证外部同步：

- 主角练出新招，对手是否看到、误判或研究？
- 主角训练消耗资源，资源方是否追债或限供？
- 主角闭关期间，盟友是否承压、敌人是否推进？
- 主角变强后，旧任务是否变成更高责任？
- 主角用新能力赢下验证场，旁观者是否改变利益判断？

训练段应形成：

> 训练目标 -> 成长证据 -> 外部反应 -> 下一轮更难问题。

如果训练成果只服务一次胜利，没有引发后续外部变化，训练段很快会变成填充。

## 13. 同步升级和卷级推进

卷级故事需要外部状态持续变化，而不是每个小单元重新找麻烦。

一个卷至少要记录三条外部升级线：

- 阶段对手线：谁在复盘主角、换策略、牵出更高层？
- 组织/制度线：规则、权限、流程、派系如何变化？
- 世界/资源线：地图、市场、舆论、资源、群众反应如何变化？

卷中转折通常来自同步升级：

- 对手从试探转为封锁。
- 中立势力被迫站队。
- 主角利用过的规则被修补。
- 资源涨价或断供。
- 旧盟友发现主角代价，合作条件变化。

卷末必须清账：

- 哪些外部变化已结清？
- 哪些对手退场、升级或转暗线？
- 哪些规则被改变？
- 哪些资源重新分配？
- 哪些舆论、关系、名声、身份会带到下一卷？

## 14. 同步升级台账

长篇项目建议在故事台账中增加“外部状态账”或在人物状态账、设定规则账里补充字段。

### 14.1 外部状态账字段

```markdown
## 外部状态账

| 节点 | 主角造成的变化 | 谁看见/知道 | 外部解释或误判 | 外部动作 | 下次压迫点 | 状态 |
| --- | --- | --- | --- | --- | --- | --- |
```

### 14.2 对手学习账字段

```markdown
## 对手学习账

| 对手/势力 | 已知信息 | 误判 | 可调用资源 | 下一次策略 | 被主角误判处 | 预定回收 |
| --- | --- | --- | --- | --- | --- | --- |
```

### 14.3 世界规则变化账字段

```markdown
## 世界规则变化账

| 规则/制度/市场 | 触发事件 | 调整内容 | 影响谁 | 主角旧优势是否受限 | 后续回收 |
| --- | --- | --- | --- | --- | --- |
```

台账不要复述整章。只记录会影响后续决策的状态差。

## 15. 常见失败模式

| 失败模式 | 读者体感 | 修正方法 |
| --- | --- | --- |
| 世界原地等待 | 像单机游戏，缺少真实感 | 主角胜利后补信息、资源、策略或规则变化 |
| 对手重复送 | 爽点疲劳，反派降智 | 对手失败后必须换策略、退场、转暗线或牵出上级 |
| 只加更高头衔 | 升级空心，压力不变 | 改变解决方式：规则、资源、舆论、关系或环境 |
| 反派全知 | 像偷看大纲 | 标明信息渠道，保留误判和盲区 |
| 新敌人空降 | 像重新开书 | 让新敌人来自旧结果、旧资源、旧规则或旧代价 |
| 组织像一个人 | 世界薄，规则随意 | 写流程、派系、权限、成本和内部利益 |
| 余波只有震惊 | 爽点空，事件不值钱 | 让旁观者改变行动或利益判断 |
| 训练后直接碾压 | 成长廉价，后续无压 | 让对手研究新能力并限制下一次使用 |
| 时间跳跃只写主角变强 | 漏章感，外部断线 | 跳后第一场景展示世界差异 |
| 过度升级 | 读者疲惫，主角没有呼吸 | 用小信号、小试探和关系变化替代每次大危机 |

## 16. 同步升级质量分级

### A 级

特点：

- 主角行动明确改变外部状态。
- 对手有信息渠道、误判和策略调整。
- 组织、规则、资源或关系至少一项发生变化。
- 下一轮压力针对主角已有优势。
- 外部变化被写入台账，并能牵引后续章节。

读者体感：

- “主角真的搅动了世界。”
- “对手不是傻子，下一轮会更有意思。”
- “胜利有余波，故事在往上走。”

### B 级

特点：

- 有对手调整或外部反应。
- 外部变化能影响下一章。
- 但资源、规则或关系变化较浅。

读者体感：

- “基本顺，但还可以更立体。”

### C 级

特点：

- 只有口头震惊或上级震怒。
- 下一轮压力仍然和上一轮相似。
- 外部状态没有进入台账。

读者体感：

- “能看，但世界有点纸。”

### D 级

特点：

- 主角变强，外部无变化。
- 对手重复同一招。
- 新敌人空降。
- 组织和规则只为当前爽点服务。

读者体感：

- “作者在给主角铺红毯。”

低于 B 级时，应重构外部反应或压缩重复冲突。

## 17. AI agent 执行流程

每次完成一个有效剧情节点后，AI agent 按以下流程处理：

### 17.1 识别主角状态变化

记录主角变化：

- 能力提升。
- 资源增加。
- 名声变化。
- 身份暴露。
- 关系变化。
- 规则理解加深。
- 旧敌人失败。
- 新地图入口打开。

### 17.2 判断谁会看见

至少列出三类观察者：

- 直接对手。
- 组织或制度。
- 旁观者或舆论。
- 盟友或队友。
- 中立竞争者。
- 潜在上级敌人。
- 市场、平台、榜单或公众。

不是所有观察者都要出场，但设计层要知道他们是否有反应。

### 17.3 写出外部解释

外部解释可以正确，也可以错误：

- 他们以为主角靠什么赢？
- 他们低估了什么？
- 他们高估了什么？
- 他们最担心主角下一步做什么？
- 他们最想利用主角什么？

### 17.4 选择同步升级类型

从七类外部变化里选择一到三项：

- 信息变化。
- 资源变化。
- 策略变化。
- 关系变化。
- 规则变化。
- 环境变化。
- 风险等级变化。

### 17.5 设计下一轮压迫点

下一轮压迫点必须针对主角已展示优势：

- 旧招被限制。
- 旧资源被封锁。
- 旧同盟被考验。
- 旧名声被反利用。
- 旧规则漏洞被修补。
- 旧情报失效。
- 旧身份带来新责任。

### 17.6 回写台账

只写状态差：

- 谁知道了什么。
- 谁误判了什么。
- 谁损失了什么。
- 谁准备采取什么行动。
- 哪条规则或资源发生变化。
- 下一章或下一单元必须回收哪个外部反应。

## 18. 可直接使用的同步升级卡

```text
反派、竞争对手与外部世界同步升级卡

1. 主角本轮状态变化：
   能力 / 资源 / 名声 / 身份 / 关系 / 信息 / 权限 / 规则理解 / 其他

2. 变化是否公开：
   公开 / 半公开 / 隐秘 / 只有少数人知道

3. 信息渠道：
   谁通过什么方式知道了结果或痕迹？

4. 外部解释或误判：
   对手、组织、盟友、旁观者分别如何理解这件事？

5. 外部变化类型：
   信息 / 资源 / 策略 / 关系 / 规则 / 环境 / 风险等级

6. 对手或势力下一次换什么策略：
   不允许重复上一轮失败招式。

7. 世界或组织有什么具体动作：
   改规则 / 调资源 / 封锁渠道 / 派人试探 / 拉同盟 / 抬价格 / 放流言 / 给任务 / 其他

8. 下一轮压迫点：
   主角哪个旧优势会受限？

9. 主角可能误判什么：
   主角是否以为自己已经解决问题？他会低估哪类反馈？

10. 需要回写台账：
    对手已知信息 / 对手误判 / 世界规则变化 / 资源变化 / 舆论变化 / 关系变化

11. 下一章呈现方式：
    一行信号 / 物件信号 / 旁观者行动 / 对手内部动作 / 规则变化 / 正面对抗
```

## 19. 审稿检查问题

审稿时逐项检查：

- 主角上一次胜利或成长后，外部世界是否有变化？
- 对手是否通过合理渠道知道了结果？
- 对手是否存在误判，而不是全知或全盲？
- 下一次阻力是否针对主角旧优势？
- 对手是否换策略，而不是重复同一招？
- 组织、制度、市场、舆论或规则是否至少有一项反馈？
- 旁观者是否改变行动，而不只是震惊？
- 新敌人、新地图、新任务是否来自旧结果？
- 时间跳跃后是否展示了外部状态差？
- 训练或升级后是否安排了外部验证和反制？
- 外部变化是否被写入台账？
- 同步升级是否服务下一轮追读，而不是增加无关复杂度？

如果超过三项为否，说明外部系统断线，需要补同步升级。

## 20. 与旧知识库规则的合并说明

本专题将既有知识库中分散的规则集中：

- 从反派与阻力系统继承“对手必须有目标、资源、战术和失败后果”。
- 从连载节奏继承“每轮爽点后必须产生下一轮更大问题”。
- 从中段卷结构继承“每章和每小单元必须有状态增量”。
- 从世界观专题继承“组织、规则、资源和地图必须影响人物选择”。
- 从连续性台账继承“变化要记录为后续事实来源”。
- 从时间跳跃专题继承“跳后必须展示状态差额”。
- 从训练专题继承“成长证据必须引发下一阶段限制”。

本专题新增的是外部同步判断：

- 主角状态变化后，谁知道了什么？
- 外部如何解释或误判？
- 对手和世界会换什么资源、策略、规则或关系？
- 下一轮压力如何针对主角旧优势？
- 外部变化如何进入台账并牵引后续？
