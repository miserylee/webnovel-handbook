# 30. 职业、行业与生活细节可信度

## 相关入口：地域、时代与社会风俗质感

涉及真实地域、历史时代、架空王朝、异世界社会、乡土现实、都市地方感、民族/宗教/异文化或地方风俗时，先调用 [31. 地域、时代与社会风俗质感](31-region-era-social-custom-texture.md)。该专题负责让背景改变人物行动、信息速度、资源成本、人情压力和读者体感，避免只换地名、时代错位、风俗猎奇和模板化场景。


状态：已确认。用途：给 AI agents 在开书、起草、修订和审稿时使用，解决“场景像空房间、职业像标签、行业像百科、生活没有质感、专业细节一眼假”的问题。

本专题联动 `docs/04-character-and-dialogue.md`、`docs/11-human-writing-upgrade.md`、`docs/13-genre-patterns.md`、`docs/20-content-compliance-risk-review.md`、`docs/28-worldbuilding-setting-exposition.md` 和 `docs/29-reader-expectation-genre-promise-selling-point.md`。

## 1. 核心原则：可信感来自被使用的细节

细节不是装饰，不是为了证明作者查过资料。商业网文里的职业、行业和生活细节，必须服务以下至少一种功能：

1. 推动剧情：细节让角色必须做某个动作。
2. 制造阻力：细节带来流程、成本、权限、风险或时间限制。
3. 塑造人物：细节显示角色熟练、外行、疲惫、谨慎、傲慢或有经验。
4. 建立场景：细节让读者知道人物站在哪里、手里有什么、正在承受什么。
5. 提供回报：读者获得“原来这个行业/生活是这样”的新鲜感。
6. 埋伏后果：细节后来变成线索、破局工具、误会来源或反转证据。

AI 写作时使用这条链路：

```text
资料来源 -> 职业/生活真实约束 -> 场景压力 -> 角色动作 -> 读者可见细节 -> 后果/回报
```

如果一个细节不能进入链路，就先放入资料库，不要塞进正文。

## 2. 可信感不是绝对真实，而是读者愿意相信

小说不等于纪实报告。网文可以夸张、架空、爽化、脑洞化，但不能让读者觉得作者连基本场景都没摸过。

判断标准：

| 层级 | 要求 | 错误示例 |
|---|---|---|
| 基础正确 | 不犯目标读者一眼能看出的常识错 | 医院抢救像办公室开会，律师随口保证胜诉 |
| 流程可感 | 读者能看见事情要经过什么环节 | 公司项目没有预算、审批、客户、时间表 |
| 物件具体 | 场景里有可交互的工具、文件、设备、环境 | “他处理完工作”一句带过所有专业动作 |
| 成本存在 | 时间、钱、人情、体力、权限和风险会产生后果 | 主角随便调档案、随便进后台、随便拿资源 |
| 人物受限 | 职业身份影响说话、行动、关系和目标 | 医生、警察、老师、主播、修士说话全一个腔 |
| 爽化有边界 | 爽点突破的是阻力，不是无视世界规则 | 主角强行违法违规还被所有人鼓掌 |

## 3. 职业身份要变成主线压力

职业不是“主角是医生/律师/主播/外卖员/炼丹师”这一行设定，而是一套持续制造事件的身份系统。

AI 设计职业身份时，要回答：

- 这个身份每天必须处理什么事？
- 谁能评价他、命令他、卡住他？
- 他掌握什么资源，又缺什么资源？
- 这份职业有什么考核、晋升、收入、风险和禁忌？
- 哪些关系会因为这个身份自然出现？
- 这个身份为什么能生成连续剧情，而不是只当开局标签？

例：

- “医学生”不是白大褂标签，而是值班、带教、病例、考试、患者家属、科室关系、伦理压力和专业边界。
- “主播”不是开播喊话，而是选题、设备、流量、平台规则、弹幕、榜单、商务、黑粉和人设维护。
- “宗门弟子”不是练功机器，而是师承、杂役、资源、任务、同门关系、门规、考核、外门内门差异。

## 4. 四层资料：事实、流程、感官、人情

AI 做资料沉淀时，不要只搜名词定义。职业和生活可信度至少要有四层资料。

| 层级 | 要找什么 | 进入正文的方式 |
|---|---|---|
| 事实层 | 职称、机构、工具、术语、地点、制度 | 少量准确名词，避免硬伤 |
| 流程层 | 一件事从开始到结束经过哪些步骤 | 用动作和时间压力体现 |
| 感官层 | 声音、气味、触感、光线、温度、空间拥挤度 | 选 1-3 个高相关细节，不铺满五感 |
| 人情层 | 谁求谁、谁怕谁、谁背锅、谁占便宜 | 转成冲突、误会、交易、面子和欠账 |

只写事实层会像百科；只写感官层会像散文；只写人情层容易悬浮。四层至少取两层，才能撑住场景。

## 5. 资料来源优先级

处理真实职业、行业、地域、制度或高风险题材时，按以下顺序使用资料：

1. 亲身经历、正式采访、实地观察、专家核对。
2. 官方公开资料、行业规范、机构页面、平台规则、公开案例。
3. 业内人士访谈、纪实报道、专业博客、纪录片、课程或讲座。
4. 同题材优秀作品的处理方式。
5. 社区经验帖、论坛问答、短视频、评论区补充。
6. AI 推测。

硬规则：AI 推测不能作为专业事实来源。无法确认的内容标“待确认”，在正文中降低精确度或改成架空版本。

## 6. 资料不要直接倒进正文

研究资料是作者的地基，不是读者的阅读任务。

可进正文的细节，必须满足至少一个条件：

- 角色正在使用它。
- 对手正在利用它。
- 它让主角付出代价。
- 它暴露人物经验差距。
- 它改变场景走向。
- 它会在后文回收。

不满足这些条件的资料，放入“资料库/后台笔记”。

## 7. 三件套：流程、物件、成本

AI 快速增强可信度时，优先补“三件套”。

```text
一个正确流程 + 一个具体物件 + 一个真实成本
```

例：

- 医院场景：挂号/分诊/检查/签字流程 + 病历夹或腕带 + 等待时间和家属沟通成本。
- 职场场景：需求评审/排期/审批/交付流程 + 会议纪要或工单 + 加班、预算或背锅成本。
- 直播场景：开播准备/推流/互动/复盘流程 + 补光灯或后台数据页 + 流量波动和人设风险。
- 修仙宗门：领任务/验身份/交材料/计功勋流程 + 身份牌或功勋册 + 灵石、名额、人情债成本。

三件套不需要长篇解释，只要压进动作里，场景就会立起来。

## 8. 生活细节：让角色像在过日子

生活细节不是日常流水账，而是让人物有身体、有钱袋、有时间表、有关系网。

常用检查维度：

| 维度 | 可写细节 | 作用 |
|---|---|---|
| 时间 | 通勤、排队、值班、宵禁、作息、截稿 | 制造紧迫感和疲惫感 |
| 钱 | 房租、饭钱、押金、路费、材料费、欠款 | 让目标和压力落地 |
| 身体 | 饿、困、冷、痛、汗、手抖、嗓子哑 | 防止人物像无体力机器 |
| 物件 | 手机、电动车、钥匙、账本、证件、药瓶、工具 | 让动作有抓手 |
| 空间 | 楼道、工位、夜市、诊室、后台、仓库、宗门杂役房 | 让场景不是空白背景 |
| 关系 | 邻居、同事、客户、师兄、家属、平台运营、房东 | 自动生成冲突和信息来源 |

生活细节要绑定角色需求。不要为了“真实”写三百字早餐，除非这顿早餐改变关系、暴露经济状态或引出事件。

## 9. 感官细节：选最有压力的那一个

五感不是清单。AI 最容易把场景写成“视觉 + 气味 + 声音 + 触感”模板，反而显假。

选择感官细节时问：角色此刻最不可能忽略什么？

- 紧张时：声音、手感、呼吸、时间流速更明显。
- 疲惫时：重量、温度、重复动作、身体疼痛更明显。
- 陌生环境：气味、空间布局、他人眼神和规则提示更明显。
- 专业场景：工具状态、流程异常、术语误用和微小风险更明显。

原则：一场选 1-3 个最相关细节，比铺满五感更有效。

## 10. 术语和行话的用法

术语不是越多越专业。好的术语让读者明白角色处在某个系统里，坏的术语让读者觉得作者在背资料。

用法：

1. 第一次出现术语，最好让它伴随动作或后果。
2. 用角色之间的误解、纠正、催促、吐槽来解释术语。
3. 专业人物可以说简称，外行人物可以听不懂或误解。
4. 术语密度随读者熟悉度调整，前期少、后期逐步加。
5. 高风险专业术语不要写成现实操作指南。

例：不要写“他开始进行复杂的项目管理流程”，而写“他把需求单退回去，原因只有四个字：没有验收人”。

## 11. 高风险行业写作边界

医疗、法律、金融、警务、教育、未成年人、危险技术、公共安全、政治历史、真实机构和真实人物相关内容，必须额外谨慎。

AI agents 应遵守：

- 不编造现实法律结论、医疗建议、投资建议或执法流程。
- 不提供可执行违法、伤害、规避监管或危险操作步骤。
- 不把真实机构、真实人物、真实案件写成未经证实的负面事实。
- 不用“专业设定”美化违法犯罪、诈骗、侵害未成年人或危险行为。
- 可使用架空机构、模糊流程、合规化处理和剧情后果降低风险。
- 正式发布前，涉及平台审核、法律、医疗、金融或版权风险的内容要重新确认最新规则。

## 12. 不同题材的细节重点

| 题材 | 细节重点 | 常见失败 |
|---|---|---|
| 都市职业/行业文 | 工作流程、考核、客户、工具、人情、行业痛点 | 职业只有标签，没有任务和成本 |
| 现实题材 | 社会话题、家庭压力、地域生活、普通人的具体难处 | 只喊议题，不写人怎么过日子 |
| 神豪/系统/文娱 | 现实平台、交易规则、曝光路径、职业门槛 | 数字乱飞，社会关系不变 |
| 直播/电竞/体育 | 训练节奏、设备、赛制、观众、复盘、团队协作 | 只写热血，不写规则和体力 |
| 古言/历史 | 礼制、称谓、财产、出行、饮食、身份权限 | 现代人换古装，制度不产生压力 |
| 玄幻/仙侠 | 宗门生活、资源流通、杂役任务、身份差异、社会关系 | 只有境界名，没有生活世界 |
| 悬疑/刑侦 | 线索来源、调查限制、时间线、证据链、程序边界 | 推理靠天降信息，办案像单人闯关 |
| 种田/经营/基建 | 物资、人工、季节、运输、损耗、账目、供应链 | 只报成果，不写投入和阻力 |
| 轻喜剧/日常 | 生活小事、空间误差、人物习惯、地域口味 | 段子漂浮，人物不像真生活中会遇到的人 |

## 13. AI 常见硬伤

1. 职业只有制服、称谓和口号，没有日常任务。
2. 行业只有名词，没有流程、成本、风险和人情。
3. 所有人都知道所有信息，没有权限差异。
4. 场景只有“办公室、医院、学校、宗门大厅”，没有具体空间。
5. 角色永远不饿、不累、不缺钱、不迟到、不处理琐事。
6. 用大段专业解释替代角色动作。
7. 让外行人物突然说出专家级术语，且无人质疑。
8. 为了爽点无视安全、法律、伦理和平台合规后果。
9. 把现实资料当世界观百科，不转化为冲突。
10. 写错目标读者非常熟悉的常识细节。

## 14. 职业/行业资料卡

```text
【职业/行业】
【题材功能】主线引擎 / 人设来源 / 爽点来源 / 阻力系统 / 氛围真实感 / 案件线索
【目标读者熟悉度】低 / 中 / 高
【资料来源】亲历 / 采访 / 官方资料 / 行业报道 / 同题材作品 / 社区经验 / AI 推测
【来源可信度】已确认 / 候选 / 待确认
【日常任务】
【关键流程】一件典型事情从开始到结束怎么走？
【考核/收益】谁评价？怎么赚钱、晋升、拿资源？
【工具/物件/文件】
【空间】人物常在哪些地方行动？
【时间节奏】忙季、值班、截稿、赛季、推荐期、考试期、宵禁等。
【人际关系】上级、同事、客户、家属、对手、平台、监管、师门等。
【权限边界】主角能做什么，不能做什么？
【成本与风险】时间、钱、体力、人情、法律、名誉、安全、伦理。
【行话/术语】最多 5 个，必须绑定动作或后果。
【可用冲突】
【可用爽点】
【合规风险】
【正文投放方式】动作 / 对话 / 物件 / 流程 / 后果 / 感官 / 台账
```

## 15. 场景可信度检查卡

```text
【场景】
【当前剧情目标】
【角色此刻最在乎什么】
【地点具体化】不是“办公室”，而是哪个位置、什么时间、谁在场。
【一个流程】这个场景正在走什么流程？
【一个物件】角色手里、桌上、门口或后台有什么能参与剧情的东西？
【一个成本】失败会损失什么？时间、钱、名额、关系、体力、名声、生命？
【一个感官细节】角色此刻最不可能忽略的声音/气味/触感/温度/光线。
【一个人情关系】谁求谁、谁压谁、谁背锅、谁欠谁？
【专业边界】有没有编造现实建议、危险步骤或高风险流程？
【是否推进承诺】它推进了主承诺、人物、关系、线索、资源或世界规则中的哪一项？
```

## 16. 审稿流程

1. 标出所有职业、行业、地域、生活、专业和制度细节。
2. 判断每个细节服务哪种功能：剧情、阻力、人物、场景、回报、伏笔。
3. 删除只炫资料、不服务场景的说明。
4. 给空场景补“三件套”：流程、物件、成本。
5. 检查人物是否受职业身份限制，而不是只穿职业外衣。
6. 检查目标读者熟悉的常识是否有硬伤。
7. 检查高风险行业是否有合规、法律、医疗、金融、未成年人或危险操作风险。
8. 把会影响后续的职业规则、机构规则、权限边界写入设定或连续性台账。

## 17. 快速自查问题

- 这场戏如果删掉职业/行业背景，还成立吗？如果成立，职业可能只是装饰。
- 主角的职业身份今天给他带来了什么任务、限制或收益？
- 本章有没有一个读者能摸到的物件、流程或成本？
- 专业细节是否被角色使用，而不是被旁白讲解？
- 有没有目标读者一眼会觉得假的常识错误？
- 生活细节是否改变人物处境或情绪，而不是流水账？
- 高风险行业内容是否避免了现实建议、危险步骤和编造结论？
- 本章细节是否推进了读者正在等待的主承诺？

## 18. 来源与迁移说明

- 起点社区《以都市职业文为例教你设置主角身份》强调职业身份不能是符号，必须写出背景细节、周围角色、现实困境，并生成连贯主线；幻想背景也需要社会化身份和生活世界。本文将其迁移为“职业身份要变成主线压力”。
- 起点社区《主角的设计，从性格、身份和思维着手》强调读者越来越要求生活化和代入感，剧情应围绕主角职业身份展开，一是真实，二是更好写。本文将其迁移为“职业身份资料卡”和“三件套”。
- 起点社区《如何把个人特色和写法技巧相结合？》强调现实品类核心是真实，行业文可从一个人看到一个现实行业，素材、人物、选题和事件都要具体。本文将其迁移为“事实、流程、感官、人情四层资料”。
- Writer's Digest《Researching Your Fiction Like a Reporter》强调研究可以让故事落在现实中，并通过访谈、观察和资料让细节沉浸进剧情，但不能把研究资料整块倒进小说。本文将其迁移为“资料来源优先级”和“资料不要直接倒进正文”。
- Writer's Digest《How to Use Facts in Your Fiction》强调事实是调味料，细节如果不能建立场景、推进情节或照亮人物，就不应进入正文；研究主要是给作者建立权威感。本文将其迁移为“可进正文的细节条件”。
- Purdue OWL《Fiction Basics》强调作者不能呈现所有细节，必须选择细节，事件间因果要合理可信，人物行动支撑情节。本文将其迁移为“可信感来自被使用的细节”。
- WAC Clearinghouse《Using Descriptive Detail》强调具体感官细节能让读者进入场景，但细节应服务目的而不是填字数。本文将其迁移为“感官细节选最有压力的那一个”。
- Reedsy《How to Plan a Novel》强调主角目标驱动行动，设定要服务冲突，真实地点可通过生活条件、常见生活方式、地图和资料建立可信图像。本文将其迁移为“生活细节和场景可信度检查卡”。
