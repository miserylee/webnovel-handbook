# 62. 日常章、过渡章与低强度章节防水专项

状态：已确认

适用对象：AI agents 在起草、续写、审稿或修订日常章、过渡章、休整章、赶路章、聊天章、吃饭章、生活流、低冲突章节、卷间缓冲、战后余波、关系缓冲、情绪沉淀、信息整理、准备章、调查间隙、轻松章和主线暂时低烈度推进时使用。

## 1. 核心原则：低强度可以慢，不能空

长篇网文不能每章都高潮。真正耐看的长篇，需要日常、休整、余波、聊天、赶路、吃饭、整理信息、关系缓冲和低烈度推进。

问题不在“低强度”，而在“低功能”。

AI 最容易把低强度章节写成：

- 角色吃饭聊天，但关系没变。
- 主角赶路，但路上没有信息、风险、环境或人物选择。
- 战后休整，但伤势、资源、关系、舆论和对手反制都没变化。
- 卷间过渡，但旧卷代价消失，新卷目标没出来。
- 日常生活很细，但细节不服务人物、关系、机制或主线。
- 一章都在解释上一章发生了什么，没有新状态。

本知识库采用以下判断：

> 日常章不是没有剧情，而是用低烈度方式改变人物、关系、信息、资源、情绪或下一步行动。

低强度章节必须至少完成一种变化：

- 目标变化：主角决定下一步做什么。
- 关系变化：信任、误会、亏欠、边界、亲密度或站队发生变化。
- 信息变化：新线索、新理解、新误判、新限制被确认。
- 资源变化：钱、药、装备、时间、权限、名声、机会发生变化。
- 情绪变化：上一轮事件的情绪债被支付、延后或加重。
- 外部变化：对手、舆论、组织、市场或规则给出反馈。
- 读者回报：人味、梗点、糖点、生活感、世界质感或角色魅力增强。

如果一章删掉后，只需要一句“他们休整了一天”就能接上，这章通常需要压缩或重构。

## 2. 什么时候必须调用本专题

出现以下情况时，AI agent 必须调用本专题：

- 章节目标是日常、休整、赶路、吃饭、购物、聊天、疗伤、收拾残局。
- 大战、公开反转、阶段高潮、卷末结算之后，需要写余波。
- 新地图或新卷开始前，需要承接旧结果。
- 主线冲突暂时降温，但不能跳过人物反应。
- 需要写生活流、种田、经营、轻喜剧、校园日常、队伍旅途、恋爱缓冲。
- 章节主要靠人物互动、关系变化、信息整理或梗点推进。
- 审稿发现“水”“日常无聊”“过渡章跳读”“吃饭聊天没用”“赶路流水账”“低冲突没有追读”。

一句话判断：

> 凡是本章没有强外部冲突，就必须先证明它的低强度功能。

## 3. 低强度章节的合法功能

低强度章节必须承担至少一种合法功能。重要过渡章最好承担两到三种。

| 功能 | 作用 | 合格表现 | 不合格表现 |
| --- | --- | --- | --- |
| 余波清账 | 回收上场结果 | 伤势、资源、关系、舆论、规则有变化 | 只说大家累了 |
| 关系推进 | 改变人物距离 | 误会加深、信任增加、边界确认、亏欠产生 | 只聊天互夸 |
| 情绪支付 | 让事件有余温 | 主角反应、后悔、愤怒、害怕、释然转成选择 | 抽象抒情 |
| 信息整理 | 把线索变成行动 | 旧线索重排，得到下一步计划 | 复述前情 |
| 世界质感 | 让环境影响行动 | 价格、人情、规矩、天气、空间改变选择 | 风景散文 |
| 人物魅力 | 展示性格和习惯 | 小动作、小选择、小缺陷服务关系或主线 | 可爱但无后果 |
| 梗点轻松 | 转换节奏并留黏性 | 笑点来自人物、关系、机制，并有后果 | 硬塞段子 |
| 资源准备 | 为下一轮铺工具 | 道具、钱、人情、权限、伤药进入可用状态 | 购物清单 |
| 外部同步 | 世界继续动 | 对手、组织、舆论、市场、规则给信号 | 世界停摆 |
| 下一目标生成 | 从旧结果生出新行动 | 章末有具体未完成动作 | 章末总结感想 |

没有功能的低强度章节，应该压缩成段落、转场或台账，不应独立成章。

## 4. 入场门槛：低强度章节开写前六问

开写前先回答六问：

1. 这章承接上一章哪个结果？
2. 这章最低限度要改变哪个状态？
3. 这章的低强度回报是什么：人味、关系、信息、梗点、世界质感还是余波？
4. 这章是否有一个小目标，而不是“大家休息一下”？
5. 这章结尾会生成什么具体下一步？
6. 如果压缩成三段，是否更好？

如果第 2、4、5 问答不出来，不要写成独立章节。

## 5. 低强度章节的七拍法

低强度章节也需要结构，但结构可以轻。

### 5.1 承接旧结果

第一屏要让读者知道这是从哪里接下来的：

- 上一战的伤。
- 上一场反转后的舆论。
- 上一卷留下的资源债。
- 上一章章末钩子的后果。
- 上一场选择造成的关系变化。

不要像重新开一段生活随笔。

### 5.2 给一个小目标

低强度章节的小目标可以很小：

- 去买药。
- 把队友送回去。
- 找一个能修法器的人。
- 把账册藏好。
- 给一个人道歉。
- 在赶路前确认路线。
- 在晚饭前问清一句话。

小目标让日常有方向。

### 5.3 遇到低烈度阻力

阻力不必是敌人冲上来：

- 钱不够。
- 伤口裂开。
- 队友不说实话。
- 店家认出主角。
- 城门盘查变严。
- 旧关系突然尴尬。
- 资源涨价。
- 旁人态度变了。
- 主角想休息但责任找上门。

低烈度阻力能让日常不散。

### 5.4 通过互动产生微变化

变化可以小，但要可见：

- 关系近一分或远一分。
- 主角知道了一个新限制。
- 队友暴露一个小秘密。
- 旧道具变成新用途。
- 主角发现自己还没从上一场走出来。
- 外部世界给出一个不妙信号。

### 5.5 给读者小回报

低强度章节的小回报可以是：

- 一个笑点。
- 一个糖点。
- 一个生活细节。
- 一个角色魅力瞬间。
- 一个小线索。
- 一个小资源。
- 一个关系小推进。
- 一个世界规则的自然露出。

小回报必须服务后续，不能只是装饰。

### 5.6 生成下一步

结尾必须把低强度变化转成行动：

- 明早去见某人。
- 立刻去查某条线索。
- 先修道具再进副本。
- 决定暂时隐瞒真相。
- 确认下一轮对手已经动了。
- 某段关系进入新的相处模式。

不要用“他们终于能喘口气了”当章末，除非下一句就是新的压力信号。

### 5.7 回写状态

写后记录：

- 关系变了什么。
- 情绪债变了什么。
- 资源变了什么。
- 信息变了什么。
- 外部世界给了什么信号。
- 下一章必须承接什么。

## 6. 日常不水的五种写法

### 6.1 任务型日常

日常中有小任务：

- 做饭是为了试探谁会留在桌上。
- 购物是为了发现资源涨价和市场封锁。
- 赶路是为了确认新地图的规矩。
- 修装备是为了暴露上一战代价。
- 排队是为了让主角看到规则如何对普通人起作用。

任务型日常的关键是：生活动作影响下一步行动。

### 6.2 关系型日常

日常中改变关系：

- 同桌吃饭，谁给谁夹菜、谁避开谁的话题、谁先起身，都能体现关系。
- 赶路时谁走前面、谁留意伤口、谁故意落后，都能体现信任和距离。
- 闲聊时谁不接梗，可能比谁说了什么更重要。

关系型日常的关键是：对话和动作必须让关系状态前后不同。

### 6.3 信息型日常

低强度场景里自然得到信息：

- 茶馆传闻。
- 店家涨价。
- 路人避谈。
- 孩子唱错的童谣。
- 公告栏换了新规。
- 队友说漏一个地名。
- 弹幕或评论区风向改变。

信息型日常的关键是：信息必须能改变计划，而不是只增加世界观。

### 6.4 人味型日常

展示角色生活质感：

- 主角受伤后还下意识把破碗洗干净。
- 队友嘴上嫌弃，却把最烫的饼撕成两半。
- 反派派来的探子不会杀人，只会因为怕冷一直跺脚。
- 女主没有大哭，只把旧合同折得很整齐。

人味型日常的关键是：小动作暴露人物，而不是堆形容词。

### 6.5 梗点型日常

用轻松感转换节奏：

- 强者不会用现代小电器。
- 系统严肃发布任务，主角正在砍价。
- 队友以为要密谋大事，结果主角拉他去买锅。

梗点型日常的关键是：笑点要改变关系、机制理解、世界反应或下一步行动。

## 7. 过渡章不水的四种功能

### 7.1 从旧结果到新目标

过渡章最重要的功能，是把上一场结果转成下一步目标。

不合格：

> 休息两天后，他们继续上路。

合格：

> 休息两天后，主角发现城门盘查名单上多了队友的名字。继续上路不再是选择，而是必须在天黑前换身份出城。

### 7.2 从大事件到人物反应

大事件后需要人物消化：

- 赢了但害怕。
- 救了人但欠债。
- 公开反转后名声上升，但隐私暴露。
- 打赢对手，但队友发现主角代价。

反应不是停顿，而是让人物为下一轮选择付心理成本。

### 7.3 从旧地图到新规则

换地图前后要逐步交接：

- 旧规则哪些仍有效。
- 新规则哪里不同。
- 主角旧优势在哪里受限。
- 旧代价如何进入新环境。

过渡章不能变成设定导游。新规则要通过行动压力露出。

### 7.4 从多线混乱到下一主线

高潮后常有很多线：

- 伤员。
- 资源。
- 舆论。
- 反派余党。
- 伏笔。
- 关系。
- 规则变化。

过渡章可以做轻量清账，但不能逐条汇报。应只挑会影响下一轮行动的两三项。

## 8. 休整章不水的规则

休整不是状态清零。休整要回答：

- 谁的伤没好？
- 谁的关系没恢复？
- 谁的资源没补齐？
- 谁的情绪还卡着？
- 谁在主角休息时行动了？
- 主角为了休整付出了什么机会成本？

休整章必须保留至少一个未恢复项：

- 伤势仍影响下一战。
- 资源只补了一半。
- 队友表面没事，但选择变了。
- 舆论暂时平息，但背后势力开始查。
- 主角休息了，但错过一个机会。

如果休整让所有问题自动消失，读者会觉得前面的代价是假代价。

---

## 9. 赶路章不水的规则

赶路可以写，但不能只写路。

赶路章的合法功能：

- 展示新地图规则。
- 暴露队伍关系。
- 让外部追兵或盘查逼近。
- 让资源消耗产生选择。
- 通过路人、传闻、价格、地形给信息。
- 让主角在移动中做决定。

赶路章的高危写法：

- 写天气和风景很久。
- 写大家走了几天。
- 写路上聊天但没有关系变化。
- 写路人传闻但后文不用。
- 写遭遇小怪但不改变状态。

判断标准：

> 如果赶路的唯一结果是“他们到了”，应该压缩。

## 10. 聊天章不水的规则

聊天章最容易水。聊天必须改变至少一项：

- 信息。
- 关系。
- 决策。
- 情绪。
- 误会。
- 站队。
- 计划。
- 价值观冲突。

### 10.1 聊天要有隐性目标

每个重要角色在聊天中都要有目的：

- 想试探。
- 想隐瞒。
- 想道歉。
- 想确认安全。
- 想争取站队。
- 想转移话题。
- 想让对方别去。

没有目的的聊天，容易变成作者用角色复述设定。

### 10.2 聊天要有转折

转折可以很小：

- 对方没有接话。
- 某人说漏一个词。
- 玩笑开过头。
- 道歉被拒绝。
- 试探得到意外答案。
- 本来想劝退，最后选择同行。

### 10.3 聊天后要能行动

聊天结束后，至少要产生一个行动：

- 改计划。
- 去找人。
- 暂时隐瞒。
- 主动道歉。
- 决定同行。
- 关系进入新边界。
- 某人独自去做一件事。

聊天后大家各回各屋，什么都不变，通常是水。

## 11. 低强度章节和章末钩子

低强度章节的章末不一定要爆炸，但必须具体。

可用钩子：

- 一句未说完的话。
- 一封到晚了一天的信。
- 店家突然改口不卖。
- 队友把药退了回去。
- 城门多了一张画像。
- 饭桌上少了一副碗筷。
- 旧伤在关键动作前疼了一下。
- 评论区风向出现第一条反常评论。
- 主角发现路线图上有一个被抹掉的村名。

低强度钩子要像轻轻敲门，不一定像砸门。但门后必须有人。

## 12. 低强度章节的手机阅读控制

低强度章节更怕手机端跳读。

规则：

- 第一屏交代本章小目标或承接压力。
- 每一屏至少有一个微变化。
- 对话段不要连续超过三轮没有新信息。
- 生活细节后要接人物反应或状态变化。
- 信息整理要拆成问题、证据、决定，不要写总结报告。
- 章末必须有具体未完成动作。

常见压缩方法：

- 把三天日常压成一个代表性场景。
- 把赶路压成路线变化加一个关键遭遇。
- 把疗伤压成伤势代价加一个关系动作。
- 把复盘压成一个争议点加一个新决定。
- 把准备压成道具、资源、风险三项清单加意外缺口。

## 13. 不同题材的低强度章节重点

### 13.1 玄幻、仙侠、武侠

重点：

- 修整时保留伤势、丹药消耗、功法隐患和宗门反应。
- 赶路时展示地图规则、灵力环境、坊市价格、势力盘查。
- 日常中用法器、门规、师承、弟子关系展示世界质感。

常见误区：

- 闭关后所有伤势自动好。
- 坊市购物写成清单。
- 师门日常只有师兄弟互夸。

### 13.2 都市、职场、商战

重点：

- 低强度章节可以写复盘会、客户回访、加班饭局、合同整理、项目准备。
- 每个细节要落到流程、责任、客户信任、预算、权限或人情变化。

常见误区：

- 职场日常只有吐槽老板。
- 加班写成流水账。
- 复盘只复述上一章，不生成下一步策略。

### 13.3 娱乐圈、直播、文娱

重点：

- 日常可以写排练、妆造、候场、后台、粉丝反馈、商务沟通。
- 低强度回报来自人设魅力、业务能力、团队关系、数据变化和舆论余波。

常见误区：

- 弹幕只夸主角。
- 后台日常不影响舞台。
- 热搜后没有商务、粉圈、对家变化。

### 13.4 宅斗、宫斗、权谋

重点：

- 低强度场景往往在请安、吃饭、赏花、送礼、换衣、账房、廊下等日常礼法里。
- 动作小，但关系、名分、规矩和站队要变。

常见误区：

- 请安场景只阴阳怪气。
- 吃饭只写菜。
- 礼物不带政治含义。

### 13.5 悬疑、探案、规则怪谈

重点：

- 低强度章节可用于整理线索、验证小假设、补时间线、展示异常日常。
- 每次整理都必须产生新问题或排除一个错误方向。

常见误区：

- 复盘全部已知信息。
- 日常异常只吓人，不给线索。
- 调查间隙无新状态。

### 13.6 女频关系、甜宠、成长线

重点：

- 日常是关系推进主战场。
- 低强度变化可以是边界、称呼、照顾方式、吃醋、回避、信任、主动表达。
- 成长线要让主角在小事里做出新选择。

常见误区：

- 只发糖不改变关系。
- 男主照顾替代女主选择。
- 日常甜宠没有外部压力和情绪债。

### 13.7 喜剧、生活流、种田经营

重点：

- 日常本身可以是主菜，但每章仍要有小目标、小麻烦、小回报、小后果。
- 经营日常要写供需、邻里、价格、工具、人情和长期收益。
- 喜剧日常要让笑点改变关系、机制、世界或下一步行动。

常见误区：

- 生活流变流水账。
- 种田只记收成。
- 喜剧只抖段子，没有后果。

## 14. 常见失败模式

| 失败模式 | 读者体感 | 修正方法 |
| --- | --- | --- |
| 日常流水账 | 像记录生活，不像故事 | 加小目标、阻力、微变化和章末动作 |
| 过渡无增量 | 跳过也不影响 | 明确从旧结果生成新目标 |
| 休整清零 | 前面代价变假 | 保留未恢复项和机会成本 |
| 赶路只写到了 | 空 | 增加路线风险、信息、资源消耗或关系变化 |
| 聊天无目的 | 像作者闲聊 | 给每个角色隐性目标和转折 |
| 复盘复述前情 | 像总结 | 只复盘一个争点，并生成决定 |
| 生活细节堆砌 | 有质感但不推进 | 让细节影响人物选择或读者理解 |
| 低强度无钩子 | 看完不想翻页 | 章末落在具体未完成动作 |
| 梗点无后果 | 好笑但可删 | 让笑点改变关系、信息或下一步 |
| 人物互夸 | 黏腻或假 | 用小动作、小误会、小代价替代夸奖 |
| 世界停摆 | 像安全屋 | 加外部信号：对手、舆论、规则、资源变化 |
| 低强度过长 | 手机端跳读 | 压缩成代表性场景，保留关键变化 |

## 15. 质量分级

### A 级

特点：

- 承接上一章结果清楚。
- 有小目标、低烈度阻力和微变化。
- 人味、关系、信息、世界质感或梗点至少一项有回报。
- 章末生成具体下一步。
- 写后能回写台账状态。

读者体感：

- “这章不爆，但我更喜欢这些人了。”
- “故事在休息，但没有停。”
- “下一章该怎么动我知道了。”

### B 级

特点：

- 有功能和小回报。
- 但阻力较轻或章末牵引一般。

读者体感：

- “能看，是缓冲。”

### C 级

特点：

- 有日常细节。
- 状态变化弱。
- 章末钩子不具体。

读者体感：

- “有点水，但还能忍。”

### D 级

特点：

- 无目标。
- 无阻力。
- 无变化。
- 无回报。
- 无下一步。

读者体感：

- “跳过不影响。”

低于 B 级时，应压缩或重构。

## 16. AI agent 执行流程

### 16.1 先判断是否值得成章

问：

- 是否承接必须写出的余波？
- 是否改变关系、信息、资源、情绪或目标？
- 是否提供读者黏性的日常回报？
- 是否生成下一步行动？

如果答案少于两项为是，优先压缩。

### 16.2 写一句话任务

格式：

```text
本章用【日常/过渡/休整/赶路/聊天】这种低强度形式，承接【旧结果】，通过【小目标和低烈度阻力】，改变【状态】，交付【小回报】，并在章末生成【下一步动作】。
```

### 16.3 设计五个微变化

至少选择两个：

- 目标微变化。
- 关系微变化。
- 信息微变化。
- 资源微变化。
- 情绪微变化.
- 外部微变化。
- 读者回报微变化。

### 16.4 控制篇幅

硬规则：

- 连续两屏无状态变化，压缩。
- 连续三轮对话无新信息，删改。
- 连续两段生活细节无人物选择，删改。
- 日常章末没有具体下一步，补钩子。

### 16.5 回写台账

只记录变化：

- 谁关系变了。
- 谁知道了新信息。
- 谁欠下新情绪债。
- 什么资源改变。
- 哪个外部信号出现。
- 下一章必须承接什么。

## 17. 可直接使用的低强度章节防水卡

```text
日常章 / 过渡章 / 低强度章节防水卡

1. 本章类型：
   日常 / 过渡 / 休整 / 赶路 / 聊天 / 吃饭 / 情绪缓冲 / 信息整理 / 准备章 / 轻松章 / 其他

2. 承接旧结果：
   本章承接上一章或上一单元的哪个结果、代价、余波或钩子？

3. 本章小目标：
   角色这章具体要做什么小事？

4. 低烈度阻力：
   钱不够 / 伤没好 / 关系尴尬 / 资源涨价 / 外部盘查 / 情绪未解 / 信息缺口 / 其他

5. 合法功能：
   余波清账 / 关系推进 / 情绪支付 / 信息整理 / 世界质感 / 人物魅力 / 梗点轻松 / 资源准备 / 外部同步 / 下一目标生成

6. 微变化：
   目标变化：
   关系变化：
   信息变化：
   资源变化：
   情绪变化：
   外部变化：

7. 小回报：
   人味 / 糖点 / 笑点 / 线索 / 生活质感 / 角色魅力 / 小资源 / 世界规则

8. 需要压缩的内容：
   哪些吃饭、赶路、闲聊、复盘、准备、疗伤可以一句带过？

9. 章末具体动作：
   这章结尾落在哪个可见动作、信号、决定或未完成问题上？

10. 台账回写：
    关系 / 情绪债 / 资源 / 信息 / 外部信号 / 下一章任务
```

## 18. 审稿检查问题

审稿或改稿时逐项检查：

- 本章是否承接上一章具体结果？
- 本章是否有小目标？
- 是否有低烈度阻力？
- 是否改变目标、关系、信息、资源、情绪或外部状态？
- 日常细节是否影响人物或行动？
- 聊天是否有隐性目标和转折？
- 赶路是否不只是“到了”？
- 休整是否保留未恢复项？
- 复盘是否生成新决定？
- 是否有小回报：人味、糖点、笑点、线索、世界质感或角色魅力？
- 连续两屏是否都有微进展？
- 章末是否有具体下一步？
- 写后是否能回写台账？

如果超过四项为否，这章应压缩、合并或重构。

## 19. 与旧知识库规则的合并说明

本专题将既有知识库中分散的规则集中：

- 从连载节奏专题继承“慢节奏可接受，但反馈要及时”。
- 从中段防水专题继承“每章必须有状态增量”。
- 从人味增强专题继承“小动作、小细节、小缺陷让人物鲜活”。
- 从情绪线专题继承“关系状态必须有阶段变化，情绪债要付利息”。
- 从喜剧专题继承“轻松感和梗点必须产生后果和回响”。
- 从移动阅读专题继承“每一屏要有微进展，章末要有具体牵引”。
- 从时间跳跃专题继承“重复过程可以压缩，关键选择和代价不能跳”。
- 从外部同步升级专题继承“休整期和过渡期世界也要继续变化”。

本专题新增的是低强度章节专用判断：

- 低强度章节是否值得独立成章。
- 日常、过渡、休整、赶路、聊天分别承担什么功能。
- 如何用小目标和低烈度阻力让安静场景有方向。
- 如何用微变化和小回报维持追读。
- 什么时候应该压缩成段落或转场。

## 20. 资料依据与改写说明

本专题参考并改写了以下资料：

- Helping Writers Become Authors 关于 scene-sequel、reaction、dilemma、decision 的资料，强调强事件之后需要反应、困境和决定，转化为网文低强度章节的“余波、情绪支付、下一目标生成”。
- Helping Writers Become Authors 关于 scene sequences 和章节钩子的资料，强调场景序列中每一段都要服务整体推进，转化为“低强度章节也要有微变化和章末具体动作”。
- Writer's Digest 和 Writing Excuses 关于 pacing 的资料，强调节奏不是一直快，而是张弛、转场、信息和情绪安排要服务读者承诺，转化为“慢可以，但不能没有反馈”。
- Reedsy 关于 scene writing 和 dialogue 的资料，强调场景要有目的、冲突、转折和人物选择，对话要推动故事，转化为“聊天章必须有隐性目标和行动后果”。
- 起点社区关于爽文节奏、矛盾冲突、大高潮小高潮和生活流/日常写作的讨论，提示中文网文中低强度章节仍要有期待感、获得感、人物黏性和下一章牵引。

注意：这些资料不直接等同于中文网文公式。本专题已将其转化为适合 AI agents 执行的中文长篇规则：安静不是问题，原地不动才是问题；低强度章节必须用小目标、微变化、小回报和具体钩子证明自己值得存在。


## 与 63 号专题的衔接：低强度章是伏笔轻回响的最佳容器

低强度日常、转场、休整、战后清点、关系缓冲章可以承担 `docs/63-foreshadowing-callback-reader-memory-management.md` 中的 C1 轻回响与部分 C3 后果提醒。

用法：让旧物被顺手整理，旧称呼在闲聊中再出现，旧规则在小麻烦中生效，旧伤口在休息时复发，旧承诺在一次选择里轻轻碰到。注意不要把低强度章写成前情提要，轻回响必须同时服务当章生活感、人物关系或下一步推进。
