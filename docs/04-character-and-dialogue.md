# 人物与对话

## 相关入口：项目风格手册与角色声音规范

设计重要人物、群像、对话、视角叙事或长期连载声音时，先调用 [37. 项目风格手册与角色声音规范](37-project-style-bible-character-voice.md)。该专题负责记录项目整体声音、旁白距离、角色声音矩阵、称呼术语样式表和可保留粗糙，避免人物同声、口癖泛滥、修订改平和多 agent 续写文风漂移。

## 相关入口：地域、时代与社会风俗质感

涉及真实地域、历史时代、架空王朝、异世界社会、乡土现实、都市地方感、民族/宗教/异文化或地方风俗时，先调用 [31. 地域、时代与社会风俗质感](31-region-era-social-custom-texture.md)。该专题负责让背景改变人物行动、信息速度、资源成本、人情压力和读者体感，避免只换地名、时代错位、风俗猎奇和模板化场景。


## 相关入口：职业、行业与生活细节可信度

涉及职业身份、行业背景、现实生活、地域场景、专业流程、工具物件、生活成本或高风险行业时，先调用 [30. 职业、行业与生活细节可信度](30-profession-industry-life-detail-credibility.md)。该专题负责把资料转化为流程、物件、成本、人情和角色动作，避免职业标签化、行业百科化和场景空心化。


人物塑造的目标不是“资料卡很完整”，而是读者在场景中能分辨：这个人想要什么，怕什么，在压力下会怎么说、怎么动、怎么选择。

## 情绪线与关系推进入口

设计或修订感情线、暧昧线、误会、亲情、师徒、搭档、宿敌转化或任何强关系互动时，必须联动 `docs/25-emotion-line-relationship-progression.md`。人物鲜活不只靠台词差异，还要看每场互动是否改变信任、边界、误会、欠账、欲望或依赖。

## 喜剧人物入口

设计吐槽役、逗哏、捧哏、天然呆、正经人、群像反应或角色癖好时，必须联动 `docs/27-comedy-gags-lightness.md`。搞笑人物不能只靠口头禅和段子，要有稳定欲望、过度反应、场景错位和后果承担。

## 1. 重要人物必须有可见差异

不要只用姓名、职业、身份标签区分人物。重要人物至少要在以下维度有差异：

- 欲望：他主动追求什么。
- 压力：什么会让他急、撒谎、妥协、攻击或逃避。
- 羞耻或恐惧：他不想让别人看见什么。
- 利益优先级：他第一眼关注钱、地位、安全、证据、感情、职责、速度、面子、控制、公平还是生存。
- 说话习惯：长句还是短句，客气还是直接，爱不爱讽刺，是否回避重点，是否喜欢威胁或找借口。
- 行动习惯：沉默、抢话、查数据、安抚、甩锅、讲笑话、讨价还价、离场。
- 压力反应：变尖锐、变温柔、变大声、变小声、变滑、变诚实、变冷酷。

写作时不要把这些都展示给读者。作者和 AI 需要知道它们，是为了在场景中选出两三个最有用的反应。

## 2. 人物卡应记录什么

推荐人物卡：

```markdown
## 人物名

- 身份/位置：
- 当前欲望：
- 当前压力：
- 恐惧或羞耻：
- 利益优先级：
- 说话习惯：
- 行动习惯：
- 与主角关系：
- 对主角的误解或真实看法：
- 弧线变化：
- 首次登场要让读者记住的动作或选择：
```

如果要让人物更有“人味”，还要补：

- 最怕暴露：他不想让别人看见什么。
- 自保动作：压力来时，他默认如何保护自己。
- 习惯性误判：他因为旧伤、经验、欲望或羞耻，容易误判什么。
- 关系欠账：他和谁有未清的亏欠、误解、旧恩、旧伤、旧承诺。
- 梗点种子：一个能反复变化使用的语言、动作、物件、职业习惯或关系错位。

人物卡不要写成“善良、坚强、聪明”这类抽象词堆。要尽量写成可执行表现：

- 不写：他很谨慎。
- 改写：重要问题不会当场表态，会先问“谁还知道这件事”。
- 不写：她很有控制欲。
- 改写：别人还没说完，她已经把记录顺序改成对自己有利的版本。

## 3. 对话要有议程和遮掩

对话不是互相传递信息。每个说话的人最好都在做一件事：

- 争取某个结果。
- 隐藏某个事实。
- 保护某段关系。
- 试探对方知道多少。
- 转移责任。
- 抢先定义事件。
- 安抚场面。
- 逼对方表态。

对话自检：

- 这句话如果删掉，场景压力会不会下降？
- 这句话只是作者解释设定，还是人物为了自己的目的而说？
- 同一句信息能不能用更符合人物身份的方式说？
- 对方是否回答了问题背后的压力，而不是字面问题？

示例思路：

- 直白说明：“愿池记录显示你拿了她的灵根。”
- 带议程说法：“长老若说记录有误，就请当场写进问愿册，日后仙盟来查，也省得我们各执一词。”

后者不仅给信息，还逼对方承担公开记录的风险。

## 4. 群戏要用反应顺序区分人物

群戏最怕“所有人同时震惊、同时沉默、同时站队”。更有效的做法是安排反应顺序：

- 谁第一时间要证据。
- 谁先安抚受害者或装作安抚。
- 谁抢先定义事件。
- 谁威胁主角闭嘴。
- 谁算利益。
- 谁怕旧账翻出来。
- 谁沉默观察。
- 谁表面站队，实际留后路。

来自 `匿名旧项目A` 的可迁移经验：

- 宗门维护小师妹不能写成集体降智。
- 普通弟子维护的是宗门天才红利。
- 大师兄维护的是秩序和自我清白。
- 长老维护的是愿池收益和代价转嫁制度。
- 师尊维护的是更高层的飞升棋局。

这类分层能让群体压迫更可信，也能让主角反击有更多角度。

## 5. 反派和工具人也要社会真实

短期对手可以功能化，但不能完全纸片化。

可用动机：

- 求生存。
- 求晋升。
- 怕承担责任。
- 维护脸面。
- 忠于组织。
- 嫉妒。
- 赚钱。
- 避险。
- 坚信某套秩序。

反派不必聪明，但要有自我解释。他可以错，但不要只为了让主角爽而蠢。

工具人风险：

- 男主只负责解释规则。
- 朋友只负责捧主角。
- 反派只负责说恶毒台词。
- 路人只负责震惊。
- 编辑、客户、长老、警察、老师只负责给流程盖章。

修正方法：

- 给工具人一个和主角目标不完全一致的小利益。
- 给他一个不完美反应：迟疑、误解、怕担责、先顾自己。
- 给他一个能改变局面的微动作：记录顺序、压住半页账、抢走证据、假装听不懂、把话题引向程序。

如果对手、反派、组织阻力或制度压力会持续 3 章以上，必须联动读取 `docs/21-antagonist-opposition-system.md`。人物鲜活不只靠台词，也靠他能动用什么资源、如何误判主角、失败后如何调整策略；否则“坏人发疯”和“路人挑衅”会把角色写成纸片。

## 6. 主角必须持续做选择

主角可以被压迫、被动开局、暂时不知道真相，但不能长期被系统、男主、外挂、命运或旁白推着赢。

主角主动性可以表现为：

- 选择什么时候说出证据。
- 诱导对手亲口承认。
- 用规则逼对方公开记录。
- 放弃短期胜利换更大证据。
- 承担使用外挂的代价。
- 在两个都不完美的选择中做取舍。
- 故意让对手局部维稳成功，再抓更大破绽。

自检问题：

- 本章主角有没有一个可见选择？
- 这个选择失败会有什么后果？
- 如果删掉主角，本章是否仍会自动发生？如果会，主角太弱。

## 7. 人物鲜活度加压检查

当试写稿出现“人物不够丰满鲜活”时，不要只增加背景故事。优先检查：

- 这个人物是否有自保动作？
- 他是否有一个会害他做错判断的习惯性误判？
- 他在本场戏除了争利益，是否还在争面子、解释权、道德高地或被看见的方式？
- 他是否和场上某人有关系欠账？
- 他是否有一个读者能记住的反差、癖好、职业习惯或固定物件？
- 他在压力下有没有非完美反应：抢话、慢半拍、装懂、装没听见、转移话题、先护住某个物件？

更多方法见 `docs/11-human-writing-upgrade.md`。

## 8. 人物自检清单

起草或修订后检查：

- 不看名字，读者能否凭台词和动作区分关键人物？
- 每个重要人物进入场景时是否带着自己的目的或压力？
- 是否有人只是在替作者交代信息？
- 群戏是否出现全员同步理解、同步沉默、同步震惊？
- 对手是否有能自洽的动机和自保动作？
- 主角是否仍在做决定，而不是被规则和解释推着走？
- 是否有一个小动作、眼神、停顿、记录动作或站位变化承担了人物塑造？

## 高潮战、大场面与群体冲突调度入口

当章节或卷级单元涉及高潮战、决战、群战、团队战、公开审判、朝堂对峙、比赛决胜、灾难救援、发布会、直播爆点、群体谈判，或任何三条以上行动线同时推进的大场面时，需要调用 `docs/60-climax-battle-large-scene-coordination.md`。该专题负责确认目标、胜负条件、空间锚点、视角锚点、群体角色分工、关键转折、动作压缩、战后结算和台账回写，防止只堆人数、招式、爆炸、喊话和震惊。

## 补充：人物设计要包含视角过滤器

人物不只需要目标、伤口、关系和口头禅，也需要“他如何看世界”的过滤器。长期视角人物应参考 `docs/69-pov-narrative-distance-information-focus.md` 补充：第一注意物、误读倾向、词汇习惯、身体焦点、回避内容、价值排序和关系偏见。

写对话和内心时，优先让人物用自己的过滤器理解场景，而不是让旁白用统一口吻解释人物。这样能减少多角色同声，也能让同一场戏在不同人物眼里产生商业网文需要的误会、张力和情绪增量。
