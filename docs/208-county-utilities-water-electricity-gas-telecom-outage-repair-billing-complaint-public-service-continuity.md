# 208 县域供水、供电、燃气、通信、停复供、抢修、缴费投诉与民生保障链专项

状态：已确认  
更新日期：2026-06-09  
适用对象：需要写县城、乡镇、农村、老旧小区、返乡开店、民宿经营、工厂生产、老人独居、暴雨寒潮、停水停电、燃气隐患、通信中断、抢修投诉、表计争议和民生保供情节的 AI agents。

## 1. 这篇解决什么问题

水、电、气、通信在小说里常被当作背景板：灯一黑就恐怖，水一停就吵架，燃气一漏就爆炸，手机没信号就强行制造孤岛。这样写很方便，但现实感很薄。

县域基础设施不是开关，而是一套民生连续性系统。它包含用户、经营企业、表计、管线、抢修、停复供通知、缴费、欠费、阶梯价格、投诉、监管、应急保供、弱势群体照护、企业生产和公共安全。它能把家庭伦理、物业邻里、返乡创业、基层干部、应急管理和企业经营拧成一条很有张力的生活链。

核心链条：

`生活/经营/公共服务依赖 -> 水电气网状态变化（故障/欠费/检修/灾害/事故/建设迁改/投诉） -> 用户体感和关键风险出现 -> 判断服务类型、权属边界和主管/经营单位 -> 报修、客服、12345、12398、电信申诉或行业主管部门分流 -> 现场排查、抢修、告知、临时保供或安全处置 -> 费用、表计、赔偿、停复供、服务质量和责任争议 -> 投诉、调解、行政处理、诉讼或事故调查接口 -> 家庭、店铺、工厂、医疗照护、舆论和长篇台账回写`

## 3. 安全边界

本篇用于小说创作和审稿，不提供电力、燃气、供水、通信、抢修、破拆、泄漏处置、带电作业、管线定位、网络破坏或规避缴费的现实操作建议。

AI 写作必须遵守：

- 不描写可模仿的私接电线、偷电偷气、私改燃气管线、破坏水表电表、破坏基站、盗割电缆、绕过计量或规避欠费停供的方法。
- 不提供燃气泄漏、爆炸、电击、触电、抢修、破拆、带电作业、管道开挖、通信干扰等危险操作步骤。
- 不把居民自行排险写成英雄爽点；高风险情节应转向撤离、报警、专业人员处置和后果。
- 不把经营单位、窗口客服、抢修人员一概写成懒政恶人。很多冲突来自权属边界、抢修优先级、道路开挖、老旧管线、系统数据和极端天气。
- 不泄露用户户号、住址、缴费记录、通信记录、弱势群体信息和抢修人员个人信息。
- 不把12345、12398或运营商客服写成万能审判。它们能受理、分流、督办或申诉，但实体处理仍要按职责和事实推进。

## 4. 写作总原则

### 4.1 水电气网先写“谁依赖它”

基础设施桥段的第一步不是停了什么，而是谁被改变了生活：

- 老人：做饭、取暖、制氧、冰箱药品、手机联系子女。
- 孩子：上网课、洗澡、夜间照明、校车和接送。
- 小店：冷柜、收银、外卖、后厨、洗消、直播。
- 工厂：生产线、冷链、设备、订单、工人工资。
- 民宿：热水、空调、Wi-Fi、消防和差评。
- 医疗机构：抢救设备、消毒、供氧、冷链药品和信息系统。
- 村庄：灌溉、饮水、广播、基站、应急联络。

写清依赖关系，停水停电才不是廉价卡剧情，而是会真正改变角色选择的压力。

### 4.2 先分清“用户侧”和“公共侧”

许多纠纷来自责任边界不清。

可分为：

- 公共管网、线路、变压器、基站、燃气主管道、供水主管：通常由经营单位或相关单位负责运行维护。
- 小区内部管线、二次供水设施、配电房、公共照明、物业管理区域：可能涉及物业、业主共有、开发商遗留、专业经营单位接管。
- 用户户内管线、家电、灶具、热水器、路由器、插座、充值卡：多与用户、产权人、租客或专业安装维修相关。
- 农村分散供水、村级泵房、小型供水工程：还涉及村集体、乡镇、供水单位、管护资金和人员。

角色先弄清边界，才能知道找谁、谁能进场、谁付钱、谁承担安全责任。

### 4.3 停供不等于断供，原因要拆开

停水停电停气通信中断有很多原因：

- 计划检修：应有通知、时间安排和恢复预期。
- 突发故障：爆管、跳闸、设备故障、基站断电、管线被挖断。
- 灾害天气：暴雨、寒潮、台风、山洪、冰冻、雷击、高温负荷。
- 施工迁改：道路施工、老旧小区改造、管网改造、工程碰线。
- 欠费或合同问题：用户缴费、表计、户号、过户、租客房东责任。
- 安全隐患：燃气隐患、电气火灾风险、水质异常、通信设施危险。
- 上游供应：水源、电源、气源、传输链路、调度限制。

不同原因决定不同情绪。计划检修的冲突是通知是否到位；突发故障的冲突是抢修速度和保供；欠费的冲突是责任和证据；安全隐患的冲突是不能为了方便冒险恢复。

### 4.4 抢修戏的重点是分工和优先级，不是危险技巧

抢修现场可以有戏，但不能教程化。

可写：

- 客服接到多户报修，判断是单户、楼栋、小区还是区域性故障。
- 抢修队先保护现场、设置警戒、联系物业、交警、城管或燃气公司。
- 医院、养老院、泵站、通信机房、学校、冷链仓库被列为优先保障对象。
- 群众只看见“怎么还没修好”，抢修人员还要等备件、停电停气安全确认、道路开挖许可或天气窗口。
- 恢复后还要排气、冲洗、检测、重启设备、通知用户和处理二次投诉。

不要写具体带电操作、燃气排险、管线切割、通信破坏和规避监管步骤。

### 4.5 缴费和表计争议能写出很强生活感

基础设施不是免费背景。可写：

- 老人不会线上缴费，充值卡或智能表切换后出问题。
- 租客搬走留下欠费，房东和新租客吵起来。
- 水表电表燃气表读数异常，用户怀疑被偷接或表坏。
- 阶梯水价、电价、气价让大家第一次看懂“用量”。
- 企业用能费用暴涨，现金流和生产计划被打乱。
- 小店停电导致冷柜货损，责任和赔偿难以认定。
- 宽带套餐、携号转网、信号覆盖、装维预约引发投诉。

生活感来自账单、户号、读数、缴费截图、催缴短信、客服录音和维修单。

### 4.6 通信不是只有“没信号”，它是社会连接

通信中断会影响：

- 手机支付、外卖接单、直播带货、网课和远程办公。
- 老人紧急求助、留守儿童联系父母、村干部发布通知。
- 12345、110、120、抢修热线、微信群和广播系统。
- 工厂订单、物流扫码、冷链监控、游客导航。
- 谣言传播和信息真空。

县域通信戏不要只写“手机没信号制造孤岛”。更好的写法是：一个基站断电、光缆中断或宽带装维迟迟不到，会让现代生活的很多链条突然失灵。

### 4.7 民生保供是基层治理，不是口号

水电气网在极端天气、春节返乡、高温寒潮、重大活动、疫情或灾后恢复中会进入保供状态。

可写张力：

- 供电公司调度保医院和泵站，普通居民等待恢复。
- 供水车进村，老人排队提水，年轻人争论谁该先打。
- 燃气公司排查隐患，餐馆老板抱怨停业损失。
- 通信抢修队翻山恢复基站，村干部用大喇叭临时通知。
- 12345工单暴增，热线人员和承办单位都被压住。
- 基层干部协调水务、电力、燃气、通信、物业、应急和民政，像拧一团湿毛线。

## 5. AI 写作前的七项判断

### 5.1 服务类型判断

先判断是哪条服务链：

- 供水：水源、水厂、管网、二次供水、水质、爆管、停水、抄表缴费。
- 供电：户表、配网、变压器、故障停电、计划检修、抢修、用电安全、企业用电。
- 燃气：管道燃气、瓶装液化气、用户设施、安检、停气、泄漏隐患、经营服务。
- 通信：移动信号、基站、宽带、光缆、装维、套餐、应急通信、网络中断。
- 排水污水：雨污管网、污水处理、内涝、堵塞、窨井、泵站，必要时联动第 207 篇。

### 5.2 影响对象判断

谁受影响决定情绪和优先级：

- 普通居民：做饭、洗澡、照明、取暖、手机支付。
- 弱势群体：老人、病人、婴幼儿、残障者、低收入家庭。
- 经营主体：小店、民宿、餐馆、工厂、合作社、冷库。
- 公共机构：医院、学校、养老机构、社区卫生站、泵站、消防设施。
- 抢修人员：夜间出勤、极端天气、群众围堵、设备风险。
- 基层干部：协调资源、安抚群众、联系企业和上报情况。

### 5.3 原因和边界判断

停供或服务差要判断：

- 是计划检修还是突发故障。
- 是公共侧问题还是用户侧问题。
- 是经营单位责任、物业责任、施工第三方责任，还是用户使用问题。
- 是缴费、合同、过户、表计还是安全隐患。
- 是否涉及灾害事故，需要转入应急链。
- 是否涉及价格、服务质量、赔偿或行政监管。

### 5.4 信息通知判断

很多冲突来自“没人告诉我”。检查：

- 是否提前公告计划停水停电停气。
- 通知渠道是短信、电话、公众号、小区群、物业公告、村广播还是上门通知。
- 老人、租客、商户、外地业主是否收到。
- 恢复时间是否反复变化。
- 客服答复是否统一，是否有工单编号。
- 事故或故障中是否及时发布权威信息，避免谣言。

### 5.5 证据和记录判断

可用证据：

- 缴费记录、催缴短信、账单、户号、表计照片。
- 报修电话、客服录音、12345工单、12398或电信申诉记录。
- 停供公告、物业通知、施工公告、抢修单、维修照片。
- 货损清单、冷链温度记录、设备报警记录、营业损失资料。
- 水质检测、燃气安检记录、电力故障记录、通信装维单。

### 5.6 出口判断

不同争议走不同出口：

- 单户故障：报修、物业或专业维修、用户侧排查。
- 区域故障：经营单位抢修、公告、临时保供、12345或行业投诉。
- 电力服务争议：供电企业客服、12398能源监管热线、监管机构或司法路径。
- 通信服务争议：运营商客服、工信部相关申诉渠道、12345或消费者投诉。
- 燃气安全隐患：燃气经营企业、住建等主管部门、应急消防和第 164 篇事故链。
- 欠费停供：账单核对、缴费、过户、租赁责任和合同争议。
- 损失赔偿：协商、调解、投诉、诉讼和证据链。

### 5.7 台账回写判断

水电气网情节之后，要回写：

- 服务是否恢复，是否反复故障。
- 谁承担费用和损失。
- 哪条管线、哪个户号、哪个表计、哪个工单成为后续证据。
- 小店、工厂、民宿和家庭计划是否被改变。
- 老人、病人、孩子是否获得后续保障。
- 物业、经营单位、基层干部和用户之间的信任是否改变。

## 6. 可复用剧情模型

### 6.1 小店冷柜停电，一夜赔掉半个月利润

适用：返乡创业、食品经营、现实经营压力。

剧情骨架：

`小店开业前囤货 -> 夜里停电 -> 冷柜报警但主角睡着没看到 -> 次日货品变质 -> 查公告发现是突发故障不是计划检修 -> 主角找供电企业、物业和保险/合同链 -> 最终发现真正漏洞是自己没有备用方案和温度记录 -> 生意爽点从卖爆变成学会经营风险`

### 6.2 老旧小区二次供水水压不足

适用：物业群像、老人生活、基层治理。

剧情骨架：

`高层住户晚高峰没水 -> 物业说供水公司问题 -> 供水公司说小区二次供水设施未移交或泵房老化 -> 业主群吵翻 -> 街道、物业、供水单位和业委会协调 -> 维修资金、改造费用和停水施工安排成为新矛盾`

### 6.3 燃气安检让餐馆停业一天

适用：小商户、食品经营、安全边界。

剧情骨架：

`餐馆生意刚好转 -> 燃气安检发现隐患 -> 老板抱怨影响营业 -> 安检人员不能为了人情放行 -> 老板试图找熟人被拒 -> 经过整改、复查和重新供气 -> 他明白安全不是吓人的牌子，而是店能不能长期开下去`

### 6.4 暴雨后基站断电，村里信息真空

适用：灾害后续、乡村治理、悬疑。

剧情骨架：

`暴雨导致停电和道路塌方 -> 基站备用电源耗尽 -> 手机信号断续 -> 村干部用大喇叭和摩托车传消息 -> 通信抢修队进村困难 -> 谣言开始扩散 -> 恢复通信后，真正的救援和物资需求才被准确上报`

### 6.5 12398投诉让电压低问题进入记录

适用：农村用电、企业生产、维权现实。

剧情骨架：

`合作社冷库频繁跳闸 -> 供电所多次上门但问题反复 -> 主角记录电压波动、设备损失和报修单 -> 咨询后走供电企业客服和12398投诉渠道 -> 监管压力推动台区改造排期 -> 解决不是当天换天，而是问题进入正式计划和责任链`

### 6.6 新租客接手旧欠费

适用：租房、商铺、房东租客纠纷。

剧情骨架：

`主角租下小门面 -> 开业时发现水电气有历史欠费或户名未过户 -> 房东说前租客问题 -> 经营单位只认户号和账单 -> 中介合同没写清交割 -> 主角补证据、找房东、查表底 -> 联动第 175 篇房屋交割链`

## 7. 场景素材库

### 7.1 停水现场

可用细节：

- 小区楼下排起接水队，塑料桶、热水壶和脸盆都拿出来。
- 物业群里有人转发旧公告，真正的抢修通知被淹没。
- 老人问“什么时候来水”，年轻人只看客服排队号。
- 供水车到了，谁先接水又变成新的争吵。

### 7.2 电力抢修

可用细节：

- 雨夜抢修车停在巷口，灯光打在湿漉漉的电线杆上。
- 群众围在警戒线外问“怎么还没好”。
- 抢修人员不断接电话，却只能重复“先保安全，再恢复”。
- 医院、泵站和老旧小区被列进优先恢复清单。

### 7.3 燃气安全检查

可用细节：

- 餐馆老板最怕午市停气，安检员最怕他抱侥幸心理。
- 老人家里软管、灶具、通风和报警器问题被写成生活习惯冲突，不写操作方法。
- 安全隐患通知单比吵架更有分量，因为它决定能不能恢复供应。

### 7.4 通信装维

可用细节：

- 宽带师傅一天跑十几户，最难解释的是“你家路由器”和“外线故障”不是一回事。
- 村里基站修好后，微信群消息一下子涌出来，真正紧急的求助反而被淹没。
- 外卖骑手、直播主播、留守儿童和村医都在等信号恢复。

## 8. AI 自查清单

写完水电气通信情节后，用这些问题检查：

- 是否先写清谁依赖这项服务，停供或故障改变了什么？
- 是否区分供水、供电、燃气、通信、排水污水和服务边界？
- 是否判断公共侧、物业侧、用户侧和第三方施工责任？
- 是否写出通知、报修、工单、抢修、恢复、回访和投诉链？
- 是否保护老人、病人、婴幼儿、冷链、医院、学校等重点对象？
- 是否避免危险操作、私接改装、偷水偷电偷气、破坏通信和抢修教程？
- 是否把客服、抢修人员、物业和基层干部写成有分工的人，而不是 NPC？
- 是否写出账单、户号、表计、缴费、过户和赔偿证据？
- 是否在事故风险出现时及时联动第 164 篇，而不是让主角自行排险？
- 服务恢复后，是否回写损失、信任、合同、经营和后续防范？

## 10. 禁写和风险

- 禁止写私接电线、偷水偷电偷气、私改燃气、破坏水表电表、盗割电缆、干扰通信、绕过计量和欠费规避的具体方法。
- 禁止写燃气泄漏排险、带电作业、管线抢修、开挖破拆、通信破坏、危险设备处理的操作步骤。
- 禁止把普通居民自行抢修、强行开阀、强行送电写成英雄行为。
- 禁止把抢修人员、客服、物业和基层干部一律写成恶人；可写失职，但要有证据、程序和后果。
- 禁止泄露真实户号、住址、缴费记录、通信记录和弱势群体隐私。
- 禁止把12345、12398、电信申诉或曝光写成万能快捷键。它们是入口、记录和督办，不是替代事实和责任认定。

## 11. 来源与依据

- 《供水条例》自 2026 年 6 月 1 日起施行，覆盖城乡供水、农村供水保障、供水工程建设、运行维护、水质安全和服务保障，是写县域供水和农村规模化供水的重要依据。
- 《电力法》《电力供应与使用条例》和新版《供电营业规则》为供电服务、用电安全、停复电、供用电合同、用户权益和营业服务提供基础框架。
- 国家能源局 12398 能源监管热线投诉举报相关资料，为供电、供气等能源服务投诉、举报、咨询和监管提供叙事入口。
- 《城镇燃气管理条例》为燃气经营、燃气使用、安全管理、设施保护、事故预防和服务责任提供依据。
- 《电信条例》《电信服务规范》和电信用户申诉处理资料，为通信服务、装维、服务质量、资费争议和用户申诉提供依据。
- 政务服务和12345相关文件为民生诉求、热线转办、承办答复、督办和回访提供背景。
- 应急管理、消防和灾害安全资料只用于风险边界：涉及燃气泄漏、触电、爆炸、火灾、抢修和灾害时，应转入第 164 篇并避免操作细节。
