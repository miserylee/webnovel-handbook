# 111. 创作调研、事实核查、来源可信度与小说转化专项

状态：已确认

适用范围：当 AI agent 需要联网查资料、写真实职业/行业/地域/历史/科技/法律/医疗/民俗/平台规则/公共事件背景，或把真实案例、新闻、研究、社区经验、官方资料转成小说素材时，使用本文件。

核心结论：调研不是把资料搬进正文，而是把可靠事实转化为故事中的限制、压力、选择、代价、质感和后果。AI 输出、模型记忆和“看起来合理”都不是来源；所有会影响现实可信度、合规、安全或读者信任的事实，都必须标注来源、时间、可信度和使用边界。

---

## 1. 本专题解决什么问题

AI agent 写商业网文时，常见资料问题有四类：

1. 胡编现实：职业流程、法律后果、医疗常识、地理距离、历史制度、行业规则一眼假。
2. 堆百科：查到资料后整段解释，正文变成说明书，剧情停滞。
3. 资料洗稿：把新闻、案例、知乎经验、竞品桥段换名后写成原创。
4. 来源混乱：把论坛段子、短视频口播、AI 回答、过期资料和官方规定放在同一可信层级。

本专题的作用：

- 原则：先明确故事需要什么事实，再查资料；先判断来源可信度，再转化成剧情功能。
- 用法：把调研拆成“研究问题 -> 来源分级 -> 事实核查 -> 素材转化 -> 正文闸门 -> 来源回写”。
- 风险：不查会假；乱查会水；直接搬会抄；只信单一来源会偏；高风险内容写细会造成合规和伤害问题。
- 检查问题：这个资料在正文里具体制造了什么压力、选择、限制、代价或情绪？如果没有，它可能只是信息垃圾。

## 3. 来源吸收结论

### 3.1 来源评估

Purdue OWL 等研究写作资料强调，使用资料前要评估作者、出版方、时效、证据、目的和适用性。迁移到小说写作中，agent 不能只问“搜到了没有”，还要问“谁说的、什么时候说的、凭什么说、为了什么说、是否适合本书语境”。

### 3.2 一手资料与二手资料

Library of Congress 等资料将 primary sources 视为来自事件、时期或亲历者的原始材料。迁移到小说中，一手资料可以提供语言、制度、生活痕迹和细节质感；二手资料可以提供解释框架；三手资料只适合快速导航，不能单独支撑关键事实。

### 3.3 横向阅读与事实核查

SIFT、lateral reading 和事实核查实践强调：看到信息后要停一下，调查来源，寻找更可靠报道，并追溯到原始语境。迁移到 AI agent：不要在同一页面里被说服，要跳出去看发布方、引用链、日期、反证和原始材料。

### 3.4 小说创作调研

Reedsy、Writer's Digest 等写作资料强调，调研能让故事更可信，但作者不应把所有研究成果塞进小说。迁移到网文：资料只在它改变人物行动、阻力、关系、环境、后果或读者理解时进入正文。

### 3.5 中文网络语境

中国互联网联合辟谣平台、国家统计局、政府部门官网、平台官方公告等来源可用于核对公共事实、数据、政策或平台规则。迁移规则：涉及实时规则、政策、行业数据、平台机制和公共安全内容时，优先查官方或权威机构，并标注日期。

## 4. 核心概念

### 4.1 研究问题

定义：为了完成某个故事任务，需要确认的具体问题。

坏问题：这个行业是什么样的？

好问题：在县城医院急诊夜班里，一个新医生最容易被什么流程限制？这种限制如何迫使主角做选择？

### 4.2 事实主张

定义：一个可以被证实、证伪或限定适用范围的具体说法。

例子：

- “某平台当前发文规则要求……”是事实主张，需要查日期和官方链接。
- “法医工作很神秘”不是事实主张，是模糊印象。
- “古代县衙审案都当场打板子”是高风险概括，需要历史时期、地区、制度和来源限定。

### 4.3 来源可信度

定义：某个来源对某类问题的可依赖程度。可信度不是固定身份，而是“来源与问题的匹配”。

例子：

- 官方公告适合查平台规则，但不一定适合判断读者真实吐槽。
- 亲历者访谈适合查体感和流程细节，但不一定代表全行业。
- 竞品小说适合查题材惯例，不适合作为现实事实来源。
- AI 回答适合启发搜索方向，不适合作为证据。

### 4.4 资料转化

定义：把外部资料从“信息”改造成小说中的故事功能。

转化链：事实 -> 限制 -> 压力 -> 选择 -> 行动 -> 后果 -> 读者感受。

### 4.5 不确定性标签

定义：agent 对资料状态的显式标注。

常用标签：

- 已确认：来源可靠，时间适用，与任务匹配，至少能支撑当前用法。
- 候选：来源有价值，但样本小、语境不完全匹配或尚缺交叉核验。
- 待确认：可能有用，但不能进入关键剧情事实。
- 冲突：多个来源说法不一致，需要写明冲突并暂停关键使用。
- 过期风险：规则、数据、政策、平台机制或技术信息可能已变化。

## 5. 来源分级

| 层级 | 来源类型 | 适合用途 | 风险 |
| --- | --- | --- | --- |
| A | 官方文件、法规、平台公告、机构数据、正式标准、原始档案 | 关键事实、规则、日期、制度、数据 | 可能难读、更新慢、缺少生活体感 |
| B | 学术论文、专业书、行业报告、专家访谈、权威媒体深度报道 | 解释框架、行业趋势、历史背景、专业判断 | 可能过于宏观，不适合直接入戏 |
| C | 亲历者访谈、职业社区、地方经验、口述材料、纪录片 | 生活细节、语言习惯、场景质感、人情压力 | 样本偏差大，需要标为候选或交叉核验 |
| D | 普通新闻、百科、科普文章、教程、问答平台 | 快速入门、关键词、检索线索 | 不能单独支撑高风险事实 |
| E | 同题材小说、影视、短剧、游戏、读者评论 | 类型惯例、读者期待、包装感、市场语感 | 不能当现实事实；容易造成融梗和洗稿 |
| F | AI 输出、模型记忆、未注明来源的搬运号、短视频口播 | 搜索启发、问题清单 | 不能作为来源，不能直接写入项目事实 |

最小规则：关键事实优先用 A/B；生活质感可用 C，但必须标注样本边界；E 只能学类型功能；F 只能帮你找下一步该查什么。

## 6. 事实风险分级

### 6.1 高风险事实

包括：法律、医疗、心理危机、金融投资、公共安全、犯罪方法、未成年人保护、真实政策、平台规则、现实民族/宗教/地域群体、真实人物、真实公司、技术安全、军事警务流程。

处理规则：

1. 必须查可靠来源，优先官方、机构、专业资料。
2. 必须标注日期和适用范围。
3. 不把操作性细节写成教程。
4. 不用单一论坛经验支撑关键剧情。
5. 不确定时降级为虚构制度、架空设定或模糊处理。

### 6.2 中风险事实

包括：职业流程、行业潜规则、学校制度、城市地理、历史习俗、地方生活、交通时间、消费水平、职场关系、赛事规则。

处理规则：

1. 至少查一个可靠来源和一个生活体感来源。
2. 把事实转成场景限制，不堆背景说明。
3. 用“某地/某时期/某行业里可能存在”替代全称判断。
4. 对过期资料标注过期风险。

### 6.3 低风险素材

包括：饮食、天气体感、普通物件、街景氛围、非关键流程、小道具、小习惯、语气词、日常噪音。

处理规则：

1. 可以用少量来源或生活经验。
2. 仍需避免地域、职业、性别、年龄刻板印象。
3. 只保留能服务视角、压力或人物的细节。

## 7. agent 调研工作流

### 7.1 明确叙事需求

先问：这次调研要解决什么故事问题？

可选类型：

- 让场景更可信。
- 让阻力更具体。
- 让人物职业动作更像真的。
- 让地域/时代/风俗不悬浮。
- 让反转不靠胡编。
- 让合规和敏感内容不过线。
- 让世界观借鉴现实规律但不复制现实事件。

输出：一句研究问题。

### 7.2 拆成事实主张

把大问题拆成 3 到 7 个可查主张。

例子：

研究问题：女主是基层劳动仲裁员，她在第一章遇到的最大流程限制是什么？

事实主张：

- 劳动争议受理需要哪些基本材料。
- 当事人最常见的证据缺口是什么。
- 基层窗口的时间压力和沟通压力是什么。
- 仲裁员能做什么、不能做什么。
- 哪些内容不宜写成法律建议。

### 7.3 选择来源组合

不要只搜一个关键词。至少组合：

- 一个规则/官方/权威来源。
- 一个解释/行业/专业来源。
- 一个生活体感或案例来源。
- 如涉及敏感或高风险内容，再查合规和安全来源。

### 7.4 评估来源

对每个来源问六个问题：

1. 作者是谁？是否有相关资质或责任主体？
2. 发布方是谁？是官方、机构、媒体、个人还是搬运？
3. 发布时间是什么？是否可能过期？
4. 证据来自哪里？是否引用原始材料？
5. 写作目的是什么？科普、营销、吐槽、宣传、争议、教程还是研究？
6. 它适合支撑哪类小说用法？事实、体感、语言、氛围、冲突，还是只适合作为检索线索？

### 7.5 横向核验

对关键事实执行 SIFT 式检查：

- 停一下：不要被第一个答案带走。
- 查来源：看发布方、作者、日期和利益关系。
- 找更可靠覆盖：看官方、机构、专业媒体或多个独立来源。
- 追溯原始语境：能回到原始公告、法规、论文、采访或数据表，就不要停在二手转述。

### 7.6 提取可用材料

不要摘长段原文。只提取以下内容：

- 限制：角色不能做什么。
- 权限：角色能做什么。
- 流程：角色必须经过哪些环节。
- 成本：时间、钱、关系、名誉、体力、风险。
- 例外：什么情况下规则会变。
- 常见误解：外行以为怎样，内行知道怎样。
- 语言：称呼、术语、黑话、口头习惯。
- 场景物：单据、工具、制服、设备、空间布局、噪音。
- 后果：做错会导致什么。

### 7.7 转成故事功能

资料进入正文前，必须回答：它服务了什么？

常见转化：

- 事实 -> 世界限制。
- 流程 -> 倒计时和阻力。
- 术语 -> 身份识别或误解。
- 数据 -> 规模感或资源稀缺。
- 生活细节 -> 视角在场感。
- 亲历者经验 -> 人情压力或职业疲惫。
- 真实案例 -> 功能变量，而不是桥段复刻。
- 历史制度 -> 权力关系和选择代价。

### 7.8 正文投放

投放规则：

1. 关键选择现场可以写细。
2. 重复流程用摘要。
3. 背景资料只给当前问题需要的部分。
4. 术语首次出现要绑定动作、后果或情绪。
5. 不在付费章用长百科吞字数。
6. 不让角色变成资料讲解员。
7. 不把危险、违法或自伤内容写成可模仿教程。

### 7.9 回写台账

调研结果要进入项目资料卡，而不是散落在聊天里。

至少记录：

- 研究问题。
- 事实主张。
- 来源链接。
- 日期。
- 可信度。
- 使用边界。
- 已转化的剧情功能。
- 未确认问题。
- 不得写入正文的高风险细节。

## 8. 资料转化模型

### 8.1 职业流程转化

资料：某岗位每天要填大量表单。

无效写法：介绍表单种类和制度历史。

有效转化：主角在关键时刻想救人/抢单/抓证据，却被一张必须签字的表卡住；他选择违规推进，就欠下后续责任。

### 8.2 地域资料转化

资料：某地冬季潮湿阴冷。

无效写法：百科式介绍气候。

有效转化：外地来的主角以为多穿就行，结果衣服晾不干、关节痛、楼道水汽重，导致他错判行动时间和人际距离。

### 8.3 历史制度转化

资料：某时期存在严格身份流动限制。

无效写法：解释制度条文。

有效转化：角色明明有能力，却因为身份文书无法进入某个空间；他必须求人、造假、绕路或公开撕破关系。

### 8.4 行业数据转化

资料：某行业利润率下降。

无效写法：大段写行业报告。

有效转化：老板不敢裁人又发不出奖金，主角的项目被砍，团队关系从合作变成互相自保。

### 8.5 真实案例转化

资料：一个真实纠纷案例。

无效写法：换名字、换地点、换职业后复刻。

有效转化：抽出“误解机制、利益结构、证据缺口、情绪触发”，换成完全不同的人物关系、情节路径和结果。

### 8.6 科技/规则转化

资料：某技术有能力边界和安全限制。

无效写法：写成说明书或神奇外挂。

有效转化：主角知道技术能解决一半问题，但另一半必须靠人判断、资源交换或冒险选择。

## 9. 高风险题材处理规则

### 9.1 医疗、心理和创伤

- 不把剧情写成诊疗建议。
- 不写可模仿的自伤、自杀或危险行为步骤。
- 不用“奇迹治疗”替代后果和恢复过程。
- 涉及创伤、心理危机和未成年人时，联动 `docs/105-sensitive-content-trauma-informed-harm-avoidance.md`。

### 9.2 法律、警务和犯罪

- 不把正文写成逃避侦查、犯罪实施或规避监管教程。
- 法律后果要限定地区、时代和虚构程度。
- 不确定时使用架空制度，并在项目文档标注“现实参考，非现实法律”。

### 9.3 金融、投资和商业

- 不写成现实投资建议。
- 数据要标注来源和时间。
- 商业规则要服务角色选择，不要堆行业报告。

### 9.4 民族、地域、宗教和真实群体

- 不用单一轶事概括群体。
- 不把刻板印象当生活质感。
- 需要区分“角色偏见”和“作品立场”。

### 9.5 平台规则和现实政策

- 平台规则、福利、审核、签约、发文限制和活动信息都可能变化。
- 使用前必须重新确认最新日期和原文。
- 知识库只沉淀可迁移写作逻辑，不替代实时规则查询。

## 11. 审稿检查清单

当审稿发现“假、悬浮、像百科、专业细节不对、历史感不对、地域不对、AI 胡编、资料太多、像新闻改写”时，agent 按以下顺序检查：

1. 这一段依赖哪些现实事实？
2. 每个事实有没有来源？来源是否适合这个用途？
3. 是否把 AI 输出或模型记忆当成证据？
4. 是否存在时间过期、地区错位、制度错位或群体刻板印象？
5. 是否有高风险内容被写成教程？
6. 是否把真实案例换皮复刻？
7. 是否把资料整段倒进正文，导致场景停滞？
8. 资料是否转化为限制、压力、选择、代价或后果？
9. 读者当前是否需要知道这些信息？
10. 哪些内容应移入项目资料卡，而不是留在正文？

## 12. 常见失败模式与修复

### 12.1 搜到即写

症状：agent 搜到第一条资料就当事实写入正文。

修复：对关键事实至少执行来源调查、日期检查和交叉核验。

### 12.2 AI 自证

症状：agent 用“常识上应该”“通常来说”“我知道”支撑剧情。

修复：把 AI 输出降级为搜索问题；真实事实必须由外部来源或项目 canon 支撑。

### 12.3 资料倾倒

症状：章节中出现大段制度、行业、历史、地理介绍，人物停在那里听讲。

修复：只保留当前选择所需信息；其余放入项目资料卡或省略。

### 12.4 单一样本泛化

症状：一个亲历帖变成整个行业、地区或群体的普遍真相。

修复：标注“个体经验”，再查更大范围资料；正文中用具体角色经验，不写群体结论。

### 12.5 竞品当事实

症状：从同题材小说学到职业流程、历史规则或地理细节，并当成现实。

修复：竞品只用于类型惯例和读者期待；事实另查。

### 12.6 过期规则

症状：平台规则、政策、技术、法规、职业流程使用旧资料。

修复：标注发布日期和访问日期；涉及现实规则时重新联网确认。

### 12.7 真实案例洗稿

症状：真实新闻或纠纷案例换名后进入正文。

修复：只抽功能变量，重构人物关系、因果链、场景顺序和结局。

### 12.8 危险细节教程化

症状：犯罪、伤害、自伤、逃避监管、危险技术被写得可照做。

修复：只写后果、情绪、阻力和道德压力，不写可操作步骤。

## 13. 最小执行规则

如果时间很少，agent 至少执行这 8 条：

1. 先写研究问题，不要直接搜泛词。
2. 高风险事实必须用可靠来源，不能靠 AI 记忆。
3. AI 输出只能当搜索启发，不能当来源。
4. 每条关键事实都要标注来源、日期、可信度和使用边界。
5. 论坛、社区、亲历帖只能代表样本，不能代表全部。
6. 竞品小说只能学习类型功能，不能当现实资料。
7. 资料进入正文前必须转化为限制、压力、选择、代价或后果。
8. 不写危险、违法、自伤或规避监管的可操作教程。

## 14. 推荐调用顺序

### 14.1 写真实职业/行业

1. 读本文件。
2. 读 `docs/30-profession-industry-life-detail-credibility.md`。
3. 如涉及敏感或合规，读 `docs/20-content-compliance-risk-review.md` 与 `docs/105-sensitive-content-trauma-informed-harm-avoidance.md`。
4. 写调研卡，再写正文。

### 14.2 写地域、时代、民俗

1. 读本文件。
2. 读 `docs/31-region-era-social-custom-texture.md`。
3. 读 `docs/78-information-density-cognitive-load-term-management.md` 控制名词密度。
4. 写入地域/时代/风俗资料卡。

### 14.3 改编真实案例

1. 读本文件。
2. 读 `docs/102-originality-source-transformation-anti-plagiarism.md`。
3. 如涉及真实伤害、未成年人或创伤，读 `docs/105-sensitive-content-trauma-informed-harm-avoidance.md`。
4. 只抽功能变量，不复刻桥段。

### 14.4 联网维护知识库

1. 先确认知识缺口。
2. 联网查至少一组可靠来源。
3. 登记到 `docs/01-source-inventory.md`。
4. 把稳定结论写入专题文档。
5. 如形成新规则，写入 `docs/99-decision-log.md`。
