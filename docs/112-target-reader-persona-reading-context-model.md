# 112. 目标读者画像、阅读场景与阅读任务建模专项

状态：已确认

适用范围：当 AI agent 需要开新书、重定方向、判断题材适配、设计开篇、写书名简介、做 A/B 版本、审稿读者反馈、解释数据波动、判断非目标读者意见、设计付费节点或维护读者关系时，使用本文件。

核心结论：目标读者不是年龄、性别、地区、收入这类人口标签，而是“在某个阅读场景里，为了某种情绪/认知/身份/关系需求，愿意为某类承诺付出时间、注意力、评论、追更或付费的读者”。AI agent 必须把目标读者写成阅读任务模型，而不是泛泛写“喜欢爽文的人”。

---

## 1. 本专题解决什么问题

很多 AI 生成的网文项目会写“目标读者：18-35 岁女性”“目标读者：喜欢玄幻爽文的人”“目标读者：大众读者”。这些描述看似完整，实际无法指导正文：

- 不知道读者为什么点开。
- 不知道读者当前最想得到什么回报。
- 不知道读者能忍受多少铺垫。
- 不知道读者对主角、关系、雷点、节奏和付费的底线。
- 不知道哪些反馈来自目标读者，哪些只是非目标读者误入。

本专题的作用，是把目标读者从“市场标签”变成“创作控制台”。

最小链路：

读者画像假设 -> 阅读入口 -> 阅读场景 -> 阅读任务 -> 核心回报 -> 耐心预算 -> 雷点边界 -> 退出触发 -> 反馈证据 -> 画像修正 -> 写作决策

## 3. 来源吸收结论

### 3.1 用户研究与 persona

Nielsen Norman Group、GOV.UK Service Manual 等用户研究资料强调，理解用户不能只靠人口统计标签，而要理解用户目标、行为、上下文、痛点和任务。迁移到网文创作中，目标读者画像应服务决策：这类读者点开作品时想完成什么阅读任务？他们如何判断值得继续？他们会在哪个节点失去耐心？

### 3.2 目标读者与图书营销

Reedsy、Writer's Digest、Jane Friedman 等写作与出版资料强调，作者需要明确作品写给谁、读者为什么会被吸引、相似作品吸引了谁，以及读者期待如何影响包装和文本。迁移到中文网文，目标读者不是“限制作品”，而是帮助 agent 明确第一章、简介、标签、更新节奏、付费节点和长线回报应该优先服务谁。

### 3.3 读者测试与反馈边界

beta reader、用户测试和读者反馈资料都提醒：反馈有价值，但必须看反馈者是否接近目标读者、是否处在正确阅读阶段、反馈是否指向同一问题。迁移到网文，不能把非目标读者的不适直接等同作品失败，也不能把目标读者的单条吐槽当成全体需求。

### 3.4 信息气味与阅读入口

Nielsen Norman Group 的 information scent 思路可迁移为：读者会根据书名、封面、简介、标签、章节名、推荐语、评论和第一屏判断“这里有没有我要的东西”。如果包装信息气味吸来错误读者，正文再努力也会被误读。

## 4. 核心概念

### 4.1 目标读者画像

定义：一个可指导创作决策的读者需求模型。

必须包含：

- 读者当前阅读任务。
- 读者期待的核心体验。
- 读者能接受的铺垫长度和信息复杂度。
- 读者最敏感的雷点和毒点。
- 读者愿意留下的证明场景。
- 读者愿意付费、追更或安利的条件。

不够用的写法：

- 年轻女性。
- 男频读者。
- 喜欢都市文的人。
- 爱看爽文的人。
- 大众读者。

更有用的写法：

- 下班后刷免费阅读平台、想快速获得身份逆袭和公开见证回报、能接受夸张设定但不能接受主角窝囊的都市脑洞读者。
- 熟悉古言宅斗套路、期待女主在家族秩序中保住尊严和主动权、愿意看细腻关系推进但讨厌反复误会拖延的女频读者。
- 喜欢规则怪谈、愿意推理规则漏洞、能接受高压氛围但不能接受关键线索空降和主角降智的悬疑读者。

### 4.2 阅读场景

定义：读者实际打开、继续、暂停、催更、弃书或付费的情境。

常见变量：

- 设备：手机为主，碎片阅读为主，长段说明耐受低。
- 时间：通勤、睡前、午休、排队、摸鱼、周末长读。
- 情绪：解压、逃离现实、找代入、找甜、找燃、找刺激、找陪伴。
- 平台：免费阅读、付费订阅、社区型平台、强标签平台、作者粉丝入口。
- 入口：榜单、推荐流、书名、封面、简介、短视频、朋友安利、作者旧作、评论区。
- 付费状态：免费试读、付费前夜、刚付费、长期订阅、完本补读。

### 4.3 阅读任务

定义：读者此刻打开作品，是为了完成什么心理任务。

常见阅读任务：

- 解压：想快速看到主角摆脱压迫、打脸、翻身。
- 代偿：想体验现实中难以获得的权力、尊严、爱情、财富、自由。
- 嗑关系：想看拉扯、偏爱、试探、误会、占有、救赎或陪伴。
- 解谜：想理解规则、猜真相、验证线索、看主角破局。
- 陪伴：想稳定追更，和角色生活在一起。
- 身份确认：想通过作品确认“我喜欢这种价值、这种关系、这种爽点”。
- 谈资：想获得可以安利、吐槽、玩梗或二创的内容。

### 4.4 耐心预算

定义：目标读者愿意等待回报的时长和复杂度。

耐心预算由以下因素决定：

- 入口承诺有多强。
- 前文是否已经兑现过。
- 类型是否允许慢热。
- 当前是否免费或付费。
- 读者是否信任作者。
- 章节是否持续给进度感。
- 信息负荷是否超过阅读任务。

### 4.5 入口误差

定义：包装、推荐或标签吸引来的读者，与正文真正服务的读者不一致。

例子：

- 标题像无脑爽文，正文是慢热群像权谋。
- 封面像甜宠，正文是高虐成长。
- 标签写悬疑，前三章主要是日常铺设。
- 简介承诺女强，正文让女主长期被动。

入口误差会制造大量“看错书”的负反馈。agent 必须先判断是正文失败，还是入口吸错人。

## 5. 总原则

### 5.1 画像服务决策原则

原则：读者画像必须能改变写法，否则就是装饰。

用法：每个画像字段都要对应一个创作决策，例如开篇速度、信息密度、主角主动性、雷点规避、章节回报、付费节点承诺。

风险：画像只写人口标签，agent 仍会写成平均化文本。

检查问题：知道这个读者画像后，我会具体改变哪三件事？

### 5.2 需求优先于身份原则

原则：比“读者是什么人”更重要的是“读者为什么此刻需要这本书”。

用法：画像优先写阅读任务、核心回报、痛点和退出触发，再写平台、年龄、性别等辅助标签。

风险：用人口标签推断喜好，容易滑向刻板印象。

检查问题：如果去掉年龄和性别，这个画像还成立吗？

### 5.3 目标读者不等于所有读者原则

原则：商业作品需要优先服务最可能留下、付费、追更和安利的一组读者。

用法：反馈冲突时，先判断反馈者是否接近目标读者，以及反馈是否击中主承诺。

风险：试图讨好所有读者会让承诺变模糊，正文变成安全但无锋芒。

检查问题：这次修改是在增强目标读者体验，还是在讨好非目标读者？

### 5.4 入口与正文一致原则

原则：书名、封面、简介、标签、章节名、推荐语和第一章必须吸引同一类读者。

用法：包装承诺和前三章兑现必须对齐；若数据点击高、读完低，优先检查入口误差。

风险：包装吸错人会让好正文被错误读者差评。

检查问题：读者点进来期待的第一口回报，正文是否在合理时间内给了？

### 5.5 画像可修正原则

原则：开书前画像只是候选，必须用数据、评论、盲评和付费表现修正。

用法：每次复盘记录画像假设是否被证据支持、修正或推翻。

风险：死守错误画像会让 agent 把真实读者反馈当噪音。

检查问题：最近 20 条有效反馈里，有哪些在改变我们对目标读者的理解？

## 6. 读者画像字段

### 6.1 最小画像

当时间很少，至少写 8 个字段：

1. 目标读者阅读任务。
2. 读者点击入口。
3. 核心期待回报。
4. 可接受铺垫长度。
5. 最敏感雷点。
6. 第一章必须证明什么。
7. 前 10 章必须兑现什么。
8. 读者为什么愿意追更或付费。

### 6.2 完整画像

建议字段：

- 画像名称：用需求命名，不用人口标签。
- 平台/频道。
- 入口来源。
- 阅读场景。
- 阅读任务。
- 核心情绪需求。
- 核心认知需求。
- 核心身份需求。
- 已熟悉套路。
- 想要的新鲜变量。
- 最低可接受质量线。
- 耐心预算。
- 信息密度承受度。
- 情绪强度偏好。
- 主角期待。
- 配角/关系期待。
- 雷点/毒点。
- 可付费条件。
- 可安利条件。
- 可接受的创新边界。
- 不可接受的背刺。
- 当前证据。
- 待验证假设。

## 7. 阅读场景矩阵

| 阅读场景 | 读者状态 | agent 应优先提供 | agent 应避免 |
| --- | --- | --- | --- |
| 通勤/排队 | 时间碎、注意力断续 | 清晰目标、短段落、强钩子、低名词负担 | 大段背景、复杂多线、长对白无状态变化 |
| 睡前 | 情绪敏感、想沉浸或放松 | 稳定声纹、情绪余波、关系黏性 | 高强度信息轰炸、过多悬而不决的焦虑 |
| 午休/摸鱼 | 快速反馈、解压需求 | 快节奏进展、明确爽点、轻量笑点 | 慢热解释、低价值过程、复杂铺垫 |
| 周末长读 | 容量较大、可沉浸 | 卷级推进、世界扩展、长线回报 | 单章强但长期原地踏步 |
| 推荐流误入 | 信任低、判断快 | 第一屏明确卖点、快速证明承诺 | 开头绕远、包装和正文不一致 |
| 老读者追更 | 有信任、有记忆 | 旧资产回响、新变化、稳定更新 | 忘记旧承诺、重复套路、人物清零 |
| 付费前夜 | 成本意识上升 | 强承诺、明确落袋、下一阶段价值 | 假钩子、拖延、回顾灌水 |
| 完本补读 | 关注完整性 | 长线闭环、口碑资产、章节导航 | 烂尾、支线不收、番外补主线 |

## 8. 目标读者分层

### 8.1 核心目标读者

定义：最可能因为本书主承诺留下、追更、付费和安利的人。

处理：优先服务。重大结构、开篇、付费节点和结局决策都要先看这类读者。

### 8.2 扩展读者

定义：不是最核心，但可能因为副卖点、作者声纹、平台推荐或题材新鲜感留下的人。

处理：可以照顾，但不能让其偏好抢走主承诺。

### 8.3 误入读者

定义：被包装、推荐流、标题、封面或标签吸进来，但真实需求与作品不匹配的人。

处理：负反馈要记录，但不应直接改主线；优先检查入口是否误导。

### 8.4 老读者

定义：来自作者旧作、系列前作或长期关注的读者。

处理：可以给熟悉感和回响，但新书仍要独立成立。

### 8.5 付费读者

定义：已经为作品付出金钱成本或订阅成本的人。

处理：其反馈不一定永远正确，但对信任、节奏、兑现和水文风险高度敏感。

## 9. 画像如何指导写作

### 9.1 指导开篇

读者画像要决定：

- 第一屏给人物、冲突、关系还是设定。
- 第一章是否必须见爽点/甜点/谜点/危机。
- 读者能不能接受先铺世界观。
- 主角第一选择要证明什么。
- 章末钩子是问题、危机、情绪还是回报。

### 9.2 指导信息密度

读者画像要决定：

- 专有名词预算。
- 世界规则首次投放量。
- 是否需要现实入口。
- 读者已知套路能否省略。
- 哪些解释必须转成行动后果。

### 9.3 指导人物设计

读者画像要决定：

- 主角最先需要展示能力、委屈、欲望、底线还是缺陷。
- 配角是提供情绪关系、反差、见证、阻力还是同盟。
- 反派能不能复杂化，何时复杂化。
- 角色讨喜、可追随和有锋芒之间如何平衡。

### 9.4 指导节奏

读者画像要决定：

- 回报间隔。
- 单章信息量。
- 支线展开长度。
- 高压和释放比例。
- 卷级目标公开时间。

### 9.5 指导付费节点

读者画像要决定：

- 付费前要落哪些袋。
- 付费后前三章必须证明什么价值。
- 读者对“水”的阈值。
- 是否能接受长线悬念延迟。
- 是否需要明确下一阶段承诺。

## 10. 画像证据来源

| 证据 | 能证明什么 | 不能证明什么 |
| --- | --- | --- |
| 竞品评论 | 读者在同题材里夸什么、骂什么 | 全市场都这样想 |
| 榜单/热书 | 当前平台存在某类需求 | 自己照写就能成功 |
| 书名简介点击 | 入口吸引力 | 正文承诺兑现 |
| 首章读完率 | 开篇是否承接入口 | 长线是否稳定 |
| 前 10 章追读 | 承诺是否形成循环 | 中后期是否不水 |
| 付费转化 | 成本前后价值判断 | 所有情绪线都无问题 |
| 评论/章评 | 读者显性体感 | 沉默读者的完整想法 |
| 读者群盲评 | 早期误解和退出点 | 统计显著结论 |
| 编辑反馈 | 结构和市场经验 | 目标读者真实情绪全貌 |
| 作者旧作读者 | 声纹和品牌期待 | 新题材一定适配 |

## 12. 审稿检查清单

审稿时，如果出现“读者不黏、点击高读完低、评论分裂、非目标读者骂、目标读者沉默、付费掉、包装像骗点”等问题，agent 按以下顺序检查：

1. 这本书的核心目标读者是否写成了阅读需求，而不是人口标签？
2. 当前包装吸引来的读者，与正文实际服务的读者是否一致？
3. 第一章是否证明了目标读者最想确认的东西？
4. 前三章是否兑现了入口承诺？
5. 当前章节回报间隔是否超过目标读者耐心预算？
6. 信息密度是否超过目标读者当前阅读场景？
7. 负反馈来自核心目标读者、扩展读者、误入读者还是非目标读者？
8. 正反馈来自什么读者？是否值得保留为资产？
9. 修改建议是在增强核心承诺，还是在模糊主承诺？
10. 是否需要改正文、改包装、改标签，还是只需记录分歧？

## 13. 常见失败模式

### 13.1 人口标签伪画像

症状：目标读者写成“18-35 岁女性”“男频读者”“大众读者”。

修复：改写成“阅读场景 + 阅读任务 + 核心回报 + 雷点 + 付费条件”。

### 13.2 讨好所有人

症状：怕得罪任何读者，导致爽点不够爽、虐点不够痛、关系不够有张力、题材味变淡。

修复：确认核心目标读者和扩展读者，非核心需求只能辅助，不能抢主承诺。

### 13.3 包装吸错人

症状：点击高，首章退出高；评论说“被骗进来”“不是我想看的”。

修复：对齐书名、封面、简介、标签和前三章承诺。

### 13.4 把非目标读者反馈当圣旨

症状：少量不适配读者提出强烈反对，agent 立刻改掉核心卖点。

修复：判断反馈来源和主承诺关系；非目标读者反馈可用于入口说明，不一定改正文。

### 13.5 忽视沉默读者

症状：只看最吵的评论，忽略读完率、追读、付费和催更。

修复：把显性评论和行为数据合并判断。

### 13.6 画像不更新

症状：开书前写过目标读者，后续数据和评论已经变化，但项目文档不修正。

修复：每次阶段复盘都更新画像假设、证据和写作决策。

## 14. 最小执行规则

如果 agent 时间很少，至少执行这 7 条：

1. 不写“所有人”“大众读者”“喜欢爽文的人”这种空画像。
2. 每个目标读者画像必须包含阅读任务、核心回报、耐心预算和雷点。
3. 开篇、简介、标签和前三章必须服务同一画像。
4. 评论反馈先判断读者类型，再决定是否修改。
5. 点击高读完低，优先检查入口误差和首章承诺兑现。
6. 付费掉，优先检查成本前后的回报承诺和水文感。
7. 画像必须随数据、评论、盲评和编辑反馈修正。

## 15. 推荐调用顺序

### 15.1 开新书

1. 读 `docs/43-commercial-project-greenlight-scorecard.md`。
2. 读 `docs/34-market-research-comp-analysis.md`。
3. 读本文件，生成目标读者画像卡。
4. 读 `docs/29-reader-expectation-genre-promise-selling-point.md`，生成承诺链。
5. 再进入开篇或包装设计。

### 15.2 开篇 A/B 测试

1. 读本文件，定义测试读者和阅读任务。
2. 读 `docs/39-opening-variant-ab-evaluation.md`，设计版本假设。
3. 读 `docs/41-reader-comment-coding-feedback-clustering.md`，整理反馈。
4. 更新画像和胜出版本理由。

### 15.3 读者反馈冲突

1. 读本文件，判断反馈者类型。
2. 读 `docs/41-reader-comment-coding-feedback-clustering.md`，标注评论。
3. 读 `docs/42-editor-reader-data-feedback-conflict-resolution.md`，处理冲突。
4. 只把稳定结论写入项目文档。

### 15.4 付费节点诊断

1. 读本文件，判断付费读者的成本状态。
2. 读 `docs/65-paid-conversion-subscription-retention.md`。
3. 读 `docs/101-reader-dropoff-poison-points-red-flags-retention-safety.md`。
4. 决定是修正文、改节奏、补回报，还是改包装承诺。
