# 167. 著作权、作者身份、授权许可、平台签约、同人改编与 AI 生成内容边界专项

适用范围：当 AI agent 需要设计、起草、审稿或维护中文网文项目，并涉及原创性、著作权归属、署名、共同创作、委托创作、职务创作、平台签约、独家/非独家授权、买断/分成/保底、信息网络传播权、改编权、影视/漫画/有声/游戏授权、同人二创、转载引用、现实歌词/图片/字体/封面素材、真实作品改写、IP 联动、版权登记、侵权争议、下架整改、AI 生成内容标识和 AIGC 参与创作记录时，使用本文件。

核心目标：让版权不再是“别抄袭”一句提醒，而是形成可执行的权利链：

`创作来源 -> 人类创作贡献 -> 作品定稿 -> 作者/权利人 -> 权利类型 -> 授权范围 -> 平台合同 -> 素材许可 -> AI 使用记录 -> 发布标识 -> 侵权争议 -> 证据保全 -> 下架/赔偿/改稿 -> 台账回写`

一句话规则：商业级网文不只要“写得像原创”，还要能说清每个可商业化资产的权利归属、授权边界、使用证据和后续改编权限。

---

## 1. 本专题解决什么问题

AI 写作和网文项目常见版权坏味道：

1. 版权口号化：只写“避免抄袭”，不检查作者身份、授权范围、素材许可和平台合同。
2. 权利打包误解：以为签约就是所有权利都归平台，或以为平台发布不影响任何改编权。
3. 改编权悬空：正文还没完本，就随口规划影视、漫画、有声、游戏和周边，但没有权利链。
4. 同人高风险：把影视、动漫、游戏、明星、历史热门 IP 的角色和设定改名后当原创。
5. 素材无来源：封面图、字体、歌词、诗句、音乐、地图、照片、设定图、宣传语没有授权记录。
6. 引用误用：把“参考资料”写成长段改写、逐句换词、保留结构，形成洗稿或搬运。
7. AI 生成免责幻觉：以为“是 AI 生成的”就天然无版权风险、无需标识、无需人类审查。
8. 多 agent 归属混乱：不同 agent 改稿、续写、重写后，不知道哪些内容来自用户、AI、来源资料或授权素材。
9. 合同字段缺失：签约、授权、改编、买断、分成、独家、期限、地域、语种、平台、违约责任都没有台账。
10. 侵权争议无后果：被指出融梗、撞设定、引用歌词、使用未授权图片后，只改几个词继续发。

本专题的任务：把“原创性检查”推进到“权利和授权管理”。

## 3. 安全边界：不是法律意见，不编造授权

版权、合同、平台规则和 AI 内容标识会随地区、平台、合同文本和时间变化。AI agent 必须遵守：

1. 不提供法律意见，不替用户判断某合同条款最终效力，不替代律师或版权专业人士。
2. 不编造授权证明、版权登记、平台审核结论、合同条款或权利归属。
3. 不声称“合理使用一定成立”“同人一定安全”“AI 生成一定无版权”“改名就不侵权”。
4. 不生成可规避版权检测、洗稿检测、平台审核或侵权追踪的教程。
5. 不将未授权歌词、长段原文、影视台词、真实图片、字体、商标、角色设定和可识别 IP 写入商业正文或宣传物。
6. 涉及正式签约、商业化发布、授权、纠纷处理、跨境平台、AI 内容标识和合同谈判时，必须查最新平台规则和权威法律资料。
7. 可以提供创作层面的风险识别、台账字段、问题清单和需要专业确认的事项。

可写：风险点、权利链、授权范围、证据保存、待确认标记。

不可写：伪造权利、规避审查、规避检测、替代法律结论。

## 4. 版权权利十线卡

设计或审稿涉及版权的项目时，至少检查十条线。

| 线 | 要问什么 | 叙事/项目功能 | 常见坏味道 |
| --- | --- | --- | --- |
| 来源线 | 内容来自原创、资料、竞品、真实案例、AI 输出还是授权素材 | 判断风险入口 | 来源混在一起 |
| 作者线 | 谁做出创造性贡献，谁有署名和控制权 | 归属和争议 | AI/人类贡献不分 |
| 表达线 | 保护的是表达，不是抽象想法、事实和套路本身 | 防误判 | 把类型套路当侵权或把独特表达当可抄 |
| 权利线 | 涉及复制、发行、信息网络传播、改编、翻译、汇编等哪些权利 | 授权拆分 | “版权”一词包办所有 |
| 合同线 | 平台签约、委托、合作、买断、分成、保底、独家如何约定 | 商业后果 | 签约只当爽点 |
| 授权线 | 授权期限、地域、语种、平台、媒介、是否可转授权 | 改编和传播边界 | 授权范围不明 |
| 素材线 | 图片、字体、音乐、歌词、封面、地图、资料图是否有许可 | 发布安全 | 素材随手用 |
| 同人线 | 是否使用他人角色、世界观、商标、明星形象或可识别 IP | 高风险过滤 | 改名当原创 |
| AI线 | AI 输出、人类选择改写、提示词和模型来源是否记录 | AIGC 风险 | AI 等于免责 |
| 争议线 | 被指侵权后如何保全证据、暂停传播、评估、改稿和沟通 | 风险处置 | 改几个词继续发 |

最小规则：任何商业发布前，必须能回答“谁创作、用到了谁的东西、授权到哪里、证据在哪里”。

## 5. 版权保护什么：想法、事实、套路和表达要分开

WIPO、中国著作权法和美国版权局资料都强调，版权保护的是具有独创性的表达，不是单纯想法、事实、原则、方法或类型套路本身。

### 5.1 四层区分

| 层级 | 示例 | 使用规则 |
| --- | --- | --- |
| 抽象想法 | 重生复仇、系统升级、先婚后爱、真假千金 | 可学习类型功能，不可照搬具体组合 |
| 事实资料 | 历史事件、行业流程、地理常识、法律制度 | 可查证后转化，但要避免长段搬运 |
| 类型套路 | 退婚打脸、秘境试炼、霸总误会、破案反转 | 可用，但要换主角欲望、因果和回报币种 |
| 具体表达 | 原文句子、桥段顺序、人物关系、专名体系、标志性梗 | 高风险，禁止改名复刻 |

### 5.2 AI agent 的判断问题

- 这个灵感是“功能”还是“表达”？
- 如果去掉来源作品的人名、专名和设定皮，骨架是否仍然相同？
- 读者是否会说“这不就是某作品换皮”？
- 相似点是行业常识、类型惯例，还是来源作品的独特排列？
- 这个设定能否通过改变主角欲望、压力来源、规则代价、关系网络和回报方式彻底重构？

检查问题：本项目吸收的是类型规律，还是来源作品的可识别表达？

## 6. 作者身份与权利归属：谁创造，谁授权，谁收钱

版权管理第一步不是“查是否侵权”，而是确认权利人。

### 6.1 常见归属场景

| 场景 | 风险点 | 台账要求 |
| --- | --- | --- |
| 用户独立创作 | 用户提供设定、正文、角色、世界观 | 标记用户原始贡献和日期 |
| AI 辅助生成 | 模型输出、用户筛选、改写、重组 | 记录提示词、输出用途、人类修改 |
| 多人共创 | 合作者、编辑、代笔、工作室 | 记录每人贡献和授权确认 |
| 委托创作 | 甲方出钱、乙方写作 | 合同约定归属、署名、修改权、交付范围 |
| 职务创作 | 员工在岗位中创作 | 需查合同和法律规则，不能凭感觉归属 |
| 平台签约 | 平台获得特定权利 | 明确签约主体、权利范围、期限和收益 |
| 改编开发 | 原小说改影视/漫画/有声/游戏 | 明确原作权利人和改编授权链 |
| 同人二创 | 基于他人 IP 创作 | 默认高风险，商业化前必须授权 |

### 6.2 多 agent 项目注意

多 agent 写作时，尤其要记录：

- 用户原始输入。
- 外部资料来源。
- 竞品分析结论。
- AI 生成片段。
- 人类确认的 canon。
- 已删除的高风险内容。
- 授权素材和未授权素材。
- 正式发布版本。

检查问题：如果半年后发生版权争议，项目能否说明每个核心设定和正文版本从哪里来？

## 7. 权利类型：不要用“版权”一词包办所有

“版权”在项目管理中应拆成具体权利。否则平台签约、改编和授权会变成糊涂账。

### 7.1 常见权利类型

| 权利/事项 | 小说项目中的表现 |
| --- | --- |
| 署名 | 作者名、笔名、联合署名、工作室署名 |
| 修改和保护作品完整 | 平台、编辑、改编方能否大改人物和结局 |
| 复制发行 | 纸书、实体出版、印刷、周边文本 |
| 信息网络传播 | 网站、App、公众号、电子书、海外平台发布 |
| 改编 | 影视、短剧、漫画、动画、游戏、有声剧、舞台剧 |
| 翻译 | 海外语种、繁体、外文版 |
| 汇编 | 合集、精选、资料设定集 |
| 表演/广播 | 有声、广播剧、朗读、直播讲书 |
| 衍生品 | 周边、角色商品、设定集、卡牌、游戏皮肤 |
| 商标/品牌 | 书名、系列名、角色名用于商业标识时可能另有风险 |

### 7.2 授权拆分例子

错误：

```text
平台拿了版权，随便改编。
```

更可执行：

```text
平台获得简体中文电子连载独家信息网络传播权，期限 5 年；影视、漫画、有声、游戏和海外翻译权待确认；作者保留署名权；收益分配和转授权需查合同。
```

检查问题：这次签约或授权到底涉及哪种权利、哪个媒介、哪个地区、多久、能否转授权、钱怎么分？

## 8. 平台签约：签的是权利和义务，不是“上岸”按钮

平台签约在网文里既是现实制度，也是商业剧情资源。不能只写“签了大神约”。

### 8.1 平台合同字段

| 字段 | 要问什么 |
| --- | --- |
| 签约主体 | 作者本人、工作室、公司、代理人，是否有权签 |
| 作品范围 | 单本、系列、衍生设定、续作、番外是否包含 |
| 授权性质 | 独家、非独家、买断、分成、保底、保底分成 |
| 授权权利 | 电子连载、出版、有声、漫画、影视、游戏、海外、周边 |
| 授权期限 | 多久，是否自动续约，终止条件是什么 |
| 授权地域/语种 | 中国大陆、港澳台、全球、简体、繁体、外文 |
| 收益分配 | 订阅、打赏、版权授权、广告、活动、改编收益 |
| 更新义务 | 字数、频率、断更、完本、请假、违约处理 |
| 修改权 | 平台/编辑能否要求删改、改名、改简介、改结局 |
| AI 条款 | 是否允许 AI 辅助，是否需标识，是否禁止训练使用 |
| 保密条款 | 合同、数据、读者群、平台资源是否可公开 |
| 违约责任 | 断更、双开平台、侵权、抄袭、未授权素材、数据造假 |
| 权利回收 | 作品下架、解约、完本后、期限届满后如何处理 |

### 8.2 签约剧情的真实压力

- 作者为了保底让出更多改编权。
- 编辑要求删改敏感桥段，影响剧情结构。
- 平台要求独家，作者不能再同步其他渠道。
- 改编机会来了，但影视权早已被打包授权。
- 作品被指融梗，平台要求暂停推荐或提交说明。
- 作者想换平台，发现旧合同还有期限和违约条款。
- 工作室代签，作者后续争议署名和收益。

检查问题：签约让角色获得了什么资源，又失去了什么控制权？

## 9. 授权许可：独家、非独家、转授权和改编权要写清楚

### 9.1 授权六件套

每次授权至少记录：

1. 授权人：谁有权授。
2. 被授权人：谁能用。
3. 授权内容：哪部作品、哪个角色、哪些设定、哪些权利。
4. 授权范围：媒介、地域、语种、期限、渠道。
5. 收益和义务：费用、分成、结算、审查、署名、宣传。
6. 违约和终止：超范围使用、拖欠、质量、下架、争议。

### 9.2 常见授权误区

| 误区 | 修复 |
| --- | --- |
| 授权电子连载就能拍剧 | 改编权单独确认 |
| 授权有声就能做广播剧和动画 | 媒介边界拆开 |
| 授权中国大陆就能海外发行 | 地域和语种拆开 |
| 非独家等于随便用 | 仍有范围、期限和署名要求 |
| 改编授权等于可乱改 | 看合同是否有审稿、署名、完整性和质量条款 |
| 授权角色名就能用商标 | 商标和版权分开确认 |

检查问题：授权是否被拆成可执行字段，而不是一句“版权卖了”？

## 10. 同人、二创和真实 IP：默认高风险，不要改名洗白

同人和二创可以作为粉丝文化存在，但商业级原创项目必须谨慎。

### 10.1 高风险内容

- 直接使用他人角色、世界观、组织名、能力体系、地图、专名。
- 使用真实明星、综艺、影视、动漫、游戏、小说 IP 的可识别元素。
- 改名但保留人物关系、设定组合、剧情顺序和标志性梗。
- 写“平替角色”但读者明显能认出原型。
- 用真实歌曲歌词、台词、海报、剧照、截图、字体和 Logo。
- 商业化同人、收费番外、付费二创、平台签约同人。

### 10.2 可安全转化的方式

- 抽取功能：例如“强导师和叛逆学生的关系张力”，而不是照搬某对师徒。
- 改变量：时代、职业、资源、权力结构、人物欲望、失败代价全部重构。
- 改规则：能力体系、组织制度、世界观逻辑重新建立。
- 改情绪引擎：同样是“宿敌变盟友”，但核心伤口、责任债、和解路径不同。
- 保留记录：原来源只作为“灵感风险”，不进入 canon。

检查问题：如果读者删掉名字仍能认出 IP，本项目就不能当原创商业正文使用。

## 11. 引用、转载、资料和真实案例改写

### 11.1 资料使用分级

| 类型 | 可用方式 | 禁区 |
| --- | --- | --- |
| 官方法律/科普资料 | 概括规则、改写成叙事限制，保留来源 | 长段照搬当正文 |
| 新闻报道 | 提炼社会机制、时间压力、制度矛盾 | 改名复刻真实受害者经历 |
| 学术/行业资料 | 转化为专业流程和可信度 | 把专有文本搬进正文 |
| 竞品小说 | 拆承诺、节奏、回报功能 | 复刻桥段、人物、设定、语气 |
| 真实案件 | 抽象风险和制度问题 | 猎奇化、暴露隐私、复刻伤害过程 |
| 网络热梗 | 作为时代氛围参考 | 把具体梗当核心卖点 |

### 11.2 引用现实文本

默认规则：正文不直接引用受保护的长段现实文本。

尤其高风险：

- 歌词。
- 影视台词。
- 小说原文。
- 诗歌长段。
- 课程讲义。
- 付费资料。
- 未授权图片说明文字。
- 真实聊天记录。

可写替代：

- 自创歌词/诗句。
- 概述氛围，不给原句。
- 用角色自己的语言转述。
- 写“那首老歌的副歌响起”，不写歌词。
- 写“判决书中的关键句显示……”，不长段复制。

检查问题：这段现实文本是否必须原文出现？如果不是，优先概述或原创替代。

## 12. 素材版权：封面、字体、图片、地图、音乐和提示词资产

网文项目不只有正文版权，视觉和宣传素材也会出事。

### 12.1 常见素材台账

| 素材 | 风险 | 台账字段 |
| --- | --- | --- |
| 封面图 | 画师授权、AI 图、图库范围、商用许可 | 来源、作者、许可、商用范围、修改权 |
| 字体 | 商用授权、嵌入、标题字、封面字 | 字体名、授权方、用途、期限 |
| 图片/照片 | 肖像权、版权、真实地点、商标 | 来源、许可、人物/地点、可修改性 |
| 音乐/歌词 | 歌曲版权、歌词引用、翻唱授权 | 不直接引用，授权待确认 |
| 地图/设定图 | 地理数据、素材模板、参考图 | 来源、可商用、是否改绘 |
| Logo/商标 | 平台、品牌、现实企业 | 是否真实商标、是否可使用 |
| AI 生成图 | 模型许可、提示词、训练争议、平台规则 | 模型、提示词、人工修改、用途 |
| 宣传文案 | 标题仿写、热梗、竞品句式 | 来源风险、原创改写记录 |

### 12.2 素材使用规则

- 未确认授权的素材只能做内部参考，不进入正式发布物。
- 商用发布前必须确认许可范围，不能只看“网上能下载”。
- AI 图、AI 字体、AI 音频也需要查平台/模型使用条款。
- 封面和宣传物涉及真实人物、明星脸、商标、平台 Logo 时进入第 20 篇。
- 素材一旦进入发布包，必须有来源和许可记录。

检查问题：如果平台或权利人要求证明授权，项目能否立刻拿出证据？

## 13. AI 生成内容：AI 不是免责牌，也不是自动作者

AI 参与创作时，最容易出现三种误解：

1. AI 生成就没有版权风险。
2. AI 生成就必然能登记版权。
3. AI 生成不需要标识和审查。

这些都不能当默认结论。

### 13.1 AI 内容管理四件事

| 项目 | 要记录什么 |
| --- | --- |
| 输入来源 | 用户原创、外部资料、竞品、真实案例、授权素材 |
| 模型输出 | 输出是否直接使用、是否像来源、是否含可识别内容 |
| 人类贡献 | 选择、删改、重组、情节设计、语言重写、结构控制 |
| 发布规则 | 平台是否要求 AI 标识、是否限制 AIGC、是否要求声明 |

### 13.2 人类创作控制

AI agent 应帮助项目保留人类控制痕迹：

- 用户确认的项目简报。
- 人物欲望和冲突引擎。
- 大纲选择记录。
- 章节任务单。
- AI 输出筛选和大幅改写记录。
- 删除高风险相似内容的记录。
- 最终人工审稿结论。

不是为了堆流程，而是为了将来证明作品不是“无来源拼贴”。

### 13.3 AI 标识和平台规则

生成式 AI 服务、深度合成和平台规则不断变化。AI agent 不得编造“无需标识”或“一定要标识”的绝对结论。

执行规则：

- 正式发布前查目标平台最新规则。
- 项目内先记录 AIGC 参与程度。
- 若平台要求标识，按平台执行。
- 若涉及封面、音频、视频、角色图等深度合成内容，额外查规则。
- 对读者和平台不要虚假声明“纯人工”。

检查问题：项目是否知道哪些内容由 AI 辅助，哪些由人类确认，哪些需要标识或禁止发布？

## 14. 侵权争议处理：先止血，再判断，再修复

被指出侵权、融梗、洗稿或素材未授权时，AI agent 不要立刻辩解或继续批量生成。

### 14.1 争议处理链

`暂停扩散 -> 保存证据 -> 标记风险内容 -> 比对来源 -> 判断相似层级 -> 查授权/合同 -> 选择下架/修改/声明/协商/专业咨询 -> 回写台账 -> 修复流程`

### 14.2 相似层级

| 等级 | 表现 | 动作 |
| --- | --- | --- |
| C0 普通类型相似 | 都是常见套路 | 记录，无需恐慌 |
| C1 低风险相似 | 单个功能类似 | 改变量，保留记录 |
| C2 中风险靠近 | 多个桥段或关系接近 | 暂停扩写，重构核心因果 |
| C3 高风险融梗 | 顺序、人物关系、设定组合明显接近 | 禁止进入 canon，重做方案 |
| C4 侵权/未授权素材 | 长段原文、歌词、图片、IP 角色等 | 下架/替换/授权/专业处理 |

### 14.3 处理话术边界

可写：

- “该内容标记为待核查，暂停发布。”
- “目前只能确认存在相似风险，需比对来源和授权。”
- “建议保留版本、来源、创作记录并咨询专业人士。”

不可写：

- “绝对不侵权。”
- “改几个词就没事。”
- “同人不盈利就安全。”
- “AI 生成所以不用管。”

检查问题：争议处理是否先保护项目和读者，而不是护短？

## 15. 平台签约与版权剧情写法

版权和签约不仅是现实风险，也可以成为小说内剧情。

### 15.1 可用剧情

- 新人作者签约时发现改编权被打包。
- 工作室代笔争署名权。
- 歌手爆红后歌词授权出问题。
- 漫画改编方想改女主性格，原作者反对。
- 平台要求删改敏感桥段，粉丝误解作者烂尾。
- 同人作者商业化后被原 IP 权利人警告。
- AI 辅助封面被平台要求标识或替换。
- 影视公司争抢版权，但原作合同期限未清。
- 反派伪造授权书，主角用创作记录反击。

### 15.2 剧情重点

版权戏的爽点不应只是“律师函打脸”，而是：

- 权利链被理清。
- 创作者控制权被争夺。
- 作品商业价值被确认。
- 合同代价被看见。
- 读者、平台、改编方和作者利益冲突。
- 主角通过专业、证据和边界守住作品。

检查问题：版权剧情是否让作品资产、人物选择和商业后果发生变化？

## 16. 古代、玄幻和架空世界迁移

即使没有现代著作权法，也有知识、技艺和文本归属规则。

### 16.1 等价物

| 现代概念 | 古代/玄幻等价物 |
| --- | --- |
| 作者署名 | 题名、落款、印章、师承、口碑 |
| 版权归属 | 家传秘方、宗门功法、戏班曲谱、书坊版刻 |
| 授权许可 | 师门准许、书坊契约、传法资格、行会许可 |
| 改编权 | 戏文改本、说书改编、曲谱改唱、话本翻刻 |
| 侵权盗版 | 盗印、冒名、偷谱、盗传功法、剽窃诗文 |
| 平台签约 | 书坊供稿、戏班聘约、宗门供奉、商会包销 |
| 同人二创 | 续写名著、借用传说人物、民间改编 |
| 登记证据 | 印章、契书、见证人、手稿、版木、师门记录 |

### 16.2 架空规则

架空世界也要回答：

- 谁有资格传授或复制。
- 谁能署名或冒名。
- 盗用后谁来裁决。
- 侵权赔偿是什么。
- 禁术、功法、戏文和秘方是否有使用范围。
- 读者是否能理解“为什么这不是随便抄”。

检查问题：这个世界是否有保护创作、传承、秘法和商业文本的规则？

## 17. 常见坏味道与修复

| 坏味道 | 读者/项目体感 | 修复方式 |
| --- | --- | --- |
| 只说注意版权 | 不能执行 | 拆来源、作者、权利、授权、素材、平台和台账 |
| 改名同人当原创 | 高风险 | 回到第 102 篇，重构功能、变量和因果 |
| 平台签约像升级 | 商业悬浮 | 写权利范围、收益、更新义务、改编权和违约代价 |
| 授权一句话 | 后续改编混乱 | 写期限、地域、语种、媒介、独家、转授权、分成 |
| AI 生成即免责 | 错误 | 记录来源、人类贡献、相似风险和平台标识规则 |
| 素材随手用 | 发布风险 | 建素材授权台账，未确认只内部参考 |
| 直接引用歌词台词 | 侵权风险 | 改为原创或概述，不长段引用 |
| 侵权争议护短 | 风险扩大 | 暂停扩散、保存证据、比对来源、专业确认 |
| 改编资产无权利链 | IP 开发空想 | 联动第 110 篇和本篇，确认谁有权授权 |
| 合同不进台账 | 长篇失忆 | 记录签约主体、权利、期限、收益和限制 |

## 18. 审稿检查清单

审稿时逐条检查：

1. 项目是否区分来源资料、竞品灵感、用户原创、AI 输出和授权素材。
2. 是否明确作者、共同创作者、委托方、平台、工作室或权利人。
3. 是否把“想法/事实/套路”和“具体表达/独特组合”分开判断。
4. 是否存在改名同人、融梗、洗稿、标志性梗复刻或专名体系借用。
5. 平台签约是否记录签约主体、作品范围、独家/非独家、期限、收益和权利范围。
6. 改编权、翻译权、有声权、漫画权、影视权、游戏权和周边权是否分别确认。
7. 是否有未授权歌词、台词、图片、字体、封面、地图、真实人物或商标。
8. 素材是否有来源、许可、商用范围、修改权和授权证明。
9. AI 生成内容是否记录输入来源、模型输出、人类修改和平台标识要求。
10. 正式发布、签约、授权或争议处理前是否查最新平台和官方规则。
11. 侵权争议是否先暂停扩散、保存证据、比对来源和标记风险。
12. 是否避免输出规避版权检测、洗稿检测、平台审核或伪造授权的方法。
13. 版权/授权风险是否影响项目立项、第一章生产许可、发布包和改编资产台账。
14. 是否把权利归属、授权范围、素材许可、AIGC 参与和争议状态写入台账。

## 21. 外部资料吸收点

本专题吸收外部资料时，只转化为创作流程和风险管理规则。

- 中国《中华人民共和国著作权法》：作品是文学、艺术和科学领域内具有独创性并能以一定形式表现的智力成果；著作权包含人身权和财产权，改编、翻译、汇编、信息网络传播等权利需要关注授权。迁移为“作品要有独创表达，权利要拆项管理”。
- 中国《著作权法实施条例》：补充作品、创作、作者和权利行使等规则；迁移为“作者身份和创作贡献要记录”。
- 中国《信息网络传播权保护条例》：网络传播作品涉及权利边界和保护机制；迁移为“网文平台发布不是无权利成本的复制”。
- 国家版权局作品登记资料：作品登记可作为权利管理材料之一，但不等于所有争议自动解决；迁移为“登记、手稿、发布时间和合同都要进证据台账”。
- WIPO Copyright 资料：版权保护创作表达，通常包含经济权利和精神权利；迁移为“署名、完整性和商业利用要拆开”。
- 美国版权局 Copyright Basics 与 AI 资料：版权不保护想法、系统、事实等本身；AI 生成材料与人类作者贡献需要区分；迁移为“AI 输出必须有人类选择、改写和创作控制记录”。
- Creative Commons 许可资料：许可可能包含署名、非商业、相同方式共享、禁止演绎等条件；迁移为“免费可见不等于商业可用，授权条件必须可执行记录”。
- 中国生成式 AI 与深度合成相关管理资料：生成式内容和深度合成服务涉及合法来源、权益保护和标识要求；迁移为“AI 参与创作要记录来源、审查风险和平台标识要求”。
- 中国《民法典》合同编：授权、委托、平台签约和版权交易最终落到合同约定与权利义务；迁移为“签约不是爽点，而是资源和控制权交换”。
- 网络出版服务相关管理资料：网络出版和平台发布有主体、内容和管理要求；迁移为“商业发布前必须查最新平台与监管规则”。

## 22. 最小执行规则

如果 AI agent 只能记住五条：

1. 第 102 篇负责“内容够不够原创”，第 167 篇负责“权利归谁、授权到哪里、证据在哪里”。
2. 平台签约和版权授权必须拆成主体、作品范围、权利类型、期限、地域、语种、媒介、收益和违约责任。
3. 同人、现实歌词、影视台词、图片字体、真实 IP、明星形象和未授权素材默认高风险，商业发布前必须授权或替换。
4. AI 生成内容不是免责牌，要记录输入来源、模型输出、人类修改、相似风险和平台标识要求。
5. 发生侵权争议时先暂停扩散、保存证据、比对来源、标记风险，再决定下架、重构、授权或专业处理。
