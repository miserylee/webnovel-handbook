# 66. 长篇时间跳跃与状态压缩专项

状态：已确认。

用途：给 AI agents 在长篇中处理“几天后、数月后、三年后、闭关结束、训练期、赶路期、备赛期、项目推进期、伤后恢复期、卷间空窗、时间线重排”时使用。目标是让时间跳跃压缩无效过程，但不抹掉人物状态、关系债、伏笔、代价、对手反制和读者追读感。

## 1. 核心判断：时间可以跳，状态不能断

长篇可以跳过无效过程，但不能跳过状态后果。

一句话规则：

> 时间跳跃不是“把时间删掉”，而是把这段时间造成的目标、资源、关系、信息、风险、伤势、舆论、对手、伏笔和规则变化压缩交代清楚。

允许跳掉：

- 重复训练。
- 重复赶路。
- 重复调查过程。
- 等待手续。
- 项目日常推进。
- 伤病恢复中的重复护理。
- 和主线无关的生活琐碎。

不能跳掉：

- 关键选择。
- 关系质变。
- 伏笔回响。
- 对手反制。
- 代价发生。
- 主角第一次理解新规则。
- 阶段回报或阶段失败。
- 会改变读者期待的事件。

---

## 2. 什么时候需要本专题

出现以下情况时，必须读取本专题：

- 需要写“几天后、几个月后、几年后”。
- 需要从卷末跳到下一卷开头。
- 需要压缩训练、修炼、赶路、调查、备赛、筹备、项目执行。
- 需要跨过角色成长阶段。
- 需要处理重生、时间线改变、时间循环、穿越前后。
- 审稿时发现“跳太快”“关系断了”“代价没了”“像漏章”。
- 多 agent 交接时，中间隔了一段未写时间。

## 3. 时间跳跃三问

任何时间跳跃前，先问三件事。

### 3.1 为什么要跳？

合格理由：

- 这段过程重复。
- 这段过程没有新的选择。
- 这段过程没有读者期待的回报。
- 这段过程可以用状态差额表达。
- 这段过程如果写出来会水。

不合格理由：

- 不知道怎么写中间过程。
- 不想处理支线清账。
- 不想写关系冲突。
- 不想解释对手反制。
- 想直接进入爽点。

### 3.2 跳过了什么？

必须列出跳过的事件类型：

- 行动。
- 训练。
- 恢复。
- 等待。
- 调查。
- 关系互动。
- 对手动作。
- 舆论发酵。
- 资源消耗。
- 规则变化。

如果跳过的是“选择”或“代价”，不能跳。

### 3.3 跳后改变了什么？

跳后必须有状态差额。

状态差额包括：

- 目标进度。
- 资源变化。
- 关系变化。
- 信息变化。
- 风险变化。
- 伤势/体能变化。
- 名声/舆论变化。
- 对手状态变化。
- 伏笔状态变化。
- 机制/规则理解变化。

如果跳前跳后没有状态差额，这段跳跃没有意义。

## 4. 时间跳跃四种类型

| 类型 | 用途 | 必须交代 | 高危风险 |
| --- | --- | --- | --- |
| 短跳 | 几小时到几天 | 行动位置、情绪余波、下一步压力 | 章间断裂 |
| 中跳 | 几天到几周 | 目标进度、资源消耗、关系状态、对手动作 | 像漏章 |
| 长跳 | 几月到几年 | 主角阶段变化、世界变化、关系债、伏笔状态 | 像重开 |
| 隐跳 | 叙述中压缩一段过程 | 过程摘要、关键节点、当前结果 | 水感或信息不清 |

不同跳跃尺度的要求不同。跳得越远，状态交代越不能偷懒。

## 5. 跳前清单

时间跳跃前，必须检查未结事项。

```markdown
跳前清单：

- 当前目标：
- 当前未结选择：
- 当前未付代价：
- 当前活跃支线：
- 当前活跃伏笔：
- 当前关系债：
- 当前对手动作：
- 当前资源状态：
- 当前伤势/体能：
- 当前舆论/名声：
- 当前规则限制：
- 哪些不能跳：
- 哪些可以压缩：
```

如果“不能跳”的内容超过 3 项，不建议时间跳跃。先写关键场景。

## 6. 跳中压缩方法

时间跳跃中，不需要把每一天写出来，但要保留关键差额。

### 6.1 状态差额句

适合短跳和中跳。

```text
这三天里，【目标进度】推进到【状态】，代价是【资源/关系/时间/伤势】，而【对手/环境】也完成了【反制/变化】。
```

### 6.2 三点压缩法

适合训练、赶路、调查、筹备。

```text
第一点：这段时间主角做了什么。
第二点：这段时间损失或改变了什么。
第三点：这段时间把主角推到哪一个新场景。
```

### 6.3 代表性场景

适合长跳。

不要写三年流水账。写一个能代表三年变化的场景。

代表性场景要展示：

- 主角现在的能力或身份。
- 旧关系的新状态。
- 旧代价仍然存在。
- 新阶段行动压力。

### 6.4 台账摘要

适合多 agent 交接。

用表格压缩：

| 项目 | 跳前 | 跳后 | 变化原因 | 后续影响 |
| --- | --- | --- | --- | --- |
| 目标 |  |  |  |  |
| 资源 |  |  |  |  |
| 关系 |  |  |  |  |
| 伏笔 |  |  |  |  |
| 对手 |  |  |  |  |

### 6.5 章首恢复句

跳后第一章开头要快速恢复读者方向。

合格句式：

```text
【时间跨度】过去，【旧卷结果】没有消失，反而让【当前压力】变得更具体。
```

不合格句式：

```text
三个月后。
```

单独一个时间标签不够，它只告诉读者时间变了，没有告诉故事为什么还连着。

---

## 7. 跳后恢复三件套

跳后第一场景必须恢复三件事。

### 7.1 方向

读者要知道现在要看什么。

- 当前目标是什么。
- 主角为什么现在行动。
- 这段时间让问题更接近解决还是更难解决。

### 7.2 状态

读者要知道跳后有什么变化。

- 主角状态。
- 关系状态。
- 资源状态。
- 对手状态。
- 伏笔状态。

### 7.3 压力

读者要马上看到行动压力。

- 倒计时到了。
- 对手出手了。
- 资格审核开始。
- 伤势影响行动。
- 人情债到期。
- 舆论发酵。
- 旧伏笔回响。

---

## 8. 时间跳跃和关系线

关系最怕被时间跳跃抹平。

跳时间前，必须记录：

- 关系起点。
- 未说出口的话。
- 未还的人情。
- 未解的误会。
- 未完成的承诺。
- 共同秘密。
- 公开站队。

跳时间后，必须回答：

- 这段时间两人有没有联系？
- 如果有，关系为何仍有张力？
- 如果没有，为什么没有联系？
- 这段空窗让关系更近、更远还是更复杂？
- 下一次见面时，关系状态如何表现？

不允许：

- 跳过关系冲突后默认和好。
- 跳过亏欠后默认消失。
- 跳过暧昧后关系回到零。
- 跳过共同秘密后没人再提。

---

## 9. 时间跳跃和伏笔/悬念

伏笔不能因为时间跳跃消失。

跳前要标记：

- 活跃伏笔。
- 待回收伏笔。
- 延期伏笔。
- 读者正在追的问题。

跳后必须至少处理一种：

- 回响一次。
- 给阶段进度。
- 明确延期窗口。
- 让伏笔产生新代价。
- 让旧误解在新时间点变得更危险。

禁止：

- 跳过伏笔回收。
- 跳后开新谜团但旧谜团不提。
- 让读者忘记旧问题后再突然回收。

---

## 10. 时间跳跃和对手/阻力

对手不会在主角跳时间时休眠。

跳后必须说明对手至少一种变化：

- 搜集情报。
- 调整策略。
- 恢复资源。
- 拉拢盟友。
- 布置陷阱。
- 改变公开叙事。
- 放弃旧打法，转入规则反制。

如果对手没有变化，要说明为什么：

- 被封印。
- 被限制。
- 误判主角。
- 正在等待时机。
- 内部受损。

不允许：

- 主角修炼三年，对手原地等。
- 主角养伤几个月，对手什么都不做。
- 对手仇恨跳后消失。

---

## 11. 时间跳跃和机制/世界规则

机制和世界规则也会随时间变化。

跳后检查：

- 系统任务是否过期。
- 资源价格是否变化。
- 组织规则是否调整。
- 地图局势是否变化。
- 行业项目是否推进。
- 战争、舆论、政策、门派、家族、公司是否有新状态。

如果世界线没有变化，长跳会显得假。

## 12. 时间跳跃类型对应表

| 场景 | 可以跳什么 | 必须保留什么 | 跳后第一压力 |
| --- | --- | --- | --- |
| 修炼/训练 | 重复练习 | 瓶颈、代价、进步证据、对手动作 | 考核、实战、资源短缺 |
| 赶路 | 路程细节 | 路上消耗、关系变化、追兵/消息 | 到达后规则压力 |
| 伤后恢复 | 重复护理 | 伤势后遗症、照顾关系、敌人动作 | 复出代价 |
| 调查 | 重复查资料 | 关键线索、误导、证人变化 | 线索到期或证人出事 |
| 备赛/备考 | 重复准备 | 排名、资源、心理压力、竞争者 | 正式开场 |
| 项目推进 | 日常工作 | 交付风险、客户/上级变化、资源消耗 | 节点验收 |
| 情绪冷战 | 日常沉默 | 未解话题、共同压力、外界推动 | 再次见面 |
| 卷间空窗 | 普通生活 | 旧卷资产、代价、伏笔、下一卷触发器 | 新阶段任务 |

## 13. 常见失败模式

### 13.1 一句“数月后”掀桌

表现：

- 旧关系没了。
- 旧代价没了。
- 旧伏笔没了。
- 新场景像重开。

修法：

- 补跳前清单。
- 补跳后状态差额。
- 第一场景带入旧资产和旧代价。

### 13.2 用时间跳跃逃避难写场景

表现：

- 关系冲突跳过。
- 失败代价跳过。
- 对手反制跳过。
- 关键选择跳过。

修法：

- 关键选择必须写成场景。
- 代价必须可见。
- 跳过的只能是重复过程。

### 13.3 跳后只有结果，没有过程可信度

表现：

- 主角突然变强。
- 关系突然亲密。
- 资源突然到账。
- 对手突然崩盘。

修法：

- 用三点压缩法交代过程。
- 用代表性场景证明变化。
- 补代价和限制。

### 13.4 跳后没有行动压力

表现：

- 跳后先介绍状态。
- 没有当前目标。
- 读者不知道为什么继续看。

修法：

- 章首恢复方向、状态、压力。
- 第一场景给触发器。
- 五页内出现行动目标或阻力。

### 13.5 长跳后人物像没活过

表现：

- 几年过去，人物欲望、关系、习惯、创伤、能力没有变化。

修法：

- 写代表性场景。
- 记录人物状态差额。
- 写一个旧习惯保留和一个新习惯形成。

## 14. 质量分级

### 高质量

- 跳前知道哪些能跳、哪些不能跳。
- 跳中压缩了过程和代价。
- 跳后方向、状态、压力清楚。
- 关系债、伏笔、对手动作、旧代价都没有消失。
- 第一场景有行动目标。
- 台账记录跳前跳后差额。

### 可用但需加强

- 跳后状态清楚，但对手变化不够。
- 关系状态保留，但缺少行动压力。
- 伏笔没有丢，但下一次回响窗口不清。
- 跳中摘要存在，但代价偏轻。

### 不合格

- 只有时间标签。
- 跳过关键选择或代价。
- 旧资产断线。
- 跳后像新卷重开。
- 台账没有更新。
- 读者不知道中间发生了什么重要变化。

不合格时，不要继续批量起草。先补时间跳跃卡和状态差额。

## 15. 时间跳跃闸门

跳后第一章或第一场景完成后，必须检查是否允许继续推进。

允许继续的条件：

- 跳前未结事项已分类。
- 跳过内容确实是可压缩过程。
- 跳后状态差额清楚。
- 旧代价至少影响一次行动。
- 活跃伏笔有下一次回响或延期窗口。
- 对手或阻力有时间内变化。
- 第一场景有当前行动压力。
- 台账已更新。

暂停修订的信号：

- 跳时间为了逃避难写场景。
- 关系债被自动抹平。
- 对手原地不动。
- 主角突然获得结果但没有可信代价。
- 跳后不知道当前目标。

闸门结论模板：

```markdown
时间跳跃闸门：

- 跳跃跨度：
- 跳跃理由：
- 跳过内容：
- 不能跳内容是否已写成场景：
- 跳前未结事项：
- 跳后状态差额：
- 旧代价保留：
- 关系状态：
- 伏笔状态：
- 对手状态：
- 当前行动压力：
- 台账是否更新：
- 结论：可继续推进 / 先补跳前清单 / 先补状态差额 / 先补关系债 / 先补伏笔回响 / 先补对手变化 / 重写跳跃段
```

## 16. AI agent 执行流程

### 16.1 设计时间跳跃前

先读取：

- `docs/core-writing/07-continuity-ledger.md`
- `docs/advanced-craft/56-volume-transition-new-stage-launch.md`
- `docs/workflows/38-chapter-production-pipeline-agent-handoff.md`
- 当前项目故事台账。

再填：

- 跳跃理由。
- 跳前清单。
- 不能跳内容。
- 状态差额。
- 跳后第一场景触发器。

### 16.2 起草时间跳跃时

每次跳跃都先写任务句：

```text
本次时间跳跃要压缩【重复过程】，保留【关键代价/关系债/伏笔/对手变化】，跳后第一场景用【行动压力】恢复读者方向。
```

### 16.3 审稿时间跳跃时

按顺序检查：

1. 为什么跳。
2. 跳过了什么。
3. 不能跳的是否已经写成场景。
4. 跳后状态差额是否清楚。
5. 关系债是否保留。
6. 伏笔是否有回响或窗口。
7. 对手是否有变化。
8. 第一场景是否有行动压力。
9. 台账是否能更新。

## 18. 一页检查清单

设计、起草或审稿时间跳跃时，逐项检查：

- 为什么要跳？
- 跳过的是重复过程，还是关键选择？
- 跳前未结事项是否列清？
- 跳后状态差额是否明确？
- 旧代价是否没有消失？
- 关系债是否保留？
- 活跃伏笔是否有回响或回收窗口？
- 对手是否在这段时间内有动作？
- 世界、机制、舆论或规则是否有变化？
- 跳后第一场景是否有行动压力？
- 是否避免只有“数月后”一个标签？
- 是否避免用时间跳跃逃避难写场景？
- 台账是否已更新？

## 19. 来源与迁移说明

本专题吸收的是通用叙事资料、场景/摘要资料、转场资料和网文连载经验，经中文网文连载场景重组后使用。

- Jane Friedman 关于 scene 与 summary 的资料：叙事可以在场景和摘要之间切换，摘要适合压缩重复过程，但必须保留关键变化，迁移为时间跳跃的状态差额规则。
- Writer's Digest 关于 fiction transitions 和 pacing 的资料：转场要帮助读者理解时间、地点和因果变化，迁移为跳后方向、状态、压力三件套。
- Helping Writers Become Authors 关于 scene-sequel 的资料：行动后需要 reaction、dilemma、decision，自然导向下一目标，迁移为跳前未结选择不能被跳过。
- Story Grid 关于 scene 与 value shift 的资料：场景或压缩段落都应产生价值变化，迁移为跳后必须有状态差额。
- Aliventures 关于 scene transitions 的资料：过渡要清楚处理时间、地点和人物状态，迁移为短跳、中跳、长跳和隐跳的不同要求。
- The Editor's Blog 关于 passage of time 的资料：时间流逝要让读者知道重要变化，迁移为时间跳跃台账和章首恢复句。
- 中国作家网关于网络文学连载和章节黏性的访谈资料：连载作品需要持续有效推进，迁移为时间跳跃不能让追读方向断裂。

迁移到单本小说项目时，不要整段复制本专题。应复制“长篇时间跳跃与状态压缩卡”，并填入跳跃跨度、跳前清单、状态差额、跳后第一场景和台账回写。

## 长篇训练、修炼与成长段落防水入口

当章节或小单元涉及训练、修炼、闭关、学习、备赛、技能升级、职业成长、系统任务或团队磨合时，需要调用 `docs/workflows/67-training-growth-anti-padding.md`。该专题负责判断训练段是否应该展开、压缩或跳过，并要求训练段具备目标、瓶颈、方法、代价、反馈、验证和下一阶段限制，防止训练流水账、闭关后直接无敌、数值升级无证明和对手原地等待。

## 反派、竞争对手与外部世界同步升级入口

当主角胜利、训练突破、暴露新能力、夺取资源、改变规则、经过时间跳跃、进入新卷或开启新地图后，需要调用 `docs/core-writing/60-antagonist-world-synchronous-escalation.md`。该专题负责判断外部世界是否同步变化：谁看见了主角变化、谁产生误判、对手换了什么策略、组织调了什么资源、规则或舆论怎样反馈、下一轮压力如何针对主角旧优势，防止世界原地等待、对手重复送、新敌人空降和升级失真。

## 日常章、过渡章与低强度章节防水入口

当章节涉及日常、过渡、休整、赶路、聊天、吃饭、生活流、低冲突推进、战后余波、关系缓冲、情绪沉淀、信息整理、准备章、轻松章或主线暂时低烈度推进时，需要调用 `docs/storycraft/62-everyday-transition-low-intensity-anti-padding.md`。该专题负责判断低强度章节是否值得独立成章，是否承接旧结果、产生微变化、交付小回报、保留人味、生成下一步并回写台账，防止日常流水账、过渡无增量、休整清零、赶路只写到了和聊天无目的。
