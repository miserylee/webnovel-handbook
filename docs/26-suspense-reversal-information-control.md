## 相关入口：世界观、设定与信息投放

涉及世界观、等级、组织、地图、金手指规则、历史背景、制度习俗或信息投放时，先调用 [28. 世界观、设定与信息投放](28-worldbuilding-setting-exposition.md)。该专题负责判断哪些设定进正文、哪些留后台、哪些必须进台账，以及如何避免开篇设定堆砌、规则漂移和名词过载。


> 用途：帮助 AI agents 设计、起草、审稿和修订悬疑、推理、规则怪谈、无限副本、权谋、阴谋线、反转爽点和信息差叙事，避免谜语人、硬藏信息、反转空降、伏笔失踪、红鲱鱼无意义、读者看不懂但作者以为高级。

## 1. 核心原则：信息差是承诺，不是欺骗

悬疑不是让读者什么都不知道，而是让读者知道“有一个问题值得追”。反转不是作者最后突然改答案，而是读者回头看时发现答案一直在场。信息差的本质是承诺：作者暂时不给完整答案，但必须持续给线索、给进度、给误导、给回收。

AI agents 写悬疑时，必须同时保护两件事：读者的好奇心和读者的信任。

```text
问题抛出 -> 线索出现 -> 误读成立 -> 信息推进 -> 反转重释 -> 真相落袋 -> 余波改变局面
```

如果读者只是“不知道”，不是悬疑；如果读者一直“不知道自己在等什么”，就是故弄玄虚。

### 用法

- 每个谜团都要有可追的问题，而不是只制造气氛。
- 每条伏笔都要登记回收位置，不能只靠作者记忆。
- 每次反转都要改变人物选择、资源状态、对手策略或世界理解。
- 每个红鲱鱼都要有剧情功能，即使误导了读者，也应带来信息、人物或阻力增量。
- 每 3-8 章至少给读者一次信息进度：确认、排除、重释、局部真相或代价。

### 风险

- 作者知道答案，读者只看到乱码，读者会弃。
- 反转没有铺垫，读者会觉得被耍。
- 伏笔隔太久不提醒，读者会忘。
- 谜语人台词过多，读者会烦。
- 红鲱鱼只是假线索，读者会觉得浪费时间。
- 线索全靠主角最后解释，过程会像答案公布而不是推理。

## 2. 六个概念先分清

| 概念 | 作用 | 读者体感 | AI 常见误用 |
| --- | --- | --- | --- |
| 悬念 | 让读者想看下一步 | 接下来会怎样 | 章末硬写“更大危机来了” |
| 谜团 | 让读者想知道答案 | 真相是什么 | 只藏答案，不给可追问题 |
| 伏笔 | 让未来回收显得合理 | 原来早就有 | 埋了不回，或回收时没人记得 |
| 线索 | 让读者或主角接近真相 | 我也能推 | 线索太少，最后靠天降证据 |
| 红鲱鱼 | 让错误答案阶段性合理 | 我猜错但服气 | 假线索无意义，只浪费时间 |
| 反转 | 重塑读者对事件的理解 | 意料之外，情理之中 | 空降设定、强行翻案 |

悬疑写作最关键的不是“藏”，而是“控制读者如何理解已知信息”。

## 3. 三层信息差：作者、主角、读者

信息差必须设计清楚，否则 AI 很容易写成“作者知道但不说”。

| 信息层 | 可用玩法 | 风险 |
| --- | --- | --- |
| 作者知道，主角和读者不知道 | 标准谜团，适合推理、规则怪谈、阴谋线 | 长期不给进度会变谜语人 |
| 作者和读者知道，主角不知道 | 戏剧反讽，适合危机、追杀、倒计时 | 主角太迟钝会降智 |
| 作者和主角知道，读者不知道 | 反转爽点、主角藏底牌 | 用多了会削弱代入和公平感 |
| 主角知道，作者也展示，读者能推但不确定 | 公平悬疑、规则解谜 | 需要严密台账 |
| 反派知道，主角和读者不知道 | 阴谋压力、幕后操盘 | 反派不能只靠黑箱全知 |

硬规则：同一谜团必须明确读者当前知道什么、主角当前知道什么、对手当前知道什么。不能只写“他露出意味深长的笑”。

## 4. 倒推设计：先知道真相，再布线索

Writing Excuses 强调可以从谜底倒推线索。AI 尤其适合使用倒推法，因为它能减少后期圆不回来的风险。

### 倒推六步

1. 写清最终真相：谁做了什么，为什么，怎么做，代价是什么。
2. 写清表面答案：读者前期最容易相信什么错误解释。
3. 列出真线索：哪些事实必须公平出现。
4. 列出误导线索：哪些事实会让读者暂时走向错误解释。
5. 安排揭示顺序：每次出现的信息改变什么判断。
6. 登记回收位置：哪一章、哪一场、由谁重释。

### 真相台账最小字段

```text
【谜团名称】
【最终真相】
【表面误读】
【读者当前知道】
【主角当前知道】
【对手当前知道】
【真线索】出现章节 / 表面意义 / 回收意义
【红鲱鱼】误导方向 / 合理性 / 附带增量
【阶段揭示】每 3-8 章给什么进度
【最终回收】章节 / 场景 / 谁解释或行动证明
【回收后影响】人物 / 关系 / 资源 / 世界 / 对手 / 情绪
```

## 5. 线索类型库

| 线索类型 | 功能 | 示例方向 |
| --- | --- | --- |
| 物证线索 | 提供客观事实 | 痕迹、物品、记录、伤口、缺失物 |
| 行为线索 | 暴露人物真实目的 | 过度解释、回避、重复动作、异常镇定 |
| 时间线索 | 限制可能性 | 不在场证明、延迟、先后顺序、时差 |
| 空间线索 | 限制路径和行动 | 密室、入口、监控盲区、地图错位 |
| 语言线索 | 暴露信息差 | 用词错误、称呼变化、知道不该知道的事 |
| 情绪线索 | 暴露隐瞒或旧伤 | 恐惧、愧疚、愤怒、保护欲失控 |
| 规则线索 | 暴露世界机制 | 禁忌、惩罚、例外、触发条件 |
| 关系线索 | 暴露同盟和背叛 | 谁保护谁、谁避开谁、谁替谁撒谎 |
| 缺席线索 | 通过“不在”提示问题 | 少了一句话、少了一个人、少了一件物品 |
| 重复线索 | 通过重复制造记忆点 | 反复出现的歌、数字、气味、动作 |

AI 起草时，不要只写“他发现了一个线索”。必须写清线索的表面意义、真实意义和它改变了什么判断。

## 6. 红鲱鱼必须有真实功能

红鲱鱼不是假货，它是“会被误读的真信息”。好的红鲱鱼即使不指向最终真相，也应该揭示人物、阻力、世界规则或另一条支线。

### 好红鲱鱼

- 表面指向错误嫌疑人，但能揭示对方真正的秘密。
- 表面解释规则异常，但后文证明只是局部现象。
- 表面是情敌或叛徒，实际暴露主角关系弱点。
- 表面是无关怪谈，实际提供世界规则边界。
- 表面是错证，实际逼出对手二次行动。

### 坏红鲱鱼

- 只为了骗读者，没有任何后续价值。
- 完全不合理，读者一看就是作者硬塞。
- 太多太乱，主线问题被淹没。
- 误导结束后不产生任何人物、资源或信息变化。
- 真相公布后，红鲱鱼无法解释为什么出现。

## 7. 反转的四个条件

反转要同时满足四个条件。

| 条件 | 检查问题 | 失败表现 |
| --- | --- | --- |
| 有铺垫 | 读者回看能找到证据吗 | 空降设定、突然新规则 |
| 有误读 | 前文是否提供了合理错误解释 | 读者从没形成判断 |
| 有重释 | 反转是否改变前文意义 | 只换答案，不改理解 |
| 有后果 | 反转后局面是否变化 | 只是惊讶一下，继续原剧情 |

### 反转类型

- 身份反转：好人/坏人、同盟/背叛者、受害者/加害者重释。
- 动机反转：行为没变，但动机变了。
- 规则反转：世界规则或副本规则被重新理解。
- 关系反转：两人关系、血缘、旧账或欠债被重释。
- 时间反转：先后顺序、循环、记忆或叙述时间被重排。
- 目标反转：主角以为要解决 A，实际真正问题是 B。
- 叙述反转：视角、记忆、身份或认知本身不可靠。

硬规则：反转不能只服务惊讶，必须服务主线推进、人物变化、机制升级或情绪回报。

## 8. 悬念节奏：大悬念和小悬念要互相喂养

起点社区节奏经验把悬念分为大悬念和小悬念：大悬念牵引长期期待，小悬念推动下一章和小单元。AI 需要同时管理两者。

| 层级 | 功能 | 节奏要求 |
| --- | --- | --- |
| 全书大悬念 | 牵引终局或最大真相 | 开篇或早期提出，定期提醒和推进 |
| 卷级悬念 | 牵引一个阶段 | 卷中有中点重释，卷末有阶段答案 |
| 单元悬念 | 牵引 3-8 章 | 必须有局部答案或代价 |
| 章末悬念 | 牵引下一章 | 下一章必须兑现、推进或转化 |
| 场景悬念 | 牵引当前场景 | 用选择、危险、缺口、异常维持注意力 |

### 进度规则

- 每次新增谜团，最好同时解决或推进一个旧谜团。
- 大悬念可以久拖，但不能久不出现。
- 小悬念不能全变成新坑，要有局部落袋。
- 章末悬念不等于假断章，下一章必须接住。
- 读者可以猜错，但不能觉得自己没有材料可猜。

## 9. 信息给法：让读者参与，而不是被动听答案

读者喜欢解谜时，会希望自己参与判断。AI 不应把所有信息都留到最后解释。

### 信息给法

| 给法 | 适合场景 | 风险 |
| --- | --- | --- |
| 场景展示 | 物证、行动、异常、规则触发 | 写得太显眼会泄底 |
| 对话夹带 | 人物口误、立场、隐瞒 | 谜语人太多会烦 |
| 行动反常 | 人物选择和表面动机不一致 | 如果不回收会像 bug |
| 旁观者误读 | 让读者和人物一起猜错 | 旁观者不能集体降智 |
| 多视角碎片 | 不同人看到不同真相 | 视角太多会乱 |
| 文件/记录 | 日记、聊天、账册、旧案、系统提示 | 不能全靠文本证据解决 |
| 环境细节 | 气味、损坏、摆放、天气、声音 | 需要回响，否则读者记不住 |

### 禁止项

- 关键答案只在作者旁白里公布。
- 关键证据从未出现，高潮时突然拿出。
- 所有人都说半句话，只为了拖延。
- 主角明明可以问，却为了剧情不问。
- 读者知道的信息比主角少太多，导致反制像开挂。

## 10. 不同题材的信息差重点

| 题材 | 信息差重点 | 高危风险 |
| --- | --- | --- |
| 悬疑/探案 | 公平线索、嫌疑人、动机、证据链 | 真相全靠最后解释 |
| 规则怪谈 | 规则边界、例外、试错代价 | 规则随机变，读者无法推 |
| 无限/副本 | 副本机制、队友博弈、隐藏任务 | 每个副本都靠新规则空降 |
| 权谋/宅斗/宫斗 | 阵营、名分、证据、利益交换 | 所有人谜语人，主角靠作者保送 |
| 玄幻/仙侠 | 世界规则、传承、秘境、身份真相 | 设定太密，反转太远，读者忘伏笔 |
| 都市异能 | 身份、舆论、证据、行业内幕 | 现实逻辑太松，证据链儿戏 |
| 情感悬疑 | 旧事、创伤、关系误读、隐瞒动机 | 为反转伤害人物可信度 |
| 喜剧脑洞 | 信息错位、误会升级、反常识规则 | 只靠巧合，不靠人物选择 |

## 11. AI 悬疑反转写作硬约束

- 不得让关键反转无铺垫空降。
- 不得让所有悬念都靠人物说半句话拖延。
- 不得让主角长期不问、不查、不验证，只等线索送上门。
- 不得让红鲱鱼没有剧情增量。
- 不得连续新增谜团而不解决旧谜团。
- 不得让读者缺少判断材料，却要求读者为反转震惊。
- 不得把章末断章当成悬疑质量本身，下一章必须兑现或推进。
- 每个重要谜团必须登记真相、真线索、误导线索、阶段揭示和回收位置。
- 涉及案件、死亡、犯罪、恐怖、精神控制、极端暴力等内容时，必须联动 `docs/20-content-compliance-risk-review.md`。
- 终局真相必须联动 `docs/24-ending-completion-risk-control.md`，不能把最大谜团留给番外或续作。

```text
【最终真相】谁做了什么；为什么；怎么做到；代价是什么
【表面答案】读者前期最容易相信什么
【读者当前知道】
【主角当前知道】
【对手/幕后当前知道】
【真线索】章节 / 表面意义 / 回收意义
【红鲱鱼】误导方向 / 合理性 / 附带增量
【阶段揭示】每 3-8 章给什么确认、排除、重释或局部答案
【章末悬念】下一章如何兑现、推进或转化
【反转条件】铺垫 / 误读 / 重释 / 后果 是否齐全
【回收场景】由行动、证据、对话还是事件证明真相

【风险检查】是否谜语人；是否空降；是否线索太少；是否红鲱鱼无意义；是否合规风险
```

## 13. 悬疑审稿流程

1. 先看核心问题：读者是否知道自己在追什么。
2. 再看真相台账：最终真相、表面答案、真线索和红鲱鱼是否齐全。
3. 再看公平性：读者是否得到足够材料参与判断。
4. 再看节奏：是否持续新增谜团却没有阶段答案。
5. 再看反转：是否满足铺垫、误读、重释和后果。
6. 再看主角：是否主动调查、验证、犯错和修正，而不是等作者喂线索。
7. 再看读者体感：是好奇、紧张、期待，还是困惑、疲惫、被耍。
8. 最后看合规和终局：敏感内容是否前置风险检查，最大谜团是否在正文解决。

## 14. 快速检查问题

- 这个谜团能不能用一句话说清？
- 读者当前知道哪些可判断材料？
- 真线索是否已经公平出现？
- 红鲱鱼是否除了误导外还有真实功能？
- 反转是否改变前文意义，而不只是换答案？
- 下一章是否接住了上一章的断章？
- 主角有没有主动验证，而不是等人解释？
- 每 3-8 章是否有信息进度？
- 读者回看时会说“原来如此”，还是“作者耍赖”？

## 信息战、舆论战与公开场合话语权攻防入口

当章节涉及证据链反杀、公开审判、庭审、朝堂对峙、寿宴摊牌、直播翻车、热搜风波、危机公关、发布会、项目验收、商战舆论、娱乐圈黑料、职场甩锅、宗门审判、弹幕评论区风向，或任何“胜负取决于谁被相信”的场景时，需要调用 `docs/61-information-public-opinion-discourse-battle.md`。该专题负责确认争点、关键受众、证据来源与边界、对手叙事、抢解释权方式、主角反制节奏、公开行动请求、落袋余波和合规风险，防止万能录音、主角独白、众人集体倒戈、热搜空转和现实舆论风险。


## 与 63 号专题的衔接：线索回收前要检查读者是否还记得

悬念、线索、红鲱鱼和反转设计时，除了检查“真相是否已倒推”“线索是否公平”“误导是否有独立解释”，还要调用 `docs/63-foreshadowing-callback-reader-memory-management.md` 检查读者记忆温度。

如果关键线索距离揭示已超过 10 章或跨卷，揭示前需要带后果的再激活或场景化唤醒。反转的目标不是让读者完全猜不到，而是让读者在揭示后觉得“我见过这个线索，只是当时误读了”。

## 补充：信息差必须标注视角权限

悬疑、反转和误会设计必须和 `docs/69-pov-narrative-distance-information-focus.md` 联动。每个关键秘密都要同时标注：作者知道什么、当前视角人物知道什么、读者知道什么、对手或幕后知道什么。

如果反转依赖“视角人物明明知道却不告诉读者”，要警惕不公平；如果反转依赖旁白临时补充规则，要警惕补丁感。更稳的做法是提前埋可回看的公平线索，让读者当时能误读，回看时能承认作者没有耍赖。
