# 去身份化政策

本仓库按开源项目维护。所有来自私有项目、内部审稿、读者反馈、作者实践、群聊或本地环境的材料，必须先去身份化，再进入公开文档。

## 目标

去身份化不是只删除姓名，而是让外部读者无法从文档中反推出：

- 具体作者、读者、编辑、群成员或账号。
- 具体旧项目、未公开作品、项目目录、稿件路径或内部代号。
- 具体读者群、编辑协作空间、聊天环境或私人组织结构。
- 本机用户名、绝对路径、工作目录、截图、文件树或私有仓库位置。
- 可把多个片段拼接起来重新识别个人或项目的组合线索。

## 允许使用的匿名标签

私有来源统一使用抽象标签：

- `匿名旧项目A`
- `匿名旧项目B`
- `匿名编辑反馈样本`
- `匿名读者反馈样本`
- `匿名作者实践样本`
- `公开平台资料`
- `官方公开来源`

不要使用真实项目名、工作目录名、群名、作者名、编辑名、读者昵称或其他内部代号。

## 必须移除的内容

提交前必须删除或改写：

- 真实姓名、昵称、账号、邮箱、手机号、微信号、QQ 号、群号。
- 本机绝对路径、云盘路径、私有仓库地址、截图文件名。
- 旧项目真实书名、人物名、章节名、未发布设定、正文片段。
- 编辑内部报告原文、读者群聊天原文、群公告、投票截图。
- 可识别学校、公司、村镇、小区、机构、门店或家庭的组合线索。
- 付费课程、未授权资料或私有文档原文。

## 可以保留的内容

可以保留抽象后的经验：

- 某类开篇为什么掉读。
- 某类人物为什么不活。
- 某类读者反馈如何聚类。
- 某类编辑意见如何转译成改稿动作。
- 某类旧项目经验如何迁移为原则、行动链、检查问题和案例卡。

保留时必须改变表达方式，避免照搬原句、原桥段、原人物和原项目结构。

## 案例写法

错误：

```text
来源：C:\Users\某用户名\Documents\某书名。
读者群里 A 说：“这里太水了。”
编辑 B 在审稿报告中指出……
```

正确：

```text
来源：匿名旧项目A。
匿名读者反馈样本显示：目标读者在本章中段出现“铺垫过长、回报延迟”的体感。
匿名编辑反馈样本指出：问题不在句子，而在本章缺少可见目标和阶段回报。
```

## Agent 使用规则

AI agent 处理本仓库时：

1. 不得恢复、猜测或询问匿名来源的真实身份。
2. 不得把匿名标签展开成具体项目名或本地路径。
3. 引用案例时，只引用抽象经验，不引用原文和可识别细节。
4. 如果发现具体路径、旧项目名、群名、账号或私人来源结构，应优先做去身份化修补。
5. 对外输出时，默认使用“匿名旧项目样本”“匿名读者反馈样本”“匿名编辑反馈样本”等表述。

## PR 检查

提交前自查：

- 是否还有本机路径？
- 是否还有旧项目真实名称或内部代号？
- 是否还有读者昵称、群名、账号或截图描述？
- 是否还有编辑报告、读者反馈或聊天原文？
- 是否可以通过多个片段拼出某个真实项目或个人？
- 是否已把经验改写成原则、用法、风险、检查问题和匿名案例？

如果答案不确定，按更保守的方式处理。
